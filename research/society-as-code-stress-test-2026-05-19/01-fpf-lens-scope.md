## §1 4 FPF primitives — metaphor mapping

### §1.1 Metaphor-as-tool (`U.MethodDescription`)

**Definition (F2):** Metaphor = analytic projection one domain (code/software) на target domain (society). Per audio_689 §1: «провести аналогию с каким-то вот программным кодом, что его вот долго писали разные разрабы, много охуенных фич добавили». Metaphor = NOT identity claim; mapping discipline.

**FPF mapping:**
- `U.MethodDescription` — metaphor = method description primitive (abstract; type-level)
- `B.3 F-G-R` — каждая metaphor mapping claim получает F-grade + Group-scope + Refutation predicate
- `A.6.B` dual-language — Russian voice surface + English FPF refs

**IP-1 compliance:** ✅ Metaphor = abstract type; cannot run. Instance bindings («Jetix-as-debugger», «Workshop-as-IDE») declared отдельно per `shared/schemas/executor-binding.yaml.template`.

[src: audio_689 §1 + FPF U.MethodDescription + IP-1 Bundle 1]

### §1.2 Society-as-system (`A.1 U.System` holonic)

**Definition (F2):** Society = system. Per audio_689 §1: «общество как систему, как она развивается, как двигается». Holonic decomposition: society ← states ← cities ← кооперативы (1000) ← clans (100) ← teams (10) ← individuals. State-myth-scale alignment (80M Germany) per voice anchor §1.

**FPF mapping:**
- `A.1 U.System` (holonic) — society = system; every layer = part-of + composed-of
- `A.16 Work-as-process` — society = bounded process; state = stocks; flows = communications + resources

**Anti-confusion checkpoint:** «Society» ≠ aggregate of individuals (Hayek emergent-order critique); ≠ single super-agent (anti-anthropomorphism). Holon-respecting.

[src: audio_689 §1 hierarchy 10→100→1000→80M + FPF A.1 holonic + Hayek 1974 «Pretence of Knowledge» open-question for Phase 5 determinism counter]

### §1.3 Code-as-method («debugging» tactic) (`A.3.1 U.Method` + `IP-2`)

**Definition (F2):** «Code» = method primitive applied к societal system. «Debugging» = specific method tactic (observe deviation → orient hypothesis → decide patch → act). Per audio_689 §1: «Jetix заявляет, что вот эти баги накопились и мы идем их исправлять».

**FPF mapping:**
- `A.3.1 U.Method` — debugging = instantiable method
- `IP-2 process inheritance` — Jetix runs debugging instance bound к societal-bug observations
- `B.5.1 Explore` — Phase 5 must also surface where method tactic fails (counter-argument inventory)

**IP-1 compliance:** ✅ Debugging = abstract tactic; Jetix instance = RUSLAN-LAYER binding (Jetix-as-debugger = specific executor authority application; never type-level claim).

[src: audio_689 §1 «идем исправлять» + FPF A.3.1 U.Method + IP-2 process inheritance + FPF B.5.1 Explore]

### §1.4 Execution semantics (`A.16 Work-as-process`)

**Definition (F2):** Society «runs» code (executes process); «bugs surface» (observable deviations); «patches deploy» (interventions). Audio_689 §1: «миллионы багов которые надо исправлять» + «время для чистки революции перестройки».

**FPF mapping:**
- `A.16 Work-as-process` — society as continuous process (NOT discrete state-machine; emergence preserved)
- `A.14 Iterative loop` — debugging is iterative; never «one-shot fix»
- `B.3 F-G-R` refutation per intervention claim
