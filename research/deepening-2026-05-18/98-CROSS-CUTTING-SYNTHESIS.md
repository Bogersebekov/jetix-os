# 98 — Cross-cutting synthesis: patterns spanning 3+ directions

## §0 TL;DR (≤200 слов)

**9 cross-cutting patterns** identified across 14 directions:

1. **Substrate-vs-governance trade-off** — viral substrate (Karpathy / WikiWikiWeb / Lean) requires governance discipline against decay
2. **Single-founder fragility** — Cybersyn / Levenchuk ШСМ / Engelbart NLS / CCP Games / Plurality (Tang+Weyl) — bus-factor 1 problem
3. **Tacit→explicit transfer lossiness** — TPS / Mondragón / Engelbart / Toyota → all suffer documentation-vs-practice gap
4. **Visible-symbol vs distributed-substrate** — Cybersyn Opsroom / Workshop physical / EVE Titanomachy — visibility = attack surface
5. **Anti-gaming necessity for trust mechanisms** — Stack Overflow / Friend.tech / SBT / Coordinape — single-score systems decay
6. **Cross-domain transfer requires 4 elements** — substrate + bridge person + venue + operational artifact (Pattern Language lineage)
7. **Pre-mortem + retrospective discipline** — Xanadu / Cybersyn / Six Sigma + Beer silence — most failures lack post-mortem
8. **Open-source + community governance shape** — Debian / NixOS / Plurality / Mondragón Cooperative Congress — explicit constitution + recall mechanism
9. **Network composition for sovereignty** — EVE Corp→Alliance / Mondragón Cooperative Congress / Network State — single unit cannot hold sovereignty

**Strategic implication:** Jetix Phase 0-1 should consciously **embed all 9 patterns into Foundation Architecture practice**, не just principle.

## §1 Pattern 1 — Substrate-vs-governance trade-off

**Surfaced in:** directions 01 (Xanadu), 02 (Cybersyn), 03 (SO+Friend.tech), 05 (Alexander lineage), 06 (Mondragón), 07 (substrate matrix), 08 (SEMAT), 13 (EVE), 14 (TPS).

**Pattern:** **Viral substrate** (low friction + high adoption velocity) **lacks governance discipline by default**. **Governed substrate** (high integrity + slow decay) **lacks viral adoption mechanism**.

**Examples:**
- WikiWikiWeb 1995 (viral, no governance) → vandalism, read-only Feb 2015
- Karpathy LLM Wiki (viral, no formal governance) → fragmentation risk
- Mondragón 1956+ (governed, slow viral) → 70-year sustained
- Debian Constitution 1998 (governed, slow viral) → 30-year sustained + fork (Devuan)

**Jetix posture:** Foundation Architecture LOCKED 2026 = governance-first. Viral substrate (Karpathy wiki) layered ON TOP of governance discipline. **Both, не either-or**.

## §3 Pattern 3 — Tacit→explicit transfer lossiness

**Surfaced in:** directions 04 (Engelbart NLS), 05 (Alexander → Beck/Cunningham bridge), 06 (Mondragón University), 08 (SEMAT slow adoption), 09 (Karpathy «learn by building»), 14 (TPS layers).

**Pattern:** Each generation of methodology transmission **loses signal** vs internal practice. Lean consultants ≠ Toyota Sensei depth. SEMAT spec ≠ Jacobson practice. FPF spec ≠ Workshop practice.

**Mitigations observed:**
- Toyota: mentor-pairing + slow promotion + internal leader growth (P9 + P10)
- Mondragón University + 12 tech centers: inter-generational continuity
- Karpathy LLM101n: «learn by building» + AI teaching assistants
- Christopher Alexander: format itself = abstraction-portable

**Jetix posture:** Workshop pattern (vision/03) must embed **apprenticeship + AI-substrate hybrid + ritualized reflection (Hansei + Kaizen)**. F-G-R schema captures explicit; mentor-pairing captures tacit. **Both required**.

## §4 Pattern 4 — Visible-symbol vs distributed-substrate

**Surfaced in:** directions 02 (Cybersyn Opsroom destroyed), 11 (Tang Taiwan digital democracy = visible target), 13 (EVE B-R5RB Titanomachy monument), 09 (Karpathy single-author authority brand).

**Pattern:** **Visible centralized symbol** = political/ideological attack surface. **Distributed substrate** = resilient but lacks rallying brand.

**Trade-offs observed:**
- Cybersyn Opsroom: photogenic but destroyed September 1973
- EVE Titanomachy: permanent monument но in-game (no real attack surface)
- Karpathy authority: individual brand drives 36K GitHub stars but creates single-point dependency
- Mondragón: 24 distinct cooperatives + Eroski + Laboral Kutxa = distributed

**Jetix posture:** Workshop pattern = practice (not single symbol). Foundation Architecture = constitutional substrate (not photogenic Opsroom). **R12 fork-and-leave** preserves substrate distribution.

## §5 Pattern 5 — Anti-gaming necessity for trust mechanisms

**Surfaced in:** directions 03 (Stack Overflow + Friend.tech), 07 (substrate matrix anti-gaming column), 06 (Mondragón wage-ratio enforcement), 13 (EVE betrayal arcs).

**Pattern:** **Single-score trust systems** decay via gamification OR financialization. **Multi-signal trust** = harder to game but more complex.

**Examples:**
- Stack Overflow karma: 15-year decay via downvote-asymmetry + privilege gating
- Friend.tech tokens: 13-month bubble + collapse via financialization
- Coordinape GIVE/GET: working anti-gaming в DAO context (epoch + peer-allocation)
- Mondragón wage ratio: anti-extraction via constitutional cap

**Jetix H8 design:** **≥3 distinct trust signals** (F-G-R + role-attestation + demonstrated results) + reset-capability + R12 anti-extraction. NO single Foundation-level «mastery score» becomes «THE» metric.

## §6 Pattern 6 — Cross-domain transfer requires 4 elements

**Surfaced in:** directions 05 (Alexander lineage), 04 (Engelbart 4-tuple), 12 (NASA SE mapping), 14 (TPS → Lean → Six Sigma).

**Pattern:** Cross-domain transfer needs:
1. **Substrate concept** in source domain
2. **Bridge person** with cross-domain literacy
3. **Right venue** with amplification (OOPSLA / GitHub Gist / Workshop)
4. **Operationally useful artifact** in target domain

**Examples of full 4-element transfer:**
- Alexander → Beck/Cunningham 1987 → GoF 1994 → Cunningham wiki 1995 → Karpathy 2026
- Engelbart NLS → Apple GUI → modern computing

**Jetix Phase 1 candidate:** «Pattern Language for Engineering Methodology»
- Substrate: Alexander format + F-G-R + AI-co-readability
- Bridge persons: Ruslan + L1 (Anatoly + Tseren)
- Venue: GitHub Gist / Karpathy-style + ШСМ ecosystem
- Operationally useful artifact: Workshop curriculum patterns

## §8 Pattern 8 — Open-source + community governance shape

**Surfaced in:** directions 06 (Mondragón Cooperative Congress 1984), 11 (Plurality CC0 self-governed book), 13 (EVE Alliance executor recall), 14 (TPS Hansei + Kaizen).

## §9 Pattern 9 — Network composition for sovereignty

**Surfaced in:** directions 06 (Mondragón Cooperative Congress federation), 11 (Plurality plural-tech), 13 (EVE Corp → Alliance), 12 (NASA — government agency, not federation).

**Pattern:** **Single unit (corp / Clan / project) cannot hold sovereignty.** Sovereignty = network composition + delegated executor + recall capability.

**Examples:**
- EVE: Corporation → Alliance с executor + 50% support
