## §2 FPF lens mapping (4 primitives)

### §2.1 Intellect-as-U.System (A.1)

Per FPF A.1 (holonic system primitive): intellect = system with **12 sub-systems** (components C1..C12); itself sub-system of «human» (super-system). Holonic discipline:
- Each Cn = own boundary + own purpose + own state
- Inter-component dependencies = system-level emergence (e.g., C4 attention enables C9 reasoning enables C10 proportion-sense)
- Failure mode at component level ≠ whole-system failure (compensable via other components)

**Provenance:** FPF-Spec §A.1 (U.System primitive); audio_690 «вся система вместе лаконично работает» (system-level frame).

### §2.2 Components-as-U.Capability (A.17)

Per FPF A.17 (U.Capability acquisition primitive): each Cn = capability to be acquired via curriculum. Capability acquisition has 4 standard stages (per A.17 acquisition lifecycle):
- **Stage 1: Awareness** — learner knows Cn exists
- **Stage 2: Procedural** — learner can perform Cn under guidance
- **Stage 3: Autonomous** — learner performs Cn unsupervised
- **Stage 4: Generative** — learner improves/teaches Cn

Curriculum design (Phase 6) must address all 4 stages per component.

**Provenance:** FPF-Spec §A.17; Karpathy LLM101n micro-curriculum pattern (cross-link к research/deepening-2026-05-18/09).
