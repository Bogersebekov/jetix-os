# Phase 1 — 30+ Hackathon variants survey

> **R1 surface.** Breadth coverage NOT selection. **30 platforms** across **6 categories**. Per-platform: 10 dimensions (origin / scale / output / IP / funding / governance / longevity / RU adoption / FPF compat / Jetix learning). [src: §3 prompt structure]

---

## §0 TL;DR

Hackathon ecosystem ≈ 30+ canonical platforms organized по category × rhythm × audience × IP mode. **Mike Swift's MLH** (~150 events/year, 100K+ student participants) и **Devpost** (~1M users registered) dominate breadth; **Kaggle** + **ETHGlobal** dominate domain depth (ML + Web3); **AI Grant / NFDG** + **g0v.tw** = highest R12 fit. **RU subculture** (Yandex / Sber / Tinkoff / DataFest) = distinct partner cohort. **Corporate hackathons** (Google Solve / Microsoft Imagine Cup / OpenAI Dev Day / Anthropic Build Days) = sponsor-substrate precedents.

**Recommendation surface (R1):** Jetix highest fit с (a) Devpost-as-substrate distribution channel, (b) Lablab.ai 7-day rhythm template, (c) g0v civic-tech-as-precedent, (d) MLH sponsor-mentor-participant triangle. Pure DIY = ETHGlobal flagship-driven scarcity model. [src: §10 cross-platform implications]

## §1 Category A — Code / ML hackathons

### §1.1 Kaggle
- **Origin:** 2010 (Anthony Goldbloom + Jeremy Howard); Google acquisition 2017. [src: en.wikipedia.org/wiki/Kaggle:retrieved_pre-2026]
- **Scale:** ~17M registered users (claimed; significantly inflated); ~50-200 active competitions; per-competition: 100-3000 teams.
- **Output type:** ML model + leaderboard rank + (rarely) paper.
- **IP model:** Per-competition T&Cs vary; sponsor often retains code (commercial competitions); learner competitions = public.
- **Funding:** Prize pool $1K-$500K (Netflix Prize $1M 2009 = historic peak); Google subsidises infrastructure.
- **Governance:** Private (Google subsidiary); kernel review = Kaggle staff.
- **Longevity:** >15y (Foundation-grade for ML community).
- **RU adoption:** High (Yandex teams active; ШАД graduates compete).
- **FPF compat:** Medium (Kaggle = method canonical; not system substrate; closed governance vs open R12).
- **Jetix learning:** Quality predicate (competition rigor) drives participant ROI perception; leaderboard = social-proof artefact. [src: concept doc A §4]

### §1.2 MLH (Major League Hacking)
- **Origin:** 2013 (Mike Swift + Jonathan Gottfried). [src: `research/hackathon-deep-2026-05-18/05-mike-swift-mlh-deep-profile.md`]
- **Scale:** ~150 events/year worldwide; ~100K+ student participants annually.
- **Output type:** Student project (hackathon weekend artefact).
- **IP model:** Participants retain; Code of Conduct enforced.
- **Funding:** Sponsor-driven (corporate API integration sponsorships).
- **Governance:** MLH org (private; for-profit acquired DEV 2026-02).
- **Longevity:** >12y (mature substrate).
- **RU adoption:** Low (MLH = US/EU university focus; RU participation limited).
- **FPF compat:** High (sponsor-mentor-participant triangle = operational substrate; FPF A.2 role taxonomy fits).
- **Jetix learning:** **Hypothesis A primary gateway**; sponsor relationships scaling pattern; Code of Conduct = trust-substrate. [src: hackathon-deep §5]

### §1.3 Devpost
- **Origin:** 2009 (as ChallengePost; rebranded Devpost ~2014).
- **Scale:** ~1M users registered; ~9K+ hackathons hosted cumulative.
- **Output type:** Project showcase + judging artefact (text + video + GitHub link).
- **IP model:** Participant-retained; hackathon organizer T&Cs vary.
- **Funding:** SaaS subscription (organizers pay; participants free).
- **Governance:** Private (Indeed acquired 2022).
- **Longevity:** >15y; consistent substrate.
- **RU adoption:** Medium (RU hackathons sometimes use Devpost as registration platform).
- **FPF compat:** Medium-high (platform = neutral substrate; не competitor; A.15 artefact storage fits).
- **Jetix learning:** Project artefact platform = scalable; participant retention drives growth; **Jetix could host events ON Devpost** vs build own platform. [src: concept doc A §4]

### §1.4 Lablab.ai
- **Origin:** ~2022 (AI-specific focus; Anthropic + OpenAI partnerships).
- **Scale:** 50-100 events; per-event ~500-5000 registrants global; per-event ~50-200 submissions.
- **Output type:** AI prototype + demo video.
- **IP model:** Participant retained; partner LLM APIs T&Cs apply.
- **Funding:** API credits sponsorship (Anthropic / OpenAI / Cohere); cash prizes secondary.
- **Governance:** Lablab.ai team (Poland-based).
- **Longevity:** ~4y (new; growth phase).
- **RU adoption:** Medium (global online events).
- **FPF compat:** **High** (7-day rhythm aligns с FPF method; Anthropic partnership = Workshop alignment). [src: hackathon-deep §4]
- **Jetix learning:** 7-day rhythm template; API-credits sponsorship model; LLM-native theming.

### §1.5 Hugging Face Spaces / Community Challenges
- **Origin:** 2022+ (Hugging Face Spaces launch).
- **Scale:** Variable; community challenges 100-2000 participants.
- **Output type:** Model + Space demo + benchmark.
- **IP model:** Open (Apache 2.0 default); model card mandatory.
- **Funding:** HF infrastructure subsidy; sponsor cash + GPU credits.
- **Governance:** HF community + curators.
- **Longevity:** ~4y.
- **RU adoption:** Medium (HF community has RU contributors).
- **FPF compat:** **High** (open-source mandate aligns с R12; A.15 artefact + model card discipline).
- **Jetix learning:** Open mandate = trust-substrate; community curator role = light-touch governance.

### §1.6 Anthropic Build Days / AI Hackathons
- **Origin:** 2023+ (Anthropic event series).
- **Scale:** Small (50-200 per event); curated.
- **Output type:** Claude-powered prototype.
- **IP model:** Participant retained.
- **Funding:** Anthropic credits + cash prizes.
- **Governance:** Anthropic team.
- **Longevity:** ~2y.
- **RU adoption:** Low (visa-friction).
- **FPF compat:** **Very high** (Anthropic = direct Workshop partner via FPF protocol exposure).
- **Jetix learning:** Curated small-scale + LLM-vendor sponsorship = template для Jetix first events.
