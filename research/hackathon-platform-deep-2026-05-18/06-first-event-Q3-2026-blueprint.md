## §2 Theme + challenge brief

### §2.1 Theme
**«AI consulting workflow automation — FPF-aligned solutions»**

### §2.2 Challenge brief (sponsor co-designed)
Real-world AI consulting workflow problem: **«Build an FPF-compliant agent that can intake unstructured client problem statement and return a structured (a) FPF primitive mapping, (b) ROY swarm dispatch plan, (c) acceptance criteria draft, (d) provenance log.»**

### §2.3 Why this theme
- **Aligns с FPF protocol** (72K строк vendored substrate exists в Jetix).
- **Real quick-money P1 problem** — pain point in consulting delivery.
- **Anthropic API credits** = natural toolchain (Claude API).
- **Sponsor differentiation**: Anthropic + Jetix joint brand.
- **Output usefulness**: best solutions could ship as Jetix tool post-event.

### §2.4 Provided resources
- FPF spec sandbox excerpt (publicly released subset for hackathon).
- Sample client problem statements (5 anonymised).
- Acceptance criteria template.
- Claude API credits (€5K-€10K equivalent).
- Hugging Face Spaces deployment access.

### §2.5 Acceptance criteria template
- Working agent (passes 3/5 sample inputs).
- FPF primitive accuracy ≥70% (manual review).
- Provenance log per output.
- Open-source code (MIT/Apache).
- 2-min demo video + GitHub link.

## §9 Success criteria + falsifiers

### §9.1 Quantitative success criteria
- ≥20 attendees physical + ≥30 online stream views.
- ≥3 demo-ready prototypes submitted.
- ≥1 sponsor follow-up commitment for Q4 event.
- ≥3 bloggers content production within 1 week post-event.
- ≥80% retention through full day.
- QF matching pool successful execution.
- 0 R12 violations.

### §9.2 Qualitative success criteria
- Sponsor satisfaction (interview post-event).
- Participant Net Promoter Score ≥7/10.
- Mentor satisfaction (would-mentor-again 100%).
- Public sentiment (Twitter / Telegram positive).

### §9.3 Falsifiers (event fails if)
- <10 attendees physical.
- Sponsor cancels mid-event.
- R12 violation event (sponsor extraction OR IP claim breach).
- 0 working prototypes.
- Mentor scarcity collapse (mentor:participant ratio < 1:10 sustained).

### §9.4 Falsifiers for Year-1 plan
- Q3 event achieves <50% success criteria → пересмотр Year-1 plan.
- Q3 + Q4 both fail → concept doc A reset к F1.
