# Phase 2 — Per-rhythm mechanism spec

> **R1 surface.** Per Ruslan voice text_009 ¶10 «на день / на месяц / на год от зависимости от сложности проекта». 6 rhythms specified across 8 dimensions. Cross-precedent examples from Phase 1.

---

## §0 TL;DR

Hackathon rhythm = orthogonal to category. **6 canonical rhythms:** day (8-24h) → weekend (48-72h) → week (5-7d) → month (4-6w) → quarter (10-13w) → year (50-52w). Each rhythm carries distinct **FPF primitive load** + **cohort fit** + **output type** + **sponsor model**. Shorter rhythms = cheaper / higher iteration / lower depth; longer rhythms = expensive / lower iteration / higher depth.

**Jetix multi-rhythm calendar (text_009 ¶10 + concept doc A §5.2):**
- Day = bloggers + sponsor activation mode (first event Q3 2026)
- Week = engineer-mode (Master Workshop pre-activation)
- Month = sponsor-project rhythm
- Year = multi-clan / Master Workshop apprenticeship

## §1 Rhythm 1 — Day (8-24h)

### §1.1 Dimensions
| Dimension | Value |
|---|---|
| Duration | 8-24h |
| Typical participants | 5-50 |
| Mentor : participant ratio | 1:3 to 1:5 (high-touch) |
| Output type | Idea / blueprint / partial prototype |
| FPF primitive load | **Low** (A.16 process only; minimal A.15 artefact) |
| Optimal cohort | Bloggers, generalists, executives, students intro |
| Cost structure | $5K-30K total (venue + catering + small prizes + mentors) |
| Sponsor type fit | Local corporate; small API credits; brand exposure |
| Examples Phase 1 | Anthropic Build Days, local Devpost meetups, college day-hacks |

### §1.2 Mechanism
- **Pre-event (1-2 weeks):** sponsor confirms; theme announced; registration opens.
- **Event day:**
  - 09:00 kickoff + theme reveal (sponsor brief)
  - 10:00-16:00 build (mentors rotate)
  - 16:00-18:00 polish + submission
  - 18:00-20:00 demos + judging
  - 20:00 announce + social
- **Post-event:** 24-48h follow-up (winner highlights, social posts, sponsor invoicing).

### §1.3 Strengths / weaknesses
- **+ low organizational complexity** → достижимо со small team
- **+ rapid sponsor ROI** → sponsor sees results within 24h
- **+ low participant commitment** → wide funnel
- **− shallow output** → rarely sustained projects
- **− low retention drive** → 1-day = ephemeral

### §1.4 Jetix fit
**HIGH** для Q3 2026 first event (bloggers + sponsor activation). Cost-aligned (~$10-20K первый event achievable). Phase 5 blueprint = day-rhythm canonical instance.

## §2 Rhythm 2 — Weekend (48-72h)

### §2.1 Dimensions
| Dimension | Value |
|---|---|
| Duration | 48-72h (54h Startup Weekend canonical) |
| Typical participants | 50-300 |
| Mentor : participant ratio | 1:8 to 1:15 |
| Output type | MVP / prototype / pitch |
| FPF primitive load | Medium (A.15 artefact + A.16 process + U.PromiseContent) |
| Optimal cohort | Engineers, designers, founders, mixed teams |
| Cost structure | $30K-100K (venue 3 days + catering + prizes + mentor stipend) |
| Sponsor type fit | Corporate API; cash prizes; recruitment-driven |
| Examples Phase 1 | Startup Weekend, MLH events, ETHGlobal flagships (3 days), Hack the North |

### §2.2 Mechanism
- **Pre-event (4-6 weeks):** sponsor lineup; track structure; mentor recruitment; participant marketing.
- **Friday evening:** kickoff + pitch round (54h Startup Weekend canon); team formation.
- **Saturday:** full build day; mentor rotations; sponsor checkpoints.
- **Sunday:** build morning; demos afternoon; judging + announce.
- **Post-event (1-2 weeks):** winner publicity; sponsor follow-through; participant onboarding (если YC pre-application).

### §2.3 Strengths / weaknesses
- **+ canonical method** (Startup Weekend 700+ cities replicated) — substrate-tested
- **+ team formation built-in** → social network artefact
- **+ MVP output** → tangible result
- **− coordination cost spikes** (>100 participants) → coord linear-superlinear
- **− mentor burnout risk** (3 full days)

### §2.4 Jetix fit
**MEDIUM-HIGH** для Q4 2026+ events. Cost step-up (~$50K+); requires sponsor + mentor lineup matured. Per concept doc A §5.2: not in initial 3-event Year-1 plan, но natural second-rhythm.

## §3 Rhythm 3 — Week (5-7 days)

### §3.1 Dimensions
| Dimension | Value |
|---|---|
| Duration | 5-7 days |
| Typical participants | 100-2000 (online accommodates) |
| Mentor : participant ratio | 1:15 to 1:30 |
| Output type | Functional prototype / submission package |
| FPF primitive load | Medium-high (A.15 + A.16 + A.17 trajectory + U.MethodDescription) |
| Optimal cohort | ML engineers, AI specialists, online-first developers |
| Cost structure | $30K-200K (mostly online; prizes + mentor + platform) |
| Sponsor type fit | LLM-vendor (API credits); cloud (GPU credits); cash prizes secondary |
| Examples Phase 1 | Lablab.ai (7-day canonical), some Devpost online, HF community |

### §3.2 Mechanism
- **Pre-event (4-8 weeks):** theme + sponsor + mentor lineup + Discord/Slack ramp.
- **Day 0 (Monday):** kickoff stream; theme reveal; team formation Discord channels.
- **Day 1-5:** build; daily standup AMAs с sponsor + mentors; checkpoint demos.
- **Day 6-7:** polish + submission + judging.
- **Post-event (1-2 weeks):** winner showcase; sponsor partnership conversations; participant retention nurture.

### §3.3 Strengths / weaknesses
- **+ online-friendly** → global reach без visa friction
- **+ functional prototype output** → demonstrable
- **+ AI-vendor sponsorship fit** → LLM credits = natural
- **− attention compete** (participants juggle day jobs)
- **− team coordination async** (timezone challenges)
- **− judging quality variable** (week-long output > demo-day attention)

### §3.4 Jetix fit
**HIGH** для Q4 2026 engineer-mode event. Aligns с Lablab.ai template + Anthropic alignment. Per concept doc A §7 Activation Gantt: «first engineer-mode hackathon Oct-Nov».

## §4 Rhythm 4 — Month (4-6 weeks)

### §4.1 Dimensions
| Dimension | Value |
|---|---|
| Duration | 4-6 weeks |
| Typical participants | 200-5000 (large online events; sponsor-project mode 20-100) |
| Mentor : participant ratio | 1:30 to 1:100 |
| Output type | MVP с sustained development potential / completed sponsor brief |
| FPF primitive load | High (A.15 + A.16 + A.17 + A.3.1 method + U.PromiseContent + A.11 commons) |
| Optimal cohort | Mixed: engineers + sponsor team + community |
| Cost structure | $50K-500K (extended mentor commitment; sponsor partner program) |
| Sponsor type fit | Sponsor-project mode: sponsor commits ~$100K to specific challenge; participants compete на real problem |
| Examples Phase 1 | Chainlink hackathons, some Gitcoin rounds, ETHGlobal hackathon series |

### §4.2 Mechanism (sponsor-project mode)
- **Pre-event (6-8 weeks):** sponsor scopes specific brief; success criteria; deliverable spec.
- **Week 1:** kickoff webinar; brief deep-dive; team formation.
- **Week 2-4:** build phase; weekly office hours; sponsor technical liaison.
- **Week 5:** polish + submission + private demo to sponsor.
- **Week 6:** judging + winner announce + IP discussion.
- **Post-event:** sponsor may hire winning team OR licence solution.

### §4.3 Strengths / weaknesses
- **+ real-problem alignment** → sponsor ROI clear
- **+ deeper output** → MVP could ship
- **+ sustained sponsor relationship** → repeat customer potential
- **− participant fatigue** (4-6 weeks side-project)
- **− sponsor IP friction** (commercial brief = IP discussion needed)
- **− coordination overhead** (sponsor-platform-participant trilateral)

### §4.4 Jetix fit
**HIGH** для Q3-Q4 2026. Per concept doc A §7 Activation Gantt: «sponsor-project month-rhythm». **Quick-money P1 alignment** — Jetix consulting clients = sponsor partners; hackathon = delivery vehicle.

## §5 Rhythm 5 — Quarter (10-13 weeks)

### §5.1 Dimensions
| Dimension | Value |
|---|---|
| Duration | 10-13 weeks (≈ 3 months) |
| Typical participants | 100-500 (intensive cohort) |
| Mentor : participant ratio | 1:5 to 1:15 (deeper mentor commitment) |
| Output type | Shippable product / startup MVP / paper submission |
| FPF primitive load | Very high (full A.15 + A.16 + A.17 + A.3.2 practice + U.MethodDescription evolution) |
| Optimal cohort | Aspiring founders, ML researchers, Workshop apprentices |
| Cost structure | $200K-2M (mentor stipends + infrastructure + grants) |
| Sponsor type fit | VC partnership; grant body; corporate R&D |
| Examples Phase 1 | YC batch (12-week), Encode Club cohort, Buildspace Nights & Weekends (10-12 weeks) |

### §5.2 Mechanism
- **Pre-cohort (4-6 weeks):** application + selection; cohort assembly.
- **Week 1-2:** orientation; project commitment; mentor pairing.
- **Week 3-10:** sustained build; weekly demos; office hours; sponsor visits.
- **Week 11-12:** polish + Demo Day + post-event reach.
- **Post-cohort:** alumni network onboarding; funding/hiring discussions.

### §5.3 Strengths / weaknesses
- **+ deep transformation** of participants
- **+ shippable output** → sustained projects
- **+ alumni network compounding** → cohort effect
- **− high cost** ($200K+ floor)
- **− selection-pressure** narrows funnel
- **− attention scarcity** (12 weeks full commitment hard)

### §5.4 Jetix fit
**MEDIUM** для 2027+. Currently expensive для Phase 1; aligns с Master Workshop activation Year-2. Concept doc A §5.2 «year-rhythm» more aspirational; quarter-rhythm = stepping stone.

## §6 Rhythm 6 — Year (50-52 weeks)

### §6.1 Dimensions
| Dimension | Value |
|---|---|
| Duration | 50-52 weeks (full year) |
| Typical participants | 10-100 (very small, Master Workshop / apprenticeship-grade) |
| Mentor : participant ratio | 1:2 to 1:5 (master-apprentice) |
| Output type | Mature product / company / portfolio / dissertation-equivalent |
| FPF primitive load | Maximum (full FPF stack; Workshop discipline; long-running A.17 trajectory) |
| Optimal cohort | Master Workshop apprentices; multi-clan teams; senior researchers |
| Cost structure | $500K-5M+ (annual mentor + infra + stipends) |
| Sponsor type fit | Long-term partnership; multi-year corporate sponsor; foundation grant |
| Examples Phase 1 | Buildspace Nights & Weekends extended cohorts (some); annual NeurIPS-tied longitudinal competitions; KDD Cup annual; **clan-wars Year mode (Charter §clan-warfare)** |

### §6.2 Mechanism
- **Pre-event (3-6 months):** sponsor + cohort + Master Workshop assembly; project pre-scoping.
- **Q1:** orientation + project commitment + multi-clan formation.
- **Q2-Q3:** sustained build; quarterly reviews; mid-year checkpoint; multi-clan internal hackathons (recursive day-rhythm events).
- **Q4:** product readiness + year-end Demo Festival + multi-clan war final.

### §6.3 Strengths / weaknesses
- **+ deepest transformation** + alumni network maximally compounding
- **+ multi-clan support possible** → multi-rhythm nested
- **+ year-budget allows multi-event nesting** (day-rhythm internal events embedded)
- **− maximum cost** ($500K+ floor)
- **− organizational capacity heavy** → requires Master Workshop activated
- **− sponsor commitment year-long = scarcer**

### §6.4 Jetix fit
**LOW для 2026; HIGH для 2027-28.** Per concept doc A §7: «Year mode» = year-rhythm activation Q1 2027. Multi-clan war = year-rhythm canonical. Concept doc A §10 confirms «Year-1 floor: 10 → 1000 participants» reachable через multi-rhythm.

## §8 Rhythm dependencies (multi-rhythm nesting)

**Year-rhythm contains** quarterly checkpoints (∋ Q1, Q2, Q3, Q4 review events) ∋ monthly mid-quarter checkpoints ∋ weekly office hours ∋ internal weekend/day-rhythm mini-hackathons.

**Mathematical fractal:** 1 year-rhythm ≈ 4 quarter-rhythms ≈ 12 month-rhythms ≈ 50 week-rhythms ≈ 200+ day-rhythms (nested embedded).

**Implication:** Jetix Year-1 starting from day-rhythm можно ladder к year-rhythm by Y2 через building intermediate rhythms. Concept doc A §7 Activation Gantt = ladder pattern.
