## §1 5 candidate scopes — FPF mapping

### §1.1 Method-as-method (`U.MethodDescription`)

**Definition (F2):** Method = abstract description of information-processing pattern applied к systems (Ruslan voice text_012: «вот непосредственно метод системного мышления взгляда на мир подхода к жизни»). Type-level artefact; no runtime executor binding by itself.

**FPF mapping:**
- `U.MethodDescription` (canonical primitive) — K-6 = method documentation artefact
- `B.3 F-G-R` schema applies per-claim within method
- `A.6.B` dual-language (human-readable Russian voice + machine-traceable FPF refs)

**IP-1 compliance:** ✅ Method = abstract type; cannot self-modify because it has no runtime presence per se. Bindings к runtime executors (humans / Jetix substrate / specific agents) declared в `shared/schemas/executor-binding.yaml.template` at instance time only.

**Operational example:** K-6 itself = method-as-method instance (the *description* of the method, not the method's execution).

### §1.2 System-as-system (`A.1 U.System` holonic)

**Definition (F2):** Method applies to *systems*. Per text_012 §2.1 «ALL = systems». Per text_013 §2.16 systems = holonic (parts decompose; composite improvement emerges from parts improvement). Per text_014 §2.19 system consumes info about (a) own working, (b) interaction protocols, (c) rules of other systems.

**FPF mapping:**
- `A.1 U.System` (holonic) — every system is part-of larger + composed-of smaller
- `A.16 Work-as-process` — system = bounded process with input/output
- Foundation Part 7 (project lifecycle) — Jetix-as-system instantiates this lens
- Foundation Part 8 (health monitoring) — system self-knowledge metric (text_013 §2.11)

**IP-1 compliance:** ✅ System decomposition respects Role≠Executor. Role-types (manager, strategist, etc.) ≠ executor bindings (claude-opus-4-7, sonnet-4-6, etc.) per `shared/schemas/executor-binding.yaml.template`.

**Recursive self-target:** Jetix-as-system applies method to itself (recursive lifecycle 8-step per text_013 §2.18). This is the *recursive* — but NOT *autonomous self-modifying* — sense per Tier 2 rule 9.

**IP-1 compliance:** ✅ Method-as-artefact spreads as *pattern*; each recipient binds executors locally per their RUSLAN-LAYER analog. No central executor authority claim.

**R12 anti-extraction:** ✅ Description-spreading = openly published method (NOT proprietary). Fork-and-leave preserved by openness.
