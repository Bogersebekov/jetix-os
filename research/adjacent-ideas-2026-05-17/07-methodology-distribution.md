# Cluster 7 — Methodology distribution (FPF universal language test_002)

> **R1 surface-only.** Surfaces existing distribution models — relevant для FPF + Workshop + L1 collaboration scaling.

---

## §0 TL;DR (≤200 слов)

Поле «methodology distribution» имеет 5 проверенных моделей:

**(1) Book + certification body:** **GTD (David Allen 2001)** — book + David Allen Company + 90 countries partners + Train-The-Trainer model (3 extra days to certify) + 35 years anniversary. **TOC (Goldratt)** — covered Cluster 4 (3M books, 23 languages, TOCICO certification).

**(2) Book + viral pattern transfer:** **Christopher Alexander «A Pattern Language» (1977)** — building architecture book → 1987 Kent Beck + Ward Cunningham extracted «design patterns» concept → Gang of Four Design Patterns (1994) → WikiWikiWeb (Ward Cunningham, **first wiki ever**, Portland Pattern Repository). **Most influential cross-domain methodology transfer in computing history.** 40+ years compounding.

**(3) Industrial consulting model:** **Toyota Production System (Toyoda/Ohno, 1950s+)** — TPS as practice; Lean Manufacturing as global brand. Distribution: **internal Toyota training → external consultants → certifications + Six Sigma fusion**. «Just-In-Time + Jidoka» core. Applied across automotive / healthcare / software / aerospace.

**(4) Manifesto + viral certification:** Agile/Scrum (covered Cluster 4) — Sutherland/Schwaber + ScrumAlliance.org + PMI Disciplined Agile. Mass adoption + soul-loss (covered).

**(5) Free + LLM-substrate (recent):** **Anthropic's prompt engineering courses (Aug 2024)** — GitHub-hosted, free, Haiku-optimized for cost. 9-chapter tutorial + AWS Workshop version + Pluralsight partnership. **Karpathy LLM Wiki (April 2026) viral GitHub Gist** distribution.

**Adjacency к FPF (vision/01-test_002):** Pattern Language → Design Patterns trajectory = direct precedent для FPF universal-language-distribution. GTD Train-The-Trainer + Toyota consulting model = patterns для Jetix Workshop. Anthropic free-course-+-GitHub model = modern viral substrate.

## §1 Key works / movements / people

### §1.1 Book + Certification body model

**GTD — Getting Things Done (David Allen, 2001 book)** — David Allen Company (David + Kathryn Allen). **35 years methodology development distribution.** **Certified Trainers + GTD Coaches in 90 countries.** **Train-The-Trainer concept: 3 extra days to certify colleagues** (via Master Trainer). Global Partners model. Productivity-focused (not engineering — but methodology-distribution analog). [src: https://gettingthingsdone.com/about/; https://www.epicpeople.nl/en/training-performance/getting-things-done/, retrieved 2026-05-17]

**TOC — Theory of Constraints** — covered Cluster 4 (Goldratt 3M books, 23 languages, TOCICO).

**Scrum / Agile certification** — covered Cluster 4 (ScrumAlliance.org, PMI).

**Six Sigma / Lean Six Sigma** — Motorola origins; Belt certifications (Yellow/Green/Black/Master Black). Enormous industrial distribution.

### §1.2 Book + viral cross-domain transfer

**Christopher Alexander «A Pattern Language» (1977)** — building architecture + urban design + community livability. Authors: Alexander + Sara Ishikawa + Murray Silverstein (Center for Environmental Structure Berkeley) + Max Jacobson + Ingrid Fiksdahl-King + Shlomo Angel. **253 patterns** in book. [src: https://en.wikipedia.org/wiki/A_Pattern_Language, retrieved 2026-05-17]

**1987 Kent Beck + Ward Cunningham extracted pattern concept → programming.** **OOPSLA conference** — «jaws dropped in software engineering community». **Gang of Four — «Design Patterns» (1994)** — Erich Gamma, Richard Helm, Ralph Johnson, John Vlissides. **Cross-domain methodology transfer = compound effect.**

**Ward Cunningham's WikiWikiWeb (1995)** — **first wiki ever**. Built specifically as «archive and discussion web application for the Portland Pattern Repository.» Wikipedia (2001) descends. **Karpathy LLM Wiki (2026)** descends. Direct ancestor of Jetix wiki/ substrate. [src: https://www.designsystems.com/christopher-alexander-the-father-of-pattern-language/, retrieved 2026-05-17]

> Verbatim (Alexander): «A group of people who use a common pattern language can make a design together just as well as a single person can within his mind.»

**This is the strongest single adjacency** для FPF universal-language hypothesis (vision/01 test_002).

### §1.3 Industrial consulting model

**Toyota Production System (Toyoda Sakichi + Kiichiro Toyoda + Taiichi Ohno, 1950s+)** — manufacturing philosophy. **Just-In-Time + Jidoka** = 2 core principles. «Elimination of waste». **Distribution:**
- Internal Toyota training (decades of in-company culture)
- External consultants (since 1980s)
- Lean Manufacturing brand (James Womack et al.)
- Six Sigma fusion (mid-2000s)
- Toyota Lean Management (TLM) global teaching

**Toyota Process Consulting** (Toyota's own consulting arm); **Center for Lean Excellence**; **Structured LEAN**; **Shoppa's Material Handling** + many regional consulting firms. **70+ years sustained distribution.** [src: https://www.shoppasmaterialhandling.com/toyota-process-consulting/; https://centerforlean.com/toyota-production-system-certificate/, retrieved 2026-05-17]

**5-step consulting pattern:** Lean Assessment → identify waste → define opportunities → design solutions → implement.

## §4 Failure cases / lessons learned

**Pattern Language partial transfer.** Christopher Alexander's own subsequent work («The Nature of Order») less influential than 1977 book. **Lesson:** the **viral artifact** (right book at right moment for right audience — pre-OOPSLA conference) carried 50-year compound. Subsequent material can't carry same momentum. **Direct lesson:** Jetix needs **one breakthrough artifact** — vision/* set must produce 1-2 «Pattern Language»-grade artifacts.

**GTD vs «Inbox Zero» dilution.** GTD methodology survives but many «GTD-adjacent» productivity systems (Bullet Journal, etc.) absorbed and diluted mindshare. **Lesson:** strong brand + clear ownership of «GTD» term preserved 35-year coherence — но adjacent ecosystem grew larger.

**Toyota consultants quality variance.** «Lean» became diluted brand — many self-declared Lean consultants without TPS depth. **Lesson:** Train-The-Trainer scales but **quality control degrades over generations**. FPF certification design must address generational degradation.

**Six Sigma backlash 2010s+.** «Six Sigma killed innovation» critique (process-over-people). **Lesson:** methodology can over-impose; FPF must preserve creative space.

**Anthropic courses adoption uncertain.** Recent (Aug 2024) launch; mid-term adoption unclear. **Lesson:** free + GitHub-hosted is necessary but not sufficient for viral; need community vehicle (Karpathy Gist had Karpathy authority).

**Karpathy LLM Wiki very recent.** April 2026 → ~1 month of community formation by this report time. Sustained adoption uncertain. **Lesson:** GitHub Gist viral пик может быть short. Sustained distribution needs maintenance + community.

**Cynefin Foundation Methods Library underused.** Despite Snowden's framework value, library is sparse. **Lesson:** distribution requires **active curation + community-building, not just publishing** (Cynefin chose consulting path).

**Toyota Way published 2004 vs internal TPS practice gap.** Even «owner» of methodology can struggle to externalize practice into transmissible form. **Lesson:** tacit knowledge transfer is **lossy**. FPF needs structural mechanisms for tacit-→-explicit (e.g., apprenticeship Workshop format).

**Agile / Scrum mass adoption + soul-loss** — covered Cluster 4. Distribution success → quality decay.
