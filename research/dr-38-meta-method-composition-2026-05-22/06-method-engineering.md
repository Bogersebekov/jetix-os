## §2 Schedrovitsky's ММК — Moscow Methodology Circle (1954-1994)

### §2.1 Origin + intellectual programme

Georgy Petrovich Shedrovitsky (Г.П. Щедровицкий, 1929-1994) founded the Moscow Methodology Circle (ММК, Московский методологический кружок) in 1954 as an offshoot of the Moscow Logic Circle (МЛК). The Circle held weekly seminars и organisational-activity games (ОДИ — организационно-деятельностные игры) from 1979 onwards [src: E1 Shedrovitsky G.P. — *Избранные труды* 1995].

Central methodological-philosophical commitments:
1. **Activity (деятельность) as primary** — knowledge derives from goal-directed action, не от passive observation
2. **Method as first-class object** — methods можно изучать, проектировать, перепроектировать; methods are made / re-made through methodological work
3. **Reflexive position** — methodologist takes external position to systems being studied; methodology = выход в рефлексивную позицию
4. **Schema as primary tool** — thinking + composing through visual / formal schemas (Shedrovitsky schemas of activity, communication, thinking)
5. **Game-form (ОДИ) as transformation method** — group-activity games to transform organisational reality

### §2.2 Key ММК concepts that map to Ruslan

| ММК concept | Mapping to Ruslan |
|---|---|
| Деятельность (activity) | Ruslan's daily action substrate — methods serve action, not abstract knowledge |
| Метод как объект | The meta-method — explicitly treats methods as objects of study/composition |
| Рефлексивная позиция | Ruslan's audio_719 IS reflexive-position act — stepping outside his methods to articulate them |
| Схема | Method V2's diagrams + Foundation v1.0 architectural maps |
| ОДИ-game-form | Cycle pattern (Method V2 §M) ≈ ОДИ at personal scale; Cycle Wave A→B→C→D structure |
| Methodologist | Ruslan as methodologist of his own life — explicit own-method articulation |

**F3 insight:** Ruslan's audio_719 act («когда я выбирал этот метод, я какой выбрал и зафиксировал?» — claim 1) IS a Schedrovitsky-rooted methodological move. It's NOT random self-introspection — it's the canonical methodologist's first operation: «выйти в рефлексивную позицию по отношению к собственному методу». **This is what Schedrovitsky called the methodologist's home territory.**

This is the strongest single tradition mapping. Phase 3 noted Beer VSM + Levenchuk = strongest combined; Phase 6 now refines: **Schedrovitsky's ММК + Levenchuk methodology + Beer VSM together comprise the direct intellectual lineage of Ruslan's meta-method.**

[src: E1 Шедровицкий Г.П. — *Избранные труды* 1995 Школа Культурной Политики; E2 Шедровицкий П.Г. — лекции по методологии 1998-2020 various]

### §2.3 The two Shedrovitskys

Petr Georgievich Shedrovitsky (П.Г. Щедровицкий, b. 1958, son of Г.П.) extends the methodological tradition into modern contexts: strategising / development / mass-mind / industrial-policy. His public lectures (1998-2020) translate ММК vocabulary for contemporary use [src: E2].

For Ruslan's substrate: P.G. Shedrovitsky's lecture series on «методология стратегирования» directly informs Levenchuk's methodology framework. The line Schedrovitsky → Levenchuk → Ruslan is direct intellectual descent.

### §2.4 The «schema thinking» discipline

Shedrovitsky's school developed elaborate visual schemas: схема акта деятельности (activity-act schema), схема мыследеятельности (thinking-activity schema), схема организационной коммуникации (organisational communication schema). These are not illustrations — они являются the actual thinking tools [src: E1 + Shedrovitsky G.P. — *Schema of activity* in Selected Works].

Cross-link к Method V2 mermaid diagrams (10+ in current state), к Foundation Architecture diagrams (Bundle 1-5), к DR-38 Phase 11 diagram pass (12-18 mermaid mandate). Ruslan's diagram-discipline = ММК schema-thinking inherited.

## §3 Levenchuk's contemporary methodology (2023-2025)

### §3.1 The three-book canon

Anatoly Levenchuk (А. Левенчук, b. 1958) wrote three foundational texts that translate ММК / Western method-engineering для contemporary Russian-language readers:

- **«Системное мышление»** Том 1+2 (2024 — 6th edition; first 2015) [src: B8 Levenchuk 2024] — systems-thinking textbook, 1200+ pages, used in Russian engineering / leadership education
- **«Методология»** (2025) [src: B9 Levenchuk 2025] — explicit method-engineering text, integrates ММК + OMG Essence + Brinkkemper SME
- **«Инженерия личности»** (2023) [src: B10 Levenchuk 2023] — personal-method composition + life-as-engineering object

### §3.2 «Метод как объект 1-го класса» chapter

Levenchuk «Методология» Гл. 4 «Метод как объект 1-го класса» («Method as first-class object») advances [src: B9 Levenchuk 2025 ch. 4]:

1. Method has **internal structure**: alphas (state-evolving entities) + activity spaces (work areas) + competencies (skills required) + work products (artefacts) + patterns (recurrent structures)
2. Methods can be **decomposed + recomposed** — analogous to functional programming composition
3. Method composition discipline:
   - Methodologist identifies need (situation → required behaviour gap)
   - Selects from method-arsenal (or constructs new)
   - Assembles via integration discipline
   - Verifies composition against fitness criteria
   - Tests in practice → iterates

This is Brinkkemper Situational Method Engineering translated к Russian-language pedagogy. Levenchuk borrows OMG Essence Kernel structure verbatim (alphas etc.) [src: cross-ref B9 + E3 OMG Essence].

## §4 OMG Essence Kernel — the Western parallel (2018)

### §4.1 The Kernel structure

OMG (Object Management Group) Essence — Kernel and Language for Software Engineering Methods — is the Western/industry-standard method-engineering substrate, currently at v1.2 (2018) [src: E3 OMG Essence v1.2 2018]:

**Kernel structure** consists of:

| Essence Element | Description |
|---|---|
| **Alpha** | Essential thing whose state evolves; abstract progress-marker (e.g., «Stakeholder», «Requirements», «Software System») |
| **Alpha state** | Defined progress level for an alpha (e.g., «Recognised», «Represented», «Involved», «In Agreement», «Satisfied for Deployment») |
| **Activity space** | Required area of work (e.g., «Understand the Requirements», «Shape the System») |
| **Activity** | Specific work approach within an activity space |
| **Competency** | Required skill (e.g., «Analysis», «Development», «Testing») |
| **Work product** | Artefact produced (document / code / test) |
| **Pattern** | Recurrent structure in method-application |

Essence Kernel defines 7 core alphas + 15 activity spaces + 6 competencies — enough to describe any software development method as composition of these elements.

[src: E3 OMG Essence — *Kernel and Language for Software Engineering Methods* v1.2 2018 Object Management Group; companion: ISO/IEC 24744:2014]

### §4.3 SEMAT — the originating project

OMG Essence was the standardised output of the SEMAT (Software Engineering Method and Theory) initiative — Ivar Jacobson + Bertrand Meyer + Richard Soley founded в 2010 to combat «methodology wars» (RUP vs XP vs Scrum vs Kanban vs etc.). The thesis: instead of debating which method, articulate the *common kernel* + use it to describe any method [src: E4 Jacobson + Spence + Bittner 2013].

«The Essence of Software Engineering» (2013) [E4] is the canonical SEMAT introduction. Key claim: «every software project executes some method, whether explicit or implicit; making the method explicit + composed-from-kernel improves outcomes».

**F3 insight:** SEMAT's thesis IS Ruslan's audio_719 act: make the implicit method *explicit* via decomposition + recomposition в standard vocabulary. Ruslan's «не открытие, личный метод» framing aligns w/ SEMAT's «every project has a method» — making it explicit IS the contribution.

[src: E4 Jacobson I. + Spence I. + Bittner K. — "The Essence of Software Engineering: Applying the SEMAT Kernel" 2013 Addison-Wesley]

## §5 Brinkkemper — Situational Method Engineering (1996)

### §5.1 The SME thesis

Sjaak Brinkkemper's "Method engineering: engineering of information systems development methods and tools" (1996 *Information and Software Technology* journal article) advances **Situational Method Engineering (SME)**: instead of using a one-size-fits-all method, construct a method *tailored* to the specific project situation [src: E5 Brinkkemper 1996].

SME's three contributions:
1. **Method fragments** — modular method-components that can be assembled
2. **Method base** — repository of fragments
3. **Method assembly process** — discipline для constructing situational methods from fragments

Henderson-Sellers + Ralyté (2014) extend this в "Situational Method Engineering" — comprehensive 4-decade synthesis [src: E7 Henderson-Sellers + Ralyté 2014].

### §5.2 Mapping

| SME concept | Ruslan substrate |
|---|---|
| Method fragments | The 16 components (each a fragment) |
| Method base | Method-arsenal (claim 2: «куча маленьких методов») |
| Assembly process | The meta-method itself (claim 2: «метод выбора методов») |
| Situational tailoring | C9 mechanism-literacy + C4 focus = situation-recognition discipline |
| Fitness criteria | C8 «развитие системы» (evolve criteria) + C6 honesty (honest fit-evaluation) |

## §6 ISO/IEC 24744:2014 — the formal standard

### §6.1 The metamodel

ISO/IEC 24744:2014 «Software Engineering — Metamodel for Development Methodologies» is the international standard formalising what Essence + SEMAT + SME describe [src: E6 ISO/IEC 24744:2014].

Key structures:
- **Methodology** = top-level container
- **Process model** = activity flow
- **Work product model** = artefacts produced
- **Producer model** = roles + competencies
- **Endeavour** = specific instance of methodology applied

Compositional discipline: methodology = composition of these models; instance (endeavour) = methodology applied к specific situation.

## §7 Kuhn paradigm composition adjacency (1962)

### §7.1 Paradigm + paradigm shift

Thomas Kuhn's "The Structure of Scientific Revolutions" (1962) advances: scientific progress proceeds через paradigm shifts (revolutionary changes в the basic conceptual framework). Normal-science = puzzle-solving within paradigm; revolutionary-science = paradigm-replacement [src: G9 Kuhn 1962].

| Kuhn concept | Ruslan substrate |
|---|---|

| Paradigm shift | Adopting new component / dropping old (C8 «развитие системы») |
| Incommensurability | Different domains' methods may not directly translate (DDD bounded contexts) |

C5 «отсечение неважного» + C8 «развитие системы» = anti-paradigm-rigidity discipline. **The meta-method is Kuhn-self-aware.**
