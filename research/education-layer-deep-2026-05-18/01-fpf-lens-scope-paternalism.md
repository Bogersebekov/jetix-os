## §1 Scope 1 — Education-as-method (U.MethodDescription + A.3.1 + U.Method)

### §1.1 FPF mapping

- **A.3.1 Method** (primary): системное мышление curriculum = canonical method для cognitive scaffolding с specific learner + mentor + practice protocols.
- **U.MethodDescription** (primary): curriculum canonical doc (per Tier 1 / Tier 2 / Tier 3 / Tier 4 module specs).
- **A.3.2 Practice** (instance per cohort): each cohort = practice instantiation of method.
- **U.Method**: methodology primitive applied across Workshop curriculum versions.

### §1.2 Implications для Jetix

- Method library: reusable templates per module; brigadier dispatches «execute Module-N method instance».
- Method evolution: capture lessons → patch method canonical → next cohort incorporates (Foundation Part 5 Compound Learning hook).
- F-G-R applied к method: F2 surface для new module variant, F3 при single successful cohort run, F5 при 5+ successful runs.
- Multi-pedagogy: same method instantiable as lecture / case study / Socratic / hackathon / mentor pairing.

### §1.3 Cross-precedent fit

- **High fit:** Meadows pedagogy (Sustainability Institute) [src: research/deepening-2026-05-18/05].
- **High fit:** Karpathy LLM101n (curriculum canonical methodology) [src: research/deepening-2026-05-18/09].
- **High fit:** ШСМ Anatoly Levenchuk (30-year methodology accumulation) [src: vision/08].
- **Med fit:** NASA APPEL (curriculum + simulation training).

### §1.4 Per-scope FPF load

A.3.1 method + U.MethodDescription + A.3.2 practice + U.Method = medium primitive load. Compatible с Foundation Part 5 (compound learning) + Part 3 (methodology library).

### §1.5 Paternalism boundary

- Method = opt-in instantiation; no enforced enrolment.
- Multiple pedagogical paths preserved (lecture / case / Socratic / build-to-understand).
- Method evolution open к dissent: phil critic seat in curriculum review per Phase 5 §7.4.

## §2 Scope 2 — Curriculum-as-artefact (A.1 U.System holonic + A.15)

### §2.1 FPF mapping

- **A.1 System** (primary): Workshop curriculum = systemic entity Phase 0 inventory O-27 candidate. Holonic decomposition: curriculum → tiers → modules → lessons → exercises.
- **U.System** (holonic): each Tier = sub-system; curriculum = super-system holding Tiers; lesson = sub-sub-system.
- **A.15 Work-as-artefact**: per-trainee artefacts (project portfolio / capstone / hackathon submission).
- **A.0 Substrate**: Cloud Cowork compute + FPF protocol + Workshop platform.

### §2.2 Implications для Jetix

- Curriculum = stable substrate; cohorts = ephemeral instances.
- Version discipline: curriculum v1.0 → v2.0; cohorts tagged по curriculum version (compound learning Foundation Part 5).
- Health monitoring (Part 8): curriculum-level SLIs (cohort completion rate, specialist emergence rate, mentor density, dropout-with-cause analysis).
- A.15 separation: trainee owns artefact portfolio; curriculum owned by Jetix platform.

### §2.3 Cross-precedent fit

- **High fit:** ШАД (Yandex elite training — curriculum-as-artefact с per-cohort tracking).
- **High fit:** Y Combinator Startup School (modular curriculum + cohort engagement).
- **High fit:** MIT OCW (curriculum-as-artefact open-source) [pattern reference].
- **Med fit:** Pattern Language books (Alexander 1977 — curriculum-as-artefact для architecture education).

### §2.4 Per-scope FPF load

A.1 + U.System + A.15 + A.0 = high primitive load (12+ Foundation parts touched per cycle).

### §2.5 Paternalism boundary

- Curriculum artefact open-source (transparency = anti-paternalism gate); learners can fork curriculum (R12 fork-and-leave).
- Diversity quotas per module source — no single methodological monoculture.
- Cultural / linguistic / philosophical compatibility surface explicit (NASA framework = secular baseline; non-conflict с most worldviews).

## §3 Scope 3 — Master-Apprentice-as-role-pair (A.2 Role + A.16 work-as-process)

### §3.1 FPF mapping

- **A.2 Role** (primary): Apprentice / Journeyman / Master / Grandmaster (4-role typology per Phase 4 §6.1).
- **A.16 Work-as-process**: apprenticeship progression process (Apprentice → Journeyman → Master).
- **A.17 Trajectory**: per-trainee multi-year journey.
- **U.SpeechAct**: teaching speech-acts («I demonstrate», «I observe», «I correct», «I sign off»).
- **U.PromiseContent**: mentor-mentee commitment (mentor promises N hours/week; mentee promises portfolio output).

### §3.2 Implications для Jetix

- Role typology = abstract role-types (per IP-1 Role≠Executor); executor bindings RUSLAN-LAYER (specific person fills Apprentice role).
- Progressive responsibility per role; quality predicate enforced by senior role (Master sign-off Apprentice → Journeyman transition).
- Per-role compensation discipline: stipend (Apprentice) → paid mentoring (Journeyman) → salary+equity capped by Mondragón ratio (Master) per R12.
- Hackathon = primary Master-Apprentice activation vehicle (cross-link Hackathon Platform deep).

### §3.3 Cross-precedent fit

- **High fit:** ШСМ Anatoly Levenchuk 30-year apprenticeship model.
- **High fit:** Medieval guild apprenticeship (1100-1800 AD) [historical pattern; failure-mode-aware].
- **High fit:** Karpathy mentor lineage (Hinton → Karpathy → next-gen ML researchers).
- **Med fit:** NASA APPEL Master-Apprentice (program manager mentoring).
- **Med fit:** Toyota Production System Sensei-Deshi (mentor pattern) [src: research/deepening-2026-05-18/14].

### §3.4 Per-scope FPF load

A.2 + A.16 + A.17 + U.SpeechAct + U.PromiseContent = medium-high. Strong fit для Part 4 (role taxonomy) + Part 7 (project lifecycle).

### §3.5 Paternalism boundary

- Master-Apprentice = voluntary entry per side (Master accepts; Apprentice consents).
- Quality predicate transparent (Apprentice knows progression criteria up front).
- Fork-and-leave preserved at role transition (Apprentice can exit без penalty; Journeyman can fork to alternative master).
- Medieval guild failure modes (rigidity / nepotism / monopoly capture) actively mitigated via open-source curriculum + multi-master choice + anonymous feedback.

## §4 Scope 4 — U.Episteme as transmission primitive (knowledge transmission + U.SpeechAct)

### §4.1 FPF mapping

- **U.Episteme** (primary): knowledge transmission FPF primitive (abstract role-type per concept doc E §2).
- **U.SpeechAct** (primary): teaching speech-acts — explain / demonstrate / question / correct / acknowledge.
- **U.Belief** (Apprentice mental model formation): per-lesson belief-state update.
- **A.11 Commons** (knowledge commons): cohort = commons; alumni network = expanding commons.
- **E.7 Civic norms** (R12 anti-extraction applied к knowledge transmission).

### §4.2 Implications для Jetix

- Knowledge transmission ≠ knowledge transfer (transmission preserves nuance + context; transfer flattens).
- Speech-act discipline: «I-demonstrate» vs «I-explain» vs «I-question» — different cognitive loads на trainee.
- Tacit ↔ explicit conversion (per TPS Hansei mechanism research/deepening-2026-05-18/14): apprenticeship surfaces tacit; documentation captures explicit; new cohort re-instantiates tacit via mentor.
- Commons evolution: alumni feedback → curriculum patch → next-cohort knowledge baseline lifted.

### §4.3 Cross-precedent fit

- **High fit:** Pattern Language (Alexander → Cunningham → Karpathy lineage — knowledge transmission через patterns) [src: research/deepening-2026-05-18/05].
- **High fit:** Engelbart H-LAM/T Training element (1962) — co-evolution Methodology / Language / Artefacts / Training [src: research/deepening-2026-05-18/04].
- **High fit:** TPS Hansei pedagogy (tacit→explicit reflection cycle) [src: research/deepening-2026-05-18/14].
- **Med fit:** Stafford Beer VSM teaching (cybernetic knowledge transmission).

### §4.4 Per-scope FPF load

U.Episteme + U.SpeechAct + U.Belief + A.11 + E.7 = medium. Cross-cuts с Part 3 (knowledge base) + Part 5 (compound learning).

### §4.5 Paternalism boundary

- Transmission ≠ imposition: Apprentice can disagree, fork, leave (per R12).
- Speech-act «I-correct» bounded by phil critic dissent preservation: correction surfaces alternative interpretations rather than asserting singular truth.
- Knowledge commons open к contribution from non-master roles (Apprentice can surface improvements; merit-based not seniority-based).
