## §1 R12-риск растёт со слоем (карта)

| Слой | R12-риск | Главная зона риска | Кто следит |
|---|---|---|---|
| **L1 Personal** | 🟢 НИЗКИЙ | только privacy / data export (один человек) | сам человек |
| **L2 Personal Expanded** | 🟢🟡 НИЗ-СРЕДН | аналитика может обнажить паттерны (self-surveillance) | сам человек |

**Сквозной закон (Platform Lifecycle):** защита должна расти **быстрее** системы. Слой выше →
больше R12-механик. L1 = почти нечего защищать; L4 = максимум защит (или нейтральный generic base).

### §2.2 Layer 2 — sweep

Те же + новая зона: **аналитика обнажает паттерны** (self-surveillance — человек видит свои
красные зоны/выгорание). Риск: аналитика становится инструментом самобичевания.

**L2 anti-patterns:** ❌ analytics как «приговор» (red zones → вина) → mitigation: тон Reviews

**Вердикт:** 🟢🟡 — приватность сохранена (данные не текут наверх), но тон аналитики важен.

**L3 anti-patterns (по экспертам AUTO-FIRE):**
- *recruitment-dynamics:* ❌ клятвы верности / спаситель-фигура / «мы против них» → mitigation:
  выход озвучен до входа, ротация ролей, нет founder-поклонения.
- *gamification:* ❌ Schelling-подсветка превращается в FOMO («успей вписаться») → mitigation:
  лимит 3-5 матчей, координация ≠ принуждение.
- *nlp:* ❌ язык «продай навыки / наняться» → mitigation: «дать команде / вписаться».
- *propaganda:* ❌ массовый фрейм «элитарный клуб избранных» → mitigation: Welcome-frame
  «все подтягиваются» (mass paradigm shift R12-aligned, per Education Paradigm O-184).
- *influence-ethics:* ❌ Inv-Cap/Mentor концентрация власти → mitigation: 5:1 cap, 12-мес потолок,
  fork-leave, авто-этик-эксперт.

**Вердикт L4:** 🔴 высокий риск, но управляем: generic base нейтрален (риск = выбор бизнеса);
overlay добавляет строгие защиты; анти-concentration guard встроен даже в generic.

## §3 Cross-layer R12: fork-and-leave preserved all the way up

Главный сквозной инвариант: право уйти со своим работает на всех 4 слоях и **не может быть
отключено вышестоящим слоем**.

```
L1 уход → все личные данные (отдельный воркспейс)
   ↑ не трогается при
L3 fork-and-leave → + доля + 30-day + Ledger строка
   ↑ не трогается при
L4 founder exit → бизнес продолжается; Scale: founder = 1 из многих
```

**Архитектурная гарантия (не «обещание»):** раздельные воркспейсы делают `fork_prevention_attempt`
технически невозможным — нельзя «забрать» данные нижнего слоя у уходящего. Это R12 на уровне
структуры, а не правил.

## §5 R12-overlay vs generic (мета-уровень)

Ключевой R12-инсайт всей архитектуры: **навязывать R12 всем = само по себе анти-R12** (идеологический
lock-in). Поэтому:
- **Generic Layer 4 base = нейтрален** — бизнес может быть любым (капиталистический / кооперативный).
- **R12 = opt-in overlay** — бизнес сам выбирает кооперативную модель.
- Это **мета-R12:** не запираем даже в «правильную» (по нашему мнению) модель. Свобода форкнуть
  generic base без R12 = высшая форма anti-extraction.

(philosophy-expert lens: это разрешает потенциальное противоречие «мы за свободу, но навязываем
кооператив» — generic нейтрален, ценности = выбор пользователя.)
