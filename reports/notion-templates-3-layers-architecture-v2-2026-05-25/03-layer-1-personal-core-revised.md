# Phase 2 — 🟢 Layer 1 Personal Core (REVISED, упрощённый)

> **Что в этой фазе.** Берём Layer 1 из v1 и **режем до минимума**. В baseline остаётся
> только то, что реально нужно человеку в первый день. Всё продвинутое — вниз, в раздел
> «что можно ещё добавить» (это инструкция, а не живые поля).
>
> **Что нового против v1:** (1) Daily Log усилен — цель дня, реально сделано, deep work,
> тип дня, нормальный шаблон страницы дня + wrap-of-day через Claude Code; (2) появился
> **Strategic под-раздел** (Vision + Goals как документ, не просто база); (3) добавлены
> **Weekly + Monthly шаблоны ревью**; (4) Knowledge получил поле «посмотреть позже».
>
> **Главный принцип фазы — упрощение.** Меньше полей в baseline = быстрее форк, меньше
> страха у новичка. Сложное никуда не делось — оно ждёт в sidebar, «добавь когда дорастёшь».

## §0 Что входит в Layer 1 (минимальный состав)

Layer 1 — это **полностью самостоятельная** личная система. Ей не нужны ни команда, ни
бизнес-слой. Человек форкает её один, заполняет за полчаса и пользуется каждый день.

| # | База | Статус | Назначение в одну строку |
|---|---|---|---|
| 1 | 📅 **Daily Log** | core | день = одна запись: цель, что сделал, энергия, связи |
| 2 | 🚀 **Projects** | core | личные проекты с детектом застрявших |
| 3 | ✅ **Tasks** | core | действия (в самом простом варианте — чек-лист внутри Projects) |
| 4 | 💡 **Ideas** | core | банк идей / inbox для сырых мыслей |
| 5 | 🤝 **Contacts** | core | личные люди — облегчённый CRM |
| 6 | 📚 **Knowledge** | core | источники + факты + «посмотреть позже» |
| 7 | ❓ **Hypotheses** | core-lite | «что хочу проверить» (нужен не всем — baseline минимальный) |
| 8 | ❤️ **Life Pulse** | core-lite | энергия / сон / настроение (мини-замер) |
| 9 | 🔁 **Habits** | core-lite | привычки: что делаю регулярно + отметка дня + streak |
| 10 | 💰 **Финансы** | opt-in | личный учёт денег (по желанию, приватно) |

Плюс **Strategic под-раздел** (это не база, а пара страниц — см. §10) и **2 шаблона
ревью** (неделя + месяц — см. §11).

**Сквозное правило для всех баз.** Каждая строка = один `.md` файл на диске (это правда),
свойства Notion = поля шапки. При конфликте — прав файл. Голос → только черновик. Claude
Code предлагает, ты решаешь. Это пять правил конституции, которые выживают в личной версии.

## §1 📅 Daily Log (REVISED — главная база)

**Назначение.** День = одна запись. Утром поставил цель, в течение дня дописываешь (в том
числе голосом), вечером — что реально сделал. Это главная точка входа во всю систему: почти
всё «касается дня».

В v1 тут было 16 полей с формулами дней недели и номеров недель. **Режем.** День недели и
номер недели Notion посчитает сам в любой момент — держать их отдельными полями незачем.
Зато добавляем то, чего реально не хватало: цель дня, что вышло на самом деле, сколько
было глубокой работы, и какой это был день по характеру.

**Схема полей (baseline):**

| Поле | Тип | Зачем |
|---|---|---|
| Date | Title (дата) | заголовок дня, `YYYY-MM-DD` |
| Цель дня | Text | что хочу сделать сегодня (ставлю утром) |
| Реально сделано за день | Text | что вышло на самом деле (пишу вечером) |
| Energy | Number (1–5) | насколько был в ресурсе |
| Deep work minutes | Number | минут реальной глубокой работы за день |
| Day type | Select: production / development / recovery / orientation | какой это был день по характеру |
| Linked Project | Relation → Projects | какого проекта касался |
| People | Relation → Contacts | кого упоминал / с кем виделся |
| Hypothesis | Relation → Hypotheses | какую гипотезу двигал |
| Ideas | Relation → Ideas | какие идеи родил |
| Habits done today | Relation → Habits | какие привычки выполнил сегодня |

**Про связку с привычками.** Поле «Habits done today» — это relation на базу Habits (§8.5).
Вечером (или по ходу дня) отмечаешь, какие привычки сегодня сделал — просто линкуешь их. Это
минимальный baseline трекинга: не нужна отдельная запись на каждую привычку, день сам помнит,
что ты выполнил. Streak-подсчёт и хитмапы — из sidebar (§13), когда захочется аналитики.

**Что выкинули из v1:** Day of week, Week number (посчитаются формулой если надо),
Gratitude, Tomorrow trigger, Day pulse, Source mode, Draft? — это всё уехало либо в шаблон
страницы дня (как секции текста), либо в sidebar. Baseline — только поля, которые реально
структурируют день.

**Про Day type — зачем select из четырёх.** Это маленькая, но важная штука. Не каждый день
должен быть «продакшн». Бывают дни разработки (учусь, проектирую), дни восстановления
(отдых — и это нормально), дни ориентации (думаю, куда вообще иду). Когда в конце месяца
смотришь, из чего состоял месяц, — видно сразу: одни сплошные production без recovery = путь
к выгоранию. Это связка с Life Pulse и с кирпичом «здоровье человека».

### §1.1 Шаблон страницы дня (важно — описываю подробно)

Каждый день в Notion — это не просто строка в таблице, а **полноценная страница**. Когда
открываешь её, видишь готовую структуру (template применяется автоматически при создании дня):

```
# 2026-05-25

## 🎯 Цель дня
(одна-две строки: что хочу сделать сегодня)

## 🧭 Точка А → Точка Б (по шагам)
Где я сейчас по главной задаче дня:  ...
Куда хочу прийти к вечеру:           ...
Шаги:
  1. ...
  2. ...
  3. ...

## ✍️ Записи дня
(дописываю в течение дня — руками или голосом-черновиком)

## 📄 Черновики и заметки
(тут живут нативные Notion-подстраницы: набросок письма,
 черновик статьи, схема на салфетке — всё, что родилось в этот день)

## 🔁 Привычки сегодня
(чек-лист привычек дня — отмечаю выполненные; связано с базой Habits)

## ✅ Реально сделано за день
(вечером — что вышло на самом деле; сравниваю с целью утра)
```

Ключевая идея — **«точка А → точка Б по шагам» прямо в дне**. Это микро-версия большой
стратегии: не «куда я иду по жизни», а «куда я иду сегодня к вечеру и какими шагами». Утром
заполнил — день получил направление. Вечером сравнил цель и «реально сделано» — увидел
разрыв (и это нормально, разрыв = материал для рефлексии, а не повод себя ругать).

**Про подстраницы-черновики.** Notion умеет вкладывать страницы в страницы. Раздел «Черновики
и заметки» — это место, где можно прямо внутри дня создать подстраницу: набросок текста,
черновик письма, схему. Они живут внутри дня, не засоряют другие базы. Если черновик дорос
до чего-то серьёзного — переносишь его в Projects или Ideas вручную. Пока сырой — пусть лежит
в дне, где родился.

### §1.2 Wrap-of-day через Claude Code (важно — описываю как работает)

Это новый механизм, которого в v1 не было в явном виде. Суть: **в конце дня Claude Code
читает запись дня и помогает её осмыслить — но ничего не решает сам.**

**Как это работает по шагам (концептуально):**

1. Вечером ты говоришь Claude Code что-то вроде «заверши день» (или это запускается командой).
2. Claude Code **читает** запись сегодняшнего дня: цель утра, записи, что сделано, какие
   проекты/люди/гипотезы упоминались, сколько было deep work.
3. Claude Code **делает анализ и предлагает**, но как **черновик**:
   - «Ты ставил цель X, а сделал Y — разрыв вот тут, возможно стоит...»
   - «Сегодня ты три раза касался проекта Z — может, повысить ему приоритет?»
   - «Проект W ты не трогал 12 дней — скоро всплывёт как застрявший»
   - «Из записей дня вытащил идею про ... — занести в Ideas как черновик?»
   - «Энергия 2 второй день подряд — Life Pulse в красной зоне, обрати внимание»
4. **СТОП.** Claude Code показывает свои предложения и **ждёт**. Ты читаешь, какие-то
   принимаешь (ack), какие-то отклоняешь.
5. Только после твоего ack изменения попадают в боевые записи.

**Почему именно так — DRAFT-only дисциплина.** Это та же дисциплина, что и в голосовом
пайплайне: Claude Code **никогда не перезаписывает боевые записи автоматически**. Он читает,
анализирует, предлагает — но финальное «да» всегда за человеком. Если бы AI сам менял проекты
и приоритеты по итогам дня — ты бы потерял контроль и доверие к системе. Поэтому wrap-of-day
= «умный помощник, который подытоживает день и подкидывает варианты», а не «автопилот».

Это прямое следствие правил: AI не пишет стратегию, AI по умолчанию не делает действий сам,
голос/анализ → только черновик. Wrap-of-day — самое полезное место, где эти правила работают
на тебя ежедневно.

**Views для Daily Log:** (1) **Today** — фильтр `Date = сегодня`, открывается как страница;
(2) **This week** — последние 7 дней; (3) **Черновики на разбор** — то, что Claude Code или
голос предложили, ждёт ack.

## §2 🚀 Projects (упрощённый baseline)

**Назначение.** Проект = тип + статус + чекпоинт + детект застрявших. Активный проект, до
которого не дотрагивался >14 дней, всплывает красным.

В v1 было 19 полей. Режем до минимума — всё остальное в sidebar.

| Поле | Тип | Зачем |
|---|---|---|
| Name | Title | название проекта |
| Type | Select: consulting / research / product / bets / personal | какого рода проект |
| Status | Select: idea / active / paused / done / archived | где он сейчас |
| Checkpoint | Select: start / in-progress / done | 3 чекпоинта (вместо 4 строгих гейтов) |
| Priority | Select: P1 / P2 / P3 / P4 | насколько важен |
| Last touched | Date (или rollup из Daily Log) | когда касался в последний раз |
| Linked goals | Relation → Goals | какой цели служит |
| Notes | Text | заметки |

**Детект застрявших** оставляем — это одна из самых полезных вещей. Если `Status = active`,
а `Last touched` старше 14 дней → проект подсвечивается. Реализуется простой формулой; в
baseline можно держать как ручную пометку, формулу — добавить из sidebar.

**Views:** Active (активные по приоритету), Board by checkpoint (канбан), Stuck (застрявшие).

## §3 ✅ Tasks (упрощённый baseline)

**Назначение.** Атомарные действия. В самом простом варианте Tasks-как-базы **нет вообще** —
задачи живут чек-листом прямо внутри страницы проекта. Отдельная база нужна, только когда
задач становится много.

| Поле | Тип | Зачем |
|---|---|---|
| Name | Title | что сделать |
| Project | Relation → Projects | к какому проекту |
| Status | Select: todo / doing / done | где задача |
| Priority | Select: now / next / later | насколько срочно |
| Due | Date | дедлайн (если есть) |

**Views:** Today (now + просроченное), By project (board), Done.

**Совет на старте:** новичку отдельная база Tasks не нужна. Начни с чек-листов в Projects;
заведёшь Tasks, когда реально упрёшься.

## §4 💡 Ideas (упрощённый baseline)

**Назначение.** Банк идей и inbox-корзина. Сырая мысль падает сюда; дозревает — промоутишь

| Поле | Тип | Зачем |

| Maturity | Select: C-raw / B-shaping / A-ready | насколько созрела |
| Domain | Multi-select: business / tech / life / research | к какой области |
| Promote to | Select: — / project / hypothesis | куда вырастить |
| Note | Text | подробности |

**Views:** Inbox (сырые C-raw), Ready to promote (зрелые A-ready), By domain.

## §5 🤝 Contacts (упрощённый baseline)

**Назначение.** Личные люди — облегчённый CRM. Кого знаю, когда последний раз общались,
что предлагаю / что могу попросить.

| Поле | Тип | Зачем |
|---|---|---|
| Name | Title | имя |
| Type | Select: family / friend / colleague / mentor / client | кто это для меня |
| Status | Select: active / warm / cold / paused | состояние отношений |
| Last touch | Date | когда общались |
| What I offer | Text | что могу дать |
| What I ask | Text | что могу попросить |

**Детект «пора связаться»** — если `Last touch` старше 30 дней у активного контакта,
подсвечивается. Формула — из sidebar; в baseline достаточно отсортировать по Last touch.

**Views:** Active, Reconnect (давно не общались), By type.

## §6 📚 Knowledge (REVISED — добавлен «посмотреть позже»)

**Назначение.** Личная вики: источники (книги/статьи/видео), главные выводы, проверенные
факты. В baseline — одна база, не три.

| Поле | Тип | Зачем |
|---|---|---|
| Name | Title | название источника / факта |
| Kind | Select: source / claim / concept | что это |
| Status | Select: queued / reading / read | стадия (для источников) |
| **Saved-for-later** | Checkbox | «посмотреть позже» — отложил, вернусь |
| Key takeaway | Text | главный вывод одной фразой |
| Domain | Multi-select | к какой области |

**Что нового — поле «Saved-for-later».** Это маленькое, но честное решение. Когда листаешь
ленту/ютуб/статьи, постоянно натыкаешься на «о, интересно, посмотрю потом». Раньше это
терялось. Теперь — галочка «посмотреть позже» + отдельный вид-фильтр «Посмотреть позже».
Раз в неделю заходишь в этот вид и разбираешь накопленное: что-то реально читаешь (Status →
reading), что-то честно выкидываешь. Это анти-свалка: отложенное не пропадает, но и не висит
вечным грузом.

**Views:** Reading queue (в очереди + читаю), **Посмотреть позже** (`Saved-for-later = ✓`),
By domain.

## §7 ❓ Hypotheses (упрощённый baseline — для тех, кому надо)

**Назначение.** «Что хочу проверить». Гипотеза одной проверяемой фразой + как пойму, что
подтвердилась + что докажет, что неверна. Это сердце познания для тех, кто работает по
методу гипотез — но **нужно не всем**, поэтому baseline минимальный (Bayesian-lite, без
тяжёлой машинерии вероятностей).

| Поле | Тип | Зачем |
|---|---|---|
| Statement | Title | гипотеза проверяемой фразой |
| Confirm criterion | Text | как пойму, что подтвердилась |
| Refute criterion | Text | что докажет, что неверна |
| Status | Select: open / testing / confirmed / refuted | где гипотеза |
| Confidence | Select: low / mid / high | насколько верю сейчас |
| Linked projects | Relation → Projects | где проверяю |

**Bayesian-lite — что это значит на пальцах.** Не нужно считать вероятности в процентах.
Достаточно поля Confidence (низкая / средняя / высокая) — это и есть «облегчённый байес».
Появилось доказательство → подвинул уверенность. Кто хочет настоящие числа и обновление
веры по Байесу — это в sidebar, «добавь когда дорастёшь».

**Views:** Open, Testing now, Closed (подтверждённые + опровергнутые).

**Если ты не методолог** — можешь вообще не заводить эту базу на старте. Гипотезы добавишь,
когда поймёшь, что хочешь проверять мир системно.

## §8 ❤️ Life Pulse (упрощённый baseline)

**Назначение.** Ежедневный мини-замер себя: энергия, сон, настроение. Ловит ранние сигналы
выгорания. Это кирпич «здоровье» с переключённым объектом: следим не за системой, а за
человеком.

| Поле | Тип | Зачем |
|---|---|---|
| Date | Title (дата) | день замера |
| Energy | Number (1–5) | уровень энергии |
| Sleep hours | Number | сколько спал |
| Mood | Select: low / neutral / good | настроение |
| Movement | Checkbox | была ли физическая активность |

**Красная зона** — если энергия ≤2 или сон <5 часов несколько дней подряд, это сигнал.
Формула-подсветка — из sidebar; в baseline достаточно глазами в недельном виде.

**Приватность.** Life Pulse **никогда не уходит наверх** (если человек когда-нибудь подключит
Layer 2/3) — это личная красная линия. Здоровье — только твоё.

**Views:** This week, Trend (календарь).

## §8.5 🔁 Habits (NEW — трекер привычек)

**Назначение.** Привычки — то, что хочешь делать **регулярно**: спорт, чтение, медитация,
английский, ранний подъём. База Habits хранит **список** привычек (что вообще отслеживаю), а
факт «сделал сегодня» живёт в Daily Log через relation «Habits done today» (§1). Так привычки
и дневник связаны: день помнит, что ты выполнил, а привычка видит, в какие дни её делали.

Это маленький, но мощный кирпич: привычки — это «прошивка себя через повторение». Где Life
Pulse замеряет состояние (энергия/сон), Habits фиксирует действие (что я регулярно делаю).
Вместе они дают честную картину: не «я хочу быть таким», а «вот что я делаю на самом деле».

**Схема полей (baseline — минимум):**

| Поле | Тип | Зачем |
|---|---|---|
| Name | Title | название привычки (спорт / чтение / медитация...) |
| Cadence | Select: daily / weekly / Nx-week | как часто хочу делать |
| Target | Text | цель словами («5 раз в неделю», «каждое утро») |
| Area | Multi-select: health / mind / work / relationships | к какой сфере жизни |
| Active | Checkbox | активна сейчас или на паузе |
| Days done | Rollup ← Daily Log (count) | сколько раз отмечена за период (авто) |
| Notes | Text | зачем эта привычка, как поддерживаю |

**Как работает трекинг (минимальный baseline).** Ты НЕ заводишь отдельную запись на каждую
привычку каждый день — это была бы та самая «задроченность». Вместо этого: в записи дня
(Daily Log) в поле «Habits done today» линкуешь выполненные сегодня привычки. База Habits
через rollup сама считает, сколько раз каждая привычка отмечена за неделю/месяц. Минимум
действий — максимум картины.

**Streak (серия) — на пальцах.** «Сколько дней подряд» — самая мотивирующая метрика привычек.
В baseline её можно смотреть глазами в недельном виде («отмечал ли каждый день»). Точный
авто-подсчёт серий + хитмап-календарь привычек — это в sidebar (§13), «добавь когда захочешь
геймификации». Важно: streak — **инструмент для себя**, не кнут. Сорвал серию — это не провал,
а данные. (Это R12-аккуратность: привычки не должны превращаться в само-эксплуатацию.)

**Views:** Active (активные привычки + Days done за неделю), By area (по сферам жизни),
Paused (на паузе — чтобы вернуться).

**Связи.** Habits ↔ Daily Log (relation «Habits done today» → rollup Days done). Опционально
Habits ↔ Goals (привычка служит цели: «бегать 3×/нед» → цель «полумарафон»). Habits ↔ Life
Pulse рядом в ежедневном ритуале (замерил состояние + отметил привычки за один заход вечером).

**Приватность.** Как и Life Pulse — Habits **личное, наверх не уходит** (если человек подключит
Layer 2/3). Что я делаю по утрам — только моё.

**Заводить — по желанию, но полезно почти всем.** core-lite: не обязательно в первый час, но
2-3 ключевые привычки завести стоит сразу — это самый дешёвый способ начать «прошивать» себя.

## §10 🧭 NEW: Strategic под-раздел (внутри Layer 1)

Это новое в v2. В v1 цели были «просто базой Goals». Теперь стратегия поднята на уровень
выше — это **набор страниц-документов**, а не таблица. Почему — объясняю ниже.

Strategic под-раздел — это **пять вещей**: три про направление (куда иду) и две про смысл и
мудрость (что хочу помнить и держать перед глазами).

### 1. 📜 Vision document (страница, не база)

Один текст: **«куда я иду»**. Не таблица, не поля — живая проза, которую пишешь ты сам.
Каким человеком хочу стать. Что для меня важно. Куда движусь в горизонте лет. Это якорь,
к которому возвращаешься, когда теряешь направление.

### 2. 🎯 Goals strategic document (страница + ссылки на Goals DB)

Тут самое интересное изменение. **Goals — это не просто база, а стратегический документ.**

Страница Goals — это текст про твои цели: что я хочу, почему именно это, как пойму, что
достиг. А **внутри** этой страницы — ссылки (linked view) на простую базу Goals DB, где
лежат конкретные цели с метриками и сроками.

**Почему цели заслуживают документа, а не просто базы — объясняю.** База — это таблица:
имя, метрика, срок, статус. Она хороша для отслеживания «достиг / не достиг». Но **цель —
это не только метрика. Это смысл.** Почему я этого хочу? Что изменится, когда достигну? От
чего откажусь ради этого? На эти вопросы таблица не отвечает — для них нужна проза. Если
держать цели только базой, теряется самое важное — *зачем*. Поэтому: проза (документ) +
данные (база) вместе. Документ хранит смысл и пишется человеком (AI не пишет стратегию).
База хранит метрики и сроки — её Claude Code может предзаполнять данными.

### 3. 🧭 Точка А → Точка Б (отдельная страница на текущее направление)

Главная страница направления. **Точка А** — где я сейчас, честно, по фактам. **Точка Б** —
куда хочу прийти. И путь между ними. Это та же «точка А → точка Б», что и в шаблоне дня
(§1.1), но на большом горизонте — не «к вечеру», а «к концу квартала/года».

Эту страницу заводишь под текущее направление. Сменилось направление — заводишь новую (старую
не стираешь, она остаётся историей: append-only).

### 4. 🧠 Философский лист (страница — что хочу помнить)

Это новое по запросу Руслана. Отдельная **страница-документ**, куда складываешь **всё, что
хочешь помнить и держать перед глазами**: жизненные мудрости, наставления (себе и от других),
принципы, правила жизни, выводы, к которым пришёл и не хочешь забыть.

Чем философский лист отличается от Vision. **Vision = «куда я иду»** (направление, цель пути).
**Философский лист = «как я хочу жить и думать по дороге»** (мудрость, принципы, опора). Vision
меняется, когда меняется направление; философский лист растёт всю жизнь — это копилка того, что
проверено и важно.

Как устроена страница (живая проза, ты пишешь её сам):

```
# 🧠 Философский лист

## Принципы (как я хочу жить)
- ...

## Мудрости и выводы (к чему пришёл)
- ...

## Наставления (себе и от других — кого хочу слышать)
- ...

## Правила, которые работают для меня
- ...
```

**Append-only по духу.** Новое добавляешь сверху своего раздела, старое не стираешь (оно может
снова стать актуальным). Перечитываешь периодически — лучше всего в месячном/квартальном ревью,
когда отвечаешь на «что я понял за этот период». Claude Code сюда ничего не пишет сам — это
**глубоко личный документ**, AI его только показывает по просьбе, не редактирует (правило
конституции №1 + приватность).

### 5. ⭐ Ключевые идеи и высказывания (отдельный документ + лёгкая база — с перечитыванием)

Тоже по запросу Руслана, и это **отдельная вещь** от философского листа. Если философский лист —
это связная личная философия (проза, читаешь целиком), то здесь — **коллекция отдельных умных
мыслей**: цитаты, ключевые идеи, важные направления, высказывания, которые зацепили. Сюда можно
**добавлять много** — и главное, **механика перечитывания**, чтобы они не оседали мёртвым грузом.

Устроено в две части:

**(а) Топ 3-5 ключевых сейчас (страница-якорь).** Наверху документа — короткий список из 3-5
**самых важных направлений / идей прямо сейчас**. То, что хочешь держать перед глазами каждый
день. Их мало нарочно — это фокус. Меняются редко.

**(б) Коллекция (лёгкая база Wisdom/Ideas — чтобы добавлять много и перечитывать).** Под якорем —
простая база, куда сыплешь всё, что зацепило. Минимальная схема:

| Поле | Тип | Зачем |
|---|---|---|
| Text | Title | сама мысль / цитата / идея одной фразой |
| Source | Text | кто сказал / откуда (книга, человек, сам) |
| Kind | Select: цитата / идея / направление / принцип | что это |
| Tag | Multi-select | тема (жизнь / работа / мышление / отношения...) |
| ⭐ Key now | Checkbox | входит в топ 3-5 ключевых сейчас |
| Added | Date | когда добавил |

**Механика перечитывания (то, ради чего всё затевалось):**
- **Вид «⭐ Ключевые сейчас»** — фильтр `Key now = ✓` (те самые топ 3-5, всегда на виду).
- **Вид «🎲 Случайная мудрость»** — Notion умеет показывать случайную запись (или сортировка-
  шаффл): зашёл — перечитал 1-3 случайных. Старое всплывает заново.
- **Привязка к ревью** — в weekly review есть строчка «прочитай 3 случайных из коллекции»;
  что-то перечитал — может, повысил в ⭐ Key now, что-то устарело — снял звезду.
- **Опционально в wrap-of-day** — Claude Code в конце дня может подкинуть одну случайную мысль
  из коллекции («на сегодня из твоей копилки: …») — лёгкий ритуал, не обязаловка.

**Зачем база, а не просто список.** Список на 100 цитат = свалка, которую никто не перечитывает.
База + resurface-механика = мысли возвращаются к тебе по чуть-чуть, и копилка остаётся живой.
Это та же логика, что «посмотреть позже» в Knowledge (§6): добавить легко, но и **достать обратно
тоже легко** — иначе смысла копить нет.

**Важно:** всю стратегическую и философскую прозу (Vision, смысл целей, точку Б, философский лист,
ключевые идеи) **пишет и курирует человек.** Claude Code только предзаполняет данные в базе Goals,
подтягивает цифры и **показывает** философский лист / коллекцию по просьбе (включая resurface
случайных мыслей) — но «куда я иду», «как я хочу жить» и «что для меня мудро» решает и формулирует
только человек. Это правило конституции №1.

## §11 🔄 NEW: Weekly + Monthly шаблоны ревью

Тоже новое в v2 на уровне Layer 1. Два готовых шаблона ревью. Не надо придумывать вопросы —
бери и заполняй. Claude Code подтягивает цифры из баз, ты отвечаешь на «почему» и «что дальше».

### 🗓️ Weekly review (воскресенье вечером, ~20 минут)

Готовый шаблон-страница со структурой:

```
# Ревью недели [даты]

## ✅ Что сделал за неделю
(топ-5 — Claude Code предзаполняет из Daily Log «реально сделано»)

## ❌ Что не сделал / что зависло
(на что не хватило — честно)

## 🚀 Проекты: движение
(какие двигались, какие застряли — подтягивается из Projects)

## ❓ Гипотезы: обновления
(что подтвердилось / опроверглось / сдвинул уверенность)

## 🎯 Топ-3 на следующую неделю
(что главное на следующие 7 дней)

## ⭐ Перечитал из копилки
(3 случайных мысли из «Ключевых идей» — что отозвалось, что повысить/снять с ⭐)
```

### 📆 Monthly review (конец месяца, ~45 минут)

```
# Ревью месяца [месяц]

## 🚀 Проекты: обзор
(скорость по типам, что закрыл, что завис — из Projects)

## 🎯 Цели: прогресс
(сверка с Goals — где продвинулся к Точке Б)

## 💰 Финансовый пульс
(если ведёшь Финансы — приход/расход за месяц)

## 🤝 Контакты: аудит касаний
(с кем давно не общался — из Contacts «Reconnect»)

## 🔁 Привычки: что держалось
(какие привычки выдержал по Cadence, какие просели — из Habits «Days done»)

## 🧠 Философский лист: что понял
(перечитал свой философский лист — что добавить за месяц, какой вывод закрепить)

## 🧭 Паттерны и уроки
(что повторяется, что менять)
```

**Как работает авто-подтягивание — механика linked views.** Шаблон ревью — это не пустая
страница. Внутри неё встроены **linked views** существующих баз: вид Daily Log за нужную
неделю, вид застрявших Projects, вид Contacts с давними касаниями. Когда открываешь ревью,
данные **уже там** — Notion сам показывает срез нужного периода. Claude Code сверх этого может
прочитать эти срезы и накидать черновик саммари («вот топ-5 сделанного, вот 2 застрявших
проекта») — но, как и в wrap-of-day, это **черновик**, ты подтверждаешь. Ты не собираешь
цифры руками — ты отвечаешь на смысловые вопросы поверх готовых данных.

**Каскад.** День сворачивается в неделю, неделя — в месяц. Каждый уровень опирается на
предыдущий: это и есть «система учится на себе» в личной версии.

## §14 Worked example: обычный день (иллюстративно)

> Без реальных данных — просто как это выглядит на практике.

Например, пользователь утром открывает **Today** в Daily Log. Создаётся сегодняшняя страница
с шаблоном. В «Цель дня» он пишет: *«закончить черновик коммерческого предложения»*. В блоке
«Точка А → Точка Б» отмечает: точка А — «есть структура, нет текста», точка Б — «готов первый
драфт», шаги — «1. вступление, 2. оффер, 3. цена». Ставит `Day type = production`.

В течение дня он что-то надиктовывает голосом — заметки падают в «Записи дня» как черновики.
Набросок самого предложения он создаёт как **подстраницу** прямо внутри дня (в разделе
«Черновики»). Заодно упоминает в записях контакт — Claude Code предлагает черновик в Contacts,
но пользователь решит позже.

Вечером пользователь открывает день и в «Реально сделано» пишет: *«вступление и оффер готовы,
цену не доделал»*. Ставит `Deep work minutes = 90`, `Energy = 3`. Видит разрыв между целью
(весь драфт) и фактом (две трети) — это нормальный материал для размышления.

Затем — **wrap-of-day**: пользователь просит Claude Code завершить день. Claude Code читает
запись и предлагает черновики: *«ты упоминал контакт — занести в Contacts? из записей вытащил
идею про шаблон предложения — в Ideas? проект "коммерческое предложение" двигался — отметить
Last touched сегодня?»*. Пользователь принимает первые два, третье отклоняет. **Ничего не
записалось без его ack.** День закрыт за пять минут осмысления, а не получаса ручной возни.

В воскресенье он откроет **Weekly review** — и увидит, что «реально сделано» за все дни уже

## §15 STANDALONE + constitutional posture

**Standalone.** Layer 1 работает **полностью один**. Человеку не нужны ни команда (Layer 2),
ни бизнес-слой (Layer 3). Ни одна baseline-схема выше не ссылается на существование других
слоёв. Это законченный продукт «Personal OS» — форкнул, заполнил за полчаса, пользуешься.

**Constitutional posture.** R1 surface only (варианты простоты surface, выбор за Русланом);
R2 STRICT (Notion API не зовётся — это спека); R6 (схемы наследуют Personal OS Plan + crm/);
R11 (никаких действий без человека; wrap-of-day и ревью — только черновики, ack обязателен);
IP-1 STRICT (роль владельца абстрактна, «пользователь» — пример применения, не реальные
данные); append-only (логи и старые направления не стираются). NO sample data — все поля и
примеры иллюстративны, ни одной реальной записи.
