# 🌍 Сеть Мастерских — full spec (R12 AUTO-FIRE)

> **Что это.** Полный разворот направления #14 «Сеть мастерских» — как одна мастерская становится
> распределённой сетью: топология, дорожная карта online→offline, мульти-язык, передвижение людей,
> роли в сети, объединение ресурсов (resources pooling), активности, направление развития, и —
> главное — **anti-паттерны R12**. Сеть — это **primary R12 surface** всего концепта: здесь живут
> масса, рост и рекрутинг, а значит здесь максимальный соблазн extraction / lock-in / секты.
>
> **R12 paired-frame STRICT (AUTO-FIRE):** каждый механизм роста/рекрутинга в этой фазе идёт
> **парой** — «как это растит сеть» + «как это НЕ становится сектой/extraction». influence-ethics
> + recruitment-dynamics + propaganda + nlp + gamification — receiver-direction по всей фазе.
> **Сквозной закон (lifecycle-doc):** защита должна расти БЫСТРЕЕ системы.

## §A Топология сети

### A.1 Mesh, не star (R12-обоснование)

Сеть мастерских — это **mesh** (ячеистая сеть равноправных cells), **не star** (звезда с центром).
Это не эстетический выбор — это R12-требование:

- **Star** = есть центр, который всё контролирует → founder/HQ может извлекать ценность из
  периферии, запирать, диктовать. Это путь к «корпорации с диктатором» (lifecycle Scale-ловушка).
- **Mesh** = каждая cell автономна, координация через общие стандарты (Charter + R12), а не через
  приказ из центра. Никто не может «выключить» cell сверху; cell может форкнуться (R12 fork-and-leave
  на уровне cells, не только людей).

```
Топология (mesh):
   Берлин-cell ── NYC-cell ── Belgrade-cell
        │  ╲      │      ╱  │
        │   ╲     │     ╱   │
   Москва-cell ── London-cell ── SF-cell
   (каждая cell автономна; координация = общие стандарты, не приказ; нет центра)
```

> **R12 paired-frame (топология):** mesh = устойчивость + анти-доминирование. **Граница:** «нет
> центра» не означает «нет стандартов» — R12 + Charter + Mondragón 5:1 = общие для всех cells
> (иначе анархия позволит локальный extraction). Координация ≠ контроль.

### A.2 Анатомия cell

- **Local cell** = одно физическое место (или одна локальная виртуальная группа до появления места).
- **Coordination layer** = общие стандарты: Charter, R12-discipline, методологический канон,
  shared tooling, общая wiki. Паттерн Mondragón (81 кооператив связаны общими принципами + внутр.
  банк, но каждый автономен).
- **Cross-cell fabric** = то, что течёт между cells: люди (передвижение §D), проекты (§G), ресурсы
  (§F), знание (лучшие наработки). Это превращает N мастерских в **одну сеть**, а не N независимых
  клубов.

### A.3 Founder-роль в топологии

Voice/lifecycle: основатель = **первый член, не глава вечно**. На Build он крутит маховик; на
Scale он «один из многих хранителей»; на Mature его уход сеть не останавливает. Топология
спроектирована так, чтобы **не зависеть от founder'а** — это и есть проверка на анти-культ (§I).

## §B Дорожная карта Online → Offline (по годам)

Voice 26.05: «сначала виртуально, потом быстро оффлайн». Структура по lifecycle (маховик):

### Year 1 — Build (виртуальная мастерская)
- Виртуальная: Notion + Discord/Telegram + Zoom + Claude Code shared workspace.
- Первые in-person meetups в Берлине (Ruslan home base) — нерегулярные, без аренды.
- Cohort 5-15, преимущественно online.
- Маховик: основатель руками. R12-риск: 🟢 низкий (нечего извлекать — денег нет).

### Year 2 — Run (hybrid)
- Берлин: первое постоянное место — **арендованный co-working** (не своя недвижимость — anti-WeWork).
- Виртуальная остаётся primary.
- 2-3 дополнительных города пробуют локальные meetups (ещё не постоянные места).
- Cohort 15-50; первый доход (ступень 5 ≈ €1500/мес).
- Маховик: основатель + партнёры + петля обратной связи. R12-риск: 🟡 средний (деньги потекли).

### Year 3-5 — Scale (distributed network)
- 3-10 постоянных физических мастерских (Берлин / Москва / Лондон / NYC / SF / Belgrade / etc. —
  *примеры городов, не план*).
- Локальная автономия (per Charter + R12).
- Inter-cell expeditions (люди ездят, §D).
- 100-1000 members.
- Маховик: система + локальные cells; основатель = 1 из многих. R12-риск: 🔴 высокий.

### Year 5+ — Mature (full network)
- 30-100+ locations.
- Кооперативная юр. структура (Mondragón-style).
- Member-owned (worker-owner модель).
- 1000-10000+ members.
- Маховик: сеть крутится сама; уход любого (даже founder'а) её не останавливает. R12-риск: 🔴
  максимальный → защита **механическая** (смарт-контракты), не на доброй воле.

> **Все цифры — сценарные (A/B/C/D), не обещания** (R11). «Быстро оффлайн» (voice) уточняется:
> первое физическое место только при подтверждённой Run-петле, и арендованное, не купленное.
> Темп — R1 (Руслан). Детальный year-by-year — Phase 8.

> **R12 paired-frame (рост):** каждый шаг масштабирования = больше людей под влиянием. **Граница:**
> на каждом этапе защита прописана ДО роста (Steward на Run, смарт-контракты на Scale), и **растёт
> быстрее системы** — иначе на Scale получим секту или корпорацию-диктатуру.

## §C Мульти-язычная координация

Voice 26.05: «multi-language (основной английский, локальные тоже)».

- **Primary: английский** — lingua franca для cross-cell (общие стандарты, межсетевые проекты,
  Charter master-версия).
- **Local: родной язык per cell** — русский Берлин-cell / английский NYC / сербский Belgrade /
  etc. Локальная культура уважается (анти-uniformity §I).
- **Translation infrastructure (AI-assisted):** Charter / R12 / методология доступны мульти-язык;
  AI-перевод как инструмент (но critical-документы — с человеческой вычиткой, не raw-машина).
- **Per-cell автономия для локальной культуры:** cell может иметь свои ритуалы, праздники, стиль —
  при сохранении общего ядра (R12 + Mondragón + методологический канон).

> **R12 paired-frame (язык):** общий язык = координация. **Граница:** английский-primary не должен
> стать инструментом доминирования англоязычных cells над локальными (культурный extraction).
> Локальный язык равноправен внутри cell; перевод — мост, не иерархия.

## §D Передвижение людей между cells

Voice 26.05: «передвижение людей между точками».

- **Travel program** — приехал в другой город → есть место «jam'нуть» (couch + верстак). Член
  Берлин-cell, оказавшись в NYC, работает в NYC-cell как свой.
- **Cross-cell projects** — коллаборация с членами из других cells (§G; worked example Phase 7 §E).
- **Sabbaticals** — провайдер стажируется в другой cell на время (обмен tacit-знанием).
- **Conferences / Demo Days** — ежегодные большие сборы всей сети (паттерн YC Demo Day).
- **Expedition format** — малая группа (5-7) путешествует вместе для глубокого обучения (§G, Phase 7 §D).

> **R12 paired-frame (движение):** мобильность = богатство сети + рост связей. **Граница:**
> передвижение добровольно, не «обязаловка для лояльности»; нет «отработай поездку преданностью».
> Гостеприимство — gift (Burning Man decommodification), не долг.

## §E Роли в сети

(IP-1 STRICT — абстрактные типы; примеры = иллюстрация.)

| Роль | Что делает | R12-нота |
|---|---|---|
| 🏠 **Local Master** | держит локальную cell, ведёт локальную мастерскую | автономен, но в рамках Charter+R12 |
| 🛡️ **Network Steward** | cross-cell governance, R12-enforcement | ловит нарушения ≤5 сек; пример: Прапион-тип |
| 🔗 **Connector** | соединяет cells, фасилитирует обмены | не «контролёр потоков», а мост |
| 💰 **Investor** | капитал-партнёрство | Mondragón 5:1 cap; нет capital-доминирования |
| 🧭 **Consultant** | услуги внутри/наружу сети | service-роль, не обязательна |
| 🔬 **Researcher** | frontier work, общий для сети результат | пример: Сева-тип |
| 🎓 **Master Mentor** | учит мастеров через cells | пример: Левенчук-тип; не гуру, один из многих |
| ✈️ **Visiting Practitioner** | временная резиденция в чужой cell | гость со статусом своего |

> **R12 paired-frame (роли):** ролевое богатство = возможности. **Граница (anti-recruitment-abuse):**
> ни одна роль не даёт власти «запереть» других; Steward enforce'ит R12, но сам под R12 (Steward
> не выше Charter); нет роли «вербовщик» — есть Connector (соединяет согласных), не «рекрутёр».

## §F Объединение ресурсов (Resources Pooling)

Voice 26.05 дословный список — «всё в кучу». Каждый ресурс = с R12-границей (приватность + добровольность):

| Ресурс | Механика | R12-граница |
|---|---|---|
| 🤝 **Знакомства** | shared CRM-интеллигенция (уважающая приватность) | данные члена = его данные; представления только с согласия (consent) |
| 🏭 **Склады / помещения** | каталог сети (член предлагает) | offer, не обязанность; владелец контролирует |
| 🏠 **Хаты / гаражи** | couchsurfing-like для членов | gift-экономика, не «отработай» |
| 🚗 **Машины** | share-economy паттерны | добровольно; нет принуждения |
| 💡 **Бизнес-идеи** | opportunity board | идея остаётся автора; форк с атрибуцией |
| 💸 **Деньги (лишние)** | micro-investment internal pool (Mondragón внутр. банк) | 5:1 cap; QF-распределение; opt-in; revenue-gated |
| 📚 **Информация** | общая база (wiki + curated streams) | curated, не дамп; R12-check на «не доим знанием» |
| 🛠️ **Инструменты** | физический tool-sharing per cell | как стена инструментов (Phase 1 §B) |
| 🎯 **Skills / expertise** | skills marketplace (per metaplan-v2) | 75/25 split; fork-and-leave |

**Ключевой принцип pooling'а:** ресурс **предлагается** в общий пул добровольно и **остаётся под
контролем владельца** (он решает, что отдать и кому). Это не «всё общее по умолчанию» (это было бы
extraction наоборот — принуждение делиться), а «легко делиться, если хочешь».

> **R12 paired-frame (pooling — критичная зона):** общий пул = огромное преимущество сети
> (Mondragón внутр. банк = почему кооперативы выживают кризисы). **Граница (R12 ядро):** (1) opt-in,
> не opt-out; (2) данные/ресурс члена = его собственность; (3) нет extraction сверх согласованной
> доли (75/25 + 5:1); (4) fork-and-leave — уходя, забираешь свою долю пула; (5) consent на любое
> представление/распределение (RUSLAN-LAYER action class `non_consensual_distribution`). Пул — это
> **gift + согласие**, не **сбор дани**.

## §H Направление сети (life-development orientation)

Voice 26.05: «direction: жизнь / развитие / голод / тренированность / дисциплина (поддерживается,
не enforced)».

- Все нацелены на **развитие** (триада O-138: жить / не умереть / развиваться).
- «**Голодные / натренированные / дисциплинированные**» — это **культурный вектор**, **не
  enforced**. Поддерживается средой (окружение тянет вверх), не навязывается правилом.
- Поддержка direction'а через **Welcome-frame** (O-144), не gate-keeping: среда приглашает и
  поддерживает, не отсеивает «недостаточно голодных».
- Используют все возможности для развития (ресурсы §F, активности §G, роли §E).
- **Frontier contribution mandate** (O-183) — культурная норма, не KPI.

> **R12 paired-frame (direction — тонкая зона):** «голод / дисциплина» как вектор = мощная
> мотивация. **Граница (критично — это где культы ломаются):** (1) НЕ enforced — нельзя выгнать за
> «недостаточную дисциплину»; (2) уважение к ритму (AP-6 — периоды низкой энергии не патология);
> (3) нет морального давления «настоящий мастер всегда голоден»; (4) выход без стыда (fork-and-leave
> без «предатель»). Direction поддерживается, человек свободен. Это **граница между здоровой
> культурой развития и сектой**, и она держится явно.

## §I Anti-patterns сети (R12 STRICT — ядро фазы)

Это самый важный раздел Сети. Каждый anti-паттерн = реальный режим отказа, в который скатываются
сети сообществ. Сквозной закон: **защита растёт быстрее системы**.

- ❌ **Central authority (star вместо mesh)** — центр контролирует периферию → extraction/диктатура.
  *Защита:* mesh-топология (§A); координация ≠ контроль; cell может форкнуться.
- ❌ **Founder cult** — основатель = вечный глава, спаситель-фигура. *Защита:* founder = первый
  член, не глава вечно; топология не зависит от него (§A.3); ротация хранителей (анти-культ);
  нет клятв верности; **выход звучит первым**.
- ❌ **Extraction beyond agreed share** — выжимание сверх доли. *Защита:* Mondragón 5:1 cap +
  75/25 split публичен; на Scale — смарт-контракт (RUSLAN-LAYER `extraction_beyond_share` +
  `wage_ratio_violation`).
- ❌ **Fork-prevention** — «после входа ты заперт». *Защита:* fork-and-leave anytime + 30-day
  opt-out window при изменении Charter; уходя, забираешь свою долю (RUSLAN-LAYER
  `fork_prevention_attempt`).
- ❌ **Closed elite gates** — «нас N избранных». *Защита:* Welcome-frame (O-144); ступени 1-4
  бесплатны; mass upliftment (O-184).
- ❌ **Uniformity** — все cells одинаковые, нет разнообразия ниш. *Защита:* локальная культурная
  автономия (§C); per-cell ритуалы; mesh.
- ❌ **Engagement-trap** — удержание любой ценой (anti-TikTok). *Защита:* метрика = рост, не время;
  лёгкий выход; добровольность активностей (§G).
- ❌ **Non-consensual distribution** — пул раздаёт ресурс/данные без согласия. *Защита:* opt-in +
  consent на каждое распределение/представление (RUSLAN-LAYER `non_consensual_distribution`).
- ❌ **Манипулятивный рекрутинг** — тёмные паттерны привлечения (anti-Pinduoduo рефералы).
  *Защита:* promoter-бонус за **качество**, не объём; 8 вопросов R12 перед любым касанием; нет
  роли «вербовщик»; Connector соединяет согласных.

```
Anti-секта чек-лист сети (всё ДА = здоровая сеть):
  [ ] mesh, не star (нет центра контроля)
  [ ] founder = 1 из многих, уход не ломает сеть
  [ ] split публичен; 5:1 cap enforced
  [ ] fork-and-leave anytime, забираешь долю
  [ ] вход открыт (Welcome-frame), ступени 1-4 free
  [ ] локальная автономия (нет uniformity)
  [ ] метрика = рост, не engagement-время
  [ ] pooling opt-in + consent
  [ ] рекрутинг = приглашение, не вербовка; нет клятв; ВЫХОД ЗВУЧИТ ПЕРВЫМ
```
