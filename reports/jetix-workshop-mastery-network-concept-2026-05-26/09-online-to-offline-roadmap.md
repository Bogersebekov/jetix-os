# Online → Offline transition — deep roadmap

> **Что это.** Детальная year-by-year дорожная карта перехода виртуальной мастерской в
> распределённую физическую сеть. По каждому году: members / locations / revenue / governance /
> R12-audit cadence + ключевые переходы + риски + митигация.
>
> **R11 STRICT — все цифры сценарные (A/B/C/D), не обещания.** Точки A/B/C/D наследуются из
> lifecycle-doc. Сквозной закон: **защита растёт быстрее системы**.

---

## §1 Принцип перехода (повтор для самодостаточности)

Переключатель этапа — **«кто крутит маховик»**, не «сколько людей»:
- **Build** — основатель руками.
- **Run** — основатель + партнёры + петля обратной связи.
- **Scale** — система + локальные cells; основатель = 1 из многих.
- **Mature** — сеть крутится сама; уход любого её не останавливает.

«Быстро оффлайн» (voice 26.05) ограничено осторожностью: первое физическое место = **арендованное**
(не купленное — anti-WeWork), и **только при подтверждённой Run-петле**. Темп — R1 (Руслан).

## §2 Year-by-year roadmap

### Year 1 — Build (виртуальная мастерская)

| Параметр | Значение (сценарно) |
|---|---|
| Members | 5-15 (когорта, преимущественно online) |
| Locations | 0 физ. + 1 виртуальная (Notion+Discord+Zoom+Claude Code); нерегулярные meetups в Берлине |
| Revenue | свои сбережения / символический; нет постоянного дохода |
| Governance | основатель руками; Charter draft; первые роли (методолог/тестер/Steward) |
| R12-audit cadence | вручную, 8 вопросов перед каждым касанием; Steward (Прапион-тип) проверяет Charter на анти-секту |

**Что происходит:** виртуальная мастерская оживает. Первые методологи проверяют метод на себе,
тестеры-гуманитарии ломают предположения, Steward сторожит R12. Видео A/B/C (блокер) записаны.
Маховик целиком на основателе.

### Year 2 — Run (hybrid)

| Параметр | Значение (сценарно) |
|---|---|
| Members | 15-50 (когорта платит) |
| Locations | 1 постоянное (арендованный co-working, Берлин) + виртуальная primary; 2-3 города пробуют meetups |
| Revenue | доход течёт; ступень 5 ≈ €1500/мес/участник; €5-15K/мес (проекция) |
| Governance | основатель + партнёры; Charter подписан 3+; первые offline retreats; кооператив legal **start** |
| R12-audit cadence | + Steward ловит ≤5 сек; Mondragón 5:1 на каждом денежном шаблоне; fork-and-leave на каждой выплате |

**Что происходит:** включается петля обратной связи. Партнёры со-создают курсы. Первое физическое
место — но арендованное, под demo и retreats, не недвижимость-бизнес. Деньги потекли → R12-риск
вырос до 🟡 среднего.

### Year 3-5 — Scale (distributed network)

| Параметр | Значение (сценарно) |
|---|---|
| Members | 100-1000 |
| Locations | 3-10 постоянных физических мастерских (разные города) + постоянная виртуальная |
| Revenue | распределяется по кооперативным шаблонам; multi-cohort; €10K+/мес (проекция) |
| Governance | mesh из cells; локальная автономия per Charter; основатель = 1 из многих; смарт-контракты вводятся |
| R12-audit cadence | Steward per clan; смарт-контракт enforce'ит 5:1 + 4 RUSLAN-LAYER class on-chain; аудит per cell |

**Что происходит:** мастерская становится сетью. Inter-cell expeditions, cross-cell проекты.
Локальная культурная автономия. R12-риск 🔴 высокий → защита смещается с «доброй воли» на
**механическую** (смарт-контракты).

### Year 5+ — Mature (full network)

| Параметр | Значение (сценарно) |
|---|---|
| Members | 1000-10000+ |
| Locations | 30-100+ |
| Revenue | member-owned кооператив; treasury + QF-распределение |
| Governance | сеть крутится сама; worker-owner Mondragón; уход founder'а её не останавливает |
| R12-audit cadence | механический по умолчанию; Steward per clan; периодический внешний аудит целостности |

**Что происходит:** полная сеть, принадлежащая своим членам. Frontier-импакт. Кандидат на Network
State substrate (gated). R12-риск максимальный → защита максимально механизирована + культурно
укоренена (анти-секта = норма, не правило).

## §3 Ключевые переходы (и что их триггерит)

| Переход | Триггер (gate) | Анти-триггер (НЕ переходим) |
|---|---|---|
| **Build → Run** | ≥1 методолог-партнёр confirmed + ≥3 тестера активны + Charter R12-проверен + Notion multi-user + звонок отрепетирован ≥5 раз + юр начато | выгорание / methodology drift (LOCKED тронуты) / R12 violation / Wave 1 без видео |
| **Первое физ. место (Run)** | подтверждённая Run-петля (когорта платит, отклик есть) | «хочу место для статуса» без петли = WeWork-ловушка |
| **Run → Scale** | 1K+ users + 3+ когорт + €10K/мес + 1-й клан + T4 появился | founder bottleneck не решён / R12-защита отстаёт от роста |
| **Кооператив legal (Scale)** | Charter 50+ подписей + стабильный multi-cohort | юр-оформление ради «выглядит серьёзно» без substrate |
| **Worker-owner conversion (Mature)** | смарт-контракты работают + cells автономны + founder готов стать «1 из многих» | founder держит контроль = корпорация-диктатура |

## §4 Риски per transition + митигация

- **Build → Run: преждевременный выход.** Риск — выйти наружу с недоделанной системой, сжечь
  дорогие контакты. *Митигация:* gate ≥80% Build-выходов; «сначала 2 человека, не 7»; видео A
  как блокер.
- **Первое физ. место: недвижимость-ловушка.** Риск — аренда становится бизнес-моделью (WeWork).
  *Митигация:* арендуем под demo/retreats, не «сдаём столы»; доход от ценности, не от метров.
- **Run → Scale: founder bottleneck.** Риск — всё держится на основателе, не масштабируется.
  *Митигация:* partner-led обязателен; маховик передаётся cells; основатель сознательно убирает
  себя из критического пути.
- **Scale: секта / extraction.** Риск — массовая динамика → культ или выжимание. *Митигация:*
  защита **механическая** (смарт-контракты 4 класса); анти-секта массово (нет клятв, нет спасителя,
  выход звучит первым); Steward per clan.
- **Scale: uniformity.** Риск — все cells одинаковые, нет разнообразия ниш. *Митигация:* локальная
  культурная автономия; mesh.
- **Mature: founder cult / контроль.** Риск — основатель не отпускает. *Митигация:* топология не
  зависит от founder'а (§A.3 Phase 3); ротация хранителей; member-owned by design.

## §5 R12 compliance escalation (как защита растёт с масштабом)

```
R12-защита по этапам (растёт БЫСТРЕЕ системы):
  Build  🟢  → 8 вопросов вручную + явные credits (соблазн: присвоить вклад методолога)
  Run    🟡  → + Steward ≤5 сек + Mondragón 5:1 на КАЖДОМ сплите + fork-and-leave на КАЖДОЙ выплате
              (соблазн: выжать аудиторию партнёра / lock-in)
  Scale  🔴  → + смарт-контракты 4 класса + Steward per clan + анти-секта массово
              (соблазн: секта + founder-диктатура)
  Mature 🔴+ → механический по умолчанию + внешний аудит целостности + member-owned
              (соблазн: максимальный — поэтому защита максимальна)
```

**4 RUSLAN-LAYER action class** (on-chain на Scale+): `extraction_beyond_share` /
`wage_ratio_violation` / `non_consensual_distribution` / `fork_prevention_attempt`. На Build/Run —
эти же классы проверяются вручную (Steward + 8 вопросов); на Scale/Mature — программно.

**Сквозной закон (повтор — потому что это ядро):** если система растёт быстрее защиты — на Scale
получится либо секта, либо обычная корпорация с диктатором. Поэтому каждый переход §3 имеет
анти-триггер «R12-защита отстаёт» = НЕ переходим.
