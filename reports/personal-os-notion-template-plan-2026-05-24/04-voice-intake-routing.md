# Phase 4 — Голос → куда что попадает (матрица)

> **Главная мысль фазы.** Руслан диктует голосом на ходу. Надо, чтобы голосовая
> заметка автоматически разложилась по правильным базам — но **никогда не перезаписала
> боевые записи сама**. Сначала всё становится **черновиком (DRAFT)**, человек смотрит
> и одобряет. Ниже — полная матрица «тип сказанного → в какую базу + что заполнить +
> по какому триггеру».
>
> **Железное правило (из голосового канона + R12):** голос = только черновик. Никаких
> авто-перезаписей. Это страховка от мусора и потери данных.

---

## §1 Как это работает (поток одной заметки)

```
Wispr Flow (голос → текст локально)
      ↓
tools/voice_intake_router.py (Claude Code разбирает текст)
      ↓
классификация: что это за контент? (по триггерам + смыслу)
      ↓
создание DRAFT-записи в нужной базе (флаг _voice_draft: true)
      ↓
запись в лог: personal-os/voice-intake/2026-05-24.md
      ↓
СТОП. Руслан утром/вечером смотрит черновики → промоутит вручную
```

**Ключевое:** между «Claude Code создал черновик» и «это стало боевой записью» всегда
стоит человек. Claude Code предлагает — Руслан подтверждает.

## §2 ⭐ Главная матрица — 14 типов контента × 9 направлений

| # | Что сказано (тип) | Триггер-слова | База-направление | Что заполняется | Ревью? |
|---|---|---|---|---|---|

| 2 | Упомянул организацию | название компании/сообщества | CRM Orgs | Name + draft-заметка | ✅ DRAFT |
| 3 | Идея | «идея:»; «а что если»; «прикольно было бы» | Ideas Bank (Tier C) | Title + body + voice_ref | ✅ Tier-промоушен |
| 4 | Гипотеза (явная) | «гипотеза:»; «хочу проверить»; «интересно, правда ли» | Hypotheses | Statement + acceptance draft | ✅ DRAFT |
| 5 | Статус проекта | имя активного проекта + «продвинулся/застрял» | Projects | last_touched + дозапись в notes | ⚠️ авто-append |
| 6 | Решение | «решил»; «выбираю»; «акаю»; «окончательно» | Strategic (decision) | Title + rationale + date | ✅ Human Gate |
| 7 | Рефлексия / журнал | «сегодня было»; «чувствую»; «понял что» | Daily Log (today) | дозапись в reflections | ⚠️ авто-append |
| 8 | Замер здоровья | «энергия»; «спал»; «настроение»; «отношения» | Life Pulse (today) | pulse-числа + note | ⚠️ авто-append |

| 13 | Смена курса (стратег.) | «новый POINT B»; «меняю направление»; «vision» | Strategic (point-b) | новая ревизия (draft) | ✅ высокий гейт |
| 14 | Связь между сущностями | «X связано с Y»; «это про ту же тему» | соответств. relation | draft-связь между записями | ✅ DRAFT |

**Покрытие:** 14 типов контента × 9 направлений (Daily Log / Projects / Tasks /
Sources / Claims / Hypotheses / Ideas Bank / CRM / Strategic + Life Pulse + Reference).
Превышает минимум промпта (≥7 типов × ≥8 направлений).

## §3 Три режима ревью (почему не всё одинаково)

Заметил в матрице три значка — это три уровня доверия к авто-обработке:

| Режим | Когда | Что значит |
|---|---|---|
| ⚠️ **авто-append** | рефлексия, пульс, статус проекта, референс | дозаписывается сразу (не перезаписывает!), потому что это безопасное добавление в свой же дневник/пульс |
| ✅ **DRAFT** | человек, орг, источник, гипотеза, задача, факт | создаётся черновик, ждёт глаза перед тем как стать боевой записью |
| ✅ **высокий гейт** | решение, смена POINT B | важное стратегическое — обязательно ручное подтверждение, никогда не авто |

**Логика:** дозаписать строчку в свой дневник — безопасно (ничего не теряется, только
прибавляется). Создать нового человека в CRM или закрыть гипотезу — уже влияет на
«боевые» данные, значит черновик + ревью. Поменять курс жизни — максимальная осторожность.

---

## §4 DRAFT-дисциплина (детально)

**Правила (из голосового канона + R12 anti-extraction):**

1. **Голос → DRAFT.** Каждая запись из голоса получает поле `Voice draft? = Y`
   (frontmatter `_voice_draft: true`).
2. **Никогда не перезаписываем.** Голос НЕ меняет существующие поля боевых записей.
   Только: (а) создаёт новый черновик, либо (б) дозаписывает в append-only поля
   (entries, reflections, history, notes).
3. **Промоушен — руками.** Руслан смотрит черновик → правит → снимает флаг
   `Voice draft?` → запись становится боевой.
4. **Очередь черновиков видна** в Command Center («Быстрый ввод») и в виде
   «Черновики на разбор» в каждой базе.
5. **Старые черновики не висят вечно:** черновик >7 дней без разбора — всплывает
   отдельным сигналом (как stuck-детект).

## §5 Wispr Flow → Command Center (полный пайплайн)

| Шаг | Кто делает | Что происходит |
|---|---|---|
| 1 | Wispr Flow | голос → текст (локально, без облака) |
| 2 | человек | текст попадает в инбокс (файл/буфер) |
| 3 | `voice_intake_router.py` | Claude Code читает текст, режет на пункты |
| 4 | Claude Code | классифицирует каждый пункт по матрице §2 |
| 5 | Claude Code | создаёт DRAFT-записи (через Notion API или в файлы) |
| 6 | Claude Code | пишет сводку: `personal-os/voice-intake/YYYY-MM-DD.md` |
| 7 | **СТОП** | — |
| 8 | Руслан (утро/вечер) | открывает сводку + очередь черновиков → промоутит |

**Где «правда»:** черновики могут создаваться сразу в файлах (`personal-os/.../  
<slug>-DRAFT.md`), а Notion получает их зеркало. Filesystem = source of truth
сохраняется (Global Rule 4).

## §6 Обработка непонятного (fallback)

Что если Claude Code не уверен, куда отнести?

| Ситуация | Что делает Claude Code |
|---|---|
| Не подходит ни под один паттерн | → Ideas Bank (Tier C) как «корзина» + пометка «нужна ручная классификация» |
| Подходит под 2+ паттерна | → создаёт DRAFT в наиболее вероятной базе + флаг «уточни» + альтернативы в заметке |
| Чувствительное (деньги/здоровье в деталях) | → DRAFT с пометкой «приватное» + ограниченное поле (не в общие виды) |
| Длинная заметка с разными темами | → режет на пункты, каждый маршрутизирует отдельно |
| Имя похоже на существующий контакт | → НЕ создаёт дубль; дозаписывает в history существующего как DRAFT + флаг «тот же человек?» |

**Принцип fallback:** при сомнении — не угадывать молча, а **создать черновик +
пометить сомнение**, чтобы человек решил. Никогда не терять заметку (Default-Deny на
действие, но не на сохранение — заметка всегда сохраняется хотя бы в Ideas Bank).

## §7 Анти-паттерны голосового ввода

- ❌ **Авто-перезапись боевых полей** — главное табу (потеря данных).
- ❌ **Авто-промоушен черновика** — человек всегда в петле.
- ❌ **Авто-решения** (закрыть проект, сменить POINT B) — только через гейт.
- ❌ **Молчаливое угадывание** при сомнении — лучше пометить «уточни».
- ❌ **Создание дублей контактов** — сначала проверить существующих.
- ❌ **Ключи/секреты в голос-логе** — если в заметке прозвучал секрет, не сохранять его
  в открытом поле.

## §8 Связь с существующим пайплайном

Текущий `tools/run_pipeline.sh` (transcribe → extract → filter → review_report)
остаётся. Новое:
- **Wispr Flow** добавляется как быстрый фронт (вместо/вместе с надиктовкой в мессенджер).
- **`voice_intake_router.py`** — новый шаг: вместо общего review-отчёта раскладывает по
  базам Personal OS (с DRAFT-флагами).
- Дисциплина «СТОП → человек читает» **не меняется** — она ядро (Part 6b Human Gate).

- **14 типов контента → 9+ направлений** (матрица §2).

  (стратегическое).
