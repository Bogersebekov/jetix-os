# Phase 7 — Как любой человек берёт шаблон под свою нишу

> **Главная мысль фазы.** Шаблон должен быть таким, чтобы **любой человек скопировал
> его к себе и за 30-60 минут настроил под свою жизнь** — инженер, исследователь,
> предприниматель, методолог или человек, наводящий порядок в личных делах. Секрет —
> **два слоя**: универсальный фундамент (его не трогают, он одинаковый у всех) и
> надстройка под нишу (её каждый кастомизирует под себя).
>
> **И железно (R12):** человек владеет своими данными, может уйти в любой момент со
> всем, что накопил, экспортировать в файлы. Никаких замков. Jetix не вытягивает данные
> из форкнутых копий — это отдельные воркспейсы.

---

## §1 Два слоя — суть fork-friendly

### 🧱 Layer 1 — Универсальный фундамент (одинаков у всех)

Это «скелет», который НЕ трогают при форке (иначе сломается совместимость и будущие
апдейты):

- Топология 10-14 баз
- Типы полей + конвенции именования (**без имён людей** — «Daily Log», не «Дневник Васи»)
- Матрица голосового ввода (универсальные классификации)
- Структура шаблонов ревью (вопросы общие, адаптируемые)
- Паттерны интеграции с Claude Code
- Дисциплина «файлы = правда»
- DRAFT-only для голоса
- R12: владение данными + fork-and-leave встроены

### 🎨 Layer 2 — Надстройка под нишу (каждый под себя)

Это то, что **меняют свободно** под свою жизнь:

- Подтипы проектов (4 базовых типа фиксированы Layer 1; подтипы — твои)
- Роли CRM (24 базовых Layer 1 + ниша-специфичные)
- Темы гипотез (твоя вселенная вопросов)
- Список активных проектов / контактов (твои конкретные)
- Корректировки вопросов ревью
- Конкретные Notion page ID (твои)

**Почему разделение работает:** Layer 1 = «грамматика» (общая), Layer 2 = «словарь»
(твой). Можно обновлять грамматику из апстрима, не теряя свой словарь.

**IP-1 STRICT:** в Layer 1 нет ни одного конкретного имени — только абстрактные роли и
структуры. Имена и конкретика — всегда в Layer 2. (Поэтому 5 примеров ниже — это
**примеры Layer 2**, не часть Layer 1.)

## §2 ⭐ Инструкция форка (шаг за шагом, 30-60 мин)

| Шаг | Действие | Время |
|---|---|---|
| 1 | **Дублировать воркспейс:** 1 клик «Duplicate» на публичном шаблоне Notion | 1 мин |
| 2 | **Переименовать корень:** «<Твоё имя> Personal OS» | 1 мин |
| 3 | **Layer 1 не трогаешь:** все базы / поля / виды оставляешь как есть | 0 мин |
| 4 | **Layer 2 — подтипы проектов:** добавь свои (см. примеры §3) | 10 мин |
| 5 | **Layer 2 — роли CRM:** добавь ниша-специфичные (студенты / инвесторы / лаб-мейты) | 5 мин |
| 6 | **Layer 2 — темы гипотез:** твоя вселенная вопросов | 5 мин |
| 7 | **Засеять Strategic:** твой POINT A (1 абзац) + POINT B + план | 10 мин |
| 8 | **Первые записи:** сегодняшний Daily Log / 3 проекта / 5 контактов / 3 гипотезы | 15 мин |
| 9 | **(опц.) Claude Code:** установить + клонировать scaffold + токен в `.env` + `/notion-sync --bootstrap` | 10 мин |
| 10 | **(опц.) Голос:** поставить Wispr Flow + тест 1 заметки | 5 мин |

**Минимум для старта (шаги 1-8):** ~45 мин, без Claude Code и голоса — чистый Notion.
Технические шаги 9-10 — продвинутая надстройка, можно потом.

## §3 ⭐⭐ 5 примеров ниш (полная Layer 2 спецификация каждой)

> Это **примеры Layer 2** (IP-1: иллюстрации, не часть универсального слоя). Человек
> берёт ближайший к себе и докручивает.

### §3.1 🛠️ Ниша 1 — Инженер / IC-разработчик (Карпатый-уровень)

| Аспект Layer 2 | Настройка |
|---|---|
| **Подтипы проектов** | code-projects / research-papers / open-source / production-systems / experiments |
| **Роли CRM** | со-контрибьюторы / ревьюеры PR / менеджеры / domain-эксперты / open-source мейнтейнеры |
| **Темы гипотез** | архитектура / производительность / масштабируемость / новые парадигмы / трейдоффы |
| **Каденс** | еженедельный tech-дайджест / месячная сверка скорости по проектам / ретро спринта |
| **Голос-паттерны** | «commit / PR / баг / рефактор / деплой» → Tasks; «прочитал paper X» → Sources |
| **Life Pulse акцент** | фокус-время / deep work часы / context-switching |
| **Что добавить в Knowledge** | code-snippets / архитектурные решения (ADR) |

### §3.2 🔬 Ниша 2 — Исследователь / PhD-трек

| Аспект Layer 2 | Настройка |
|---|---|
| **Подтипы проектов** | papers-in-progress / эксперименты / literature-reviews / коллаборации / конференции |
| **Роли CRM** | научрук / комитет / коллабораторы / ревьюеры / лаб-мейты / соавторы |
| **Темы гипотез** | исследовательские вопросы / методология / воспроизводимость / теор. рамки |
| **Каденс** | еженедельный literature-дайджест / месячное ревью экспериментов / per-paper milestone |
| **Голос-паттерны** | «paper / эксперимент / гипотеза / данные / цитата» → Sources/Hypotheses/Claims |
| **Life Pulse акцент** | устойчивость (PhD = марафон) / изоляция / прокрастинация |
| **Что добавить в Knowledge** | Claims с обязательным source_quote (научная строгость) |

### §3.3 💼 Ниша 3 — Предприниматель / соло-фаундер

| Аспект Layer 2 | Настройка |
|---|---|
| **Подтипы проектов** | клиенты / продукты / маркетинг / операции / фандрейзинг |
| **Роли CRM** | клиенты / инвесторы / эдвайзеры / партнёры / наймы / поставщики |
| **Темы гипотез** | валидация рынка / ценообразование / каналы / unit-эконика / retention |
| **Каденс** | еженедельное ревью продаж / месячный cohort retention / квартальный burn |
| **Голос-паттерны** | «звонок / лид / сделка / метрика / клиент сказал» → CRM/Tasks/Claims |
| **Life Pulse акцент** | финансовый пульс (runway) / стресс / work-life граница |
| **Что добавить в Strategic** | очередь решений активнее (быстрые pivot-решения) |

### §3.4 📐 Ниша 4 — Методолог / преподаватель (Левенчук-уровень)

| Аспект Layer 2 | Настройка |
|---|---|
| **Подтипы проектов** | курсы / когорты / разработка-методологии / письмо / консалтинг |
| **Роли CRM** | студенты / со-преподаватели / методологи-коллеги / клиенты / выпускники |
| **Темы гипотез** | педагогика / дизайн-программы / измерение исходов / онтологии понятий |
| **Каденс** | per-cohort retention check / per-course outcome review / месячный разбор фидбэка |
| **Голос-паттерны** | «когорта / сессия / исход / фидбэк студента / понятие» → CRM/Reviews/Concepts |

| **Что добавить в Knowledge** | Concepts как первоклассная база (точные определения — ядро методолога) |

### §3.5 🤝 Ниша 5 — Гуманитарий / управление жизнью (Дмитрий-стиль)

| Аспект Layer 2 | Настройка |
|---|---|
| **Подтипы проектов** | отношения / здоровье / личные-финансы / хобби / семья |
| **Роли CRM** | семья / близкие-друзья / профессионалы (врачи / бухгалтеры / эдвайзеры) |
| **Темы гипотез** | дизайн-образа-жизни / протоколы здоровья / качество отношений |
| **Каденс** | еженедельный life-pulse / месячный аудит отношений / квартальная сверка ценностей |
| **Голос-паттерны** | «здоровье / отношения / пульс / решение / самочувствие» → Life Pulse/Daily Log |
| **Life Pulse акцент** | центральная база (а не вспомогательная) — это ядро ниши |
| **Что упростить** | Knowledge до минимума (Sources только); фокус на Daily Log + Life Pulse + CRM |

### §3.6 Сравнение 5 ниш одним взглядом

| Ниша | Центральная база | Что усиливают | Что упрощают |
|---|---|---|---|
| 🛠️ Инженер | Projects + Tasks | Knowledge (code) | Life Pulse |
| 🔬 Исследователь | Hypotheses + Sources | Claims (строгость) | Tasks |
| 💼 Предприниматель | CRM + Projects | Strategic (решения) | Concepts |
| 📐 Методолог | Concepts + Reviews | Knowledge (определения) | — |
| 🤝 Гуманитарий | Life Pulse + Daily Log | CRM (отношения) | Knowledge |

**Вывод:** один и тот же Layer 1 обслуживает 5 очень разных жизней — меняется только
**что в центре** и **какие подтипы/роли/темы**. Это и есть универсальность.

## §4 R1-решение — какие ниши включить в раздачу

Промпт рекомендует все 5 (Инженер + Исследователь + Предприниматель + Методолог +
Гуманитарий). Финал — за Русланом (§9.E п.3). Можно:
- Раздавать все 5 готовых overlay'ев.
- Начать с 1-2 (например, Гуманитарий для Дмитрия + Методолог для Левенчука).
- Дать только Layer 1 + инструкцию «собери свой Layer 2».

(AP-6 — держим варианты открытыми.)

## §5 Анти-паттерны при форке (чего НЕ делать)

- ❌ **Переделывать Layer 1** — ломает совместимость и будущие апдейты
- ❌ **Удалять поля provenance** (источник / уверенность) — ломает дисциплину фундамента
- ❌ **Ломать зеркало в файлах** — рвёт синк с Claude Code
- ❌ **Убирать DRAFT-only для голоса** — риск потери данных
- ❌ **Хардкодить Notion page ID в Layer 1** — должны быть только в Layer 2 / `.env`
- ❌ **Добавлять механики замков/lock-in** — нарушение R12
- ❌ **Пропускать засев Strategic** — без POINT A/B система теряет навигацию
- ❌ **Имена-личности в Layer 1** («Мой дневник» вместо «Daily Log») — ломает fork-friendly

## §6 Апдейты и миграция

- Layer 1 апдейтится из апстрима (Jetix поддерживает универсальный шаблон).
- Форк подтягивает апдейты Layer 1, **не затирая Layer 2** (свой словарь сохраняется).
- Скрипт миграции: `tools/personal_os_upgrade.py` (Phase 6).
- Обратная совместимость: Layer 1 v2 читает данные v1.
- **Апдейты добровольны** — никто не форсит обновление (R12: не запираем в версию).

## §7 ⭐ R12 в fork-friendly дизайне (на человеческом)

R12 = «нельзя запирать человека и выжимать из него больше согласованного». В контексте
шаблона это значит:

| Принцип R12 | Как реализован в шаблоне |
|---|---|
| **Нет замков** | твои данные — твои; экспорт в `.md` файлы в любой момент |
| **Нет извлечения** | Jetix НЕ получает данные из форкнутых копий (отдельные воркспейсы) |
| **Fork-and-leave** | перестал пользоваться — данные остаются у тебя в твоём Notion |
| **Портативность данных** | `.md` файлы = переносимый канонический формат (не привязан к Notion) |
| **Апдейты добровольны** | обновляешь Layer 1 когда хочешь, не принудительно |
| **Прозрачность** | весь шаблон открыт, видно как устроен (не чёрный ящик) |

**Парная рамка (R12 paired-frame).** Раз шаблон будет раздаваться людям — он не должен
стать инструментом вовлечения-в-секту или вытягивания. Поэтому: ноль механик
удержания, ноль FOMO, ноль «закрытого клуба». Человек берёт инструмент, делает себя
сильнее, и уходит когда хочет. Это проверяется этическим чеком (Phase 5 §7).

- **R12 встроен:** владение данными / fork-and-leave / портативность / добровольные
