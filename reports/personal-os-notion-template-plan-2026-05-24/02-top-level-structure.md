# Phase 2 — Структура: какие базы и как они связаны

> «скелет» Notion-воркспейса: какие базы данных, как они называются, как связаны и
> как выглядит главная страница-хаб. Я даю **пул из 14 баз-кандидатов** и **3 варианта
> сборки** (8 / 10-12 / 14 баз) с честными плюсами-минусами. Финальный набор выбираешь
> ты — это R1-решение (§9.E промпта, пункт 1).

## §2 ⭐ 3 варианта сборки (R1 — Руслан выбирает)

Один и тот же набор кирпичей можно собрать плотнее или раздельнее. Три варианта —
от «минимум баз, максимум простоты» до «максимум раздельности, максимум контроля».

### §2.A Вариант LITE — 8 баз (минимум сущностей)

Слияния:
- Projects **+** Tasks → одна база (задачи как под-элементы / чек-листы)
- Concepts **+** Sources **+** Claims → одна база **Knowledge** (поле type)
- CRM People **+** Orgs → одна база **CRM** (поле kind: person/org)

Итого 8 баз: **Daily Log / Projects / Knowledge / Hypotheses / CRM / Reviews /
Strategic / Life Pulse** (+ Ideas и Reference как простые страницы, не базы).

| ✅ Плюсы | ⚠️ Минусы |
|---|---|
| Меньше навигации, быстрый старт | Задачи теряются внутри проектов |
| Идеально для форка (проще скопировать) | Knowledge-база «перегружена» тремя типами |
| Меньше связей = меньше путаницы | CRM People/Orgs в одной базе — связи неудобнее |

**Кому:** новичкам, гуманитариям, тем, кто хочет «просто начать».

### §2.B Вариант STANDARD — 11 баз (рекомендуемый баланс)

Базы 1-13 минус слияния, минус Tasks (задачи = чек-листы в проектах + Daily Log),
минус Concepts (сливаем в Sources на старте):

**Daily Log / Projects / Sources / Claims / Hypotheses / Ideas Bank / CRM People /
CRM Orgs / Reviews / Strategic / Life Pulse** (Reference = страница).

| ✅ Плюсы | ⚠️ Минусы |
|---|---|
| Каждая сущность отдельно — чистые связи | 11 баз — надо привыкнуть к навигации |
| Hypotheses отдельно (запрос Руслана) | Чуть дольше форкается |
| CRM People/Orgs раздельно — как в `crm/` | |

**Кому:** Руслану и людям, кто уже жил в системе. Золотая середина.

### §2.C Вариант FULL — 14 баз (максимум контроля)

Все 14 баз из §1 раздельно, включая Tasks, Concepts, Ideas Bank, Reference как базы.

| ✅ Плюсы | ⚠️ Минусы |
|---|---|
| Максимальная гранулярность и связи | Сложно форкать (много настройки) |
| Tasks как отдельный поток (GTD-стиль) | Риск over-engineering для одного человека |
| Concepts отдельно (полноценная wiki) | Навигация требует дисциплины |

**Кому:** инженерам/методологам, кто любит максимальную структуру.

### §2.D Сравнение одним взглядом

| Критерий | LITE (8) | STANDARD (11) | FULL (14) |
|---|---|---|---|
| Скорость старта | 🟢 быстро | 🟡 средне | 🔴 дольше |
| Простота форка | 🟢 легко | 🟡 средне | 🔴 сложнее |
| Чистота связей | 🔴 смешанные | 🟢 чистые | 🟢 чистые |
| Риск перегруза | 🟢 низкий | 🟡 средний | 🔴 высокий |
| Под кого | гуманитарий/новичок | Руслан/опытный | инженер/методолог |

**Это R1-решение.** Моя задача — разложить варианты, не выбрать за тебя. (AP-6 —
держим варианты открытыми.)

## §3 Command Center — главная страница-хаб

Это **Part 9 в Notion-форме**: одна страница, которую открываешь первой утром. Всё
видно с одного взгляда, никуда не нужно нырять.

### §3.A Блоки Command Center (сверху вниз)

| Блок | Что показывает | Откуда тянет (связь) |
|---|---|---|
| 🎯 **Фокус дня** | сегодняшняя запись Daily Log + 1-3 главных дела | Daily Log (today) + Tasks (priority) |
| 🎤 **Быстрый ввод** | кнопка/виджет голосового черновика + инбокс DRAFT'ов | очередь DRAFT (Phase 4) |
| 🚀 **Активное состояние** | топ-3 проекта + топ-3 гипотезы + 5 тёплых контактов | Projects + Hypotheses + CRM (filtered) |
| ⏰ **Что пора отревьюить** | напоминание: неделя/месяц/квартал/год | Reviews (по датам) |
| 🧭 **Стратегический якорь** | POINT B + план текущего квартала | Strategic (filtered) |
| ❤️ **Пульс жизни** | последние оценки энергии/сна/отношений | Life Pulse (latest) |
| 🔴 **Застрявшее** | проекты active без касания >14 дней + застрявшие контакты | Projects + CRM (stuck filter) |

### §3.B Принцип дизайна хаба

- **Только просмотр + быстрые действия.** Хаб ничего не хранит сам — он показывает
  отфильтрованные виды из баз (Notion linked views).
- **Один экран без скролла** (по возможности) — фокус дня + быстрый ввод сверху.
- **Сигналы важного:** «что пора отревьюить» и «застрявшее» — красные флажки, чтобы
  важное не утонуло.

## §5 ⭐ Граф связей между базами (relations)

Связи — это то, что превращает набор баз в **систему**. Без них — 14 отдельных
табличек. С ними — всё переплетено и навигируемо.

### §5.A Карта связей (текстом; mermaid NT-2 в Phase 8)

| Связь | Тип | Зачем |
|---|---|---|
| Projects ↔ Tasks | 1:N | у проекта много задач |
| Projects ↔ Daily Log | N:N | дневная запись упоминает активные проекты |
| Projects ↔ Hypotheses | N:N | гипотезы проверяются в контексте проектов |
| Projects ↔ CRM People | N:N | у проекта есть коллабораторы |
| Hypotheses ↔ Sources | N:N | гипотеза копит доказательства из источников |
| Hypotheses ↔ Claims | N:N | подтверждённая гипотеза → проверенное утверждение |
| Claims ↔ Sources | N:N | у утверждения есть источник (provenance) |
| Concepts ↔ Sources/Claims | N:N | понятие связано с источниками и утверждениями |
| CRM People ↔ CRM Orgs | N:N | человек работает в организации |
| CRM People ↔ Daily Log | N:N | сегодня общался с человеком (касание) |
| Ideas Bank ↔ Projects/Hypotheses | N:N | идея вырастает в проект или гипотезу |
| Reviews ↔ Projects/Hypotheses/Life Pulse | 1:N | ревью периода сворачивает всё это |
| Daily Log ↔ Life Pulse | 1:1 (день) | у дня есть оценка пульса |
| Strategic ↔ Projects | N:N | проекты служат стратегической цели (POINT B) |

### §5.B Главные «оси» системы (3 потока)

1. **Поток дня:** Daily Log → касается Projects + Hypotheses + CRM + Life Pulse.
   Это ежедневная пульсация — всё, что произошло сегодня, цепляется к записи дня.
2. **Поток познания:** Hypotheses → Sources → Claims → Concepts. «Что хочу узнать»
   → собираю доказательства → подтверждаю → формирую понятие. Цикл обучения.
3. **Поток рефлексии:** Reviews сворачивает Projects + Hypotheses + Life Pulse за
   период → влияет на Strategic (POINT B). Это цикл «учусь на себе» (Part 5).
