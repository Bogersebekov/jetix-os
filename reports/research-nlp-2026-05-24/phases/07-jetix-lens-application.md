## §7.4 Q3 — Welcome-frame O-144 enhancement (NLP-derived items only где R12 + empirical pass)

The Welcome-frame O-144 9-section playbook from propaganda research bucket already covers most relevant ground. Phase 7 NLP supplements it with:

### O-144 §2 «Listen-before-pitch» enhancement

**Replace** generic NLP rapport claims **with** Bolton reflective-response protocol:
- Posture of involvement (attending)
- Door-openers — non-coercive invitations to talk
- Paraphrase what you heard (in your own words)
- Reflect feelings («It sounds like you felt X when...»)
- Reflect meanings (combine content + feeling)
- Summative reflection periodically

**SKIP** во время Welcome-frame:
- Calibration «reading» on partner micro-cues for compliance leverage
- Predicate-matching as covert influence (instead: paraphrase naturally)
- Eye-cue interpretation
- Pacing-leading к outcome

### O-144 §3 «Partner-perspective audit» (new addition)

**Add** Perceptual Positions discipline (Dilts) — STRIPPED of identity-binding:

Before sending Welcome-frame invitation, run **4 positions**:
1. **First position (self)**: «What am I asking for? What am I offering?»
2. **Second position (partner)**: «If I were them receiving this, how would it land?»
3. **Third position (neutral observer)**: «From outside, does this look honest? Coercive? Manipulative?»
4. **Fourth position (system / Jetix)**: «Does this preserve fork-and-leave? Does this honor R12?»

This is a **modified Bolton-style** discipline — not an «install Dilts levels» but a 4-perspective check before action.

### O-144 §5 «Paired-frame disclosure» enhancement

**Reinforce existing** R12 paired-frame discipline:
- For every offer, surface limitation in same message
- For every ask, surface what's offered in return
- For every commitment-seeking, surface fork-and-leave option
- For every claim, surface what would falsify it

**SKIP во время paired-frame:**
- Sleight of Mouth «overcome objections» — partner's objection IS information, not problem to dissolve
- «Future-pace them into the offer» — only future-pace your own next-step
- Anchoring positive state to «yes» response — let response emerge без anchored state

### O-144 §9 «Forward-pacing» enhancement (workshop graduation)

**Add** voluntary forward-pacing к the workshop closing:
- Imagine yourself 30 days from now applying X
- Imagine yourself 6 months from now if you've integrated X
- Imagine yourself opting out — what would that look like? («fork-and-leave preserved» reinforcement)

This is **Gollwitzer implementation intentions** + **fork-and-leave reinforcement** combined.

## §7.6 Q5 — Development→Promotion O-160 (perceptual positions + R12 paired-frame)

The transition from «development partner» к «promoted Clan member» (per O-160) is a high-R12-sensitivity moment. Phase 7 NLP adds:

### Use perceptual positions to design transition signaling

Before promoting a development partner:
1. **First position**: «Is this person ready for full Clan membership?»
2. **Second position**: «How would they experience this promotion? Pressure? Honor? Burden?»
3. **Third position**: «From outside, does this promotion preserve fork-and-leave?»
4. **Fourth position (Jetix)**: «Does adding this person maintain R12 cohort discipline?»

If any position raises concern → don't promote.

### R12 paired-frame for promotion offer

When extending promotion invitation:
- Surface what's gained AND what's required
- Surface fork-and-leave option explicitly («if this doesn't fit, walk; relationship preserved»)
- Time-limit decision moderately — but never via «massive pain associated with not deciding now»
- Disclose what would falsify the offer

### What NOT to do during promotion

- ❌ Anchor positive state to «yes» response (covert anchoring)
- ❌ Apply Sleight of Mouth to dissolve any reluctance
- ❌ Mirror their preferred metaprograms in the pitch covertly
- ❌ Use Milton-pattern embedded commands («you can join us … now»)
- ❌ Identity-bind via Logical Levels («you ARE a Clan member, you just haven't said yes yet»)

These are all SKIP-list class-2 covert patterns.

## §7.11 Operational R12 Discipline Test (NLP-specific)

Plus к propaganda Phase 7's 9-clause + 16-point + 8-point tests, Phase 7 NLP introduces an **empirical-status filter** as 5th operational discipline:

### NLP Empirical-Status Filter (5-clause test)

For ANY NLP-derived technique under consideration for Jetix import:

1. **Independent empirical source?** Can you cite a non-NLP-affiliated peer-reviewed source for the FUNCTIONAL effect? **Y → proceed / N → SKIP**
2. **Better-validated equivalent in adjacent literature?** Is there a Bolton / Rogers / CBT / ACT / Locke-Latham / Gollwitzer / Clear / Galinsky equivalent? **Y → import from there instead / N → continue**
3. **Falsified by independent research?** Has it been refuted by Sharpley / Heap / NRC / Wiseman / Sturt / Roderique-Davies? **Y → HARD SKIP / N → continue**
4. **R12-compatible in transparent / voluntary / fork-and-leave-preserving deployment?** **Y → continue / N → SKIP**
5. **R12-compatible in covert deployment OR built-for-covert deployment?** If covert deployment violates R12 AND technique's defining feature is covertness → **SKIP**

5-of-5 PASS → IMPORT
Any FAIL → SKIP
