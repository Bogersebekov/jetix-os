## А

### Альфа (Alpha)
- **Determination (Левенчук «Методология 2025»):** «предмет метода, который может быть и физическим объектом (системой), и абстрактным объектом (описанием). […] Альфа позволяет управлять вниманием создателя в ходе исполнения длинных цепочек операций.» `[R-B §2.1 L162-166]`
- **SEMAT origin (Jacobson 2012):** «essential element of the software-engineering endeavor… relevant to an assessment of its progress and health» `[R-B §2.1 L170]`
- **Modern ШСМ kernel (LJ 2024-04):** 4 primary alphas — воплощение системы / описание системы / функция/метод работ / работы системы `[02-livejournal §Post 1]`
- **FPF encoding:** word ABSENT; mechanism via A.2.5 U.RoleStateGraph + A.15.1 U.Work + past-participle state names

### Активный вывод / Active Inference
- **Per R-A §1.2:** Friston's Free Energy Principle; incorporated by Левенчуком as physics-based foundation для Gen-3 systems thinking ontology
- **Bridge person:** Ivan Metelkin (Active Inference Institute co-founder + МИМ mentor)

### Архитектура мышления / Architecture of Thinking
- **Per FPF Spec L120:** «FPF is closer to an architecture for reasoning: a set of reusable patterns and working forms that help teams turn tacit thinking into shared, reviewable work»

## Б

### Бесконечное стратегирование / Infinite Strategizing
- **Per R-A §4.1:** «Continuous, never-ending practice of orientation and decision-making. Levenchuk emphasizes 'бесконечное стратегирование' — the process is more important than any artifact produced. Against 'strategy documents' as endpoint.»

### Bounded Context (см. также U.BoundedContext)
- **Per FPF Spec L447:** primary mechanism для establishing local CWA (Closed-World Assumption)
- See also: Контекст ограниченный

## В

### Воплощение системы / System Embodiment
- **Per R-D §2.2:** «воплощение системы привязывает все рассуждения по схеме к физической реальности (воплощение системы — 4D индивид)»
- **Replaces:** Essence «Software System» alpha
- **States:** в виде сырья → в виде частей → демонстрируемо → готово → эксплуатируется → выведено из эксплуатации `[R-D §3.1]`

### Внешние проектные роли / External Project Roles
- **Per R-D Section 3.1:** Replaces «Stakeholders» в most recent ШСМ kernel editions. Sub-alphas для specific roles: финансист заказчика, архитектор, etc.

### Возможности / Opportunity
- **Per R-D:** Customer-area alpha; preserved from Essence verbatim
- **States (Essence original):** Identified → Solution Needed → Value Established → Viable → Addressed → Benefit Accrued

## Г

### Граф создания / Creation Graph
- **Per R-B §3.1:** «по мотивам OMG Essence 2.0:2024. Агенты-создатели (тогда – люди и их организации, сейчас включаем и AI) как надсистемы целевой системы»
- **Three-level mereology:** Надсистема ↔ Целевая система ↔ Системы создания
- **Five typed edges:** part-of, creates, operates-in, methodologically-uses, constrained-by
- **FPF encoding:** A.1 Holonic Foundation + A.14 Advanced Mereology + B.2 MHT

### Guard-Rails (E.5.*)
- **Per FPF Spec L745:** part of FPF Constitution alongside Eleven Pillars (E.2)
- **E.5.1 DevOps Lexical Firewall:** core never cites CI pipelines/file formats
- **E.5.2 Notational Independence:** forbids locking into OWL/JSON-LD/category-theory

## Д

### Действия / Actions (in U.RoleEnactment)
- **Per FPF Spec C.3.x:** Kinds & KindSignature ground action descriptions

### Determined by usage / «определяется статистически»
- **Per psybertron review (Левенчуковский verbatim):** «The meanings of terms are determined statistically, not precisely» through examining «diagrams, texts, and sets of expanded statements containing the term of interest» `[02-livejournal §Post 6]`

### DRR (Design Rationale Record)
- **Per FPF E.9:** Durable decision documentation artifact. Contains rationale + alternatives + open issues. Used когда normative change или durable canon rationale must be published.

## Е

### Episteme (см. U.Episteme)
- **Per FPF Spec L1095:** sub-archetype of U.Holon; knowledge claim holon
- **Slot structure (C.2.1):** DescribedEntity + GroundingHolon + ClaimGraph + Viewpoint

### Eleven Pillars (E.2)
- **Per FPF Spec L745:** part of FPF Constitution. 11 normative content-constraining principles.
- **Identified:** P-1 Cognitive Elegance, P-2 Didactic Primacy, P-7 Pragmatic Utility, P-8 Cross-Scale Consistency, P-10 Open-Ended Evolution. Full P-1 through P-11 list TO-COLLECT Phase B.

## З

### Захват внимания / Attention Management
- **Per R-A §4.1:** primary function of системное мышление = «attention management in complex collective projects»

## И

### Инженер-менеджер / Engineer-Manager
- **Per R-A glossary:** МИМ's target practitioner identity — «someone who does both systems engineering and systems management»
- МИМ 2025-2026 rebrand was «Мастерская инженеров-менеджеров»

### Интеллект-стек / Intellect Stack
- **Per R-A §4.1:** «hierarchical set of 16-17 transdisciplines constituting 21st-century cognitive competence. Replaces STEM as the standard.»
- **Current 2026 count:** 16 (system-school.ru/stack verified 2026-05-17)
- **FPF's own 5-layer version:** L1 Structure & Reality / L2 Knowledge & Reasoning / L3 Action & Execution / L4 Strategy & Rationality / L5 Governance & Purpose `[FPF Spec L718]`

### Исполнитель / Executor
- **Per R-B §1.1 (Левенчук):** Agent who carries a role; «всё что из методов умеет делать агент – это личность агента»
- **NOT the role itself** — bind at task-creation, separate concerns

### IWE — Intelligent Workflow Engine
- **Per R-A §5.1:** «AI personal guide that tracks learner progress, answers questions, provides route recommendations — the practical implementation of the exocortex concept within the school»
- **Per Левенчуковский TG 2026-05-17 C5:** «там примерно всё так же устроено [как наша система]. Но внутри там интеллект опирается на тот же FPF — и понятно, за счёт чего его IWE умнее конкурентов»
- **Platform:** embedded в aisystant.system-school.ru (paid)
- **MIM 10th conf 2026 talks:** Tserenov «AI in МИМ: IWE and personal guidance»; Sergeev «IWE as personal assistant for agricultural consultants»

## К

### Команда / Team
- **Per Essence + R-D:** Endeavor-area alpha; in ШСМ extended to include AI agents
- **States (Essence original):** Seeded → Formed → Collaborating → Performing → Adjourned

### Контекст ограниченный / U.BoundedContext (A.1.1)
- **Per FPF Spec L447:** «primary mechanism for establishing a local CWA. Within a Bounded Context, a specific set of models, rules, and invariants is declared to be authoritative.»

### Конструктив / Construct (агента)
- **Per R-D glossary:** Physical object enacting a role (person, team, AI system)

## Л

### Личность / Personality (агента)
- **Per Левенчук «Методология 2025»:** «всё что из методов умеет делать агент (совокупность мастерства агента) – это личность агента»

## М

### Мастерская инженеров-менеджеров (МИМ)
- **Per R-A §1.1:** Rebrand 2025 from ШСМ (Школа системного менеджмента) → МИМ (Мастерская инженеров-менеджеров)
- **Per 9th MIM conf 2025 theme:** «От школы менеджмента к клубу создателей»
- Tseren Tserenov = Managing Partner; Anatoly Levenchuk = Scientific Director

### Метод / Method
- **Per Левенчук «Методология 2025» (R-D §5.2):** primary term; treats «practice / process / workflow / engineering process / development process / methodology / strategy / culture» as synonyms or specializations
- **Hierarchy:** Метод разработки (top) → Практика (sub) → Стиль (variant)
- **FPF U.Method + U.MethodDescription (A.3):** «universal, representation-agnostic way to describe *how* an action is performed»

### Методология / Methodology
- **Per LJ 2024 1725757:** layer enabling agents «to describe ways of work of creators/agents in graphs of creating successful systems»
- **Three method-selection approaches:** Applied (domain knowledge) / Fundamental (system thinking) / Distributed Representations (AI/NN)

### Мышление письмом / Writing-as-Thinking
- **Per Левенчук «Системное мышление 2024 т.1»:** «Системное мышление происходит путём мышления моделированием… и письмом (с текстами на естественных языках, но с отслеживанием типов объектов и видов отношений объектов в этих текстах), поэтому внимание… удерживается… всё время проекта»
- **FPF dedicated section (Spec L515-533):** «Thinking Through Writing: The FPF Discipline of Conceptual Work»
- **Quality test (R-B §8.5):** «Узнал ли я что-то новое в процессе написания?» — yes = real writing-as-thinking
- **AI constraint (R-B §5.5):** «Без внешнего по отношению к LLM контуру обработки текста — никак, LLM всегда обманет»

## Н

### Надсистема / Suprasystem
- **Per R-D §4.2:** Top-level в граф создания — system into which target system is embedded; hosts Customer-area alphas

### NQD / Novelty-Quality-Diversity
- **Per FPF C.17-C.19 + R-A §5.2:** Open-ended evolution patterns; Q-diversity portfolio framework
- **Per Spec L770:** «OEE is a Pillar; NQD/E/E-LOG are the engine parts»

## О

### Обеспечивающая система / Enabling System
- **Per R-D §4.2:** Creator-level system в граф создания; hosts Endeavor-area alphas (Work, Team, Method, Technology)

### Определение системы / System Definition
- **Per R-D §3.1:** Replaces Essence «Requirements» alpha. Per Левенчуковский (2015 SE Essence paper): «source code is a work product of System Definition, not System Embodiment»

### Организационное звено / Organizational Unit (оргзвено)
- **Per R-D glossary:** Construct enacting role (synonym для конструктив)

### Operating System for Thought
- **Per FPF Spec L674 + L738 + Readme L9:** Левенчуковский canonical FPF self-description

## П

### Past-participle convention
- **Per LJ 2024-04 (Левенчуковский):** state-naming convention — «потребность определена», «быть выбранным», «быть закупленным», «быть проверенным в работе»
- **Why:** machine-readable; alpha-state checklist = AI agent onboarding protocol

### Pattern (FPF sense)
- **Per FPF Spec L348:** «A pattern is not a tutorial and not a "best practice" blog post; it is a **contract**: Problem frame → Problem → Forces → Solution, ending with a Conformance Checklist.»
- **Canonical template:** E.8

### P-2W / Principles to Work (E.TGA)
- **Per FPF Readme L350:** Canonical reasoning flow «P2W» = principles-to-work; unpacked via Transduction Graph Architecture (TGA / E.TGA)

### Понятизация / Conceptualization
- **Per R-A §1.2 + intellect-stack:** foundational transdiscipline; «practice of forming and stabilizing concepts»

## Р

### Работы / Work
- **Per R-D §3.1:** Endeavor-area alpha; per Левенчуковский «works of target system» (during operation) vs «works of creation system» (during development) — distinguished

### Развитие для развитых / Development for the Developed
- **Per 10th MIM conf 2026 + R-A §5.2:** Advanced practitioner track для МИМ masters and reformers; Левенчуковский талк title

### Разбор / Razbor (structured review)
- **Per LJ 2025-09 + R-A §2.2:** Weekly mentor-led 2-hour session in residencies. Review of participants' actual work projects (NOT lectures)

### Резидентура / Residency
- **Per R-A §2.2:** МИМ's primary learning format (replaced traditional courses). 10-19 weeks; weekly разборы; tracked R0 → R1 → R2 → advanced
- **Per LJ Autumn 2025 1777145:** ~10-15 hours/week minimum commitment

### Role-Method-Work seam (A.15)
- **Per FPF Spec L91:** Pattern enforcing clean separation of role definition / method capability / work enactment

### Роль / Role
- **Per Левенчук «Методология 2025» (R-B §1.1):** «Если говоришь роль, то за ролью скрывается метод»
- **R-B formula:** «Роль = сигнатура метода × интерес к системе × набор мастерства. Конкретный исполнитель — носитель роли, но не сама роль.»
- **FPF U.Role (A.2):** + U.RoleAssignment (A.2.1) + U.RoleStateGraph (A.2.5) + Agential Role (A.13)

## С

### СМД (Системо-Мысле-Деятельность) / SMD methodology
- **Per R-A §1.2:** Shchedrovitsky tradition; engaged + critiqued by Левенчуком; he rejects dialectical-Marxist epistemology в favor of Popperian falsificationism + engineering pragmatism

### Собранность / Collectedness
- **Per R-A glossary:** «practice of attention management and self-organization»
- **Residency R0:** «Рациональная работа / Собранность» — entry-level discipline, no prerequisites

### SoTA pack / State-of-the-Art Pack
- **Per FPF Part G:** Domain-specific knowledge package с CG-Spec/CG-Frame + harvest patterns + selectors + telemetry
- Per Левенчуковский: «not as one-time benchmark — as managed portfolio»

### Стейкхолдеры / Stakeholders
- **Per Essence + R-D §3.1:** Customer-area alpha; preserved in most ШСМ editions; later editions sometimes reframe as «внешние проектные роли»

### Стратегирование / Strategizing
- **Per Левенчук «Методология 2025» (R-B §4.1):** «метод выбора метода»
- **Strict ordering:** Стратегирование → Планирование → Работа
- **Trigger-based, NOT scheduled** (else = «методы-конвульсии»)
- **AI agents do NOT strategize** — no identity/commitment/skin-in-game
- **FPF encoding:** B.5.2 Abductive Loop + B.4 Canonical Evolution Loop + E.9 DRR + C.17-C.19 NQD/E-E-LOG

### Стратегирование рабочего развития (residency R0)
- See Residency entry

### Системное мышление / Systems Thinking
- **Per Левенчуковский (definitive 2024):** «transdiscipline for attention management in complex collective projects. Provides a shared ontological vocabulary for describing systems across all levels of reality using the same primitives. Not a recipe but a scaffolding for thinking.»

### Системный менеджмент / Systems Management
- ШСМ original namesake; Левенчуковский «Системный менеджмент 2023» book

### Системная инженерия / Systems Engineering
- ШСМ R2 residency content; «Системная инженерия 2022» book

## Х

### Холон / Holon (см. также U.Holon)
