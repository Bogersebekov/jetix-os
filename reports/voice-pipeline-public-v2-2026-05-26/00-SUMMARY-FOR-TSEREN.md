# Voice Pipeline — что это и как устроено

**Какую проблему решает.** Два разрыва. Первый — скорости: думаешь и говоришь 150-200 слов/мин,
печатаешь 30-50. На разрыве идеи теряются целиком или глохнет поток, пока причёсываешь формулировку.
Второй — места: лучшие мысли приходят не за столом, а на прогулке, в поездке, перед сном — где писать
нечем. Pipeline закрывает оба: голос на скорости мышления, телефон всегда с собой, формулировать
красиво не нужно (это работа следующей стадии). Результат — доля пойманных интересных мыслей
стремится к ~100% вместо привычных 2-3 из десятка.

**Главное — это не «заметки», а база данных.** Каждая голосовая → структурные единицы → встраиваются
в связанную wiki (паттерн LLM-wiki Карпатого: 9 типов сущностей, типизированный граф рёбер). И тут
**двойная фильтрация**, ядро всего:

- **Filter 1 — интеграция в базу.** Новая единица прогоняется против всей существующей wiki (у меня
  ~1100+ единиц). Система определяет: новое (→ создать + связать) / уже было (→ дубль/усиление/
  уточнение) / противоречит (→ сигнал — я сам себе возразил месяц спустя, обычно это теряется) /
  расширяет. На выходе не голый текст, а мысль на своём месте в картине мира.
- **Filter 2 — линза.** Реальный конфиг (keywords + веса + threshold + top-N), не метафора. Задаёшь
  призму — «вопросы недели» / «гипотеза, которую тестирую» / «открытые вопросы без ответа» — и база
  приносит топ-N релевантного из месяцев накопления, уже связанного рёбрами. Принцип: pipeline
  стабилен, меняется только линза.

**Почему это инструмент исследователя, а не дневник.** Фиксируются не идеи, а вопросы + наблюдения +
гипотезы + опровержения. Это ровно «книжки наблюдений» Дарвина и да Винчи (и Zettelkasten Лумана,
70 книг с картотеки; и Second Brain Форте). Тот же проверенный паттерн: structured capture →
time-decoupled answering (вопрос записан сегодня, ответ найден через месяцы, когда накопился контекст)
→ cross-domain linking → re-derivation. **Ново только ускорение:** Дарвину нужны были годы на
связывание — Claude находит родственное за минуты; 1100+ единиц вручную неуправляемы — авто-связывание
делает их workable. Но граница чёткая: **машина связывает, смысл рождается у человека.** Augmentation,
не замена. Это не «революция» — это extension паттерна, которому 180 лет.

**Технически (точно).** Захват — Wispr Flow (desktop) + Telegram (mobile). Транскрипция —
Groq `whisper-large-v3`. Извлечение/фильтрация — Claude через **CC headless на Max-подписке (без
API-ключа)**; модели `sonnet-4-6` (extract) и `opus-4-7` (filter+meta-analysis). Конвейер:
транскрипция → extraction (12 категорий) → **СТОП на ревью** → filter+dedup → review-отчёт →
**СТОП на ревью** → ручная дистрибуция. **Два human-gate обязательны.** Стоимость на дефолтном пути —
почти только Groq Whisper (~$1-2/день); Claude амортизирован в подписке.

**Ограничения (честно).** LLM теряет нюанс/иронию — поэтому verbatim (F2) хранится всегда, можно
переизвлечь. Голос = 30-50% шума, фильтруется на extraction. Batch lag — часы/дни (не для real-time).
Авто-дистрибуцию (`distribute.py`) **отключили намеренно**: через неделю автономной работы 40-60%
items оказались «звучит правильно, но не точно» — загрязнили бы базу. Ручной review между стадиями —
non-negotiable. Это substrate, не peer-reviewed методология.

**Про зрелость.** Репозиторий новый (~40 дней), но работа — год+: 1100+ заметок про мастерскую и
методы работы с информацией накоплены до репо (жили в Notion/vault). Репо = миграция этого substrate'а
на открытый сервер + синтез через LLM. То есть acceleration, не initiation. При этом согласен: для
партнёрств substrate не дозрел, один автор, peer-review не пройден. Открытый репозиторий — это и есть
приглашение проверить.
