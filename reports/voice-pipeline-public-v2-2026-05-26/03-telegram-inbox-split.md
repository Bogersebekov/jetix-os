# §B Базовый механизм — Telegram как единый inbox с разделением по намерению

> Где живёт capture-слой цепочки из §A. Принцип: **одна точка входа, разделённая по намерению.**
> Не десять приложений, а один мессенджер, который всегда в кармане; внутри него — несколько
> каналов по типу мысли. Эта секция честно разделяет, что здесь **дизайн/дисциплина captureа**,
> а что **уже автоматизировано в коде**.

---

## §B.0 Зачем именно Telegram

Telegram выбран не из любви к мессенджеру, а по трём практическим причинам:

1. **Всегда под рукой.** Тот же инструмент, в котором ты и так переписываешься, — значит, нулевой
   барьер входа. Не нужно открывать отдельное приложение для заметок (это и есть тот самый барьер,
   на котором умирают 90% систем заметок).
2. **Голос — first-class citizen.** Голосовая запись в один тап, без диалогов «сохранить / отменить».
   Записал, отпустил — улетело в чат. Поток не прерывается.
3. **Универсальный контейнер.** В тот же чат улетает не только голос: пересланное видео, ссылка на
   статью, контакт человека, скриншот, PDF. Одна труба для всего входящего материала.

Альтернатива на input-стороне за рабочим столом — **Wispr Flow** (Windows, push-to-talk): голос →
текст прямо в активное окно или файл. Telegram закрывает мобильный / «в движении» сценарий; Wispr
закрывает десктоп / «за столом». Вместе — capture покрыт во всех контекстах §A.2.

## §B.1 Разделение по намерению — 3 части + единый inbox

Ключевая идея Руслана (voice 26.05): мысли бывают **разного типа**, и тип лучше всего фиксировать
**в момент captureа**, а не вычислять потом. Поэтому inbox разделён на части по намерению — ты
выбираешь канал в ту же секунду, когда решаешь записать.

### Часть 1 — 🪞 Рефлексия / Отчёт жизни

*Общение с самим собой.* Сюда уходит:
- Самочувствие, уровень энергии, ритм («сегодня сел тяжело, к обеду разогнался»).
- Что улучшить, что не работает («эта встреча была пустой тратой — больше не назначать без agenda»).
- Daily / weekly рефлексия — как прошёл день, что получилось, где провалился.

**Намерение:** разговор с собой. Цель обработки — не «извлечь задачи», а свести в periodic
review-сводку: паттерны энергии, повторяющиеся проблемы, личные наблюдения. В системе это маппится
на категории `Личные наблюдения` + `Принципы` (см. §B.3).

### Часть 2 — 🔭 Общие идеи / Видение / Мудрости

*Долгая дуга.* Сюда уходит:
- Видение жизни и работы — куда всё идёт, кем хочу стать.
- Общие мысли о методологии, стратегии, устройстве мира.
- Философское, мудрости, long-arc инсайты («мастерство — это не объём знаний, а выбор нужного в момент»).

**Намерение:** не привязано к конкретному проекту, живёт годами. Цель обработки — интеграция в wiki
как концепции и видение (категории `Видение` / `Инсайты` / `Идеи`). Именно из этой части родилась,
например, метафора «мега-мастерская» (Phase 4 §D.4).

### Часть 3 — 📁 Per-проектные чаты

*Конкретика под контекст.* Отдельный чат под каждый активный проект (jetix-foundation / quick-money /
research / brand / …). Голосовая запись летит **сразу в правильный контекст**.

**Намерение:** routing решается в момент captureа — ты знаешь, про какой проект думаешь. Цель
обработки — project-specific KB: задачи, гипотезы, решения, идеи для проекта попадают в нужную папку
без ручной сортировки (категории `Задачи` / `Решения` / `Стратегические гипотезы` / `Идеи для проектов`).

### Plus — 📥 Единый inbox для внешних материалов

Помимо голоса, в Telegram сваливается всё внешнее **в одну точку**:
- 🎥 видео (пересланные / сохранённые)
- 🔗 статьи (ссылки)
- 👤 контакты (пересланные карточки людей)
- 📄 любые файлы / скриншоты / PDF

**Принцип «не разбросано».** Главная ценность — не сила какой-то одной фичи, а то, что **всё входящее
в одном месте**. Не «видео в закладках браузера, статьи в Pocket, контакты в телефоне, голос в
диктофоне». Одна труба → один разбор → ничего не теряется по углам десяти приложений.

## §B.2 Workflow: capture → routing → periodic pull → per-part обработка

```
[момент мысли]
     │  выбираешь канал по намерению (Часть 1/2/3 или общий inbox)
     ▼
[Telegram чат]  ←── голос / видео / ссылка / контакт / файл
     │  раз в несколько дней (или автоматизированно) — выгрузка
     ▼
[inbox/voice/]  ←── аудиофайлы попадают в рабочую папку pipeline
     │
     ▼
[конвейер]  ──→ каждая часть обрабатывается по своим правилам:
     ├─ Рефлексия   → review-сводки (Личные наблюдения / Принципы)
     ├─ Видение     → wiki-интеграция (Видение / Инсайты / Идеи)
     └─ Проектные   → project-specific KB (Задачи / Решения / Гипотезы)
```

**Periodic pull — ритм «раз в несколько дней».** Не каждую минуту (это сломало бы асинхронность из
§A.4), а пачкой. Накопилось → выгрузил → прогнал конвейер → прочитал отчёт → принял решения. Это и
есть batch-эффективность: конвейер видит несколько дней мыслей сразу и замечает связи между ними.

## §B.3 Что уже автоматизировано, а что — дисциплина captureа (честно, R6)

Здесь важно не приукрашивать. Разделим:

| Слой | Статус | Деталь |
|---|---|---|
| **Голос → текст** | ✅ автоматизировано | `tools/transcribe.py` берёт всё из `inbox/voice/` (Groq whisper-large-v3) |
| **Текст → 12 категорий** | ✅ автоматизировано | `tools/extract.py` тегирует каждую единицу одной из 12 категорий |
| **Категория → review-разделы** | ✅ автоматизировано | `tools/review_report.py` раскладывает по 7 секциям (Задачи / Идеи / KB / Решения / Видение / Контакты / Прочее) |
| **Контакты → CRM-черновик** | ✅ автоматизировано | step 2b: `crm_items[]` → `<slug>-DRAFT.md` (DRAFT-only) |
| **3-частный Telegram split** | 🟡 дисциплина captureа | Разделение по чатам — практика Руслана на входе; в код приходит как поток файлов в `inbox/voice/` |
| **Per-part разные правила обработки** | 🟡 частично | Реализовано через **категории** (extract) + **линзу** (orchestrator), а не через source-chat metadata. Намерение части ≈ ожидаемый набор категорий |
| **Telegram → `inbox/voice/` мост** | 🟡 ручной / полу-авто | Выгрузка из Telegram в рабочую папку — отдельный шаг (вручную или скриптом); не часть основного pipeline-кода |

**Вывод честный:** *концепция* «единый inbox, разделённый по намерению» — это **операционная
дисциплина captureа** Руслана, и она работает. *Реализация* per-part обработки достигается не через
«pipeline знает, из какого чата файл», а через **12 категорий на extraction** + **lens-driven фильтр**
(Phase 3) — намерение части воспроизводится через ожидаемый набор категорий. Telegram-бот как полностью
автоматический мост (chat → routing → folder) — это **upgrade-кандидат** (см. Phase 6 §F.3:
«Telegram bot integration ~2 часа setup»), а не уже работающая магия. Методолог, который захочет
воспроизвести, должен понимать эту границу.

## §B.4 Почему «один inbox по намерению» лучше двух крайностей

Две тупиковые крайности, между которыми сидит этот дизайн:

- **Крайность А — всё в одну кучу без разделения.** Один общий поток, ничего не размечено. Capture
  лёгкий, но разбор — ад: рефлексия, проектные задачи и философия перемешаны, конвейеру приходится
  угадывать намерение. Точность тегирования падает.
- **Крайность Б — десять приложений по типам.** Рефлексия в Day One, задачи в Todoist, идеи в
  Obsidian, контакты в CRM, видео в закладках. Разделение идеальное — но capture тяжёлый (какое
  приложение под эту мысль?), и в момент инсайта на прогулке ты не будешь выбирать из десяти.
  Барьер убивает охват (§A.3).

«Один inbox по намерению» берёт лучшее: **барьер captureа почти нулевой** (всё в Telegram, выбор из
3-4 чатов — мгновенный), но **намерение уже размечено** на входе, так что разбор знает контекст.
Это компромисс, оптимизированный под главную метрику — **охват captureа стремится к 100%**.

---

## §B.5 Связь с остальной системой

- **Вверх (Phase 1 §A):** Telegram = физический носитель цепочки ценности; именно он делает «телефон
  всегда с собой» → «~100% capture».
- **Вниз (Phase 3 §C):** из inbox материал уходит в БД-конвейер с двойной фильтрацией. Намерение
  части подсказывает, какая линза релевантна (рефлексия → lens «энергия/ритм»; проектный чат → lens
  «текущий фокус проекта»).
- **Вбок (Workshop frame):** единый inbox — это «приёмное окно мастерской»: всё, что мастер приносит
  с улицы (наблюдение, человек, статья, инструмент), складывается в одну точку и дальше расходится
  по верстакам.
