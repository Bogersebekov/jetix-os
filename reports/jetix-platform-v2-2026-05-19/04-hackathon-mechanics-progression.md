> Per Ruslan (text_010 §1): «сперва онлайн небольшие → потихоньку передвигаются в более масштабные offline». 5 Tiers spec — Tier 1 micro-hackathons → Tier 5 offline major projects. FPF lens B primary (Method).

## §0 Overview

| Tier | Rhythm | Participants | Format | Q3 2026→Q3 2027 cadence |
|------|--------|--------------|--------|--------------------------|
| **T1** Online micro | Day | 5-20 | Online async + 4-8hr sprint | Bi-weekly (cumulative ~25/year) |
| **T2** Online standard | Weekend | 20-100 | Online 36-48hr | Monthly (12/year) |
| **T3** Hybrid | Week | 50-200 | Online + Berlin meet | Quarterly (4/year) |

**Progression principle:** participants graduate Tier-to-Tier via demonstrated contribution + skill-test gating + cohort recommendation. Compounding cohort effect: Tier 5 participants drawn primarily from Tier 1-4 graduates [src: text_010 §1 «потихоньку передвигаются»].

**FPF discipline (Phase 4 cross-ref):** every Tier event ≡ FPF-discipline practice ground; participants gain FPF fluency Tier-by-Tier.

## §1 Tier 1 — Online Micro-Hackathons

### §1.1 Participant profile (Phase 1 cross-ref)

- **Primary:** Cat 1 (Domain experts entering); Cat 3 (Builders junior-mid); Cat 21 (Open-source contributors)
- **Secondary:** Cat 20 (Communities — recruiting funnel direct entry); Cat 14 (Connectors)
- **Cohort size:** 5-20 participants
- **Skill-floor:** demonstrated technical or methodological output (single artefact)
- **Entry mechanism:** open application + 1-question screen («show your best 1-week project»)

### §1.2 Resource inputs (Phase 2 cross-ref)

- R2 Compute (small allocation per participant — €50-200 GPU-credits)
- R6 Time (4-8 hours focused sprint + async work)
- R10 Mentorship (1-2 mentors / 5-10 participants ratio)
- R26 Methodology (lightweight FPF intro)
- R24 Code (platform substrate — registration + submission + voting)
- R29 Event hosting (online — Discord / Slack / video)
- R1 Money (prize pool €100-500; covers compute + small token recognition)

### §1.3 Output type / scale

- Single artefact per team (notebook / repo / prototype / dataset / methodology proposal / docs PR)
- Documentation requirement: README + 1-paragraph FPF F-G-R reflection
- Public-archive (all submissions Creative Commons / open-source default)
- Community visibility: showcase channel + brief presentation

### §1.4 Online vs offline mix

- **100% online** (async + 1-2 live sync sessions)
- Geographic neutrality (timezone-friendly windows)

### §1.5 Cohort building mechanism

- Random team-assignment OR self-organised teams (default: random for serendipity)
- Mentor-team pairing protocol (FPF U.PromiseContent — mentor commits time-window)
- Post-event: cohort-graduate list maintained; T2 invitation priority

### §1.6 Cross-Tier progression criteria (T1 → T2)

- Submitted artefact + F-G-R reflection
- Peer-feedback completed (FPF SpeechAct discipline)
- Mentor recommendation OR community-vote threshold
- T2 entry: invitation-priority for T1 graduates

### §1.7 Sponsor profile fit (Phase 1 Cat 8/16 cross-ref)

- Compute providers (Cat 16 sub-class) — credit grants
- Mid-tier corporates seeking talent-pipeline access
- Tool vendors (Cat 16) — license grants
- Foundations (Cat 19) — mission-aligned

**Sponsorship cost:** €1K-5K per Tier-1 event (12-25 events / year = €25K-125K annual sponsor flow).

**R12 alignment:** sponsor data-extraction = NONE (participant data confidential); sponsor brand-association only.

### §1.8 Success / falsifiability metrics

| Metric | Target Q3 2026 | Failure trigger |
|--------|----------------|------------------|
| Participant count / event | 10-20 | <5 sustained = format failure |
| Artefact submission rate | >80% | <50% = format broken |
| F-G-R reflection completion | >70% | <40% = FPF discipline failing |
| Mentor satisfaction | NPS >40 | <20 = mentor pool burning |
| T2 progression rate | >30% | <10% = funnel broken |

**Refute predicate:** if 3 consecutive T1 events miss participant-count or submission targets → halt + redesign.
