### §1.1 What Jetix provides

- **Infrastructure:** Compute access (R2), Office Site (R3 Berlin Grundstück), Tools licensing (R4), Physical hardware pool (R5)
- **Code substrate:** Platform implementing FPF protocol (R24), open-source-default
- **Methodology library:** FPF + IWE + VSM + Cynefin canon (R26)
- **Audit trail:** Provenance + transparency infrastructure (L6 protocol layer)
- **Convening capacity:** Event-hosting (R29) — online + Berlin Grundstück

### §1.2 What this enables

- Participants don't need to bootstrap infrastructure
- Cohort time spent on actual work (not coordination overhead)
- FPF discipline practiced на shared substrate
- Compounding (substrate improves with each cohort)

### §1.3 IP-1 boundary

- **Pre-authorized:** infrastructure provisioning per published rules
- **Default-Deny:** substrate-architecture changes (humans decide, especially Ruslan + Foundation governance)

## §6 Role 6 — NOT Decider (IP-1 STRICT canonical)

### §6.1 Pattern (Recursive Engine 2026-05-18)

Jetix surfaces candidates / options → humans ack → execution

Recursive engine pattern applies везде where decision-substrate emerges:
- Roy swarm surfaces project candidates → Ruslan acks (Pillar A)
- Matching algorithm surfaces match suggestions → both parties ack (L3 protocol)
- Cohort governance proposes changes → cohort votes (L3 protocol)
- Strategic reflection surfaces direction options → Ruslan + cohort ack (Pillar A quarterly)

### §6.2 What «decider» means

- **Setting direction:** what to build, what mission, what priorities
- **Selecting partners:** which Cat 16/17/18 to engage
- **Allocating capital:** how Phase 7 revenue distributed
- **Crafting strategy:** Pillar A North Star, Direction Cards, Lock Entries
- **Authorising significant action:** any Foundation-level write

### §6.3 What «NOT-decider» does NOT mean

- Jetix can execute pre-authorized substrate functions (infra / protocol / matching algo)
- Jetix surfaces candidates (NOT silent abstention)
- Jetix provides decision substrate (data / analysis / options) — Pillar A inputs
- Jetix runs the platform mechanism (operating-system level)

### §6.4 IP-1 boundary

Strict — described в §10 28-entry boundary cases.

- Fork-and-leave preserved (any participant can exit with their work-product + proportional asset claim)
- Mondragón wage-ratio cap (≤6:1 baseline within cohort)
- QF (Quadratic Funding) revenue distribution

### §7.2 Anti-extractor manifests

- **In revenue:** Phase 7 monetization R12 audit per variant

- **In exit:** Foundation governance ensures fork-and-leave preserved

### §7.3 NOT-extraction does NOT mean

- Jetix CAN earn sustainability revenue (consensual; transparent)
- Jetix CAN charge for premium services (transparent pricing)
- Jetix CAN receive sponsorship (R12-compliant)
- Jetix CAN return value to participants (incentive alignment)
