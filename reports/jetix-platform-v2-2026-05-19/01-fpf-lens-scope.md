## §1 Что описываем — четыре scope-lens (комбинированный default)

Jetix Platform v2 описывается одновременно через **четыре FPF lens-scope**. Любая single-lens projection теряет существенное; default = combination.

### §1.1 Lens A — Platform-as-system (`U.System` + `A.1 System`)

Jetix = **holonic supersystem** (FPF A.1; B.5.1 explore over multiple sub-holons). Состоит из:

- substrate-layer: shared infra, FPF protocol, audit trail [src: voice anchor text_010 §1 — «Jetix берёт organisational задачу»]
- cohort-layer: participants (20+ categories per Phase 1) [src: text_010 §1 «нам нужны все сука умнейшие люди»]
- protocol-layer: FPF interaction discipline (Phase 4)
- value-layer: humanity-development anchor (text_011 §1)

**Provenance** [src: voice anchor text_010 §1 line 32 «Jetix как бы берёт organisational задачу»]
**F-grade:** F2 (surface; not LOCK) — substrate-pattern surface, not Octagon LOCK content.

### §1.2 Lens B — Platform-as-method (`U.MethodDescription`)

Jetix = **methodology bundle** (FPF U.MethodDescription) для:

- recruiting + activating cohorts (Phase 1 categories methodology)
- resource-exchange protocol (Phase 4 FPF)
- hackathon progression online→offline (Phase 3)
- partner outreach reveal pattern (Phase 8)

**Provenance** [src: text_010 §1 «вот как будет проходить хакатон да что соответственно на платформе люди будут сперва там онлайн как-то ну работать над проектами над небольшими да потом это будет потихоньку передвигаться все в более более масштабные блять в оффлайн проекты»]
**F-grade:** F2.

### §1.3 Lens C — Platform-as-cohort-mechanism (`A.2 Role` + `U.SpeechAct`)

Jetix = **role-coordination scaffold** — конкретный set ролей и promise/commitment artefacts через которые cohorts формируются и operates.

- Roles via Phase 1 taxonomy (20+ classes); each Role = FPF A.2
- SpeechActs: offers / asks / promises / commitments
- Cohort formation = recursive matching engine [src: `decisions/strategic/JETIX-RECURSIVE-SELF-DEVELOPMENT-ENGINE-2026-05-18.md` IP-1 STRICT]

**F-grade:** F2.

### §1.4 Lens D — Platform-as-protocol (`A.6.B Bounded Context Bridge` + `U.PromiseContent`)

Jetix = **bridging protocol** для cross-system / cross-organisation / cross-culture партнёрств — bounded context bridge между разными мирами (corporate / academic / community / individual / governmental).

[src: `decisions/strategic/JETIX-SYSTEM-MERGER-PROTOCOL-FPF-2026-05-18.md` — H9 candidate; bridge modes: full-FPF / partial-FPF / opaque-bridge]

## §2 IP-1 STRICT boundary (HARD)

**Rule (constitutional, Pillar C Tier 2 §1-2):** Jetix = U.System substrate-as-thing; **NOT autonomous decider**. Humans = decision actors.

### §2.1 Pre-authorized (Jetix substrate-decisions OK)

- Infrastructure provisioning (compute, hosting, FPF runtime)
- FPF protocol minor updates (within acked frame)
- Dispute mediation (по pre-published rules)
- Audit trail emission
- Matching algorithm suggestions (surface к humans для approval)
- Hackathon logistics execution (post-Ruslan ack thematic + budget)
- Communication delivery (per pre-published templates)

### §2.2 Default-Deny (escalate к Ruslan + community)

- Project direction (что строить — humans decide)
- Hackathon themes (humans choose)
- Workshop curriculum (humans design)
- Partner selection (humans approve)
- Monetization mix (humans decide weights)
- Cohort acceptance criteria changes
- Values anchor modifications
- R12 enforcement parameter tuning

### §2.3 Edge cases (escalate-and-log)

- Cross-domain priority conflicts
- Resource allocation conflicts (limited compute / time / mentor hours)
- Mode negotiation (full-FPF vs opaque-bridge per partner)
- Saturation control (when «каждому запихаю» hits quality floor)
- Goal-divergence handling (when partner values diverge from Jetix values)

**Pattern:** Jetix surfaces options + impact estimates → humans (Ruslan + cohort governance) ack → execution. Recursive engine pattern applies [src: `decisions/strategic/JETIX-RECURSIVE-SELF-DEVELOPMENT-ENGINE-2026-05-18.md`].
