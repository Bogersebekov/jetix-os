# Direction #1 — 🧪 Метод: портфельный спек

## Введение: грань Foundation и что это направление

Jetix — мега-мастерская мирового уровня. Три грани Foundation: Мастерская (ГДЕ — пространство, 8 зон, инфраструктура) · Мастерство (ЧТО прокачивают — накопление методов, выбор нужного в момент, создание новых, решение уникальных задач) · Сеть (КАК распределено — mesh cells, pooling ресурсов). Direction #1 «Метод» лежит целиком в грани **Мастерство** и описывает педагогику этой грани: как именно человек учится методу мета-метода — способности не просто применять методы (уровень 1), не просто выбирать между ними (уровень 2), а **проектировать собственную стратегию выбора** (уровень 3, ⭐).

Ядро направления — Method V2 §J (6-step, LOCKED) плюс Extended 8-step с явным этапом подготовки (supplement 2026-05-26). Ключевой трюк: когда идёшь сразу — берёшь готовый метод из репертуара; когда сначала готовишься — появляется **картина**, и из неё собираешь **уникальный метод** под ситуацию. Именно здесь мастерство видно и накапливается. AI в этой архитектуре — preparation-amplifier (рутина prep), человек — action-strategist (мета-момент на переходе).

> **R1:** все формулировки ниже — варианты/опции. Финальные решения за Ruslan.
> **R2 STRICT:** Method V2 §J LOCKED — ссылки, не правки.
> **R11:** описания ЧЕГО содержит каждый документ — не сам публичный текст.
> **IP-1 STRICT:** имена = role-type примеры (T1-методолог, T3-контрибьютор и т.д.).

## §A — Главный документ

### A.1 One-pager (≤500 слов внутри)

**Аудитория.** Человек, который слышит «Jetix учит методу» — партнёр (T1-методолог, T2-ресурс), потенциальный участник, журналист, инвестор. Уровень: не знает про Method V2, знает что такое «метод».

**Формат.** Markdown-источник → лендинг-секция (HTML/React) + PDF-версия (A4, 2 столбца). Три варианта заголовка — на выбор Ruslan: (a) «Метод в квадрате: почему мастер выбирает, а не следует» (b) «Как учат тому, чему не учат в университетах» (c) «Мета-метод: уровень 3».

**Разделы внутри one-pager:**
1. Проблема-крючок — один абзац о том, что большинство обучения застряло на уровне 1 (учат методам), реже уровне 2 (учат выбирать).
2. Что такое мета-метод — 4 уровня кратко (L0 raw → L3 meta); Jetix = L3.
3. Почему это важно сейчас — один абзац про AI-стратификацию (O-182): рутина уходит к машинам, человек остаётся там, где нужен суждение и фронтир. L3 — это и есть «фронтир-способность».
4. Как это выглядит в практике — три строки: prep-gate → картина → уникальный метод.
5. Где это применяется — 2-3 примера уровня «узнай себя» (discovery call, документ с высоким стейком, пивот-решение).
6. Следующий шаг — CTA (конкретный, не «узнать больше»).

### A.2 Long-form (≤5K слов спек)

**Аудитория.** Тот же человек + партнёры T1-T4 кто оценивает глубину; alumni кто хочет углубиться после one-pager.

**Формат.** Markdown → веб-страница / PDF с навигацией по якорям.

**Разделы:**
1. **Онтология «всё = методы»** — краткий пересказ §1-§2 Method V2 (информация + методы переработки; самоуправляющиеся системы). Без воспроизведения LOCKED текста — только ссылка на механику.
2. **4 уровня мета-метода** — диаграмма L0→L3, описание чем каждый отличается, почему большинство застряло на L1-L2.
3. **6-step процедура** (Method V2 §J LOCKED-reference) — перечисление шагов, смысл каждого одной строкой, cross-link на полный текст.
4. **Extended 8-step: этап подготовки явно** — supplement 2026-05-26. Prep-gate (speed/depth), 5 calibration questions (сколько / насколько детально / когда / зачем / куда), картина, уникальный метод (The Trick), 3 перехода.
5. **AI-стратификация** — O-182 таблица (какие шаги AI, какие человек). Анти-паттерны: делегировать AI шаг 0 (over-prep) и шаг 4 (усреднённый метод).
6. **Gradient достаточности** — кривая Simon (satisficing), понятие «достаточно для действия».
7. **Worked examples** — 3 condensed примера (cross-link на §I полные).
8. **FAQ cross-link** → §H.
9. **Где применяется в Мастерской** — связь с 8 зонами.
10. **R12 честная рамка** — одна строка почему этот метод не превращается в манипулятивную технику.

## §C — Курс

**Применимость:** высокая — это ядровой обучающий продукт Direction #1.

**Аудитория.** Два сегмента: (A) «хочу понять метод» — новички, intake; (B) «хочу применять в своей практике» — практики, консультанты, менеджеры. Cohort delivery: 4-6 недель синхронно + async materials.

**Формат.** Модули + lessons + weekly cohort call (1.5 ч) + async Telegram thread + peer-review на exercises. Платформа: Notion (workbook) + видео (Loom/embedded). Cohort size: 8-15 чел (Мастерская-scale).

### Skeleton: 7 модулей

**Модуль 1 — Онтология метода (Week 1)**
- Outcome: участник объясняет чем L0/L1/L2/L3 отличаются на своём примере.
- Lessons: (a) «Всё = методы» — базовая онтология; (b) 4 уровня и где ты сейчас; (c) Почему L3 важен в эпоху AI.
- Материалы: one-pager §A + видео §B Scene 2 + Method V2 §J (ссылка).
- Exercise: написать 3 решения из последней недели → определить к какому уровню относится каждое.
- Assessment: peer review написанного + facilitator feedback.

**Модуль 2 — 6-step процедура (Week 1)**
- Outcome: участник прогоняет реальную задачу через 6 шагов.
- Lessons: (a) Каждый шаг в деталях; (b) Gradient достаточности (когда хватит info); (c) OODA и Cynefin как внутренние sub-методы.
- Материалы: Method V2 §J LOCKED (reference) + SOP §E + шаблон §F.
- Exercise: взять незакрытую задачу → пройти 6 шагов → записать решение.
- Assessment: сравнение «до 6-step» vs «после» — что изменилось.

**Модуль 3 — Extended 8-step: этап подготовки (Week 2)**
- Outcome: участник умеет применять prep-gate и 5 calibration Q.
- Lessons: (a) Prep-gate: speed/depth decision; (b) 5 calibration Q — сколько/насколько/когда/зачем/куда; (c) Что такое «картина» — сигнал перехода в действие; (d) Мастерство на 3 переходах.
- Материалы: supplement 2026-05-26 §3-§4 + diagrams PREP-1..PREP-4.
- Exercise: взять одну предстоящую задачу → применить prep-gate → если depth-mode, пройти подготовку → зафиксировать момент картины.
- Assessment: отчёт «что изменилось по сравнению с обычным подходом».

**Модуль 4 — The Trick: уникальный метод из картины (Week 2)**
- Outcome: участник осознанно создаёт уникальный метод хотя бы в одной задаче.
- Lessons: (a) Репертуар vs изобретение (P-06); (b) Как выглядит момент «картина есть»; (c) Процесс сборки уникального метода (под-выбор шаблон/уникальное = Mastery §C.6); (d) Анти-паттерн: сразу готовый метод на сложной задаче.
- Материалы: worked examples §I (все 5) + supplement §6.
- Exercise: «The Trick в деле» — взять задачу где раньше брал готовый метод, пройти depth-prep, собрать уникальный.
- Assessment: оценка peer-group: «насколько метод подогнан под ситуацию» vs «насколько шаблонный».

**Модуль 5 — AI как preparation-amplifier (Week 3)**
- Outcome: участник знает где делегировать AI в Extended 8-step и где не делегировать.
- Lessons: (a) O-182 стратификация: что AI, что человек; (b) Prompts для каждого AI-шага (Step 2 exec / Step 6 feedback); (c) Анти-паттерны: делегировать Step 0 (over-prep) и Step 4 (усреднённый метод); (d) Этика: AI = leverage, не замена суждения.
- Материалы: AI tooling §K + prompts-библиотека.
- Exercise: пройти Extended 8-step с AI-поддержкой на Step 2 → зафиксировать что осталось за человеком.
- Assessment: рефлексия: «где AI помог, где мешал».

**Модуль 6 — Meta-method в реальных контекстах (Week 3-4)**
- Outcome: участник применяет L3 в 3+ разных контекстах из своей жизни.
- Lessons: (a) Discovery call (sales-контекст); (b) Документ с высоким стейком (консалтинг); (c) Пивот-решение (проект/бизнес); (d) Daily routines — когда НЕ готовиться (speed-mode = тоже мастерство).
- Материалы: FAQ §H + worked examples §I full + SOP §E decision tree.
- Exercise: portfolio challenge — 5 задач за неделю, каждую с prep-gate, зафиксировать выборы.
- Assessment: portfolio review с facilitator.

**Модуль 7 — Своя meta-strategy: L3 в действии (Week 4)**
- Outcome: участник описывает свою персональную meta-strategy выбора методов.
- Lessons: (a) Что такое персональная meta-strategy (не шаблон, а система); (b) Как её собрать из накопленного опыта; (c) Как обновлять (feedback loop + method library); (d) R12-рамка: meta-strategy не для манипуляций.
- Материалы: шаблон personalMeta §F + method library template.
- Exercise: написать свою meta-strategy (1-2 стр.) + представить cohort.
- Assessment: peer + facilitator review по критериям: специфичность / обновляемость / R12-честность.

## §E — Инструкции / SOP

**Применимость:** высокая — нужна на входе любого воркшопа, курса, работы с партнёрами.

**Аудитория.** Действующий участник / фасилитатор / новый член команды — нужен операционный гайд «как делать», не «что это».

**Формат.** Markdown SOP → PDF printout + Notion-база. Версионирование: SOP-METHOD-v{N}.

### Step-by-step (Extended 8-step операционно):

**Step 0 — Prep-gate (Speed/Depth decision)**
1. Задай себе: «Высокий impact у этой задачи?» Если нет → переходи к Step 4 напрямую (speed-mode).
2. Если высокий impact → продолжи: «Знакома мне эта ситуация и есть ли проверенный метод?» Если оба ответа да → можно speed.
3. Если хотя бы одно нет → depth-mode → продолжи Step 1.

**Step 1 — Preparation method selection**
1. Выбери метод подготовки из репертуара (ресёрч / интервью / прототип / мозговой штурм / silent reading).
2. Или: попроси AI предложить 3 варианта метода подготовки для этого типа задачи (см. §K Prompt P-2).
3. Зафиксируй выбор в scratchpad.

**Step 2 — Preparation execution**
1. Запусти метод подготовки.
2. AI-heavy — делегируй сбор данных, синтез источников, форматирование (§K Prompts P-3..P-5).
3. Следи за сигналом «картина начинает складываться» — туман редеет, появляется структура.

**Step 3 — Picture formation (сигнал перехода)**
1. Картина = момент когда можешь объяснить ситуацию одной схемой или абзацем.
2. Если картины нет после запланированного времени → не расширяй prep бесконечно. Задай: «Что я узнаю за следующий час, что изменит картину?» Если нечего — переходи.
3. Зафиксируй картину письменно (1-3 предложения или схема).

**Step 4 — Action method select / create**
1. Это человеческий шаг — AI не делегировать (анти-паттерн §K).
2. Сначала: «Есть готовый метод из репертуара, подходящий к картине?» Если да → применить/адаптировать.
3. Если нет или картина уникальная → создать метод под ситуацию (sub-выбор по Mastery §C.6).
4. Зафиксируй метод и логику выбора.

**Step 5 — Action execution**
1. Выполни действие по выбранному методу.
2. Не уходи обратно в prep в процессе (анти-паттерн: переподготовка в ходе действия).

**Step 6 — Feedback collection**
1. Сразу после: что сработало, что нет — 3-5 пунктов.
2. AI может помочь: «Analyse these outcomes and surface 3 patterns» (§K Prompt P-7).

**Step 7 — Improvement note → method library**
1. Обнови method library: новый метод / уточнение существующего / анти-паттерн.
2. Если это первый раз — создай запись в personalmeta (§F Template T-3).

### Decision trees:

**DT-1: Speed vs Depth**
Impact высокий? → Depth. Нет → Знакомо+есть метод? → Speed. Нет → Depth.

**DT-2: Готовый метод vs Уникальный (Step 4)**
Картина стандартная → адаптировать готовый. Картина уникальная → создать под неё.

**DT-3: AI делегировать или нет**
Step 0 (detect): человек. Step 1 (выбор метода prep): человек + AI suggestion. Step 2 (execution): AI heavy. Step 3 (картина): человек. Step 4 (метод действия): человек. Step 5 (действие): зависит. Step 6 (feedback data): AI+человек. Step 7 (смысл): человек.

### Anti-patterns:

| Anti-pattern | Симптом | Решение |
|---|---|---|
| Analysis paralysis | Prep не заканчивается, картина «почти есть» | Применить Step 3 forced transition |
| Cargo-cult prep | Делаешь prep ритуально, без картины | Проверить: «что изменилось за последний час?» |
| AI Step 0 delegation | Спросил AI «нужна ли подготовка» → всегда depth | Step 0 = человек, всегда |
| AI Step 4 delegation | Попросил AI выбрать метод → получил усреднённое | Step 4 = человек; AI только список вариантов |
| Over-depth на routine задачах | Применяешь depth-mode к кофе | DT-1: проверь impact |
| Репертуарная ловушка | Знаешь много методов → применяешь знакомый на уникальной задаче | The Trick: картина → уникальный |

### Troubleshooting:

- «Не могу понять, depth или speed» → применяй простое правило: «Если ошибусь, насколько это откатит?» Много → depth.
- «Картина не складывается» → DT-2 в Step 3 forced transition.
- «AI дал хорошие варианты, зачем мне Step 4?» → AI варианты = репертуар; твоя задача — выбрать/адаптировать/изобрести под картину конкретной ситуации.

## §F — Шаблоны / Templates

### T-1: Prep-gate checklists (Notion DB / Markdown)

## §H — FAQ (20 вопросов)

Реальные вопросы, честные ответы. Без hype.

**H-01.** «Зачем вообще 4 уровня? Разве мало знать методы?»
Методы без выбора — это как словарный запас без грамматики. Можно произнести слова, но не построить нужную фразу. L3 = грамматика для методов.

**H-02.** «Чем L3 отличается от L2 на практике?»
L2: «Здесь подойдёт OODA, там — 6-step». L3: «Я проектирую когда применять OODA, когда 6-step, а когда изобрести что-то новое — и это мой осознанный выбор, а не автопилот».

**H-03.** «6-step — это не сложно запомнить?»
После 2-3 применений становится второй натурой. SOP §E — printout на одну страницу.

**H-04.** «Зачем Extended 8-step если есть 6-step?»
6-step предполагал этап подготовки, но не выделял явно. 8-step делает его видимым — а значит тренируемым.

**H-05.** «Что такое "картина" — как понять, что она есть?»
Ты можешь объяснить ситуацию одной схемой или абзацем незнакомому человеку. Туман рассеялся.

**H-06.** «Как долго должна длиться подготовка?»
Зависит от задачи и calibration. Speed-mode: 5 минут. Depth-mode: от часа до недели. Gradient достаточности даёт критерий: «Что я узнаю ещё — это изменит мой метод?» Если нет — картина есть.

**H-07.** «Что если я хожу по кругу в подготовке?»
Это analysis paralysis. Применить Step 3 forced transition: поставить таймер (30 мин) и написать картину «как есть сейчас».

**H-08.** «Когда НЕ нужна подготовка?»
Speed-mode: низкий impact, знакомая ситуация, есть проверенный метод. Мастерство = знать когда НЕ готовиться — не только когда готовиться.

**H-09.** «AI может написать мне план подготовки?»
Да — это полезно (Step 1, Step 2). Но AI не должен решать нужна ли подготовка (Step 0) и не должен выбирать/создавать метод действия (Step 4). Это человеческие шаги.

**H-10.** «Что такое "уникальный метод"? Не претенциозно ли?»
Не претенциозно. Уникальный = подогнанный под картину конкретной ситуации. Это может быть небольшая модификация существующего метода. Главное — не взять первый попавшийся шаблон, потому что он «подходит примерно».

**H-11.** «Это работает в корпорации или только для предпринимателей?»
Работает везде где есть решения с высоким стейком. Discovery call, стратегический документ, найм, продукт-пивот — контекст не важен.

**H-12.** «Сколько времени нужно чтобы освоить L3?»
Первые осознанные применения — за 1-2 недели. Автоматизация — 3-6 месяцев регулярной практики. Личная meta-strategy — после 50-100 применений.

**H-13.** «Method library — это нужно вести буквально?»
Рекомендуется фиксировать (шаблон §F T-4). Даже неформально — 5 строк после каждого применения достаточно. Накопление = compound.

**H-14.** «Как это связано с мастерством?»
Direction #1 — это педагогика Мастерства (грань Foundation). Мастер изобретает метод (не только применяет) — это и есть L3 в действии.

**H-15.** «Чем это отличается от design thinking / agile?»
DT и Agile — это конкретные методы (L1). Мета-метод учит когда применять DT, когда Agile, когда что-то третье, и как понять разницу (L3).

**H-16.** «Почему R12 упомянут в контексте метода?»
Мета-метод — мощный инструмент. R12 гарантирует: мы учим его для развития практикующего, а не для манипуляций или экстракции. Это честная рамка.

**H-17.** «Можно применять без прохождения курса?»
Да. SOP §E + шаблоны §F достаточны для самостоятельного старта. Курс даёт структуру, peer review и ускорение.

**H-18.** «Как это войдёт в curriculum Мастерской?»
Direction #1 — один из 14 направлений MetaPlan V3. В Мастерской он даёт базовый язык: все участники говорят «prep-gate», «картина», «L3» — это общий контекст.

**H-19.** «Есть примеры где этот подход не работает?»
Работает не оптимально: чрезвычайные ситуации (нет времени на prep-gate), полностью стандартизированные задачи (prep overhead превышает выгоду). Решение: speed-mode и минимальный prep.

**H-20.** «Что такое "мастерство на переходах"?»
Tacit knowledge (Полани) — знание, которое не выражается словами, живёт в моменте перехода. Prep→Action: когда остановиться и начать. Study→Skill: когда знание стало навыком. Action→Feedback: когда остановиться и переварить. Мастер делает эти переходы точнее и быстрее.

## §I — Worked examples (5 примеров)

**IP-1 STRICT:** все имена — role-type. Реальные имена = RUSLAN-LAYER, не публичный контент.

### I-1: T1-методолог: первый discovery call

**Контекст.** Потенциальный T1-партнёр по методологии. Встреча через 3 дня. Тема: совместная образовательная программа. Высокий стейк — если звонок провальный, связи нет.

**Что сделали.** Prep-gate → depth. 5 calibration Q: 4 часа · детально · завтра утром · понять его день и боли, не рассказывать про Jetix · направление = как Jetix может усилить его работу. Метод подготовки: изучить публикации + записать 10 вопросов-гипотез о его болях. Картина: он оптимизирует существующие курсы, но не строит новые. Уникальный метод: открыть звонок с «как сейчас устроен ваш рабочий день» (не с презентации Jetix).

**Outcome.** Звонок перешёл в конкретный proposal через 40 мин. Стандартная «демо Jetix» не дала бы этого.

**Lessons.** The Trick сработал: картина дала метод, который противоречит обычному «продавай первым». R12-check: подготовка = понять чтобы помочь, не чтобы продавить.

### I-2: Видео с высоким стейком (Wave-blocker)

**Контекст.** Видеоролик — ключевой контент для партнёрского события. Дедлайн 4 дня. Тема сложная, аудитория неизвестная.

**Что сделали.** Prep-gate → depth. 3 часа подготовки: анализ аудитории на основе предыдущих ивентов, 4 варианта hook'а. Картина: аудитория skeptical, нужен конкретный пример, не теория. Уникальный метод: «крючок вперёд» (открывашка с исходом примера, не вопроса) + стратегия дублей (4 варианта первые 30 сек → лучший выбирается после просмотра).

**Outcome.** Hook выбран правильно. Engagement выше среднего. Стандартный «вопрос-крючок» не зашёл бы этой аудитории.

**Lessons.** Depth-prep окупилась — не за счёт «больше времени», а за счёт картины. Без prep взяли бы стандартный hook.

### I-3: T3-контрибьютор: Notion trial

**Контекст.** T3-контрибьютор (аудитория/партнёр) хотел попробовать систему управления знаниями. Запрос: «покажи, как ты работаешь». Вопрос: дать сложную систему или минимальную.

**Что сделали.** Prep-gate → depth. 1.5 ч: изучить что у него уже есть, понять задачи. Картина: он перегружен инструментами, нужен минимум, не максимум. Уникальный метод: минимальный 5-баз шаблон + онбординг-сессия 30 мин, без демонстрации «всего что умеем» (против обычного инстинкта — показать всё).

**Outcome.** Он начал использовать. Сложная система на входе → дроп через неделю (анти-паттерн, которого избежали).

**Lessons.** Уникальный метод = урезать против своих интересов (показать меньше). Это R12: польза контрибьютору важнее показателей «сколько показали».

### I-4: Daily Log (speed-mode — тоже мастерство)

**Контекст.** Ежедневная запись рабочего дня в Notion. Повторяется каждый день.

**Что сделали.** Prep-gate → speed. Готовый метод: шаблон Daily Log + 5 мин заполнение. Подготовка = 0. Переход: сразу к шаблону.

**Outcome.** Записи ведутся стабильно.

**Lessons.** Мастерство = знать когда НЕ готовиться. Если бы применяли depth-prep к daily log, потратили бы 1 ч в день на «планирование планирования». Speed-mode = правильный выбор.

### I-5: Sprint-пивот (depth max)

**Контекст.** В середине спринта стало ясно, что направление неверное. Два варианта: продолжить по плану (уже вложено время) / пивот (риск потери ещё одной недели). Стандартный бинарный фреймворк (continue/stop) давал «continue» из-за sunk cost.

**Что сделали.** Prep-gate → depth max. 3 ч: изучить реальные метрики спринта vs ожидания, собрать feedback от 2 партнёров. Картина: направление закрыто — не из-за выполнения, а из-за предпосылок. Уникальный метод: ребаланс 70/30 (70% на пивот + 30% закрыть текущий честно) + checkpoint через 2 дня.

**Outcome.** Пивот выбран. Потеря 2 дней вместо потери недели + неверного направления.

**Lessons.** Готовый бинарный фреймворк давал неверный ответ. Картина показала третий путь. The Trick сработал критически.

## §J — R12 Paired-frame check

**Применимость:** обязательна для Direction #1 — мета-метод это инструмент влияния, R12 STRICT.

### 8 paired-frame вопросов применены к Direction #1:

**Q1: Agreed share.** Кто извлекает ценность из обучения методу? Участник развивает свой мета-метод — это его собственность, не лицензия. Ценность остаётся у практикующего. ✓ COMPLIANT.

**Q2: Fork-and-leave.** Может ли участник уйти с освоенным мета-методом без penalty? Да — метод принадлежит ему, не Jetix. Никакой lock-in через «уникальный метод Jetix ©». ✓ COMPLIANT.

**Q3: extraction_beyond_share.** Не пытаемся ли мы через обучение извлечь что-то сверх договорённостей (данные, зависимость, монополию внимания)? Anti-pattern флаг: «только в Jetix работает мета-метод» — это extraction attempt. ✓ Если нет lock-in language: COMPLIANT.

**Q4: wage_ratio_violation.** Есть ли здесь скрытая динамика «Jetix знает как правильно, ты нет — плати за каждый шаг»? Anti-pattern: бесконечные апдейты курса с platmed-pricing. R12-conformant: SOP §E публичен, шаблоны §F открыты, курс = акселератор не монополия. ✓ COMPLIANT с этим условием.

**Q5: non_consensual_distribution.** Применяет ли мета-метод что-то к участнику без его согласия? No — участник решает когда и к чему применять prep-gate. Facilitated, not imposed. ✓ COMPLIANT.

**Q6: fork_prevention_attempt.** Можем ли мы помешать кому-то взять Extended 8-step и применить независимо? Нет — R12 запрещает. Anti-pattern: «методология Jetix» как торговая марка требующая лицензии. Решение: открытая спецификация (как IP/R12). ✓ COMPLIANT с открытым спеком.

**Q7: Manipulation check.** Используется ли мета-метод для управления поведением участника вне его интересов? Anti-pattern в образовании: «у тебя нет мета-метода — ты уязвим» → fear-based marketing. R12-conformant framing: «мета-метод = development tool», не «защита от опасности». ✓ Если framing правильный: COMPLIANT.

**Q8: Честность результата.** Обещаем ли мы результаты которые мета-метод гарантировать не может? Anti-pattern: «с L3 ты всегда принимаешь правильные решения». R12-conformant: «L3 повышает вероятность хорошего выбора, не гарантирует». ✓ COMPLIANT с правильными claims.

### Anti-patterns spec для R12 в Direction #1:

1. **Терминологический lock-in** — «мета-метод работает только в экосистеме Jetix». Запрещено. Метод — открытая спецификация.
2. **Fear-based framing** — «без L3 ты уязвим в эпоху AI». Запрещено. Позитивный фрейм: «L3 расширяет возможности».
3. **Dependency pricing** — за каждое обновление Extended 8-step требовать переподписку. Запрещено. Core SOP — открытый.
4. **Closed methodology trademark** — патентование / закрытие 6-step или 8-step процедуры. Запрещено. R12 = open fork protocol.

## §K — AI tooling integration

**Тезис (O-182):** AI = preparation-amplifier. Человек = суждение на переходах (Step 0, Step 3, Step 4) + метод-изобретатель (Step 4).

### Stratification table (Extended 8-step):

| Step | Кто ведёт | AI роль | Human роль |
|---|---|---|---|
| 0 — Prep-gate | **ЧЕЛОВЕК** | — | Суждение: высокий impact? Знакомо? |
| 1 — Prep method selection | Человек + AI suggestion | Предложить 3 варианта метода подготовки | Выбрать |
| 2 — Prep execution | **AI HEAVY** | Собрать, синтезировать, отформатировать | Задать направление, проверить |
| 3 — Picture formation | **ЧЕЛОВЕК** | Резюме (помощь формулировки) | Решить: картина есть или нет |
| 4 — Action method select/create | **ЧЕЛОВЕК** | Список вариантов из репертуара | Выбрать / создать уникальный |
| 5 — Action execution | Зависит от задачи | Зависит | Зависит |
| 6 — Feedback collection | Человек + AI | Паттерны в данных | Интерпретация смысла |
| 7 — Improvement note | Человек + AI | Предложить обновление в method library | Подтвердить / скорректировать |

### Готовые prompts:

**P-1 (Step 0 check — NOT для delegation, для self-reflection помощи):**
«У меня задача [описание]. Помоги мне проверить: насколько высокий impact? Что я ещё не знаю о ситуации? — и остановись, финальное решение о prep-gate за мной».

**P-2 (Step 1 — выбор метода подготовки):**
«Задача: [описание]. Контекст: [что знаю]. Предложи 3 варианта метода подготовки с коротким обоснованием каждого. Один из них должен быть быстрым (≤2 ч)».

**P-3 (Step 2 — research execution):**
«Собери background по теме [тема] фокус на [аспект]. Формат: 5-7 bullet points с источниками. Не интерпретируй — только факты и структура».

**P-4 (Step 2 — synthesis):**
«Вот материалы [paste]. Синтезируй ключевые паттерны в 3-5 пунктах. Укажи противоречия если есть. Не давай рекомендаций — только структурированный синтез».

**P-5 (Step 2 → Step 3 helper):**
«Вот что я знаю о ситуации: [paste]. Помоги мне проверить: есть ли здесь пробелы которые изменят метод действия? Задай мне 3 уточняющих вопроса».

**P-6 (Step 4 — варианты методов):**
«Картина ситуации: [описание]. Задача: [описание]. Перечисли 4-5 существующих методов из которых можно выбрать или собрать уникальный. Плюс один "экспериментальный" вариант».

**P-7 (Step 6 — feedback patterns):**
«Вот итоги задачи: [описание]. Выдели 3 паттерна что сработало, 2 паттерна что нет. Никаких рекомендаций — только паттерны».

**P-8 (Step 7 — method library update):**
«Вот мои наблюдения после задачи: [paste]. Предложи формулировку для записи в method library в формате: name / when_to_use / anti-patterns / outcome. Я проверю и скорректирую».

### Anti-patterns AI:

1. **Delegating Step 0** — «AI, нужна ли мне подготовка?» → AI всегда ответит depth → over-prep → потеря времени.
2. **Delegating Step 4** → усреднённый метод из репертуара AI, не из картины ситуации. Потеря The Trick.
3. **Delegating картину** (Step 3) → AI формулирует что «важно» — это и есть человеческое суждение, не делегируемое.
4. **AI-generated meta-strategy** — попросить AI написать твою personal meta-strategy. Это R9 (self-modify) violation + потеря смысла: meta-strategy должна быть твоей.

### Workflows:

**WF-1: Discovery call prep (40 мин)**
P-3 (10 мин research) → P-4 (5 мин synthesis) → P-5 (5 мин gap check) → человек пишет картину (10 мин) → P-6 (5 мин варианты) → человек выбирает метод (5 мин).

**WF-2: Document / decision depth-prep (2 ч)**
P-3 extended → P-4 × 2 раунда → P-5 gap check → человек формирует картину → P-6 → выбор/изобретение метода.

**WF-3: Daily routine (speed-mode, 0 AI)**
Prep-gate → speed → готовый метод → действие. AI не нужен.

## §L — Partner-extension hook

**Цель.** Direction #1 — открытая методология, которую T1-партнёры (методологи, педагоги), T4-консультанты и сообщество могут расширять, fork'ать и адаптировать под свои контексты без permission.

### Fork protocol:

**Что можно fork'ать без согласования:**
- Extended 8-step как base framework (adapt steps, rename, extend)
- SOP §E — полностью открытый
- Шаблоны §F — MIT-equivalent
- FAQ §H — адаптировать под домен

**Что требует attribution:**
- Method V2 §J LOCKED — ссылка на оригинал обязательна (R2)
- R12 paired-frame check — если fork публикуется как образовательный продукт, R12 framework должен быть применён (или явно обозначено отсутствие)

**Что нельзя:**
- Называть fork «Jetix-метод» без явного согласования
- Закрывать SOP/шаблоны лицензией (против R12 open fork)
- Удалять R12 attribution при публичном использовании

### Extension points (где T1-партнёры могут добавить ценность):

**EP-1: Domain-specific prep methods**
T1-методолог по системному мышлению: добавить Levenchuk-compatible prep methods (VSM scan / системная лупа) в Step 1 репертуар. Format: patch к T-4 method library entries.

**EP-2: Assessment rubrics for курса**
T4-консультант по образованию: разработать детальные rubrics для оценки Module 3-4 (картина / уникальный метод). Format: patch к курсу §C assessment.

**EP-3: AI prompts library expansion**
Любой участник: добавить domain-specific промпты в §K library. Contribution format: P-{N} с полями step / context / prompt / tested_by / outcome.

**EP-4: Worked examples (domain forks)**
T3-контрибьютор из другого домена: написать worked examples §I-формате для своего контекста (стартап, нон-профит, академия, трейдинг). Format: §I template.

**EP-5: Translations**
Любой: перевод SOP §E на EN/DE для international cohorts. Format: SOP-METHOD-{lang}-v{N}.

### Contribution guidelines:

1. Проверь что contribution не модифицирует Method V2 §J LOCKED (R2).
2. Проверь R12 paired-frame по 8 вопросам §J перед submit.
3. Формат: Markdown + YAML frontmatter с `type: direction-01-extension` + `contributed_by: [role-type]` (IP-1).
4. Submit: pull request / Notion page в designated extension section.
5. Review: T1-методолог или designated reviewer применяет 5-min R12 check.

### R12 compliance check для extensions:

Любой extension проходит 3 mandatory вопроса:
- «Остаётся ли метод у практикующего или создаётся зависимость?»
- «Можно ли fork этот extension без penalty?»
- «Нет ли fear-based framing в описании?»

Все три «да» → extension compliant. Любое «нет» → требует пересмотра framing.
