# Direction #12 — 🏛️ Мастерская: портфельный спек

## Введение: грань Foundation и что это направление

Jetix — мега-мастерская мирового уровня. Три грани Foundation: **Мастерская** (ГДЕ — пространство, 8 зон, инфраструктура) · **Мастерство** (ЧТО прокачивают) · **Сеть** (КАК распределено — mesh cells, pooling). Direction #12 «Мастерская» — это **грань ГДЕ**, тело всего Vision'а. Именно она операционализирует foundational metaphor: не «мы строим AI-платформу», не «мы делаем курс» — а **«у нас есть мастерская, приходи работать»**.

Одна строка (R1 surface): «Jetix — мега-мастерская мирового уровня: место, где люди становятся мастерами в эпоху AI, вместе двигают фронтир и не дают системе доить или запирать себя».

Зачем это направление нужно отдельно. Все 11 существующих направлений до сих пор висели в воздухе — методы, правила, видение, платформа. Мастерская — первая конкретная точка входа: **«вот место, вот что ты там делаешь, вот кем становишься»**. Это снимает риск «эзотерично», потому что человек понимает: инструменты (#2 Платформа) стоят на стене мастерской; методы (#1 Метод) — педагогика этого зала; R12 (#8) — правила, по которым место не превращается в ловушку. Мастерская = хаб #3 (рядом с #1 Метод и #8 R12).

GAP статус: ❌ create — физического места ещё нет (Year 1 виртуальная). Видео-тур (§B) — defer до появления достаточного контента/визуальных артефактов. R12 paired-frame: fork-and-leave без клейма, Welcome-frame на каждом входе.

> **R1:** все формулировки ниже — варианты/опции. Финальные решения за Ruslan.
> **R11 STRICT:** ниже — спецификации ЧЕГО содержит каждый документ, не сам публичный текст. Цифры — сценарные.
> **IP-1 STRICT:** имена ролей = role-type примеры (T-методолог, T-тестер, T-исследователь и т.д.), не конкретные люди.
> **R12 paired-frame STRICT:** каждая секция, где есть риск extraction / lock-in / cult / guru — получает явный anti-pattern check.

## §A — Главный документ «Мастерская Jetix»

**Что это:** 1-pager + long-form spec. Не публичная страница, не лендинг — операционный документ, который определяет, что такое мастерская, что в ней происходит и как она устроена. Аудитория: потенциальные участники + партнёры + сам Ruslan (founder-as-resident-master — первый читатель = строитель).

**Структура 1-pager (≤400 слов):**
- Заголовок + one-liner (R1 surface выше)
- 3 ключевых вопроса: Что это / Для кого / Что ты там делаешь
- 8 зон — один абзац перечислением
- Чем НЕ является — 7 пунктов (co-working / bootcamp / элитный клуб / школа с гуру / WeWork / фабрика продуктивности / engagement-trap)
- Где начать (entry point: Visitor → первый шаг)

**Структура long-form spec (≤3K слов, 8 разделов):**

1. **Метафора и корни.** Пять вещей, обычно разбросанных по жизни, соединённых в одном месте: цех (инструменты) + исследовательский центр (изобретают) + клуб (network, роли) + тренировочный зал (deliberate practice) + место для тела и тишины (спорт/медитация/отдых). Исторические корни: ремесленная гильдия / подмастерье→мастер, bottega Возрождения (обучение = участие в реальном деле), atelier, makerspace. Tacit knowledge (Полани): большая часть мастерства не передаётся текстом — только через apprenticeship + действие. Зафиксировано в Method V2 §4 как обоснование «hands-on, не лекции».

2. **8 зон (WK-1 диаграмма встроена):**
   - 🛠️ Стена инструментов — существующие станки (Notion-шаблоны, ROY-агенты, voice pipeline, OCR, CRM, Wiki, Claude Code, FPF); механика «поставить станок»: предложил → сообщество оценило → встал на полку с именем
   - 🔬 Исследовательский центр — frontier work, не повторение чужого
   - 🎯 Зона мастерства — deliberate practice, структурированная прокачка
   - 🤝 Зона встреч / клуб — network + ролевое экспериментирование (примерить роль инвестора/консультанта без перестройки жизни)
   - 🏋️ Тренировочный зал — skill-drills, спарринги, разборы
   - 💪 Спортзал — тело, триада «не умереть»
   - 🧘 Медитация/йога — стоическая дисциплина, ритм, уважение к периодам низкой энергии
   - 🌴 Отдых — community bonding, восстановление; список зон открыт

3. **Активности.** Нарабатывать мастерство / создавать / изобретать · встречать интересных людей · побыть в роли инвестора/партнёра/консультанта · соревнования (развивающие, не токсичные) · решать сложное · учить других (Feynman-эффект — лучший способ выучить, двигатель mass upliftment) · создавать сообщества.

4. **Роли + прогрессия (WK-5 диаграмма).** Вертикаль: Visitor → Apprentice (1-4 free) → Practitioner → Master → Master of Masters. Горизонталь: Investor / Partner / Consultant / Researcher / Steward. Роли не пожизненные и не взаимоисключающие. Прогрессия не навязана — среда поддерживает движение, Welcome-frame.

5. **Tacit knowledge механика.** Почему мастерская — единственная форма, где tacit передаётся: текст передаёт explicit; actions + apprenticeship + peer-feedback передают tacit. Без физической или виртуальной среды совместного действия — tacit не идёт.

6. **Online → Offline дуга (сжато).** Year 1: виртуальная, 5-15 участников, маховик строит основатель. Year 2: первое физическое место = арендованное (anti-WeWork, только при подтверждённой Run-петле). Year 3-5: mesh. Подробно — §E и §8 Workshop concept.

7. **Founder-as-Resident-Master.** Основатель не «запустил и ушёл» — остаётся жить на виду: пользуется теми же инструментами, работает в тех же зонах, показывает работу. Доверие = прозрачность живого примера, не маркетинговый нарратив.

8. **Чем НЕ является (anti-patterns).** 7 пунктов из KEY ANCHORS + ссылка на §J (R12).

**Формат:** MD + YAML frontmatter + встроенные WK-диаграммы (Mermaid). Источник: §1 JETIX-WORKSHOP-MASTERY-NETWORK-CONCEPT-2026-05-26.md.

## §B — Видео-тур (виртуальная мастерская)

**Статус: DEFER.** Физического места нет (Year 1). Видео-тур создаётся после появления достаточного визуального/операционного контента (первые реальные участники, реальные инструменты, документированная работа).

**Когда делать:** при переходе Year 1 → Year 2, либо при накоплении ≥5 участников-типажей с документированными mastery-моментами.

**Структура (спек для будущего):**

- **Hook (0:00–0:45):** открытый вопрос («что мешает стать мастером не через 10 лет, а через 2?»), ответ одной строкой + cut к первой зоне
- **Scenes — 8 зон (0:45–5:00):** каждая зона — 20-30 сек. Не демонстрация продукта — показ того, что происходит: человек ставит станок / пара решает задачу на спарринге / тишина медитационной зоны. Voiceover = конкретный пример (role-type, IP-1), не абстрактное обещание
- **Roles scene (5:00–5:45):** визуальная прогрессия Visitor→Master of Masters + горизонтальные роли; акцент «не пожизненное, не взаимоисключающее»
- **Closing (5:45–6:30):** one-liner Vision + entry point (как попасть) + fork reminder (Welcome-frame, можно уйти в любой момент)

**Формат:** сценарный документ (MD), не готовый монтаж. Аудиовизуальный продакшн — отдельная задача.

## §C — Курс (Workshop Onboarding)

**Тип:** онбординг-курс, не образовательный курс. Цель: новый участник (Visitor → Apprentice) понимает зоны, находит свой первый станок, делает первое deliberate-practice действие.

**Структура (5 модулей):**

1. **Что такое мастерская.** 15 мин. Метафора + 5 составных частей (цех/исследование/клуб/тренировка/тело). Интерактив: «найди себя» на WK-1 диаграмме. Не лекция — guided exploration.

2. **8 зон: экскурсия.** 20 мин. По каждой зоне — один конкретный пример действия (role-type, IP-1). Участник отмечает: «в каких 3 зонах я хочу начать?»

3. **Стена инструментов: первый станок.** 30 мин. Практика: взять один из существующих инструментов, попробовать, оставить feedback (формат «предложить улучшение»). Если приносит свой — протокол «поставить станок» (§E).

4. **Роли: примерить без страха.** 20 мин. Вертикаль и горизонталь. Механика «не пожизненное»: можно попробовать роль консультанта на 1 сессию, роль исследователя на 1 неделю — без обязательств. Welcome-frame + fork-reminder.

5. **Первый mastery-момент.** 30 мин. Deliberate practice drill — одно конкретное упражнение из Зоны мастерства (например: «сформулировать гипотезу за 5 мин — 3 захода с разбором»). Метрика: «немного больший мастер, чем до модуля».

**Формат:** структурированный MD-документ (не SCORM, не LMS). Воспроизводится через Notion или аналог. Продолжительность ≈ 2 часа суммарно + домашнее задание «поставить первый станок».

## §D — Книга

**Применимость:** по данной теме книга не является приоритетным артефактом на Phase 1-2. Мастерская как концепция — операциональна, живёт в действии и среде, не в тексте. Длинный нарратив имеет смысл только после накопления достаточного числа mastery-историй (role-type-примеры, cross-cell проекты, expedition сценарии).

**Вектор (если будет):** не «манифест» и не «учебник» — скорее «репортаж изнутри» (в формате Остервальдера или Pixar Braintrust), где истории участников показывают, как мастерская работает в реальности. Tacit knowledge не объясняется — демонстрируется через истории.

**Отложено** до Year 2-3 при наличии достаточного substrate'а историй.

## §E — Инструкции / SOP

**Назначение:** операционный слой мастерской. Три типа SOP:

### SOP-1: «Поставить станок»

Механика добавления нового инструмента в Стену инструментов.

- **Шаг 1 Предложение.** Участник заполняет tool-card: название / что делает / для каких задач / где живёт (ссылка или файл) / автор/атрибуция. Формат YAML frontmatter + краткое описание.
- **Шаг 2 Оценка сообществом.** ≥3 участника пробуют инструмент за 7 дней. Feedback template: сработало / не сработало / для какого типа задач. Не «нравится» — а «применил и вот результат».
- **Шаг 3 Решение.** Если ≥2 из 3 подтвердили пользу → на полку с именем автора. Если нет → возвращается с feedback (не «отклонено», а «нужна доработка»). Решение принимает Steward-роль (не единоличный гуру).
- **Шаг 4 Полка.** Инструмент появляется в Стене инструментов: карточка + статус + кто поставил + когда последний раз обновлялся.
- **Anti-pattern:** инструмент ставит только основатель → монополия на инфраструктуру → extraction (R12). Механика поставить станок = распределённая Стена.

### SOP-2: «Встать в роль»

Механика горизонтального ролевого экспериментирования.

- **Decision tree:**
  - Хочу попробовать роль → выбрать из горизонтали (Investor / Partner / Consultant / Researcher / Steward)
  - Срок попытки: от 1 сессии (2 часа) до 1 месяца
  - Объявить в зоне встреч: «беру роль X на период Y, ищу задачу»
  - Найти matching задачу/человека через AI-матчинг (§K) или вручную
  - После срока: рефлексия (3 вопроса: что получилось / что нет / продолжать ли)
  - Решение полностью за участником, без давления (Welcome-frame)
- **Anti-pattern:** «теперь ты навсегда наш консультант» → lock-in. Роль заканчивается, если участник не продлевает — без клейма.

### SOP-3: «Запустить deliberate practice сессию»

- Выбрать навык из Зоны мастерства
- Определить конкретный drill (не «работаю над X», а «делаю упражнение Y с измеримым результатом Z»)
- Найти пару (tester-роль, IP-1) или работать с AI-feedback (§K)
- 25-45 мин: максимальная концентрация, затем разбор
- Фиксация: что получилось чуть лучше. Метрика сессии — не «часы», а «заметный прогресс по одному аспекту»

### Anti-patterns (общие для всех SOP):

- Гуру-ловушка: один человек решает, кто достоин. Противоядие: Steward-роль (сменяемая, не пожизненная) + community feedback.
- Engagement-trap: метрика «время в мастерской». Противоядие: метрика «mastery-момент» — что ты умеешь, чего не умел.
- Закрытость: «мы сами решаем кто войдёт». Противоядие: Visitor = open, Apprentice 1-4 = free, fork-and-leave без клейма.

## §F — Шаблоны / Templates

**F1 — Workshop Space Mockup.** Визуальная карта мастерской (8 зон + связи). Формат: Mermaid-диаграмма (WK-1 как основа) + Notion-страница с секциями per зона. Что включает: название зоны / назначение / типичные активности / инструменты / кто там сейчас. Обновляется участниками.

**F2 — Zone-Activity Notion DB.** База данных: зоны × активности × роли. Колонки: Зона / Активность / Подходящие роли / Уровень (Visitor..Master) / AI-поддержка (yes/no) / Примеры (role-type, IP-1). Позволяет фильтровать: «что можно делать Apprentice в Тренировочном зале».

**F3 — Role-Progression Tracker.** Трекер прогрессии участника. Персональный (не публичный рейтинг). Секции: текущий уровень вертикали / активные горизонтальные роли / последний mastery-момент / следующий drill / история ролей. Формат: Notion-страница или YAML-файл в личном пространстве. Видимость: только сам участник + Steward (если участник разрешил).

**F4 — Tool-Card Template.** YAML-шаблон для карточки инструмента (станка): name / description / use-cases / location / author / added-date / last-updated / feedback-count / status (proposed / active / deprecated).

**F5 — Onboarding Checklist.** Для нового Visitor → Apprentice: посетил все 8 зон / нашёл первый инструмент / сделал первый drill / встретил ≥1 человека / знает как уйти (fork). Последний пункт обязателен — Welcome-frame.

## §G — Презентация (10-20 слайдов)

**Аудитория:** потенциальные участники (warm) + потенциальные партнёры (partner-extension, §L). Не pitch-deck инвестору — это объяснение концепции для человека, который уже слышал про Jetix и хочет понять, что такое мастерская.

**Структура (15 слайдов):**

1. **Заголовок + one-liner.** «Jetix — мега-мастерская...» (R1 surface)
2. **Проблема.** Мастерство в 2026 — либо одиночный путь, либо дорогой MBA, либо TikTok-алгоритм. Среды для tacit knowledge больше не существует в нужном масштабе.
3. **Метафора: 5 вещей в одном месте.** Цех / исследование / клуб / тренировка / тело — визуально, не текстово.
4. **Исторические корни.** Гильдия / bottega / atelier / makerspace — «мы не изобретаем, мы восстанавливаем + обновляем под AI-эпоху».
5. **8 зон (WK-1 диаграмма).** Один слайд, все 8 зон, минимум текста.
6-13. **8 слайдов — по одной зоне.** Каждый слайд: название зоны + что там делают + конкретный пример (role-type, IP-1) + AI-роль в этой зоне (§K).
14. **Роли: вертикаль + горизонталь.** Схема WK-5, объяснение «не пожизненные, не взаимоисключающие».
15. **Чем НЕ является + entry point.** 7 anti-patterns + один призыв к действию (как стать Visitor).

**Anti-patterns слайд-дека:** длинные bullet-списты → замени схемами. Слайды с «мы лучшие, потому что...» → убери, нет сравнения с конкурентами. Слайд «join now» с срочностью → Welcome-frame, не FOMO.

## §H — FAQ (10-30 вопросов)

Честные ответы. Не маркетинговые — реальные ответы, которые снимают confusion.

**Категория: что это**
- Это co-working? → Нет. Co-working = аренда стола. Мастерская = среда для роста мастерства. Инструменты, коллаборация, прогрессия — не просто стол с WiFi.
- Это bootcamp? → Нет. Bootcamp выдаёт диплом за прохождение программы. Мастерская не выдаёт дипломов — мастерство видно по тому, что умеешь, не по бумажке.
- Это школа? → Нет. Нет одного преподавателя, нет расписания лекций. Peer-mastery: учат все всех (Feynman). Есть структура (зоны, SOP, роли) — нет фигуры гуру.
- Это элитный клуб? → Нет. Visitor = открытый вход. Apprentice 1-4 = бесплатно. Элита не определяется деньгами или связями — только мастерством, которое ты нарабатываешь.
- Это WeWork? → Нет. WeWork = недвижимость как бизнес, монетизация пространства. Мастерская не продаёт квадратные метры — продаёт среду роста.
- Это продуктивность-фабрика? → Нет. Спорт/медитация/отдых = равноправные зоны. Мастер — человек целиком, не биоробот-оптимизированный.
- Это engagement-trap? → Нет. Метрика «время в приложении» — запрещена. Метрика = «насколько ты вырос». Если мастерская не помогает расти — уходи.

**Категория: где физически**
- Это онлайн или офлайн? → Year 1: виртуальная (все инструменты, все активности — онлайн). Year 2+: первое физическое место = арендованное помещение в Берлине (не своё здание). Переход — только при подтверждённой Run-петле.
- Когда будет физическое место? → Когда виртуальная мастерская докажет работоспособность: ≥15 активных участников, ≥3 когорты, стабильный Run. Цифры сценарные.
- Если я не в Берлине? → Виртуальная мастерская доступна отовсюду. Физические cells (Year 3+) — mesh из локальных мастерских в разных городах.

**Категория: роли и прогрессия**
- Нужен ли опыт чтобы войти? → Нет. Visitor = нулевой порог. Apprentice 1 = первый шаг. Мастерство нарабатывается здесь.
- Что если я хочу только попробовать роль консультанта? → Горизонтальные роли = временные. Берёшь на 1 сессию или 1 месяц, пробуешь, решаешь продолжать или нет. Никто не давит.
- Можно ли уйти в любой момент? → Да. Fork-and-leave = базовое право. Без клейма, без потери наработанного знания (твои материалы — твои).
- Можно ли быть Practitioner и одновременно Investor? → Да. Роли не взаимоисключающие. Вертикаль = уровень мастерства; горизонталь = функция, которую берёшь.

**Категория: инструменты и AI**
- Где хранятся инструменты? → Стена инструментов = каталог с карточками (§F4). Сейчас: Notion-шаблоны, ROY-агенты, voice pipeline, CRM, Wiki, Claude Code, FPF. Список открыт — каждый может поставить новый станок (§E SOP-1).
- AI заменяет участников? → Нет. AI делает рутину (бронирование зон, матчинг, digest). Участник делает frontier: решает сложное, создаёт, изобретает, передаёт tacit knowledge.
- Это про то чтобы стать AI-специалистом? → Не только. Мастерство = любая область. AI = инструмент в цеху, не конечная цель.

## §I — Worked Examples (3-5 сценариев, IP-1 STRICT)

Все имена = role-type (IP-1). Цифры = сценарные (R11).

**Сценарий 1: Типичный день Practitioner-типа (виртуальная мастерская)**

08:40 — deliberate practice drill: Зона мастерства, упражнение «сформулировать гипотезу за 5 мин», 3 захода с разбором. AI-feedback между заходами (инструмент в Стене). Итог: один конкретный аспект стал лучше.

09:15 — deep-work: Исследовательский центр, 90 мин фронтир. Рутинная часть уже сделана AI заранее (digest, выборка источников). Человек — на frontier: синтез, гипотеза, изобретение.

10:45 — пара с T-тестер-типом (25 мин): ловят tacit blind spot. Формат: один объясняет, второй задаёт вопросы пока не зависнешь. Где завис — там и прячется tacit.

12:00 — обед/спорт/тишина. Спортзал + медитация. Не «потерянное время» — часть мастерства.

13:00 — ролевой блок: сегодня Consultant-роль. Помогаешь T-исследователь-типу структурировать задачу. Не постоянная роль — взял на сегодня, пробуешь.

14:30 — улучшил инструмент (Стена инструментов): нашёл баг в voice-pipeline, починил, выложил обновление. Через час берут из другой cell.

16:00 — учит новичка (Feynman): Apprentice-тип спрашивает про deliberate practice. Объясняешь — и сам обнаруживаешь то, что не мог сформулировать до этого. Лучший способ выучить.

17:30 — wrap. Метрика дня: «немного больший мастер, чем утром». Не «отсидел 8 часов».

**Сценарий 2: Путь Visitor → Master (12 месяцев, сценарный)**

Месяц 0 (Visitor): зашёл, посмотрел, не понял зачем. Нашёл один инструмент — voice pipeline. Попробовал на своей задаче.

Месяц 1 (Apprentice-1): поставил прошивку (прошёл онбординг §C). Нашёл Зону мастерства для себя — исследовательская работа. Сделал первый drill.

Месяц 3-6 (Practitioner): пробует роли вбок. Месяц как Researcher, потом как Consultant на 2 сессии. Понял: Researcher — это моё.

Месяц 6-12 (первый mastery-момент): решил задачу, которая раньше казалась непосильной. Не «получил диплом» — люди начали приходить за советом. Репутация = то, что умеешь.

Год 2+ (Master / Researcher-роль): помогает новым участникам, контрибьютит в Исследовательский центр, участвует в expedition. На любом шаге — fork-and-leave без клейма.

**Сценарий 3: Founder-as-Resident-Master**

Ruslan (как role-type «основатель», не конкретный человек публично) строит виртуальную мастерскую для себя: ставит первые инструменты (ROY-агенты, voice pipeline), документирует первые mastery-моменты, делает первые deliberate practice сессии. Мастерская видна изнутри, не через маркетинговый текст — «вот что я делаю сегодня в Зоне мастерства». Доверие = прозрачность живой работы.

**Сценарий 4: Cross-cell проект (Year 3+, сценарный)**

T-методолог (Берлин) + T-исследователь (другой город) + T-тестер (третий город). 3 месяца, конкретная задача: разработать и проверить новый учебный метод. Mesh-координация: никто не начальник. Ресурсы из общего пула (opt-in + consent per R12). Frontier outcome → общий результат; revenue-share per Mondragón (75/25, 5:1 cap). Кто хочет — fork с атрибуцией. Показывает: сеть = fabric, рождает то, что в одиночку не родится.

## §J — R12 Paired-Frame Check

R12 (anti-extraction, additive 2026-05-12) применяется к Мастерской на 8 уровнях.

**8 вопросов:**

1. **Может ли участник уйти в любой момент без потерь?** → Да. Fork-and-leave = базовое право (§H). Его материалы — его, репутация — его, наработки не «принадлежат мастерской». При форке: атрибуция (знание происхождения) + возможность унести. ✅

2. **Есть ли механизм, который лочит участника?** → Anti-pattern: «ты уже вложил 3 месяца — нельзя уйти». Противоядие: прогрессия = репутация (не активы мастерской), роли не пожизненные. Sunk-cost framing = запрещён. ✅

3. **Кто контролирует Стену инструментов?** → Распределённый механизм (SOP-1): любой участник предлагает, community оценивает, Steward (сменяемый) утверждает. Монополия основателя на инфраструктуру = extraction risk → предотвращён. ✅

4. **Есть ли фигура гуру, на которую замкнуто всё?** → Нет. Peer-mastery. Основатель = Resident Master (один из мастеров, не верховный судья). Решения по SOP — community + Steward. ✅

5. **Может ли мастерская «выгнать» участника за несогласие?** → R12 тонкая зона (§3 Workshop concept): нельзя выгнать за несогласие с методологией. Можно — за нарушение базовых правил (harassment, extraction). Правила прозрачны и симметричны. ✅

6. **Метрика = engagement или рост?** → Метрика = mastery-момент, не время в приложении. Engagement-trap = запрещён как тип монетизации/удержания. ✅

7. **Welcome-frame везде?** → Онбординг, §C, FAQ, SOP — каждый входной документ содержит: «можно уйти в любой момент». Не мелким шрифтом — явно. ✅

8. **Партнёрские cells — mesh или star?** → Mesh (§L). Каждая cell автономна, ресурсы в общий пул по opt-in, нет центрального контролёра. Star-топология = extraction architecture → запрещена. ✅

**Anti-patterns (явные красные флаги):**

- Элитный клуб: «мы сами решаем кто достоин» → Противоядие: Visitor = open, критерии прозрачны.
- Гуру-ловушка: один человек = источник всей истины → Противоядие: peer-mastery, Steward сменяемый.
- Engagement-trap: «проведи больше времени в мастерской» → Противоядие: метрика = mastery-момент.
- Sunk-cost lock: «ты уже вложил — нельзя уйти» → Противоядие: fork-and-leave без клейма, явно.
- Extraction через роли: «ты теперь навсегда наш консультант (бесплатно)» → Противоядие: роли временные, компенсация по Mondragón-ratio при реальной работе.

## §K — AI Tooling Integration

**Принцип:** AI = рутина. Человек = mastery на станках.

Разделение по зонам:

| Зона | AI делает | Человек делает |
|---|---|---|
| 🛠️ Стена инструментов | каталог, поиск, рекомендация инструмента под задачу | выбор, применение, оценка результата, улучшение инструмента |
| 🔬 Исследовательский центр | digest источников, выборка, структуризация черновика | синтез, гипотеза, frontier insight, изобретение |
| 🎯 Зона мастерства | scaffold drill-упражнения, feedback между заходами, progress tracking | deliberate practice, рефлексия, mastery-момент |
| 🤝 Зона встреч / клуб | матчинг по ролям и задачам, calendar coordination | реальный разговор, tacit-передача, ролевое исследование |
| 🏋️ Тренировочный зал | статистика прогресса, подбор спарринг-партнёра | спарринг, разбор, совместное решение |
| 💪 Спортзал | трекер физической активности (если интегрирован) | тренировка, тело |
| 🧘 Медитация | напоминания, scheduling | тишина, ритм, стоическая дисциплина |
| 🌴 Отдых | предложение retreat-форматов, community events | отдых, bonding, экспедиция |

**AI-матчинг ролей:** алгоритм (not «рекомендательная система с engagement-метрикой», а утилитарный матчинг) соединяет запрос «ищу партнёра для drill в Зоне мастерства, Researcher-тип» с подходящими участниками по профилю и доступности. Матчинг — инструмент, не алгоритм удержания.

**Ограничения AI в мастерской:**
- AI не оценивает мастерство участников (rule 8 Pillar C: агент не оценивает peer без human review). Mastery-момент фиксирует сам участник или peer, не алгоритм.
- AI не принимает решения о прогрессии в роли — только участник + community.
- AI не хранит «личное дело» без явного разрешения участника (rule 6 §4.1: no aggregate unstructured long-term memory).

**Текущий substrate (Year 1):** ROY-агенты (brigadier + 5 экспертов + 8 book-driven) уже делают рутину для Ruslan. Расширение на других участников — Year 2+ при наличии инфраструктуры.

## §L — Partner-Extension Hook

**Назначение:** как партнёр открывает свою cell-мастерскую в своём городе/контексте.

**Zone Template для новой cell:**

Минимальный комплект при открытии cell:
1. ≥4 из 8 зон активированы (обязательны: Стена инструментов + Зона мастерства + Зона встреч + один из «тела» (спорт/медитация/отдых))
2. Стена инструментов: минимум 3 инструмента (можно взять из общего пула мастерских)
3. SOP-1 и SOP-2 адаптированы под локальный контекст (язык, культура) при сохранении canonical role structure
4. Роли вертикаль + горизонталь = canonical (не изменяются per cell — иначе нет interoperability между cells)

**Role Canon:**
- Вертикаль и горизонталь одинаковые во всех cells (canonical из §A long-form spec)
- Локальные additions возможны как «extended roles» с явным атрибутом local: true в role-card
- Extended local roles не получают cross-cell recognition автоматически — только по opt-in других cells

**R12 Compliance для partner-cells:**

| Требование | Что означает |
|---|---|
| Mesh, не star | Берлин-cell (основная) не является «головным офисом». Каждая cell автономна. Решения о ресурсах — per cell, не централизованно. |
| Opt-in ресурсный пул | Участие в общем пуле ресурсов (инструменты, капитал, знания) = добровольное, с явным consent, с явными условиями. |
| Fork-and-leave для cell | Cell-партнёр может выйти из сети без штрафа. Атрибуция сохраняется, инструменты (если разработаны совместно) — по договору об open-source атрибуции. |
| Revenue-share cap | При совместных проектах между cells — Mondragón-ratio: 75% к участникам труда, 25% в treasury; wage-ratio cap 5:1. |
| Нет контроля curriculum | Основная cell не навязывает методологию обучения локальной cell. Canon = role structure. Педагогика = per cell + Method V2 как ресурс (не обязательство). |

**Процесс открытия cell (сценарный):**
1. Партнёр знакомится с Zone Template и Role Canon
2. Активирует ≥4 зон в своём контексте
3. Подписывает R12 compliance checklist (явный opt-in в mesh + отказ от star-подчинения)
4. Первый виртуальный запуск (≥5 участников, 1 cohort)
5. Интеграция в общий каталог cells (карточка cell: город / зоны / контакт / язык)
6. Cross-cell активности — по opt-in: expedition, cross-cell проекты, ресурсный пул

**Anti-patterns при масштабировании:**
- Франшиза-ловушка: «вы платите роялти за использование названия» → R12 violation (extraction beyond agreed share). Противоядие: open-source атрибуция + voluntary revenue-share.
- Curriculum diктат: «ваши участники обязаны проходить наш курс» → нарушение автономии cell. Противоядие: Canon = минимум (роли + R12), всё остальное per cell.
- Центр как арбитр конфликтов cells: если все конфликты решаются через Берлин-cell → star де-факто. Противоядие: у каждой cell есть Steward-роль для внутренних конфликтов; cross-cell конфликты — через согласованный protocol (не через hierarchy).

## Закрытие: что это направление даёт системе

Direction #12 «Мастерская» — не ещё одна страница в метаплане. Это **тело Vision'а**, которое делает всё остальное конкретным. Метод (#1) живёт в Зоне мастерства. Платформа (#2) = Стена инструментов. Партнёры (#5) = роли в мастерской и сети. Видение (#6) получает физическую форму. R12 (#8) = правила, по которым место не становится ловушкой.

Три вещи, которые этот спек фиксирует:

1. **Мастерская = операциональная метафора, не маркетинговый слоган.** 8 зон, SOP, роли, онбординг — всё существует как спецификация для построения реальной среды.

2. **GAP честно признан:** физического места нет (Year 1). Видео-тур — defer. Первый физ. place = арендованное при Run-петле. Никакого fake-launch.

3. **R12 пронизывает каждый артефакт:** fork-and-leave явно в §H и §J; mesh-топология в §L; anti-patterns в каждом SOP; Welcome-frame в §C. Мастерская не может стать ловушкой, если R12-check вшит в операционку с самого начала.
