## Патч M-2 — Шаблоны × Уникальное → §C «Выбор метода» (новый под-раздел §C.6)

**[TARGET: Mastery §C, после §C.5 — добавить §C.6]**

> **§C.6 Шаблоны × уникальные задачи (первый под-выбор).** Прежде чем выбирать метод, мастер решает:
> **это шаблонная задача или уникальная?** (a) Шаблонное → зашаблонить/автоматизировать/AI, «чтобы
> голову не ебать» (voice). (b) Уникальное → выбрать/собрать/изобрести метод под конкретный случай.
> Большинство значимых задач уникальны (voice: «все задачи жизни сложные и уникальные»; каждый
> клиент/человек — уникален, S-09 → НЕ один скрипт на всех). Сам выбор «шаблон/уникальное» = мастерский
> навык: зашаблонил уникальное = cargo-cult; уникализировал шаблонное = жжёшь голову на ерунде. Это
> уточняет default-vs-deliberate quadrant (§C.4). (S-08, S-09; Phase 3 §A.)

**R12-нота:** прямое следствие для CRM/partner-facing — per-person подход, не one-size-fits-all.

## Патч M-3 — 3 оси накопления → §D «Вечная тренировка» (расширение)

**[TARGET: Mastery §D, добавить под-блок]**

> **3 оси накопления (что именно копится).** Мастерство копится параллельно по трём осям: **(а)
> база знаний** (что знаешь — ось §B) · **(б) навыки** (что умеешь — deliberate practice §D) ·
> **(в) люди/network** (кого знаешь/можешь привлечь — Сеть #14). Ценность = знания × навыки × сеть
> (**перемножаются**, не суммируются): знание без навыка = теория; навык без сети = соло-потолок;
> сеть без знаний = пустые контакты. Это встраивает Network-направление прямо в прокачку мастерства.
> (S-11; Phase 3 §B.)

## Патч M-4 — Нелинейная прогрессия → NEW §M (т.к. §J занят)

**[TARGET: Mastery — добавить новую секцию §M после §L «Мост»]**

> **§M Нелинейная прогрессия по темам (темы vs уровни).** Развитость — **не одна лестница** (уровень
> 1→2→3), а **множество тем, в каждой своя глубина** (voice: «не с уровня на уровень, а с темы на
> тему... тема — много топиков разных»). Внутри одной темы вертикаль есть (Apprentice→Master по
> теме, Workshop §D); **между** темами — горизонталь (исследуешь новые). Следствия: (1) multiple
> valid peaks — глубоко в теме A и в начале темы B = норма; (2) **сравнивать мастеров разных тем
> глупо** (S-13) → anti-ranking culture в Сети; (3) Welcome-frame усиление — нет «я внизу», есть «я
> в начале вот этой темы». [WK-S-3 mermaid — дерево тем, не лестница.] (S-12, S-13; Phase 3 §C.)

**Обоснование §M, а не §J:** §J уже = «Reference deep» (density pass предыдущего прогона). Новая
секция = §M (после §L), чтобы не конфликтовать. **R1:** Ruslan может выбрать иную букву/место.

## Патч M-5 — Anti-credentialism + measurable proxy → §I Anti-patterns (расширение)

**[TARGET: Mastery §I, добавить anti-pattern + позитивный прокси]**

> - ❌ **Credentialism** — мерить мастерство дипломами/титулами/«сколько курсов прошёл». **Чем
>   заменить (positive proxy, S-15):** мастерство ≈ **количество × тяжесть уникальных задач, которые
>   человек реально решил** (portfolio > diploma). Не «какой уровень/титул», а «какие непохожие и
>   тяжёлые задачи берёт и закрывает». (Voice: «чем больше уникальных задач может + чем тяжелее → тем
>   прокаченнее мастер».)

**R12-нота:** усиливает Workshop §F «не bootcamp с дипломами» — теперь с тем, **чем** заменить
диплом (track record решённых уникальных задач).

## Патч M-6 — Curiosity-driven + infinity → §D / §E (усиление)

**[TARGET: Mastery §D вечная тренировка + §E — добавить механизм мотивации]**

> **Curiosity loop (двигатель вечной тренировки).** новая задача → интересно → решаешь
> (прокачиваешься) → учишься для следующей → берёшь сложнее → снова интересно → ∞. Двигатель —
> **интерес (intrinsic), не долг**. Вечная тренировка вечна не «через силу-дисциплину», а потому
> что новые задачи интересны сами по себе (SDT competence+autonomy; Flow при challenge≈skill).
> **Infinity-frame:** «бесконечные развития, улучшения» (voice) — нет финишной черты; мастерская =
> lifelong residence, не graduation. Конечная цель — **«больше вариантов на жизнь, и просто лучше
> живёшь»** (voice), не титул/деньги (триада O-138). (S-10, S-16; Phase 3 §E.)
