## Кланы (твоя «основная часть»)

Клан = автономная кооперативная единица (своя культура/методы/экономика 5:1 + право форкнуться). НЕ команда
(команду доят сверху; клан — нельзя, mesh не star). 7 фаз: formation → governance → свобода → соревнование
(уважение, no poaching) → сотрудничество → fork-spawn (Mondragón) → dissolution. Floor enforced platform-wide
= триада + R12 + уважение; всё остальное = inner-clan freedom. Платформа = качалка/склад (infra + Charter +
events), не диктует методы/темы. Klan Charter template (2-level: floor-mandatory + inner-freedom) = новый
cross-cutting doc.

## Геймификация (R12 — главное)

Самый опасный direction: тот же инструмент мотивирует ИЛИ манипулирует. R12 = дизайн-ограничение с первой
строки, не «прикрутить потом». **Тест:** убери метрику «время в системе» — механика всё ещё ценна? skill tree
(рост мастерства) → да; streak который «сгорает» → нет (страх потери = dark pattern). ❌ Запрет: addictive
loops, variable-reward, sunk-cost, FOMO, social-pressure, pay-to-win, vanity metrics. ✅ Дисциплина: intrinsic
motivation, meaningful progression, opt-out always, **метрика = «насколько вырос» не «время в приложении»**.
Virtual economy (tokens) = defer до R12-design session. Anti-Dark-Patterns audit = gate каждого game-mechanic.

## Хакатоны (revenue engine)

Events = сам продукт, не маркетинг (платят за результат: решил задачу + встретил людей + наработал мастерство).
Event cycle: Initiate → Match (FPF «за 2 часа в проекты») → Solve (day/month/year) → Reward (QF + 5:1 cap) →
Recurse. Sponsor + paid entry + IP/talent placement. Target ≥$30K/event (сценарно). first event ≤90d (Q3 2026,
bloggers+sponsor day-rhythm). Clan-wars = inter-clan хакатоны. Substrate готов (JETIX-AS-HACKATHON-PLATFORM).
