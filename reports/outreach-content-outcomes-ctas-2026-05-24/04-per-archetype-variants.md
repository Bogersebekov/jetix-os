## §1 Archetype 1 — Инженер (Engineer)

### §1.1 Profile

**Who:** Software / hardware engineer, 3-15+ years experience. Likely senior IC or staff-level. Reads Hacker News / Lobste.rs / engineering blogs. Knows Karpathy / Bret Victor / Joel Spolsky / John Ousterhout. Skeptical of management-speak. Values craftsmanship + correctness + simplicity.

**Pain points:**
- Cargo-cult «agile» rituals
- AI-generated code reviewed как junior PR (zero amplification)
- Lost time on tools that don't compound (10 new tool per year, none retained)
- Struggling to articulate why senior > junior beyond «experience»
- Feeling deskilled by AI tools (concern)
- Tech debt accumulates without leverage gain

**Trust signals что работают:**
- Code / architecture quality on display (Karpathy compound learning instantiation)
- Engineering-thinking project (compound substrate)
- AI Market PLAN distillation (Sutton bitter lesson respected)
- IP-1 abstract roles (engineer recognizes good abstraction)
- Foundation Architecture v1.0 (structured deeply)

**Anti-trust signals (Halt):**
- Management-speak («synergy» / «10x» / «move fast and break things»)
- Cult vocabulary («framework» without rigorous definition)
- «AI-powered» buzzwords без concrete leverage articulation
- No code / no repo / no concrete artifacts

### §1.2 Voice adaptation

**Vocabulary:**
- ✅ «engineering thinking» / «compound learning» / «architecture» / «leverage» / «refactoring» / «test discipline»
- ✅ «AI as compiler — translates intent to artifacts» (Karpathy-tier framing)
- ✅ «method-of-methods = architecture-of-architectures»
- ✅ «технический долг intellect-level» (compound «cargo-cult code» concept)
- ❌ «прошивка» (use «base layer» or «foundation» instead для engineer)
- ❌ ШСМ vocabulary
- ❌ «paradigm shift» (too marketing-ish)

**Examples used:**
- Senior vs junior delta = selection skill, not knowledge bulk
- Architecture decisions = method-of-methods in code
- AI generates CRUD; human designs domain model
- Refactoring = compounding learning back into code
- Engineering principles outlast frameworks (TCP/IP outlasts MVC variants)

**Tone:**
- Precision / concrete / minimal handwaving
- Show, don't tell (links к concrete artifacts)
- Self-deprecating humor about engineering pitfalls (cargo-cult / over-engineering)
- Respect for prior art (cite Karpathy / Ousterhout / Hickey / etc.)

## §2 Archetype 2 — Исследователь (Researcher)

- Literature reviews taking 6 months
- Methodology gaps not surfaced until reviewer comments
- AI tools not integrated into research workflow (Zotero / Obsidian + LLM gap)

- Replication crisis: peers / own work not reproducible
- Career pressure for paper count vs quality

**Trust signals:**
- R6 provenance discipline (per-claim citations)

**Anti-trust signals:**
- Hand-wavy synthesis without provenance

- Unfalsifiable claims
- Lack of methodology articulation

**Examples used:**
