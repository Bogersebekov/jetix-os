# Phase 3 — Agent topology proposal

> **R1 surface only — Ruslan picks final set + names + scopes via Phase 6 AWAITING-APPROVAL packet ack.**
> Format per candidate: `name (kebab-case) / role-archetype (IP-1 abstract) / domain lens / sub-cluster anchors / required-books / signature methods / boundary cases / R12 check / mode coverage / dispatch routing notes`.

## §3.1 Design constraints (non-negotiable)

1. **IP-1 strict:** ALL candidates = role-archetype slugs (NOT executor instances). Phase 4 system.md drafts further preserve. Executor binding (which model, which file) = RUSLAN-LAYER per `shared/schemas/executor-binding.yaml.template`.
2. **Ashby variety target ≥20** (per Part 4 §H.2): current 5 ROY experts × 4 modes = 20 cells; adding 5-10 more candidates raises cell-count to 40-60, expanding requisite variety per existing routing-table architecture.
3. **R12 paired-frame discipline:** Any influence-tactics-dispatching agent MUST surface paired frame (technique + counter / abuse-mode / consent boundary) before promotion. Phase 4 enforces via system.md draft.
4. **Mode coverage:** Each candidate must declare which of 5 modes [critic / optimizer / integrator / scalability / writing-support] it operates in. Per routing-table `task_shape_dispatch_matrix`.
5. **Brigadier orchestration preserved:** All new candidates dispatch THROUGH brigadier (hub-and-spoke per Part 4 §H + Global Rule §4.2). NO direct agent-to-agent calls.
6. **Default-Deny novel actions:** Per Tier 2 R11 + Part 6b §I.2. Each candidate carries explicit "out-of-scope" list (what it does NOT do).

## §3.2 Candidate evaluation rubric (for §3.3 ranking)

Each candidate scored on 6 axes (0-3 scale):

| Axis | What it measures |
|---|---|
| **A. Book-mass** | How many books in 80-corpus directly support this agent's lens (≥6 = 3; 4-5 = 2; 2-3 = 1; 0-1 = 0) |
| **B. ROY-coverage-gap** | How much existing 9-agent ROY swarm fails to cover this lens (zero = 3; partial = 2; mostly = 1; full = 0) |
| **C. Constitutional-fit** | How cleanly the agent fits Tier 2 rules (no R12-extraction risk = 3; bounded with R12 paired-frame = 2; surface-tension = 1; fundamental-conflict = 0) |
| **D. Cross-cluster-utility** | How many other agents would routinely consult this one (≥3 = 3; 2 = 2; 1 = 1; standalone = 0) |
| **E. Active-Jetix-project-relevance** | Maps to active projects in CLAUDE.md (quick-money / research / brand / community / ai-tools / life-os / engineering-thinking / bets): direct relevance count |
| **F. Risk-of-overlap-with-existing** | How safe re: not duplicating existing ROY expert function (no overlap = 3; partial-overlap = 2; near-duplicate = 1; duplicate = 0) |
