## §0 TL;DR (≤200w)

**4 sequencing options surfaced:**
1. **Sequential strict B → A → C** (safe, slow ~5 weeks) — covers всё но slowest
2. **Parallel ALL B+A+C** (fast, demanding ~1.5-2 weeks) — fastest но focus dilution risk
3. **Mixed B-first → A+C parallel after** (balanced ~3 weeks) — recommended default suggestion

- Plan B docs (B3 Welcome-frame + B4 outreach) feed Plan A V1 script (Days 1-2)
- Plan B templates (B5 Discovery script) feed Plan C C3 handoff structure
- Plan C feedback (C8) feeds Plan A V2 (refined claims)

**6 decision criteria:** time / energy / risk / leverage / R12 safety / feedback loop quality.

**Default brigadier surfaces:** Mixed option (B-first → A+C parallel after) per balanced trade-offs — но **Ruslan picks final** per Pillar C Tier 2 rule 1.

## §2 Sequencing options (4 alternatives)

### §2.1 Option 1: Sequential strict B → A → C (safe, slow)

**Timeline:** ~5 weeks total
- Week 1 (24-30.05): Plan B Days 1-7
- Week 2-3 (31.05-13.06): Plan A V1+V2+V3-V7
- Week 4-6 (14.06-4.07): Plan C build + Дмитрий trial + Сева loop + synthesis

**Pros:**
- ✅ Focus undivided — one plan at a time
- ✅ Each plan informs next (B → A → C)
- ✅ Lowest energy demand (no parallel overhead)
- ✅ Highest reversibility (kill any plan if not working)

**Cons:**
- ❌ Slowest to Wave 1 launch (Plan B Wave 1 Day 2; Plan A delayed)
- ❌ Plan C real-test evidence delayed к Week 4-6 (Wave 1 launch already done)
- ❌ Дмитрий + Сева warm window may pass (if waited 3+ weeks)

**Best when:**
- Energy unclear / unsustainable parallel
- Reversibility priority
- No urgency to Wave 1

### §2.2 Option 2: Parallel ALL B+A+C (fast, demanding)

**Timeline:** ~1.5-2 weeks total
- Day 1 (24.05): B1+B2+B3 (docs) + A1+A2+A3 (video setup) + C1+C2 (Notion+CC)
- Day 2 (25.05): B4+B5+B6+B7 (Wave 1 docs send) + A4+A5+A6 (video record+edit+R12) + C3+C4 (handoff)
- Days 3-7: All 3 plans in parallel response cycles
- Week 2: V2 + Plan C trial completion + Plan B follow-ups

**Pros:**
- ✅ Fastest to Wave 1 + real-test evidence
- ✅ Maximum compound leverage (3 paths to Wave 1)
- ✅ Plan A + B redundancy (text+video parallel for same Wave 1 candidates)

**Cons:**
- ❌ ⚠️ Focus dilution risk (3 plans simultaneously)
- ❌ ⚠️ Energy demand maximum (Plan A on-camera + Plan B writing + Plan C iteration)
- ❌ ⚠️ Quality risk на каждом плане (rushed execution)
- ❌ ⚠️ Plan C check-ins (15min/day) compete with Plan B response cycle

**Best when:**
- Energy sustained + high motivation
- Wave 1 urgency
- Trust в parallel execution capability

### §2.3 Option 3: Mixed B-first → A+C parallel after (balanced — default suggestion)

**Timeline:** ~3 weeks total
- Days 1-2 (24-25.05): Plan B B1-B7 (Wave 1 docs send)
- Days 3-5 (26-28.05): Plan A A1-A8 (video record + post + outreach) + Plan C C1-C4 (Notion + handoff)
- Days 6-9 (29.05-1.06): Plan C check-ins ongoing + Plan B response cycle + Plan A V2 plan
- Days 10-14 (2-6.06): Plan C C8-C9 (feedback + iterate) + Plan A V2 record/edit + Plan B follow-up docs
- Days 15-21 (7-13.06): Plan C C10-C12 (offer + Сева loop + synthesis)

**Pros:**
- ✅ Plan B Wave 1 launch fastest (Day 2 sends out)
- ✅ Plan A V1 catches second-wave momentum
- ✅ Plan C runs parallel — real-test evidence accumulates
- ✅ Each plan reinforces others (B docs feed A script + C handoff; A V2 informed by C feedback)
- ✅ Manageable energy distribution

**Cons:**
- ⚠️ Days 3-5 = peak load (3 plans active)
- ⚠️ Plan A V1 launch overlaps с Plan B response cycle (mitigation: V1 = unlisted; не bulk-broadcast)
- ⚠️ Plan C check-ins concurrent с Plan A recording

**Best when:**
- Balanced energy
- Want Wave 1 launch but also real-test evidence
- Sustained 3-week commit

### §2.4 Option 4: C-first (lab test)

**Timeline:** ~4 weeks total
- Days 1-2 (24-25.05): Plan C C1-C4 (Notion + handoff Дмитрий)
- Days 3-9 (26.05-1.06): Plan C C5-C7 trial running + Plan B B1-B3 prep (central docs)
- Days 10-12 (2-4.06): Plan C C8 feedback + Plan B B4-B7 Wave 1 send (informed by trial evidence)
- Days 13-15 (5-7.06): Plan A V1 record + post (informed by Дмитрий feedback + Wave 1 response)
- Days 16-28 (8-20.06): Plan C C9-C12 Сева loop + Plan A V2

**Pros:**
- ✅ Wave 1 launches с real-test evidence (highest credibility)
- ✅ Plan A V1 script informed by Дмитрий feedback (refined claims)
- ✅ Plan B docs informed by trial-frame learnings
- ✅ R12-cleanest path (real-world voluntary engagement first)

**Cons:**
- ❌ Slowest to Wave 1 (Day 10-12)
- ❌ Дмитрий warm window critical (если cold — defer whole plan)
- ❌ Plan B + Plan A delayed
- ❌ Higher Ruslan focus к Plan C check-ins (~15min/day × 5-7 days)

**Best when:**
- Real-test evidence highest priority
- Дмитрий warm + bandwidth confirmed
- Wave 1 quality > speed
- Long-game / authority play

User can re-weight criteria per priority:

- Если Ruslan = «energy preservation»: Energy demand × 2 weight → Option 1 favored
- Если Ruslan = «authority > speed»: Leverage + R12 × 2 weight → Option 3/4 favored
- Если Ruslan = «real-test evidence priority»: Feedback loop × 2 weight → Option 4 favored

### §4.3 Question framework для Ruslan picks (6 questions)

1. **Энергия + recording setup сейчас (Day 1)?**
   - Если ✅ — Plan A viable Day 2 → Options 2/3
   - Если ❌ — Plan A defer → Options 1/3 (Plan B первый)

2. **Дмитрий + Сева warm + bandwidth?**
   - Если ✅ — Plan C add → Options 2/3/4
   - Если ❌ — Plan C archive → Options 1/2/3 без Plan C

3. **Quick win или authority play?**
   - Если quick → Options 2/3
   - Если authority → Options 3/4

4. **Real-world test evidence нужна перед Wave 1?**
   - Если ✅ — Plan C первый → Option 4
   - Если ❌ — Plan C parallel или later → Options 1/2/3

5. **3 weeks sustained focus achievable?**
   - Если ✅ — любой option viable
   - Если ❌ — Option 1 (sequential) или Option 2 (fast burst) preferred

6. **R12 cleanliness priority?**
   - Если highest → Option 1 (thorough) или Option 4 (lab discipline)
   - Если normal → Options 2/3 OK
