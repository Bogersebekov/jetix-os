## §2 Per-template structure draft (universal template anatomy)

### §2.1 Database schema (properties + views)

**Common properties:**
| Property | Type | Purpose |
|---|---|---|
| Title | Text | Item name |
| Status | Select | Lifecycle status (e.g., draft / active / archived) |
| Type | Select | Sub-categorization |
| Date created | Created time | Audit trail |
| Date modified | Last edited time | Audit trail |
| Tags | Multi-select | Cross-cutting categorization |
| Linked items | Relation | Cross-db references |
| Author / owner | Person | Accountability |

**Common views:**
| View | Purpose |
|---|---|
| All / Master list | All items unfiltered |
| Active | Filter status = active |
| Per-status pipeline | Kanban-style |
| Recent (last 7 days) | Time-windowed |
| Per-tag grouped | Grouped по primary tag |

### §2.2 Pages structure

**Master pages (per template):**
- Landing / overview page (welcome + how-to-use)
- Sub-template launcher pages (one per category)
- Walkthrough / tutorial page (embedded documentation)
- Settings / config page

**Per-item page templates:**
- Standard item page (с metadata block + body + linked items)
- Variant pages для specific types (e.g., per-archetype Charter template variants)

### §2.3 Automations (Notion formulas + rollups)

**Common automations:**
| Automation | Formula / Rollup | Purpose |
|---|---|---|
| Days since last touch | `dateBetween(now(), prop("Last touch"), "days")` | CRM-style stale detection |
| Streak count | Rolling formula across daily logs | Habits tracking |
| Total value (sum) | Rollup linked items | Metrics aggregation |
| Status auto-update | Formula based on linked date | Lifecycle auto-progression |
| Tag aggregation | Rollup multi-select | Cross-cutting view |
| Days remaining | `dateBetween(prop("deadline"), now(), "days")` | Deadline tracking |

### §2.4 Sample data (5-10 example entries per template)

Each sub-template включает 5-10 sample entries:
- Examples cover diverse statuses / types / tags
- Anonymized / non-personal data
- Demonstrates how-to-use без assuming context
- Removable («Sample» tag для bulk delete)

### §2.5 Documentation embedded (walkthrough page)

Per template — one **Walkthrough page** containing:
- Why this template exists (problem solved)
- How to use (step-by-step)
- Customization guide (что можно adapt)
- Limits / caveats (что НЕ делать)
- Linked V_demo video (if Plan A V_demo ready)
- R12 frame: «voluntary opt-in / fork-and-leave / your data your-ownership»

### §2.6 Video tutorial (optional, ref Plan A V_demo)

If Plan A V_demo recorded — embed link in Walkthrough page. Otherwise — text walkthrough sufficient.

## §3 Tool Concept articulation (Plan-of-Day Шаг 6)

### §3.1 The offer (verbatim concept)

> **«Notion template + Claude Code subscription + setup/training = улучшенный инструмент управления вниманием / финансами / проектами / отношениями + AI-amplified daily workflow + mastery practice. Взамен — работа над Jetix + spread system. R12 paired-frame: voluntary opt-in / fork-and-leave / no extraction / 30-day opt-out.»**

### §3.2 Components of the tool

| Component | Spec | Cost |
|---|---|---|
| **Notion template (per category)** | 6 categories per §1; partner picks needed sub-set | Free (template export) |
| **Claude Code subscription** | Anthropic Max tier (~$200/mo) или Pro tier (~$20/mo per partner) | Partner pays own subscription OR Foundation covers for L4-L5 |
| **Setup session** | 1-2h handoff session (live) — installs template + walks через Claude Code | Foundation provides |
| **Training (5-7 day trial protocol)** | Daily 15-min check-ins + 1h feedback session at end | Foundation provides |
| **Ongoing support** | Async Q&A channel (Telegram / Slack) | Foundation provides |
| **Mastery practice** | Per Method V2 §J operational triangle — actual practice не just tool | Partner self-driven |

### §3.3 Value-for-time-exchange (R12-compatible)

**What partner gets:**
- Tool (Notion + Claude Code) instantly usable
- Setup + training (~5-10h Foundation time)
- Ongoing support (async)
- Mastery practice framework
- Cohort association (peer learning)

**What partner gives (in exchange):**
- Work on Jetix substrate (per their tier — % of attention / output to Foundation reinvestment)
- Spread system (promoter bonus mechanic per Economic V10)
- Per Charter (Mondragón ratio cap 5:1 / 30-day opt-out / fork-and-leave)

**Exchange rate** — per Charter signed (tier L1-L7 specific):
- L1 Engineer-builders: 6+ months hands-on / 10% symbolic
- L4 Founding partners: 6+ months commitment / 10% founding stake
- L5 Cohort partners: 3+ months / 15-20%
- L6 Cohort members: 1+ month / 20-25%
- L7 Workshop users: Per-engagement service fee €1500/mo (NO long commit; transactional)

### §3.4 R12 paired-frame в Tool Concept

| R12 clause | Tool Concept compliance |
|---|---|
| Voluntary opt-in | ✅ Partner self-chooses trial + Charter |
| Reversible | ✅ Tool stays partner's (Notion template owned по export); CC subscription cancellable |
| Member-portable | ✅ Notion = exportable; data = partner-owned; substrate Wiki = public |
| Falsifiable | ✅ 5-7 day trial = explicit hypothesis test |
| Glossaried | ✅ Walkthrough page + Welcome-frame ref |
| Impartial | ✅ Same offer/standards across partners (per tier) |
| Substrate-real | ✅ Real templates / real Claude Code / not toy |
| Fork-and-leave preserved | ✅ Partner exits → keeps everything (template + CC subscription own) |
| Disclose не conceal | ✅ Foundation cost transparency; reinvestment publicly logged |

**Pass clean.** Tool Concept = R12-cleanest option (real-world voluntary engagement structure).

### §4.3 Handoff session protocol (per Plan C C3)

**Pre-handoff (Day 0):**
- Schedule confirmed (1h block)
- Дмитрий accessible на video call
- Recording consent ack'd (R12)
- Template URL prepared (per category subset)
- Claude Code subscription created (за Дмитрий или Foundation-covered)

**Handoff session (1h live):**
1. Welcome + testing-posture re-emphasis (5 min)
2. Tool overview (5 min)
3. Notion template walkthrough — category by category (15-20 min)
4. Point-A → Point-B framework application к Дмитрий-context (10-15 min)
5. Claude Code basics demo (5-10 min)
6. Q&A + adjustments (5-10 min)
7. Daily check-in window agreement (5 min)
8. Wrap + 5-7 day trial commitment (5 min)

**Post-handoff (Day 0 evening):**
- Notion template URL shared
- Claude Code login confirmed working
- Calendar slot 15min/day check-in confirmed для 5-7 days

### §4.4 5-7 day trial protocol

| Day | Дмитрий activity | Foundation activity | Output |
|---|---|---|---|
| Day 1 | Populate point-A (current state) | 15-min check-in (Q&A) | Point-A documented в template |
| Day 2 | Continue point-A + start point-B (target state) | 15-min check-in | Point-A complete; point-B started |
| Day 3 | Complete point-B + daily reporting begin | 15-min check-in | Point-B documented |
| Day 4 | Daily reporting + time tracking | 15-min check-in | Real work данные accumulating |
| Day 5 | Continue daily reporting | 15-min check-in | Patterns emerging |
| Day 6 | Feedback prep + 1h feedback session | 1h feedback session | Iteration notes + template v0.1 fixes если needed |
| Day 7 | Optional iteration test | (passive) | – |

### §4.5 Feedback collection mechanism

**During trial (Days 1-5):**
- 15-min check-in: simple «что работает / что ломается / что неясно»
- Notion comments on template pages (Дмитрий-side annotations)
- Voice memos shared async (опционально)

**End of trial (Day 6-7):**
- 1h feedback session (recorded, consent)
- Structured questions:
  1. Что работало? (positive findings)
  2. Что ломалось? (negative findings)
  3. Что было неясно? (jargon / framework gaps)
  4. Что бы добавил? (feature requests)
  5. Что бы убрал? (overload signals)
  6. Continue or stop? (decision signal)
- Notes documented в `crm/people/dmitriy-DRAFT.md` §11 history

**Post-feedback:**
- Iterate template к v0.1 per findings (Plan C C9)
- Formal offer draft (Plan C C10) с tier selection per Ruslan
- R12 8-item check per offer (per Lev-master L7 matrix)
