## §4 Jetix SoTA-tracking gaps (5 critical surfaces)

### §4.1 Gap 1 — Replication discipline (Phase 5 §5)

**Что отсутствует:** Methodology compositions applied к real partner-clan → **formal replication?** Currently каждое использование Method V2 stands alone; нет formal cross-cohort replication.

**Pain:** SoTA-claim «Method V2 = SoTA» не falsifiable без replication track-record. Кажется хорошим, но **may be brittle to scale**.

**Phase 5 §5 fix candidate:** `wiki/experiments/` cluster — log every methodology application + outcome. Aggregated → replication metric.

### §4.2 Gap 2 — External benchmarks (Phase 4 §5)

**Что отсутствует:** Jetix-internal artefacts (Method V2, FPF) **not benchmarked externally** vs established alternatives (Scaled Agile, ISO 24744, OMG Essence, OpenAI eval suites).

**Pain:** SoTA-claim relative to whom? Without external comparison, claims могут be locked-in self-validation.

**Phase 4 §9 fix candidate:** Create comparisons cluster `wiki/comparisons/method-v2-vs-essence-vs-sebok.md` etc. Year-stamped, evidence-tied.

### §4.3 Gap 3 — Time-decay default + revalidation cadence

**Что отсутствует:** Most Jetix artefacts have `LOCKED` date but **no expiry / revalidation date**. Левенчуковский 5+5=10y curve uncalibrated for Jetix domain.

**Pain:** «Foundation LOCKED 2026-04-28» — when does it need revisit? 1 year? 2 years? Currently implicit «forever until proven obsolete» = anti-SOTA discipline.

**Phase 6 §4 fix candidate:** Adopt МИМ-style **2-year revalidation cadence** для core artefacts (Foundation, agents, principles). Year-stamp + scheduled re-review.

### §4.4 Gap 4 — Public SoTA-claim discipline для partner-facing

**Что отсутствует:** Currently NO explicit rule «what does it mean to say Jetix-offering = SoTA in X» для partner-facing claims.

**Pain:** Risk of dilettantism + Huyen-style "cutting edge" misclaim. R12 (anti-extraction) + AP-6 (dissent preservation) require honest SoTA-disclaimer.

**Phase 4 §9 fix candidate:** 10-rule discipline (§9 phase 4) applied к partner-facing materials. E.g. «Method V2 = SoTA-в-methodology-composition-as-of-2026-05-24» — year, scope, comparison-baseline explicit.

### §4.5 Gap 5 — Cross-paradigm SoTA-translation

**Что отсутствует:** Multiple paradigms exist within Jetix substrate (Левенчук + СМД + ML + cooperative governance + cybernetics). **Cross-translation layer пока неявная.**

**Pain:** Partner-clan merger (FPF B.3) requires translation; SoTA-claim в одной paradigm может НЕ соответствовать SoTA в другой.

**Phase 2 §3 + §10 fix candidate:** FPF Steward role explicitly — translation discipline. SOTA-claim multilingual artefact.

### §4.6 Gap 6 — Quantified agent qualification (МИМ-style 3-axis)

**Что отсутствует:** ROY agents не имеют quantified 3-axis profile (agency × scale × method-discipline). Только qualitative descriptions in `system.md`.

**Pain:** Без quantification — какой agent более «mature»? Какой ready для делегирования? Currently impressionistic.

**Phase 6 §2 fix candidate:** Implement 3-axis profile в `agents/<id>/qualification.yaml` updated per cycle. EQF-like 7-step scale per agent.

### §4.7 Gap 7 — Adversarial preservation tooling (AP-6 operational)

**Что отсутствует:** AP-6 declared (preserve dissent), но operational mechanism — где dissent stored? Searchable? Used?

**Pain:** Без tooling, dissent buried в archive/. Не surfaces during decision-making.

**Phase 2 §5 (Feyerabend) fix candidate:** `wiki/dissent/` cluster + index. Adversarial-frame preservation in F-G-R schema.
