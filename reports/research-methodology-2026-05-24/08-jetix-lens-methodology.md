  ontology)

- Russian-language audience constraint (Левенчук primary in Russian)

### §3.2 Option B — «Jetix as parallel extension of Russian methodology tradition»

**Articulation:** Jetix = independent parallel branch of Russian methodology
lineage (ММК → СМД → multiple continuators including Jetix as one). Jetix-
distinctive contributions = R12 anti-extraction governance + programmable

**Weaknesses:**

- May appear pretentious без validated track record
- Loses some Russian-methodology lineage prestige

### §3.3 Option C — «Jetix as fully hybrid methodology»

**Articulation:** Jetix = explicitly hybrid syntheses: Russian methodology

## §7 5 CRITICAL GAPS — methodology dimensions weak in current Jetix substrate

### §7.1 Gap 1 — Embodied skill dimension

**Substrate observation:** Jetix Workshop currently primarily **verbal / written**.
Embodied skill (Merleau-Ponty body schema + Polanyi tacit transmission +
Schön practicum demonstration) underrepresented.

**Manifestation:**
- Workshop = text-based curriculum + remote calls
- No physical co-presence patterns
- Voice pipeline transcribes но не captures embodied dimension
- ROY swarm operates fully text-mediated

**Mitigation R1 options (owner decides):**
- M1: In-person Workshop sessions (limited scale)
- M2: Video demonstration corpus (Polanyi pedagogy approximation)
- M3: Accept embodied-gap as feature (text-first cohort scaling priority)

### §7.2 Gap 2 — Cohort stage-awareness curriculum

**Substrate observation:** Workshop curriculum currently assumes **one-pathway
learners**. Dreyfus 5-stage model (Novice / Advanced Beginner / Competent /
Proficient / Expert) not explicitly informing curriculum design.

**Manifestation:**
- Single curriculum per Workshop
- Same methodology pedagogy для всех participants
- No stage-diagnostic at intake

**Mitigation R1 options:**
- M1: Develop stage-diagnostic intake protocol + 5-track curriculum
- M2: Develop 2-track minimum (Novice/Advanced Beginner vs Competent+)
- M3: Accept single-pathway; pre-select cohort by stage

### §7.3 Gap 3 — СМД-schematisation discipline

**Substrate observation:** Jetix uses mermaid + markdown. But **СМД-schematisation
conventions** (position + activity + situation + knowledge layered schemas) not
adopted. Russian methodology tradition's distinctive notational discipline missing.

**Manifestation:**
- Mermaid diagrams freeform (no positional / activity / situation conventions)
- Schematisation grammar implicit
- Visual artefacts not always have explicit "from which methodological position"

**Mitigation R1 options:**
- M1: Develop Jetix-СМД schematisation convention; adopt explicitly
- M2: Develop subset (3 of 4 schema types) for selective use
- M3: Accept gap (mermaid + markdown sufficient для current scale)

### §7.4 Gap 4 — Direct collaboration с Левенчук + ШСМ

**Substrate observation:** Jetix heavily mines Левенчук corpus but **no direct
collaboration** acked. Silent-inspiration model operative.

**R1 considerations:**
- Direct collaboration would strengthen Path 1 (continuator)
- Silent operation preserves Path 2/5 independence
- Trade-off between credibility + autonomy

**Mitigation R1 options:**
- M1: Reach out to Левенчук; propose collaboration / curriculum cross-licensing
- M2: Maintain silent-inspiration model; cite extensively in substrate
- M3: Partial — periodic public attribution + occasional cross-collaboration на
  specific topics

### §7.5 Gap 5 — Praxeology layer absence

**Substrate observation:** Левенчук M2025 line 289 explicitly DOES NOT use
праксиологию (Mises/Rothbard) as foundation; opts для systems engineering
tradition. Jetix inherits this gap.

**Consequence:** Jetix lacks formal **theory of action / value / exchange** as
foundation. R12 anti-extraction LOCK addresses some of this но не at axiomatic
foundation level.

**R1 considerations:** is praxeology axiomatic foundation needed? Or is current
"systems engineering + R12 ethics" sufficient?

**Mitigation R1 options:**
- M1: Add explicit praxeology layer; integrate Mises-Rothbard axioms
- M2: Add functional equivalent (constructor theory + cybernetic economics)
- M3: Accept absence; rely on systems-engineering + R12
