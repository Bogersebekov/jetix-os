## §1 Literature foundations

### §1.1 Cialdini commitment-consistency principle

**Pattern.** Once a person publicly commits к a position OR makes a small initial agreement, they're significantly more likely to honor subsequent larger requests aligned с that commitment.

**Implication for sequence design.**
- Start с smaller, easier asks (15-min call > 1-hr meeting > full partnership)
- Reference target's prior public commitment (paper / talk / tweet) to anchor consistency frame
- Soft audience first → tone calibration + learning → harder audience после
- First-conversation feedback → iterate template → improved later conversations

[src: Cialdini «Influence: Science and Practice»; modern operationalisation per `02-channel-tactics-research.md` §1.2]

### §1.2 Schein process consultation

**Pattern.** Helping relationship = built через interactive process — consultant (Jetix in our context) and client (target/partner) jointly assess what role is most helpful at present (doctor-patient / expert / process). Helping role evolves over time.

**Three modes Schein identifies:**
- **Doctor-patient mode** — target tells you what's wrong; you diagnose + prescribe
- **Expert mode** — target asks for specific deliverable; you provide
- **Process consultancy mode** — target + you collaboratively examine situation; jointly design intervention

**Implication for sequence design.**
- First conversation = NOT pitch; first conversation = assessment + helping relationship building
- Tone: «hey, I'm working on similar stuff, what are you working on?» NOT «I have THE solution, sign here»
- Soft audience first lets you practice **process consultancy mode** before harder audience где expert mode pressure higher
- Pre-existing partnership frame (O-75) directly maps к Schein helping relationship mindset — NOT extraction-mindset

[src: Schein «Process Consultation Revisited: Building the Helping Relationship» 1999; Semantic Scholar / Internet Archive freely available]

### §1.3 DARPA program manager pattern

**Pattern.** DARPA program managers operate as **idea-seeders** across academic / industry / government — reach across disciplines to uncover challenges at the seams. Fuel innovation ecosystem by empowering diverse collaborators.

**Sequence pattern observed:**
1. PM identifies high-leverage problem area
2. PM seeds idea с 5-10 trusted intellectual partners (informal conversations)
3. Feedback loop → idea refinement
4. PM hosts workshop / kickoff event → broader cohort recruitment
5. Cohort selected → funded → multi-year program execution
6. Results published → ecosystem amplification

**Implication for Jetix sequence design.**
- Phase 1 — 1-2 deep conversations с trusted partners (Дмитрий + Левенчук) = idea-seeding
- Phase 2 — Tier-1 cluster recruitment = workshop-equivalent (Karpathy + Buterin + Anthropic)
- Phase 3 — cohort formation (Engineer Workshop founding cohort 5-15)
- Phase 4 — daily cadence L2 amplification (DARPA program execution-equivalent)
- Phase 5 — public results = cascade 150 → 15 → 1M

[src: darpa.mil/about/program-managers; openbookpublishers «DARPA's Process for Creating New Programs»]

### §1.4 Community-led growth (open-source + creator economy)

**Pattern.** Sustainable cohort growth follows:
1. Founder authenticity + personal investment in 5-15 first cohort members
2. Cohort members reproduce founder's work to their networks (organic amplification)
3. Each cohort member ~10× extends reach
4. Stages crystallise into community pillars (mentors / contributors / users)

**Critical fail patterns:**
- Founder skips early hand-holding → cohort disengages → no amplification
- Aggressive volume early → reputation damage → cohort recruitment poisoned
- Extraction-only framing (R12 violation) → community rejects → fork happens

**Implication.**
- First 15 partners get ~80% of Ruslan's deep attention в Phase 1 (per «100h floor» concept Platform v2 §21.3)
- Daily cadence (Phase 4) → quality > volume; even at 10-20 touches/day discipline required
- R12 paired-frame discipline preserves long-term cohort trust

[src: open-source maintainer pattern observed across multiple ecosystems (Linux kernel / Kubernetes / Rust); creator economy referenced in `02-channel-tactics-research.md`]

### §1.5 Reichheld NPS — referral / amplification measurement

**Pattern.** Net Promoter Score = «how likely would you recommend [X] to a friend?» 0-10 scale.
- Promoters (9-10) — generate organic referrals
- Passives (7-8) — neutral
- Detractors (0-6) — negative WOM

**Application к Jetix:** measure «amplification factor» (per Phase 7) by tracking L2/L3 contacts publicly mentioning Jetix → these are promoters; their volume = forward indicator of cascade health.

[src: Reichheld «The Ultimate Question 2.0»; modern operationalisation per CRM SaaS playbooks]

## §2 Sequence design — Jetix-specific (per audio_697 C25)

```
Phase 1 (Week 1-2):    Дмитрий (humanitarian) — soft audience FIRST
```

### §2.2 Sequencing rationale (per literature foundations)

**Why Дмитрий first (per audio_697 C13 + C25):**
- Soft audience = lower-stakes test
- Tone calibration: humanitarian framing test before engineering framing test
- Schein helping-relationship mode practiced before harder audience
- O-86 Project-of-Humanity frame validated в actual conversation
- Custom pitch principle (O-94) iterated с feedback
- **Risk-controlled learning** before commitment к harder audience

**Why Левенчук second (per audio_697 C22):**
- Methodology authority — engagement signals validate Jetix substrate quality
- Ecosystem amplification potential: Левенчук circle ≈ 1500+ Russian engineers / Aisystant cohort / МИМ
- Direct cross-link к K-6 Method substrate + Левенчук inventory v2 (5 GAPS surface'нуты)
- If endorsement → significant credibility shift для Phase 3 Tier-1 cluster
- Pre-existing partnership frame O-75 paired directly с methodology overlap

**Why Tier-1 cluster third (per audio_697 cascade):**
- Leverage Левенчук intro (if achieved) — warm intro vs cold (4-7× reply rate per literature)
- Accumulated learning from Phase 1+2 → polished pitch
- C.1+C.2 ready by Week 4+ (gap analysis Phase 1)
- Multi-channel coordination (Twitter passive build + cold email + Telegram где applicable)
- Karpathy lineage credentials accrue when Jetix substrate references his work substantively

**Why L2 daily cadence then (per audio_686 + Pillar C max 20):**
- Phase 1-3 = high-investment per-contact (~30 min - 2 hr per touch для Tier-1)
- Phase 4 = scalable; 10-20 touches/day × 5-8 min per touch (per Phase 6 cadence)
- Manager attention budget (max 20 active tasks) sustainable
- Cohort effect: each L2 amplifier extends reach 10× (community-led growth pattern)

**Why mass cohort Q3 2026+:**
- Phase 4 amplification reaches saturation in addressable L2 community
- Workshop founding cohort 5-15 ready to recruit Wave 2+ (recursive cohort building)
- Cascade 150 → 15 → 1M = EOY 2026 KEYSTONE target

### §2.3 Phase-gate criteria

| Phase | Gate to proceed |
|---|---|
| 1 → 2 | Дмитрий conversation produced ≥1 actionable feedback OR ≥1 ecosystem intro | 
| 2 → 3 | Левенчук substantive engagement (≥30 min call OR substantive written exchange) | 
| 3 → 4 | At least 1 Tier-1 cluster member positive response (no need for full partnership; signal of engagement) |
| 4 → 5 | L2 cadence stable ≥4 weeks с amplification factor ≥3 (Phase 7 metrics) |

### §2.4 Reversibility / pivot triggers

**Pivot trigger Phase 1:** Дмитрий conversation reveals O-86 frame doesn't land → revise frame BEFORE Phase 2 launch.

**Pivot trigger Phase 2:** Левенчук engagement neutral → defer Tier-1 cluster launch (Phase 3); regroup substrate strengthen K-6 method evidence + Workshop runbook before retry.

**Pivot trigger Phase 3:** All Tier-1 cluster non-responsive → audit template (back to Phase 4 templates review) + escalate К-2 PITCH-BLOCKING patterns + consider channel mix shift (e.g. Twitter passive build longer before cold email).

**Pivot trigger Phase 4:** L2 cadence < 5 touches/day for 2+ weeks consecutive → burnout signal → halt + reflect + reduce cadence target до sustainable level.
