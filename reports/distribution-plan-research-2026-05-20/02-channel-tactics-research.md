## §1 Cold email

### §1.1 2026 benchmarks (literature)

- **Reply rate baseline:** 1-5% average B2B cold email; elite >10%; average 3.43%. [src: instantly.ai/cold-email-benchmark-report-2026]
- **Warm intro replies:** 34% vs 5% cold. [src: growleads.io/blog/warm-outreach-vs-cold-email]
- **Optimal length:** ≤80 words elite; 50-125 words for max reply rate. [src: outreach playbooks 2026]
- **Follow-up sequences:** 3-5 step sequences achieve 8.3% reply rate vs 4.1% without. [src: hunter.io/state-of-cold-email]
- **2026 signal:** infrastructure (domain config / sender reputation / timing) > copywriting in importance. [src: unifygtm.com/cold-email-2026]

### §1.2 Cialdini 6 principles applied

| Principle | Jetix application | R12 paired-frame check |
|---|---|---|
| **Reciprocity** | Lead с offer (substrate / research summary / intro к relevant person) BEFORE ask | ✅ offer-first inverts default ask-first; aligned R12 |
| **Commitment & consistency** | Reference target's prior public commitment (paper / talk / tweet) to surface shared meta-project | ✅ honors target's authored stance |
| **Social proof** | Mention 1-2 mutual peers OR shared community (Karpathy lineage / Левенчук circle / RU systems thinking) | ✅ if accurate; ❌ if fabricated |
| **Liking** | Specific reference к target's actual work (recent paper / tweet thread / podcast appearance) | ✅ basic respect; not manipulation |
| **Authority** | Foundation v1.0 LOCKED + ROY swarm + 5 concept docs F2 + Platform v2 = traction signals | ✅ verifiable substrate |
| **Scarcity** | DEFAULT-OFF — «срочность» framing risks reading as manipulative; only mention if real (Q3 2026 hackathon cohort sized) | ⚠️ AP-6 caution; honest scarcity OK, false scarcity violates R12 |

### §1.3 Cold email Jetix template structure (≤120 words)

```
Subject: [Specific reference to their work] + [1 verb hook]

[1-line opening: specific reference, not generic flattery]

[2-3 lines: shared meta-project frame O-75 — partnership pre-exists]

[Offer explicit: substrate / artifact / intro / specific value]

[Ask explicit + small: 15-min call OR feedback on specific artifact]

[Signature + 1-line traction signal verifiable]
```

**R12 paired-frame check:** offer line + ask line both explicit; no extraction-only ask.

## §2 Twitter / X

### §2.1 2026 benchmarks (literature)

- **Engagement rate:** small accounts (<10K) 1.2% avg / medium 0.6% / large 0.3%; platform-wide 0.5%. >1% = strong. [src: enrichlabs.ai/blog/twitter-x-benchmarks-2025]
- **Algorithm priority:** engagement velocity in first 30-60 min + replies weighted 15× more than likes + sentiment analysis rewards constructive. [src: posteverywhere.ai/blog/how-the-x-twitter-algorithm-works]
- **Inbound DM pattern:** for B2B founders, value = inbound DMs from content-attracted prospects (pre-qualified) > outbound cold DMs. [src: teract.ai/resources/grow-twitter-following-2026]
- **Trend:** average engagement dropped 18% since 2024; top 10% gain. [src: enrichlabs]

### §2.2 Jetix Twitter approach

**Two-mode operation:**

1. **Outbound:** strategic tagging Tier-1 (Karpathy / Buterin / Anthropic principals) in substantive content threads — NOT generic congratulations
2. **Inbound funnel:** content threads (Jetix substrate fragments + K-research insights + Foundation Arch posts) → followers DM Ruslan → pre-qualified conversation

**Cadence:** 2-3 substantive threads/week ≥ 5-tweet length; reply tactically в Tier-1/L2 discussions с substance (not just emoji or «great point»).

**Tagging discipline:** tag Tier-1 ONLY when content directly references their work / is genuinely useful to them; never spray-tag.

### §2.3 Thread structure template

```
Tweet 1 (hook): specific claim or counter-intuitive insight
Tweet 2-3 (substrate): evidence / citation / cross-link к Jetix substrate
Tweet 4 (frame): O-75 partnership frame OR O-86 zoom-out
Tweet 5 (ask): «curious how [Tier-1 named] thinks about this» — invite engagement
Tweet 6 (link): «full substrate here» → Jetix doc link
```

**R12 check:** content provides standalone value even if reader never engages; not bait-and-switch.

## §3 Telegram DM

### §3.1 2026 benchmarks (literature)

- **Open rate:** Telegram >80% vs email ~20% — 4× higher. [src: crmchat.ai/articles/telegram-ads-vs-telegram-outreach-2026-guide]
- **RU CIS tech / crypto / Web3 communities** — primary platform; FRI 116K+ traders; ICO Speaks RU CIS audience. [src: bitdegree.org/crypto/tutorials/best-crypto-telegram-groups]
- **Direct outreach** consistently outperforms paid ads in Web3/AI sectors. [src: crmchat.ai]
- **Critical success factors:** profile optimization / message personalization / respect rate limits / A/B testing / systematic follow-up. [src: marketingagent.blog/2026/01/08]

### §3.2 Jetix Telegram approach

**Primary use case:**
- Tier-1 named partners (Russian-speakers especially: Левенчук / Цэрэн / Дмитрий) — direct DM
- L2 RU systems thinking community amplification
- Engineer cohort coordination (Workshop founding cohort RU subset)

**NOT use case:**
- Cold spray DMs к unknown people — violates Cialdini principles + R12 + reputation risk
- Channel ads buys — defer to organic content + warm intro pattern

### §3.3 Telegram DM template

```
Привет [Имя],

[1 line: specific reference — recent post / talk / shared connection]

[2-3 lines: pre-existing partnership frame O-75 explicit — «мы оба работаем над X»]

[Offer explicit + concrete: «вот substrate / link / artifact которое возможно useful»]

[Ask explicit + minimal: «было бы интересно 15-20 мин созвон / краткий feedback»]

Руслан
[Link к 1 anchor doc для context]
```

**R12 check:** opening не flattery-only; offer concrete; ask proportional; no «time-pressure» language unless real.

## §4 Podcast guest-spot

### §4.1 2026 benchmarks (literature)

- **Acceptance rate:** well-matched pitches → ~70% positive host responses; 36% rejections = «wrong fit for show». [src: blog.justreachout.io/how-to-be-a-guest-on-a-podcast]
- **Tier-segmented blended booking conversion:** 16-24% on 50-show list. [src: forkoff.xyz/blog/podcasts/podcast-booking-system-founders-2026]
- **Pitch length:** ≤120-200 words; 3-5 short paragraphs. [src: lowerstreet.co/how-to/podcast-pitch]
- **2026 reality:** AI-generated spam saturated hosts' inboxes; personalization standard at all-time high. [src: justthink.ai]

### §4.2 Jetix podcast strategy

**Two-mode:**

1. **Guest-spot pitches** — Ruslan as guest on AI / engineering / systems-thinking podcasts (Karpathy adjacent / EA-adjacent / Russian tech / systems thinking)
2. **Host invitations** — for Jetix-owned podcast (Q3+ 2026; bets P4 priority; not Phase 1 scope)

### §4.3 Podcast pitch template (≤200 words)

```
Subject: [Specific episode reference] + [1 angle hook]

[1-line opening: specific past episode + what resonated]

[2-3 sentences: insight you bring — concrete claim, not bio]

[Frame O-75: shared meta-project anchor]

[Offer: 1 specific story / data / unique angle — «here's the thing your audience hasn't heard»]

[Ask: 30-min recording; flexible scheduling]

[1-line credibility signal: verifiable]

[Calendar link OR offer to suggest 3 dates]
```

**R12 check:** value to audience explicit (not just plug Jetix); offer-first; ask proportional.

## §5 IRL events (hackathons / conferences / meetups)

### §5.1 Pattern (literature + general practice)

- Conference / hackathon attendance ≠ outreach; **structured pre-event + during + post** outreach = outreach
- Pre-event: identify 5-10 attendees worth meeting → outreach 1-2 weeks before requesting 15-min coffee
- During: 1-2 substantive conversations > 20 superficial introductions
- Post-event: 48-hour follow-up email + Telegram DM with specific reference to conversation

### §5.2 Jetix IRL strategy (Q3 2026+)

**Priority events (sizing Q3 2026):**
- AI engineering conferences (Karpathy-adjacent / OpenAI DevDay / Anthropic events)
- Berlin Mittelstand + DACH government events (Platform v2 §02 Cat 16-17 audiences)
- Russian systems-thinking community events (МИМ / Левенчук circle / Левенчук-adjacent Aisystant)
- AI safety / EA events (Anthropic / EAG / LessWrong London)
- Crypto + Web3 events (Devcon Buterin-adjacent; only где AGI substrate framing resonates)

**Cadence Q3 2026:** target 1-2 IRL events per month after Workshop founding cohort established.

### §5.3 IRL approach template

**Pre-event email/DM (≤80 words):**
```
[Привет / Hi] [Имя],

Заметил [event name] на [date]. Я еду / буду в [city] для [reason — Workshop / Hackathon launch / substrate exchange].

Тема которой работаю overlap с твоей: [1-sentence specific].

15-20 мин кофе по теме [X]? Гибкий по времени — [3 time slots].

[Calendar link]

Руслан
```

**R12 check:** ask proportional к event format; offer не just «pick your brain»; specific overlap claim verifiable.

## §7 Cross-channel coordination

**Pattern:** multi-channel touch sequence is more effective than single-channel cold sequence.

**Recommended Jetix touch sequence per Tier-1 partner (when warm intro unavailable):**

1. Twitter follow + engage с 1-2 substantive replies (passive familiarity build)
2. Telegram DM (if available) — pre-existing partnership frame opening
3. Cold email (if Telegram unavailable OR Telegram declined) — same frame + attachment к C.1 one-pager
4. IRL event meet (if event upcoming) — pre-event DM
5. Podcast guest cross-amplification (если applicable)

**Timing:** spread 2-3 weeks between channel touches; don't burst all channels in 48 hours (= harassment pattern).

## §8 R12 paired-frame discipline per channel

| Channel | Offer line example | Ask line example | Audit pattern |
|---|---|---|---|
| Cold email | «Прикладываю one-pager по [X] которое возможно useful» | «15-min созвон если интересно по [Y]» | Both lines present; ask proportional |
| Twitter | thread provides standalone insight even if no reply | «curious how [X] thinks about this» tagged | Content-value-first |
| Telegram | «вот substrate которое overlap с твоим [X]» | «20-min созвон по [Y]» | Direct + minimal |
| Podcast | «specific story / data / angle для audience» | «30-min recording» | Audience-value-first |
| IRL | «overlap с твоей темой [X]» | «15-min кофе» | Pre-event proportional |

**Universal rule:** если template draft missing explicit offer OR explicit ask → flag «requires revision before use» (per Phase 4 audit).
