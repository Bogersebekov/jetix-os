## §3 Overview layering (несущая структура P-1 + логика всего пакета)

Per Ruslan voice 29.05 + ACK — Overview подаётся **слоями**, и 3P **накладывается** на базу, а не сам
по себе:

```
СЛОЙ 1 (база)   методология работы с информацией
                 «всё = информация + методы её обработки»
                 интеллект (человек/система) пытается развиваться,
                 используя лучшие методы для собственного развития
                          │
                          ▼ накладывается
СЛОЙ 2 (3P)     продукты / процессы / проекты
                 = информация + методы работы с ней В ПРИМЕНЕНИИ
                 жизнь человека = его главный продукт;
                 управление = люди + системы + процессы + технологии;
                 «ты управляющий своей жизнью, но и другие хотят ею управлять»
                          │
                          ▼ из этого вырастает
СЛОЙ 3 (Jetix)  суть Jetix
                 мега-мастерская + сеть кооперативных кланов + возможности
                          │
                          ▼ отсюда — конкретика
СЛОЙ 4 (нужно)  что нужно на текущий момент
                 фундамент (tech / fin / legal) + помощь + партнёры
```

> ⚠️ Критично: **сначала база (информация + методы → саморазвитие интеллекта), потом 3P как её
> применение.** 3P НЕ самостоятельная рамка — он наложение на базовый слой.

## §4 Tone discipline (из VOICE-PIPELINE-PUBLIC — gold-standard)

Что переносим в каждый из 5 документов:
- **Честная рамка.** «Вклад честно небольшой», «substrate ≠ peer-reviewed», «середина Build». Не «купи».
- **Founder-as-Exhibit / anti-marketing.** «Вот как устроено, проверь, скажи где не так» — не воронка.
- **Vivid concrete.** Один живой пример в каждом документе («как это в жизни»).
- **Free goodbye.** Выход звучит легко; no pressure.
- **R12 floor.** NO «миллионер / корпорация / станешь богатым». Только «возможностей — куча, потенциал
  колоссальный» (per O-244 REFRAME-STRICT + ACK решение #4).
- **Humble scale.** «Занимаюсь основами / фундаментом», не «строю единорога».
