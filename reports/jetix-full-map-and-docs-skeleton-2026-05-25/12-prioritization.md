## §5 Что это меняет (R1 surface для Руслана)

**Факт #1:** Build exit держится на ~5 P0. Из них **видео A — единственный, что блокирует
4 другие сущности сразу**. По чистой топологии зависимостей — это первый рычаг. *(Это
наблюдение зависимости, не команда приоритета — приоритет выбирает Руслан.)*

**Факт #2:** ~⅔ missing-документов (Run/Scale) — **намеренно P2-P4 сейчас**. Их рано
делать: курс пишет T1-партнёр в Run, HR — при найме, DAO/legal — в Scale. Делать их в
Build = тратить время не на тот этап (ловушка «накопить ещё»).

**Факт #3:** есть «вторая волна» дыр (HR/Legal/Financial-ops), невидимая в текущем
DOCS-CLASSIFICATION, но критичная при переходе Build→Run (найм 5 ассистентов требует
compensation structure + hiring docs).

**Варианты для Руслана:**
- **Вариант A (sequential):** P0 Build по одному, видео A первым.
- **Вариант B (parallel tracks):** видео A (Руслан) ∥ Charter текст (после R12-ревью) ∥ legal (Steuerberater) — три независимых трека.
- **Вариант C (quick-wins first):** discovery script + Steuerberater email + D-LOCK promotion (дёшево) → потом видео A.

*Финальный выбор — Руслан.*
