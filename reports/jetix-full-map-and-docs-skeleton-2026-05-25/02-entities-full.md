## 1. 🟦 Метод

### Определение
Метод в Jetix — это **способ что-то сделать с информацией**, который мы трактуем как
самостоятельный объект: его можно изучать, передавать, комбинировать и сравнивать по
качеству. Главный метод Jetix (формулировка O-107) — «метод объединения методов по
улучшению системы самой себя»: то есть не «как сделать X», а «как выбирать и собирать
способы делать что угодно, и применять это к самой системе, которая выбирает». Внутри —
четыре уровня: уровень 0 (сырое действие) → 1 (применяю метод) → 2 (выбираю метод) →
**3 (метод выбора методов / мета-метод)**. Большинство образовательных систем живут на
уровнях 1-2; ключевое отличие Jetix — уровень 3. Поверх него — **мета-контроль**
(«управляю не X напрямую, а Y, который управляет X» — даёт множитель рычага на каждом
подъёме) и **принцип внешней управляющей системы** (управляемая система структурно
имеет слепые зоны, и независимая внешняя система добавляет в них разнообразие — это
кибернетически доказуемо).

### Что внутри
- **Метод как объект 1-го класса** — есть вход, правило преобразования, выход, оценка качества (F-G-R); носитель метода ≠ сам метод (IP-1).
- **Мета-метод (уровень 3)** — 6-шаговая процедура: определить систему влияния → собрать достаточно информации → обработать → поставить целесообразную цель → выбрать лучший метод → применить + замерить + поправить.
- **8 компонент мета-метода** (кластерная форма из 16 полных, голос audio_719): радикальная проработка / фокус на важном / честность (фундамент) / рычаг (leverage) / развитие системы / изучение системы / «1000% голодный» / захват-победа (с R12-рамкой). Полная форма — 16 компонент в 9 кластерах.
- **Кандидат C17 «восстановление»** — критический пробел: без компоненты восстановления есть риск выгорания (failure mode A4). Три варианта закрытия surfaced в DR-38.
- **Мета-контроль / управление через уровень-выше** — L0 операции → L1 peer-лидеры → L2 стратегия (Руслан + brigadier) → L3 конституциональный уровень (Foundation + Pillar C + R12).
- **Принцип внешней системы (O-128)** — три прочтения: сильное (Эшби «закон необходимого разнообразия» + Гёдель «неполнота») / среднее (для внутреннего substrate) / слабое (Матурана-Варела «структурное сопряжение», для публичной R12-чувствительной коммуникации).
- **Градиент достаточности** — не «идеальная информация», а «достаточная» (Саймон, bounded rationality): кривая «качество × объём» загибается.
- **Метафора Франкенштейна / солянки** — метод собран из многих малых; это «живая композиция», не мёртвая.
- **Прямая линия родословной** — Щедровицкий (ММК) → Левенчук → Руслан; вокабуляр «метод / методология / инженерия личности» взят напрямую.

## 4. 💰 Заработковые модели

### Определение
Доходная архитектура Jetix построена на трёх принципах: **(1)** рекурсивные 25% —
каждый партнёр отдаёт 25% в общий фонд, из которого рекурсивно выделяются слои
компенсации; **(2)** тройная роль — один человек одновременно воркер (получает 75%),
инвестор (доля фонда) и промоутер (бонус за приведённых); **(3)** замкнутый контур —
ценность остаётся внутри Jetix-акторов без утечки наружу. Поверх — потолок неравенства
5:1 (Mondragón) и право уйти без штрафа (R12). Минимум 8 различимых моделей дохода.

## 5. 🚀 Платформа

### Определение
Платформа Jetix — не одна вещь, а **три режима одной системы**: Build / Run / Scale.
Переключатель не «сколько людей», а **«кто крутит маховик»**. Build = основатель руками,
методология на бумаге, метод снаружи не прогнан. Run = живёт петля «делаем → смотрим
отклик → правим», когорта 5-50 платит, 1-2 партнёра T1 со-создают курс. Scale = несколько
кланов параллельно, система растёт сама, основатель — один из хранителей. Слои: Personal
OS (Layer 1-2, Notion + Claude Code для индивида) → Team OS (Layer 3, multi-tenant) →
Кланы (Scale, автономные единицы с Charter-overlay).

### Что внутри
- **Build / Run / Scale** с критериями выхода (≥80%): Build exit = ≥1 T1 + ≥3 T3 + Charter проверен R12-экспертом + Notion multi-user + звонок отрепетирован ≥5 раз + юр начато. Run exit = 1K+ users, 3+ когорты, €10K/мес+, 1-й клан. **Текущее состояние: Build, середина (25.05).**
