# 🏗️ Phase 8 — 12 категорий документов Jetix

> **Что это.** Двенадцать категорий, в которые раскладываются ВСЕ документы «нормальной
> корпорации» Jetix. Для каждой: зачем она / кому / в каком формате / привязка к 4
> категориям DOCS-CLASSIFICATION (🟢/🛠️/📚/⚙️) и к референсным паттернам Phase 7. Это
> каркас для Phase 9 (per-doc списки).

---

## §0 TL;DR

12 категорий, сгруппированных по «кому в первую очередь»:
- **Наружу (🟢):** Executive, Brand/Marketing, Partner-facing
- **Для работы (🛠️):** Platform/Product, Community/Cohort, Operational, HR/People
- **Глубина (📚):** Methodology/IP, Research/Knowledge, Financial (часть)
- **Инфраструктура (⚙️):** Legal/Governance, Risk/Compliance

**R12 paired-frame STRICT** применяется к 4 категориям: Partner-facing, Community/Cohort,
Brand/Marketing, HR/People (там, где есть динамика влияния/вербовки).

---

## §1 📜 Executive / Strategic

- **Зачем:** объяснить «куда и зачем идёт компания» — для себя, партнёров, инвесторов.
- **Кому:** Руслан (первично), T1/T2 партнёры, T2 капитал.
- **Формат:** Markdown (internal) + PDF/лендинг-секция (наружу). Аналог: Tesla Master Plan, Berkshire annual letter.
- **DOCS-CLASS:** 📚 (internal стратегия) + 🟢 (публичная версия).
- **R12:** не требуется (нет динамики влияния).

## §2 🧪 Methodology / IP

- **Зачем:** зафиксировать метод как передаваемый и лицензируемый актив.
- **Кому:** T1 методологи (selective deep), education-cohort (через адаптацию), internal.
- **Формат:** Markdown canonical + human-language версия + видео. Аналог: Anthropic Constitutional AI, Apple HIG.
- **DOCS-CLASS:** 📚 (canonical) + 🟢 (видео A для людей).
- **R12:** не требуется напрямую (но «прошивка» education → см. §11).

## §3 🏗️ Platform / Product

- **Зачем:** описать продукт (Personal/Team OS, Workshop) — что это, как работает, как настроить.
- **Кому:** T3 тестеры/cohort, T1 co-creators, internal.
- **Формат:** Notion DB (шаблоны) + Markdown specs + changelog. Аналог: Apple API docs, Tesla product specs.
- **DOCS-CLASS:** 🛠️ (шаблоны) + ⚙️ (внутренняя архитектура).
- **R12:** не требуется.

## §4 👥 Community / Cohort

- **Зачем:** правила и инфраструктура сообщества — как вступить, как себя вести, как выйти.
- **Кому:** cohort members, T3/T4, mentor matrix.
- **Формат:** Charter (Markdown/PDF) + Notion onboarding + Code of Conduct. Аналог: John Lewis Constitution, Mondragón member agreement.
- **DOCS-CLASS:** 🛠️ (Charter/onboarding) + 🟢 (публичная Constitution).
- **R12:** ⚠️ **STRICT** — здесь главная зона вербовки/культа. anti-cult discipline + influence-ethics auto-fire.

## §5 💰 Financial

- **Зачем:** деньги системы — модель, бюджет, runway, цены, контракты, инвойсы.
- **Кому:** Руслан, Treasurer (Run), T2 капитал, аудит.
- **Формат:** Markdown (модель) + spreadsheet (бюджет/runway) + PDF (invoice/contract). Аналог: Mondragón financial transparency, Berkshire intrinsic value.
- **DOCS-CLASS:** 📚 (модель V10) + 🛠️ (invoice/budget templates).
- **R12:** косвенно (5:1 cap, distribution policy).

## §6 ⚖️ Legal / Governance

- **Зачем:** юридическая оболочка + правила управления + след решений.
- **Кому:** Руслан, Steuerberater/юрист, Steward, DAO (Phase 2+).
- **Формат:** Legal docs (PDF) + Markdown (Stage Gate records, Steward decisions log) + on-chain (Phase 2+). Аналог: Mondragón statutes, John Lewis governance records.
- **DOCS-CLASS:** ⚙️ (Foundation/governance) + 🛠️ (контракт templates).
- **R12:** косвенно (Charter legal = enforcement R12).

## §7 🎨 Brand / Marketing

- **Зачем:** как Jetix выглядит и звучит наружу — единый голос, визуал, сообщения.
- **Кому:** mass audience, T2 аудитория, все каналы.
- **Формат:** Brand book (PDF) + messaging guide (Markdown) + лендинг + pitch deck + одностраничники. Аналог: Apple brand book.
- **DOCS-CLASS:** 🟢 (всё наружу).
- **R12:** ⚠️ **STRICT** — messaging не должен манипулировать (pain-primary но честный); propaganda-expert + influence-ethics.

## §8 🔬 Research / Knowledge

- **Зачем:** глубина — research-deep'ы, wiki, книги, decisions log.
- **Кому:** internal (первично), curious deep-divers (1-2 ref на разговор).
- **Формат:** Markdown (research mains) + wiki (9 типов сущностей) + git log. Аналог: OpenAI/Anthropic research papers (но internal).
- **DOCS-CLASS:** 📚 (всё depth).
- **R12:** не требуется (но propaganda/nlp deeps = R12 STRICT в применении).

## §9 🤝 Partner-facing

- **Зачем:** всё, что показываем партнёру — оффер, discovery script, onboarding, agreement.
- **Кому:** T1-T4 по типам (audience × category матрица DOCS-CLASS §7).
- **Формат:** Markdown (оффер/script) + PDF (agreement) + видео C. Аналог: Berkshire acquisition criteria.
- **DOCS-CLASS:** 🟢 (оффер) + 🛠️ (agreement/script).
- **R12:** ⚠️ **STRICT** — 8 вопросов перед касанием; recruitment-dynamics + influence-ethics auto-fire.

## §10 📊 Operational

- **Зачем:** ежедневная работа — CRM, outreach, pipeline, support.
- **Кому:** internal, sales-роли, support (Run).
- **Формат:** Notion DB + Markdown (CRM) + spreadsheets. Аналог: Apple retail playbook.
- **DOCS-CLASS:** 🛠️ (рабочие шаблоны).
- **R12:** косвенно (outreach = R12 8 вопросов).

## §11 🎯 HR / People

- **Зачем:** роли, найм, перформанс, компенсация (кооперативная Mondragón-надстройка).
- **Кому:** Руслан, hires (Run, 5 ассистентов Month 1), Team OS роли.
- **Формат:** Markdown (role defs) + Notion (hiring) + spreadsheet (comp). Аналог: Mondragón worker-owner handbook.
- **DOCS-CLASS:** 🛠️ (role defs) + ⚖️ (comp policy = R12).
- **R12:** ⚠️ **STRICT** — wage-ratio 5:1 + non-extraction в comp structure.

## §12 🚨 Risk / Compliance

- **Зачем:** реестр рисков + R12 compliance log + audit trail + privacy.
- **Кому:** internal, Steward, Прапион (R12 ревью), аудит (Phase 2+).
- **Формат:** Markdown (risk register) + JSONL (R12 log, `swarm/approvals/log.jsonl`) + PDF (privacy). Аналог: Anthropic RSP, Trust & Safety.
- **DOCS-CLASS:** ⚙️ (compliance infrastructure).
- **R12:** ⚠️ это **сама** R12-инфраструктура.
