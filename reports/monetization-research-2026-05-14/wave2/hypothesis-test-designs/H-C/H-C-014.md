## H-C-014 — Realm exclusion — extreme cases; fork-and-leave preserved

**Hypothesis statement:** Severe violations (fraud / harassment / repeated extraction) → Council vote → Realm exclusion. R12 preserves IP + rank earned + fork-and-start elsewhere.
**Why test:** Last-resort deterrent; visible punishment for major defection.
**Minimum viable test:** Define exclusion process; ideally zero exclusion cases Phase 1 (small Realm); if case arises — handle per protocol.
**Success metric:** Exclusion process defined + acked by Council; if used, member's IP preserved per R12; legitimacy maintained.
**Failure metric:** Exclusion legitimacy crisis; biased application; member sues; Realm splits over exclusion decision.
**Confounders:** Small sample; case specifics dominate.
**Precedent test:** Wikipedia editor ban (sustained, occasional controversy); Stack Overflow account suspension; corporate ethics committee dismissal; Mondragón cooperative expulsion (very rare).
**Phase fit:** Phase 1+ (process defined; sparingly used).
**Required L1 fit:** Any (Council vote).
**R12 retest:** Verify R12 explicit: (a) excluded member keeps IP/rank earned; (b) appeal process exists (external arbitration); (c) fork-and-start preserved; (d) supermajority vote required (≥2/3 Council).
**Cross-pollination:** Combines with H-C-013 (decay precedes exclusion); blocks if H-M-017 Standards Body abuses exclusion (cartel risk); preserves R12 fork-and-leave anchor.
**Bayesian prior:** **Medium-High** — Wikipedia / Mondragón precedent.
