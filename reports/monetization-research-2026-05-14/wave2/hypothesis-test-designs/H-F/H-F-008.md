## H-F-008 — Federated identity protocol — Realm passport

**Hypothesis statement:** Member identity portable across Clans + external services; cryptographic signature (SSI / W3C DID-VC); reduces re-onboarding friction.
**Why test:** Reduces switching cost; enables Realm reputation portability externally.
**Minimum viable test:** Phase 2+; deploy W3C VC-based credential schema; integration с Estonia e-residency / Microsoft Authenticator.
**Success metric:** ≥50% of members adopt verifiable credential; external service integration works ≥1 case (e.g., LinkedIn skills); credential portable on fork.
**Failure metric:** Adoption <10%; technical complexity blocks; standards drift makes VC non-interoperable.
**Confounders:** SSI / VC standards immature; member technical sophistication varies.
**Precedent test:** Estonia e-residency digital ID (100K+ residents, 10y); W3C Verifiable Credentials (deployed 2019+, growing); OpenID Connect; Microsoft Entra Verified ID.
**Phase fit:** Phase 2-3+.
**Required L1 fit:** Any.
**R12 alignment:** ✓ credential portable; member-controlled.
**Cross-pollination:** Combines with H-M-008 (cert issuance); combines with H-F-002 (rank portability); amplifies external recognition.
**Bayesian prior:** **Medium** — Estonia / W3C VC growing but unproven at scale; technical complexity barrier.
