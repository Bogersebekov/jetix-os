## §1 Что такое «платформа Jetix» — три разных объекта на трёх этапах

Главная ловушка слова «платформа»: оно звучит как одна неизменная вещь. На деле под ним
прячутся **три разных системы**, и они отличаются не размером, а тем, **что внутри
работает само, а что держится на руках основателя.**

| Этап | Что это в FPF-терминах | На человеческом |
|---|---|---|
| **🏗️ Build** | system-under-construction | Система, которую ещё **собирают руками.** Основатель — главный (и почти единственный) механизм. Методология зафиксирована на бумаге, но ещё не проверена третьими людьми. Notion-шаблоны существуют как дизайн, не как рабочие копии у пользователей. Денег нет или они из своих сбережений. |
| **▶️ Run** | operating-system (с обратной связью) | Система, которая **работает как цикл.** Мастерская идёт с реальными участниками, не только с основателем. Деньги текут (ступень 5 ≈ €1500/мес). Методологию одновременно проверяют несколько человек. Charter подписан. Появляется кибернетическая петля: делаем → смотрим отклик → правим → снова делаем. Основатель уже не единственный движок. |
| **📡 Scale** | self-propagating system (само-распространяющаяся) | Система, которая **растит сама себя.** Несколько кланов/сообществ параллельно, не один центр. Аудитория растёт через блогеров/предпринимателей, а не через рассылки основателя. Доход распределяется по шаблонам кооператива. Потолок 5:1 и «не доим / не запираем» защищены механически (смарт-контракты, если включён этот путь). Основатель — один из многих хранителей, не единоличная власть. |

**Ключевой переключатель между этапами — не «сколько людей», а «кто крутит маховик».**
В Build маховик крутит основатель. В Run маховик крутят основатель + первые партнёры + петля
обратной связи. В Scale маховик крутится сам, и его уже нельзя остановить уходом одного
человека (включая основателя). Это и есть смысл слов construction → operating → self-propagating.

## §2 Frame of Reference (FoR) — с какого угла смотрим на каждый этап

Один и тот же этап выглядит по-разному, смотря откуда смотришь. FPF говорит: всегда называй
угол зрения (Frame of Reference). Для платформы полезны пять углов; на каждом этапе главный
угол свой.

| Угол зрения (FoR) | Вопрос, на который отвечает | Главный на этапе |
|---|---|---|
| **Система** | Что это за объект целиком и где его границы? | Build (что мы вообще собираем) |
| **Роль** | Кто что делает (функции, не имена)? | Run (кто крутит маршрут участника) |
| **Метод** | Какой методологией это работает? | Build → Run (метод фиксируется → проверяется) |
| **Рабочий продукт** | Какие артефакты на выходе (видео, шаблон, Charter)? | Build (что не стыдно показать) |
| **Паттерн** | Какая повторяющаяся форма масштабируется? | Scale (что копируется в каждый клан) |

**На пальцах.** В Build главный вопрос — «что мы строим и какие артефакты на выходе»
(угол: система + рабочий продукт). В Run — «кто и как ведёт человека по ступеням» (угол:
роль + метод). В Scale — «какая форма копируется сама в каждый новый клан» (угол: паттерн).
Если перепутать углы — будешь, например, в Build бесконечно проектировать роли когорты,
которой ещё нет, вместо того чтобы записать видео A.

## §3 FPF-примитивы, применимые на каждом этапе

FPF даёт набор «кирпичей мышления». Не все одинаково важны на разных этапах. Вот какие
работают где.

| Примитив | Build | Run | Scale |
|---|---|---|---|
| **Роли (Role ≠ Executor, IP-1)** | роли = типы (методолог, тестер); имена = примеры | роли укомплектованы людьми; multi-hat начинается | роли ротируются; «нет роли на всю жизнь» |
| **Методы** | Метод V2 зафиксирован, не проверен 3-ми лицами | метод проверяется N людьми одновременно | метод форкается под каждый клан |
| **Рабочие продукты** | видео A/B/C + 5 шаблонов + лендинг (делаем сами) | партнёры создают курс + кейсы | community-варианты + многоязычность |
| **Границы (bounded contexts)** | граница = «личная система» одного человека | граница = «общий котёл» одной когорты | граница = клан со своим Layer 2 |
| **Провенанс (F-G-R, R6)** | каждое утверждение с источником | кейсы с провенансом | кейсы 1000+, провенанс автоматизирован |
| **Слой этики (R12)** | 8 вопросов перед касанием; риск низкий | 8 вопросов + Mondragón 5:1; риск средний | 8 вопросов массово + смарт-контракты; риск высокий |

**Главное наблюдение.** Этический слой (R12) — единственный примитив, чья **важность растёт
с этапом.** Всё остальное либо стабильно, либо упрощается. Это не случайно: чем больше людей и
денег в системе, тем больше соблазн «выжать» и «запереть». Поэтому защита должна расти быстрее
системы — иначе на Scale получится секта или обычная корпорация с диктатором.

## §4 F-G-R mandate — почему каждое утверждение несёт три метки

FPF требует: серьёзное утверждение не висит в воздухе, оно несёт три метки.

- **F (Formality, формальность)** — насколько строго зафиксировано. F2 = черновик/пул;
  F8 = конституционный LOCK. Этот документ — **F2-F3** (синтез поверх substrate, не LOCK).
- **G (Group-scope, область применимости)** — для кого верно. Весь этот план = область
  `prompt-platform-lifecycle-stages-2026-05-25`.
- **R (Reliability, надёжность)** — насколько проверено. R-low = гипотеза; R-high =
  многократно подтверждено.

**На человеческом:** когда ниже написано «в Run появляется доход €5-15K/мес» — это **не факт,
а проекция (F2, R-low)**, и так и помечено. А «Foundation v1.0 LOCKED 28.04» — это **факт
(F8, R-high)**. Смешивать их нельзя: тогда мечты выглядят как обещания, и партнёр потом
поймает на завышении. Поэтому в основном документе цифры будущего идут с пометкой «проекция,
не обещание» — ровно как в consolidated-hl §8 («это направление, не обещание»).

## §5 Анти-паттерны (флаги из памяти) — чего этот документ НЕ делает

Из накопленного опыта — три типичные ошибки, которые здесь намеренно обходим:

1. **Объёмы без контекста.** «1509 коммитов», «402 книги», «162 вики» — это не достижение
   само по себе. Цифра идёт только когда отвечает на вопрос «и что это даёт на этом этапе».
2. **Утверждения без F-G-R.** Каждое крупное утверждение о будущем — с пометкой формальности
   и надёжности. Будущее = проекция, не факт.
3. **Сравнение без совпадения масштаба.** Не сравниваем сегодняшний Build (0 участников) с
   будущим Scale (100K) как равные — это разные системы. Сравнения всегда внутри одного этапа.

Плюс — границы из самого prompt: **никакого нового ресёрча**, никаких новых книг/источников,
никаких правок LOCKED-документов и Foundation, никакого авто-запуска следующих действий
(видео не записываются, Wave 1 не отправляется, Notion API не трогается).

## §6 Что эта рамка даёт следующим фазам

После этой фазы у нас есть общий словарь:
- «Платформа» = три разных системы (construction / operating / self-propagating), а не одна.
- Переключатель этапов = «кто крутит маховик», не «сколько людей».
- Угол зрения называется явно (система / роль / метод / продукт / паттерн).
- R12-защита растёт быстрее системы — это сквозной закон всех следующих фаз.
- Будущее = проекция с метками F-G-R, не обещание.
