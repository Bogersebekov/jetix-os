# 📥 Phase 0 — Что прочитано, что входит в публичный скоп, для кого

> **Что это.** Базовая фаза метаплана. Здесь — что я прочитал из substrate, какой
> **фильтр** отделяет публичное от внутреннего, и **кто читатель** этих документов. Без
> фильтра любой вариант структуры превращается в кашу «свалим всё, что есть». Фильтр —
> это то, что делает метаплан метапланом, а не свалкой.

**Главный вывод чтения (он же задаёт весь метаплан):** у Jetix **глубина закрыта, упаковка
наружу — нет**. Внутренние сущности (Метод / Корпорация / R12 / Research) имеют почти полный
doc-coverage; outward-сущности (Платформа / Образование / Партнёры / Community) — дырявые.
Значит публичный набор документов не пишется с нуля — он **переупаковывает уже готовый
substrate в человеческий жанр для конкретной аудитории**. Метаплан отвечает на один вопрос:
**по какой логике резать** этот substrate на публичные документы.

## §2 Фильтр публичного скопа

Решающее различение — **что выходит наружу, а что остаётся внутри**. Беру разрез из
DOCS-CLASSIFICATION (4 категории) и накладываю на 12 сущностей.

### ✅ Входит в публичный скоп (про это пишем публичные документы)

| Сущность / тема | Почему публичное |
|---|---|
| 🧪 **Метод / Методология / Образование** | главный оффер; то, ради чего партнёр приходит (включая прошивку + cross-skill + 7 ступеней) |
| 🚀 **Платформа** (Personal/Team OS + Workshop + Community) | что человек получает руками |
| 💼 **Бизнес** — как устроена компания + как работает | партнёру нужно понять, во что он входит |
| 💰 **Заработок** — модели дохода на пальцах (75/25, triple-role, 5:1, Workshop €1500) | прямой интерес партнёра; **но не финансовая отчётность** |
| 👥 **Партнёры / Команда** — 4 типа, что просим / что даём | приглашение к сотрудничеству |
| 🎯 **Видение / Стратегия** — куда идём + зачем | рамка, в которую всё складывается |
| ⚖️ **R12 как обещание** («не доим / не запираем / уйти без штрафа») | публичное обязательство, не внутренний механизм |

### ⛔ НЕ входит (внутреннее или отложено)

| Что | Почему вне скопа |
|---|---|
| 📑 **Legal documents** | defer — потом найдём специалистов (per Ruslan voice 25.05) |
| 💵 **Financial reporting** (budget / invoice / bookkeeping / statements) | defer — отдельные специалисты; модели дохода ≠ отчётность |
| 🔬 **Research raw** (5 deeps / 80+ книг / wiki 750 сущностей) | internal-only; наружу — максимум 1-2 curated ссылки в footnote |
| 🎨 **Brand book / pitch deck / one-pager** | defer — отдельная сессия (бренд) сегодня вечером |
| ⚙️ **Foundation infrastructure** (11 Parts / Pillar C / FPF / schemas / Halt-Log-Alert / default-deny) | internal-only; показать = выглядеть как «закрытый клуб со своим жаргоном» (анти-паттерн §9 DOCS-CLASSIFICATION) |

**Правило фильтра (одно).** Перед тем как документ попадает в публичный набор — проверяем
категорию. 📚 substrate и ⚙️ system **не идут напрямую**: из них берётся суть и переписывается
в 🟢 пояснительный жанр. Глубина остаётся доступной как footnote-ссылка («хочешь глубже —
вот Method V2 §J»), но не как стартовый материал.

## §3 Персона аудитории — для кого пишем

Публичные документы пишутся **не для всех сразу**, а под умного партнёра, который должен с
нуля понять Jetix за 5-15 минут и решить «мне интересно / нет». Три под-персоны (из
audience×category matrix + 4 типов партнёров):

| Персона | Кто это | Время | Что ему надо первым |
|---|---|---|---|
| 👁️ **Любопытный** | случайно попал, ещё не партнёр | 5 мин | «что это вообще» + видение одним взглядом |
| 🤔 **Серьёзный кандидат** | T3 тестер / T1 методолог, готов вникать | 30-60 мин | метод + платформа + как сотрудничать + деньги |
| 🔬 **Methodology-savvy / R12-мост** | T1 методолог уровня Левенчука / Прапион | deep dive | полная глубина + честные refs в substrate |

**Якорь персоны (важно для R12).** Читатель — взрослый умный человек, которого мы
**приглашаем, а не втягиваем**. Поэтому публичный набор:
- не давит срочностью / дефицитом / «успей записаться»;
- ставит выход («уйти можно на любой ступени без штрафа») **рано и заметно**;
- даёт человеку решить самому, не перекладывает на него когнитивную нагрузку «разберись в 250 документах».

Это R12 paired-frame на уровне самой структуры: публичный = partner-facing = R12 STRICT.
Любой вариант структуры в Phase 2-6 проверяется этим якорем.

## §4 Что Phase 0 даёт следующим фазам

1. **Фильтр** (§2) → Phase 5 substrate mapping тянет только из ✅-зоны; ⛔-зона не маппится.
2. **3 под-персоны** (§3) → прямой каркас Варианта C (по аудиториям) и entry points в Phase 4.
3. **Вывод «глубина закрыта, упаковка — нет»** → все 3 варианта = переупаковка, не новое
   производство. GAP-флаги в Phase 3/5 показывают, где даже переупаковывать нечего (надо создать).
4. **R12-якорь персоны** → критерий «public-readiness» в comparison matrix (Phase 6).
