# 🧭 Phase 4 — Как партнёр реально читает (worked examples)

> **Что это.** Для каждого варианта — **проигранный сценарий**: партнёр получил ссылку,
> открыл первый документ — что видит, что читает дальше, куда ведёт кнопка. Плюс оценка
> времени на маршрут, точки решения («глубже / достаточно») и точки входа для разных
> аудиторий (T1 методолог / T3 тестер / R12-мост).
>
> **R12-линза (сквозная).** В каждом сценарии проверяю: есть ли **выход без давления** на
> каждом шаге? Приглашаем или втягиваем? Это критерий, а не украшение. IP-1: имена = примеры.

## §1 ВАРИАНТ A — «партнёр заходит на полку»

**Сценарий (T3 тестер, условно «Дмитрий»):** получил ссылку на главную с 6 разделами.

1. Видит 6 плиток: Метод · Платформа · Бизнес · Заработок · Партнёры · Видение. **Решает сам**, с чего начать.
2. Как гуманитарий-тестер кликает «Платформа» (ему интересно «что я получу руками») → читает A2 (~7 мин) → видит Personal OS.
3. CTA в конце A2: «хочешь понять, зачем это» → прыгает в «Метод» (A1, ~10 мин).
4. Дальше любопытство ведёт в «Заработок» (A4, ~7 мин) → «о, тут ещё и деньги».
5. CTA «поговорить» → discovery-звонок.

**Время маршрута:** ~25-30 мин (3 из 6 разделов; остальные — по желанию).
**Точки решения:** на каждой плитке («интересно / пропускаю»); выход — просто закрыть, ничего не теряет.
**Сильное:** партнёр идёт за своим интересом, не по чужому сценарию.
**Слабое:** если он **не знает**, что Jetix вообще, 6 плиток без проводника = «а с чего начать?» — любопытный может уйти.

**Точки входа по аудитории:**
- **T1 методолог** → сразу «Метод» (A1) → «Видение» (A6). Хочет идею и масштаб.
- **T3 тестер** → «Платформа» (A2) → «Заработок» (A4). Хочет потрогать и понять выгоду.
- **R12-мост** → «Бизнес» (A3) + ищет R12-якорь. Хочет проверить анти-секту.

## §2 ВАРИАНТ B — «партнёр идёт по дорожке»

**Сценарий (новый T1 методолог, условно «Maxim», с нуля):** получил ссылку на вход B1.

1. **B1 Старт** (~5 мин): одна страница — что такое Jetix, для кого, зачем. В конце: «дальше — как работает метод».
2. **B2 Глубже** (~12 мин): метод + прошивка + 7 ступеней. Как методолог цепляется за cross-skill. CTA: «посмотри, что есть руками».
3. **B3 Что есть** (~10 мин): платформа Personal/Team OS. CTA: «как в это войти».
4. **B4 Как сотрудничать** (~10 мин): 4 типа партнёров → узнаёт себя в T1. **Здесь R12-обещание** («не доим / не запираем»). CTA: «про деньги».
5. **B5 Деньги** (~8 мин): 75/25, 5:1, fork-and-leave. CTA: «разбор-звонок».
6. **B6 Куда идём** (~8 мин): Master Plan, этапы. Конец воронки.

**Время полного маршрута:** ~50-55 мин (линейно, все 6 шагов).
**Точки решения:** конец каждого шага — «дальше / достаточно». Выход чистый: каждый шаг самодостаточен.
**Сильное:** ведёт за руку; к моменту «Деньги» партнёр уже прогрет; R12 появляется ровно тогда, когда речь про партнёрство — органично.
**Слабое:** методологу, которому нужна сразу глубина метода, приходится ждать до B2; нельзя «прыгнуть к деньгам» — точнее можно, но структура к этому не подталкивает.

**Точки входа по аудитории:**
- **T1 методолог** → B1→B2 (метод цепляет), может пропустить B3, вернуться к B6.
- **T3 тестер** → проходит всё линейно — идеальный кейс для воронки.
- **R12-мост** → B1 → сразу ищет B4 (R12-обещание) → проверяет, потом назад.

## §3 ВАРИАНТ C — «партнёр выбирает свою дверь»

**Сценарий тройной — три персоны, три двери:**

**👁️ Любопытный (5 мин):** открывает C1 → одностраничник + видео-welcome 3 мин + 5 тезисов.
Решает: «прикольно, может позже» или «хочу серьёзно → дверь C2». Уходит без потерь.
- **Время:** 5 мин. **CTA:** одна — «хочу глубже».

**🤔 Серьёзный кандидат (T3/T1, 40-60 мин):** входит в C2 → видит 4 блока (метод/платформа/
партнёрство/заработок) → читает в любом порядке → видео A/B/C по ходу → FAQ → discovery-звонок.
- **Время:** 40-60 мин. **Точки решения:** после каждого блока. **CTA:** разбор-звонок.

**🔬 Methodology-savvy (R12-мост / методолог, deep dive):** входит в C3 → «глубокая навигация»
показывает карту: метод-канон §J / R12 4 класса / экономика §11 / governance. Ныряет туда,
где сомневается. Видит честные refs в substrate.
- **Время:** 1-3 часа (selective). **CTA:** содержательный разговор как с равным.

**Сильное:** каждый получает ровно свою глубину; методолог не обижен витриной, любопытный не утоплен в глубине; R12 виден на всех трёх уровнях (фраза → раздел → механика).
**Слабое:** человек должен честно оценить «кто я» (T3 может зайти в C3 и испугаться, или в C1 и заскучать); правки контента надо синхронизировать между слоями.

**Точки входа по аудитории = сами двери:** аудитория и есть навигация. Это и сильная, и слабая сторона (нужно, чтобы человек правильно себя определил).

## §4 ВАРИАНТ D — «навигатор у входа»

**Сценарий:** партнёр на входе видит вопрос «ты сейчас кто?» (3 двери C) + опцию «просто
покажи всё» (полки A) + «не знаю — проведи меня» (маршрут B).

- **«Я только смотрю»** → дверь C1 (5 мин).
- **«Я серьёзно»** → дверь C2, и если выбирает «проведи» → идёт по воронке B (метод→…→видение).
- **«Я методолог»** → дверь C3 (глубина + refs).
- **«Покажи всё»** → полки A (6 направлений, сам выбирает).
- Везде в шапке — ссылка-якорь **«Наше обещание»** (R12), как safety у Anthropic.

**Время:** зависит от двери (5 мин — 3 часа). **Сильное:** покрывает все маршруты сразу,
каждый находит свой; R12-якорь всегда на виду. **Слабое:** навигатор на входе сам может стать
точкой трения («слишком много выборов на старте»); дороже в реализации; для Build-этапа (партнёров
единицы) может быть избыточно.

## §5 Сводка времени и проводника

| Вариант | Время быстрого пути | Время полного | Кто ведёт | Выход без давления (R12) |
|---|---|---|---|---|
| **A** | 7-10 мин (1 раздел) | ~40 мин (все 6) | сам партнёр | ✅ закрыл вкладку |
| **B** | 5 мин (B1) | ~50-55 мин (вся воронка) | структура (дорожка) | ✅ выход на каждом шаге |
| **C** | 5 мин (C1) | 40-60 мин (C2) / 1-3ч (C3) | выбор двери | ✅ выбрал глубину сам |
| **D** | 5 мин | по маршруту | навигатор + выбор | ✅ + R12-якорь всегда виден |

**R12-вывод (для critic-mode):** все 4 варианта проходят базовую R12-проверку «выход без
давления». Чище всего — **B** (выход структурно встроен в каждый шаг) и **D** (R12-якорь
всегда на виду). **A** полагается на то, что партнёр сам закроет (пассивный выход — ок, но
R12 не подчёркнут). **C** ставит выбор глубины на партнёра (хорошо для автономии, но требует
самооценки). Ни один не нарушает R12; различие — в том, насколько **обещание заметно**.

**Анти-паттерн, которого избегаем во всех (из DOCS-CLASSIFICATION §9):** не вываливать 5+
ссылок сразу; не смешивать витрину и глубину в одном сообщении; не давать substrate до
пояснительного слоя. Каждый вариант это соблюдает — различие лишь в механике.
