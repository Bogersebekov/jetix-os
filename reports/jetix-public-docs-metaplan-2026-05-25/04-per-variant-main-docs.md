# 📋 Phase 3 — Главные документы по каждому варианту

> **Что это.** По каждому варианту — полный список главных документов с one-liner'ом: что
> в нём, кому primary, откуда substrate, формат, целевая длина, и **GAP-флаг** (есть ✅ /
> частично ⚠️ / нужно создать ❌). Плюс поддерживающие страницы (2-4 на главный).
>
> **Ключ к чтению (важно).** Состав документов у вариантов **почти одинаковый** — это один
> и тот же пул (Phase 2 §0). Поэтому **канонический пул я выписываю один раз на Варианте A**,
> а для B и C показываю **переразметку** того же пула + что уникально для варианта. Это
> честнее (не плодим иллюзию, что варианты требуют разных документов) и короче.
>
> GAP-легенда: ✅ substrate готов, нужна переупаковка · ⚠️ частично есть · ❌ создавать с нуля.

| | Главных docs | Из них готовых ✅ | Уникальных новых (vs пул A) | Доп. усилие vs A |

| **B** | те же 6 + 2 связки | как A + 2 лёгких | страница входа + CTA-карта | **низкое** (+2) |
| **C** | сгруппированы в 3 слоя + 2 | как A + 2 + анти-дубль | одностраничник + глубокая навигация | **среднее** (+2 + дисциплина слоёв) |

**Главный факт для тебя (R1):** выбор варианта **почти не меняет, какие документы писать** —
меняет, **как их группировать и сколько связующих элементов добавить**. Самые рычажные
GAP-документы одни и те же во всех вариантах: **видео A · публичный Master Plan · «Как устроен
Jetix» · Charter текст · R12-обещание-страница · discovery script универсальный**. Это значит:
выбор структуры можно делать спокойно — он не сжигает работу по наполнению, а только задаёт
рамку, в которую наполнение ляжет.
