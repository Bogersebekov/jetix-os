# 📐 Phase 6 — Скелеты главных документов (11 направлений)

> **Что это.** Самая тяжёлая фаза v2 (Ruslan: «плюс-минус по каждому»). Для каждого из 11
> направлений — **полный скелет primary main doc**: что внутри, какие секции, какие заголовки,
> какие bullet-подсказки в каждой секции, какая Notion-структура, формат, длина, cross-refs.
>
> **КРИТИЧНО (R11):** это **скелет** — заголовки + подсказки «что писать в секции» — а **НЕ сам
> публичный текст**. Где стоит `[подсказка: …]` — это инструкция автору наполнения, не контент.
> Финальный текст пишется отдельными prompt'ами per direction (Phase 9 Wave-sequencing), часть —
> Ruslan-authored (Vision one-liner, ценности).
>
> **Единый шаблон скелета (8 элементов)** применяется к каждому направлению:
> 1. **Title + tagline** — название + подзаголовок-крючок
> 2. **Hero** — обложка/1-liner/30-сек value prop
> 3. **TOC / Navigation** — оглавление + куда дальше
> 4. **§1 Intro** — зачем этот документ / для кого / read time
> 5. **§2-N Content sections** — основные секции с placeholder-заголовками + bullet-подсказками
> 6. **Closing** — next step / CTA / куда идти
> 7. **Appendix** — cross-refs / further reading / footnote-глубина
> 8. **Metadata** — версия / обновлено / язык / форматы

---

## §0 Как читать скелеты

Каждый скелет ниже = «болванка документа», в которую потом ляжет человеческий текст. По каждому
направлению указаны: **дверь-видимость** (из Phase 2 матрицы), **substrate-источник** (откуда
тянуть, Phase 8 детализирует), **GAP-статус** (✅ готов к переупаковке / ⚠️ частично / ❌ с нуля),
**R12-paired** (где обязателен ethical-counter-frame). Формат-легенда: `§` = секция документа;
`[подсказка: …]` = что писать; `→` = cross-link.
