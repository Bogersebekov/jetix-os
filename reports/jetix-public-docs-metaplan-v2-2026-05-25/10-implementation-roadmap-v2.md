## §0 Два принципа sequencing

1. **Quick wins сначала, блокеры параллельно.** ✅ готовые доки (#4 Заработок, #5 Партнёры из
   Phase 8) дают публичный результат сразу, пока ❌ блокеры (видео A/B, #3 Бизнес, #11 Master Plan)
   делаются параллельно. Это снимает риск «ничего наружу, пока всё не готово».
2. **Видео A = критический путь.** PLATFORM-LIFECYCLE §2: видео A блокирует 3 вектора. Значит
   видео A — в Wave 1, даже если остальное в #1 Метод не готово. Не блокировать им текстовые доки.

**Расхождение с §3 промпта (surface для R1):** промпт предлагал Wave 1 = Vision+Видение+Master
Plan+Метод+R12. GAP-картина (Phase 8) показывает: Vision/Master Plan/Метод-текст = ❌/⚠️ (тяжёлые,
с нуля), а #4/#5 = ✅ (лёгкие). **GAP-оптимальный порядок:** Wave 1 = ✅ готовые (#4/#5) + R12 +
видео A старт; тяжёлые манифесты (#11/#3) = Wave 2-3. Это **surface — выбор твой** (§5.1).

## §6 R1 decisions для roadmap (surface)

1. **Wave 1 порядок** — GAP-оптимальный (✅ #4/#5 первыми) ИЛИ промпт-порядок (Vision/Master Plan/
   Метод первыми)? (рекоменд: GAP-оптимальный — быстрый результат, тяжёлое параллельно).
2. **Первый prompt наполнения** — какой док? (рекоменд: «Наше обещание» #8 как рычажный якорь ИЛИ
   Partner Offering polish #4 как самый готовый).
3. **Видео A** — записываешь ты сам (R1, блокер) — когда? (это вне scope роя, твой input).
4. **One-liner Vision + триада** — твои формулировки (prose_authored_by: ruslan) — когда?
5. **public/internal граница Правил** — фиксируем сейчас или после первой когорты?
