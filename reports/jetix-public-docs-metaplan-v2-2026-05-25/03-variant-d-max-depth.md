# 🏗️ Phase 2 — Variant D Hybrid, развёрнутый на максимум

> **Что это.** Центральная фаза v2. v1 описал variant D в 4 абзацах как «дорогой гибрид,
> отложить до Run». v2 берёт D как **picked** (Ruslan voice) и разворачивает на максимум:
> **11 направлений-полок × 3 двери-аудитории × 1 маршрут** — с явным назначением каждой
> ячейки (что видно / скрыто / по ссылке) на каждом пересечении.
>
> **Ключевая идея D (не меняется с v1):** документ пишется **один раз** и лежит на своей **полке**
> (направление); показывается под нужным **углом** через свою **дверь** (аудитория); а **маршрут**
> — это рекомендованная тропа для того, кто не знает, с чего начать. Полки = хранилище. Двери =
> витрина. Маршрут = проводник. R12-обещание = якорь, видимый из любой точки.

двери, чтобы не перегружать) · **— ** (не существует для этой полки/двери).

## §3 Измерение 3 — МАРШРУТ (проводник внутри двери B)

Рекомендованная тропа для того, кто не знает, с чего начать. Это **порядок показа полок**
внутри двери «Кандидат», не отдельная структура (в отличие от v1, где B был самостоятельным
вариантом). Маршрут v2 (расширен с v1 6 шагов):

```
[0] Видение (#6)  →  «зачем это всё» — 1-page, цепляет смысл
        ↓ decision: интересно → дальше / достаточно → ухожу (R12 выход)
[1] Метод (#1)  →  «как мы думаем и чему учим» — сердце оффера
        ↓ метод откликается → смотрю инструмент
[2] Платформа (#2) + Образование (#7)  →  «что есть руками + как учим»
        ↓ вижу инструмент → как войти
[3] Партнёрство (#5)  →  «4 типа + что просим/даём» — тут R12 органично
        ↓ хочу про деньги
[4] Деньги (#4)  →  «75/25, 5:1, fork-and-leave» — прогрет к этому моменту
        ↓ готов поговорить → CTA discovery
[5] R12-обещание (#8)  →  «что мы НЕ делаем» — закрывает доверием, не давлением
        ↓ вписываюсь в долгую игру → Master Plan (#11) / discovery-звонок
```

**Почему такой порядок:** Видение даёт смысл → Метод даёт суть оффера → Платформа+Образование
делают абстракцию осязаемой → Партнёрство показывает «как именно вместе» → Деньги (когда уже
прогрет, не в лоб) → R12-обещание закрывает доверием. **Выход встроен в каждый шаг** — это R12
на уровне структуры маршрута (приглашаем, не втягиваем). Ценности (#10) и Правила (#9) — не в
основном маршруте, а по ссылке (для тех, кто хочет «во что верите / по каким правилам»).

- **Методолог** видит 10 ✅ — максимум честной глубины; скрыто только System infra (вне матрицы).
- **#8 R12** — единственная полка ✅ во всех трёх дверях (якорь, как Anthropic safety).
- **#9 Правила** — 🚫 для Любопытного (правила — не вход), 🔗/✅ дальше.

## §5 R12 как сквозная нить (не только полка #8)

R12 живёт в двух режимах одновременно (это и есть гибрид D):
1. **Как полка #8** — отдельная заметная страница «Наше обещание» (для тех, кто хочет коммитмент явно).
2. **Как сквозная нить** — в каждой полке R12-paired-frame:
   - #4 Заработок: каждая цифра + «как уйти с долей».
   - #5 Партнёры: «кого зовём» + «кого НЕ берём» + «уйти без штрафа».
   - #7 Образование: ступени бесплатны 1-4, выход на каждой.
   - #2 Платформа: fork-friendly (не lock-in, в отличие от WeChat).
   - #10 Ценности: cooperative > extractive как belief.

**Enforcement (influence-ethics-expert auto-fire):** любая полка, касающаяся партнёрства /
рекрутинга / образования / денег / community, обязана нести paired-frame (что НЕ делаем +
защита читателя). Отсутствие пары на R12-touching полке = сигнал к доработке (на уровне
архитектуры, не Halt — это план, не действие).

### Пример B (дверь 🤔 Кандидат) — тестер платформы, прошёл по маршруту

## §8 Cross-direction links — почему полки связаны именно так (rationale)

Связи между полками — не украшение, а **навигационная логика** (чтобы партнёр не упирался в
тупик). Ключевые рёбра и зачем:

- **#1 Метод ↔ #7 Образование** — самое важное разведение v2: ЧТО (канон) vs КАК (педагогика).
  Партнёр, пришедший «за идеей» (#1), переходит в «а как меня научат» (#7). Референс: МИМ канон ↔
  МИМ курсы; Mondragón principles ↔ Mondragón University.
- **#4 Заработок ↔ #8 R12** — неразрывная пара: каждая цифра дохода тянет за собой обещание «не
  доим». Без этого ребра #4 читается как MLM-pitch (R12 вопрос 7 нарушен).
- **#5 Партнёры ↔ #4 Заработок ↔ #8 R12** — треугольник доверия: тип партнёра → его доля →
  гарантия выхода. Это ядро partner-facing R12 STRICT.
- **#6 Видение → #11 Master Plan** — намеренная лестница: короткий обзор (витрина) ведёт в
  long-form (для тех, кто хочет «всю дугу»). Не дубль — разные глубины (Tesla: продукт-страница →
  Master Plan манифест).
- **#10 Ценности → всё** — ценности = «почему» под каждой полкой; ссылаются отовсюду, как
  Anthropic Core Views.
- **#9 Правила ↔ #3 Бизнес ↔ #8 R12** — правила операционализируют governance (#3) и R12 (#8);
  это «как именно мы держим обещание на практике».

**Анти-тупик правило:** ни одна полка не должна быть «листом без выхода» — у каждой минимум 2
исходящих cross-link (вперёд по маршруту + к R12/Ценностям). Это прямой урок Stripe (любая docs-
страница имеет «next» + «related»).
