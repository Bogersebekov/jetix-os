## §1 VISION DOCUMENT — спецификация

### §1.1 Core Vision statement (3 уровня детализации)
Документ открывается ядром в трёх масштабах (читатель берёт нужный):
- **One-liner (для двери A, витрина):** «Учим людей думать системно, с AI как усилителем —
  и строим кооператив, где доля честная, а уйти можно всегда». *(формулировка = R1 Руслана; здесь
  placeholder-структура, не финальный текст — R11)*
- **1-параграф (для двери B):** разворачивает one-liner — что такое метод-метод (O-107), почему
  AI = усилитель а не замена (O-128/B-1), почему кооператив а не пирамида (B-3), для кого.
- **1-страница narrative (для двери B/C):** полная история «зачем» — gap в мире, почему именно
  этот подход, куда ведёт. Тон Berkshire (лично, честно).

### §1.2 What we are building (3 горизонта)
Структура «что строим» в трёх временных масштабах (из STRATEGIC-PLAN горизонтов):
- **1 год (2026→2027):** платформа Personal/Team OS работает; когорта 10-20K; 1-2K платящих
  партнёров; метод проверен реальными людьми; первый клан. *(Scenario B Realistic, 40%)*
- **5 лет (2026→2031):** мульти-клановая сеть; кооператив юридически оформлен; leveraged
  дистрибуция; 200K-1M когорта; R12 on-chain enforcement операционен.
- **25 лет (long arc):** массовый сдвиг парадигмы (O-184) — системное мышление как норма;
  метод само-воспроизводится; Network State implications; IP-лицензирование глобально.

### §1.3 Why this exists (gap analysis)
Honest gap — что в мире не хватает + почему мы:
- **Gap 1 — образование учит функциям, не мышлению** (O-176): мир накапливает знания, AI обесценил
  накопление; нужна прошивка (как думать), не дамп (что знать).
- **Gap 2 — AI используют как «зачитерить», не как усилитель** (O-181): массовая ошибка —
  делегировать мышление; правильно — освободить время для фронтира.
- **Gap 3 — платформы извлекают, не делятся** (R12/China anti-урок): super-apps запирают; нет
  массовой платформы с fork-and-leave + честной долей.
- **Почему мы:** Левенчук-foundation метода + ROY swarm + R12 анти-extraction + кооп-экономика —
  комбинация, которой ни у кого нет (Method V2 §11 дифференциация от МИМ).

### §1.4 For whom (5+1 архетипов)
- **Primary (5):** методолог-creator (T1) · ресурс-держатель (T2) · smart-тестер (T3) ·
  будущий консультант (T4) · cohort-член (масса, O-184).
- **+1 secondary mention:** R12-критик/мост (Прапион-архетип) — не «клиент», а валидатор/совесть.
- **IP-1:** архетипы = роли, не персоны; имена (Maxim/Дмитрий/Левенчук/Цэрэн/Прапион) = примеры.

### §1.5 How (high-level mechanics)
Четыре несущих механизма (без жаргона):
1. **Методология** — метод-метод (O-107) + 7 ступеней + прошивка.
2. **Платформа** — Personal OS → Team OS → кланы (AI-augmented exocortex).
3. **Кооп-governance** — 75/25 + 5:1 + 1-член-1-голос + fork-and-leave (R12).
4. **AI-augmentation** — AI как усилитель внешней системы (O-128), не замена.

### §1.6 What success looks like (≥3 measurable, НЕ только revenue/users)
Намеренно не-только-деньги (анти-hype, ценность «развиваться» а не «накопить»):
- **Метрика 1 — paradigm shift:** доля когорты, реально перешедшая на question-first/системное
  мышление (не «прошли курс», а «изменили способ думать»). Измеримо через before/after.
- **Метрика 2 — R12-целостность:** 0 нарушений extraction; % партнёров, осознанно знающих свой
  fork-and-leave путь; ratio фактический ≤ 5:1.
- **Метрика 3 — frontier contribution:** число вкладов партнёров во фронтир (наука/технологии/
  клановые продукты) — O-183 в действии.
- **Метрика 4 (вторично) — устойчивость:** revenue-positive + runway > X мес («не умереть»).
- **Метрика 5 (вторично) — масштаб:** когорта/платящие (но как следствие 1-3, не самоцель).

### §1.7 What we won't compromise (границы)
Прямая проекция Values §4 anti-beliefs + R12:
- R12 anti-extraction (никогда не доим сверх доли).
- Mondragón 5:1 (никогда extreme inequality).
- Fork-and-leave (никогда lock-in).
- Триада ценностей (жить/не-умереть/развиваться).
- AI disclosure (никогда не выдаём AI за человека внешне).
- Анти-манипуляция (никогда engagement-trap / фейк-срочность).

## §2 MASTER PLAN PUBLIC — спецификация (Tesla-style, 4 части)

> Tesla-формат: нумерованные части, от первого лица, конкретные вехи, честно про
> неопределённость. Каждая часть ~500-800w. Part 1+2 публичны; Part 3+4 gated (anti-hype —
> чтобы публика не строила нереалистичных ожиданий); long-arc speculation internal.

### Part 1 — «Где мы сейчас и что строим» (Build phase, ~6 месяцев) — PUBLIC
- **Состояние:** Точка А (Build, середина). Есть: метод-канон V2, Foundation v1.0, 17 агентов,
  180 контактов, substrate. Нет: видео A (блокер), внедрённых Notion, Charter текст, Wave 1.
- **Что строим:** видео A/B/C → Personal OS 5 баз рабочие → Charter v1 → первый trial (T3) →
  Wave 1 outreach (2 человека, не 7) → 1-2 T1 + 3-5 T3 → Build exit ≥80%.
- **Вехи:** Точка B (~15-22.06) = Build exit готов.
- **Тон:** честно — «мы в начале, метод не прогнан снаружи, вот что делаем».

### Part 2 — «Когорта работает, деньги текут» (Run phase, ~1-2 года) — PUBLIC
- **Состояние:** петля обратной связи включена; когорта 5-50 → растёт; €1500/мес (ступень 5);
  1-2 партнёра со-создают курс.
- **Что строим:** Workshop регулярный → когорта 500 → 10-20K; первый клан; revenue €1.5-3M MRR
  (Scenario B); кооп-структура оформляется.
- **Вехи:** Point C (MVP операционен) → Point D (public alpha, Wave 2-5 cascade).
- **Тон:** «вот как масштабируется метод через людей, с сохранением R12».

### Part 3 — «Мульти-клан + кооператив legal + leveraged дистрибуция» (Scale, ~3-7 лет) — GATED
- **Состояние:** N кланов параллельно; доход по кооп-шаблонам (5:1); основатель = один из
  многих хранителей; маховик крутится сам.
- **Что строим:** кооператив юридически (Mondragón-like) → Charter 50+ членов → R12 smart-contract
  операционен → leveraged дистрибуция через блогеров/предпринимателей → 200K-1M когорта.
- **Почему gated:** конкретные цифры/сроки Scale = high uncertainty; публично = hype-риск.

### Part 4 — «Массовая платформа + Network State» (long arc, ~10-25 лет) — GATED/INTERNAL
- **Состояние:** массовый сдвиг парадигмы (O-184); метод само-воспроизводится; рекурсивный спавн кланов.
- **Что строим:** Network State implications (Balaji-pattern, 5 of 7 NS steps); IP-лицензирование
  глобально; $1T trajectory non-pyramidal (D-19).
- **Почему gated/internal:** спекулятивно; наружу только как «направление», не как обещание.
  Balaji-outreach = Phase 2+ deferred (memory project_balaji_outreach_target).
