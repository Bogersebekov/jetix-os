# Phase 7 — Как новый человек входит в систему ⭐⭐

> **Простыми словами.** Когда приходит новый участник — нужен понятный вход: первая неделя
> по дням, гайд по его роли, документ про этику (что мы делаем и чего НЕ делаем), и
> опционально — общая сессия для знакомства. Здесь важнее всего одно: **вход не должен
> превращаться в вербовку или культ.** Поэтому на этой фазе автоматически включается
> recruitment-dynamics-expert — он следит, чтобы формирование когорты было добровольным и
> без сектантских механик.

---

## §1 ⚠️ Главная рамка: когорта, а не секта

Формирование когорты — зона повышенного риска (сюда легко прокрадываются культовые
механики). Поэтому **с первой минуты** соблюдаем:

- **Выход называется вслух сразу** — на онбординге человек узнаёт, что может уйти в любой
  момент со своей долей, **до** того как что-то подпишет.
- **Никаких клятв верности** и «обрядов посвящения через жертву».
- **Никакой эскалации преданности** («ты теперь свой, докажи»).
- **«Когорта-основатель 2026» — это признание вклада, а НЕ элитарный клуб.** Тонкая, но
  важная грань: гордиться вкладом — нормально; «мы избранные против них» — это семя секты.

Кого зовём (профиль «paradigm pioneer» по Баркеру): гибкие в мышлении, любопытные,
толерантные к неопределённости, готовые пробовать до полного доказательства. Скрининг-вопрос
на discovery-звонке (Future-Vision Question): **«Что сегодня невозможно в твоей сфере, но
если бы стало возможным — изменило бы её фундаментально?»**

## §2 📅 Первая неделя — по дням

Дефолт для любого нового участника (всего ~8-12 часов за неделю):

| День | Что | Время |
|---|---|---|
| **День 1** | Bootstrap личной системы (Personal OS — форк шаблона + настройка, формат мастерской 2ч) | 2ч |
| **День 2** | Тур по командному пространству (видео 1ч + Q&A 30мин) | 1.5ч |
| **День 3** | Тень на первом проекте (наблюдаешь активный SG-2 проект 1-2ч, без обязательств) | 1-2ч |
| **День 4** | Заполнение биржи offer/need (гайд + само-аудит) | 1ч |
| **День 5** | Первый разбор находок (вместе смотрим daily brief) | 30мин |
| **День 6** | Чтение Charter + R12-этики (независимое чтение) | 1-2ч |
| **День 7** | Первый вклад (маленькая задача — роль Contributor baseline) | варьируется |

**Принцип:** от наблюдения к маленькому вкладу. Никакого «прыгни в омут». День 3 — тень
**без обязательств**, День 7 — самая маленькая задача. Это анти-burnout вход.

## §3 📖 Гайд по каждой роли (8 гайдов)

У каждой роли — свой короткий гайд. Кратко, что в каждом:

| Роль | Что в гайде |
|---|---|
| **PM Guide** | Stage Gates / протоколы координации / R12-аудит / разрешение конфликтов / учёт вклада |
| **Investor Guides (4: Capital/Time/Network/Knowledge)** | конкретные обязательства + механика revenue share + R12-границы для каждого типа |
| **Contributor Guide** | приём задач / учёт вклада / навигация по проекту |
| **Advisor Guide** | лёгкое участие / R12-граница (не лезть в управление, no scope creep) |
| **Facilitator Guide** | структура сессии / R12 paired-frame (анти-культ / Schelling-чек) |
| **Mentor Guide** | R12-границы наставнической власти (нет эксклюзивности / fork-and-leave / потолок 12 мес) |
| **Observer Guide** | критерии перехода (Observer → Contributor / Investor / …) |
| **Steward Guide** | R12-enforcement / протоколы Halt-Log-Alert / аудит Charter / peer-Steward review |

Каждый гайд **обязательно** содержит секцию «твои права и твой выход» — что ты всегда
можешь сделать (уйти / усомниться / форкнуть).

## §4 🧭 R12 ethics doc — обязательное чтение перед входом

Документ на ~1500 слов простым языком (обязателен **до** вступления). Структура:

1. **Что Team OS делает:** объединяет людей через роли + биржу + монетизацию.
2. **Чего мы НЕ делаем:**
   - не извлекаем сверх согласованной доли
   - не запираем людей (fork-and-leave в любой момент)
   - не нарушаем потолок 5:1
   - не распределяем чужое без согласия
3. **Что значит «fork-and-leave»:** можешь уйти в любой момент, забрать свою долю вклада,
   без штрафа, 30-дневное окно при изменении правил.
4. **Mondragón 5:1 простыми словами:** потолок неравенства — самая большая выплата не
   больше 5× самой маленькой. Как в кооперативе. Никаких «директор в 350 раз больше».
5. **Что такое Charter:** что подписываешь / что обязуется проект / что обязуешься ты.
6. **Роль Steward:** кто следит за соблюдением / как поднять тревогу / как эскалировать.
7. **Разрешение споров:** последовательность шагов (Steward → peer-Steward → внешний медиатор).
8. **Красные флаги:** что должно настораживать (манипуляция / культовые динамики / признаки
   извлечения).
9. **Твои права:** что ты можешь всегда — уйти / усомниться / возразить / форкнуть.

**Тон:** не «правила, которые ты обязан», а «вот как мы защищаем тебя». Этика подаётся как
гарантия участнику, не как требование к нему.

## §5 🎉 Ритуал онбординга (опционально, общая сессия)

Раз в 2 недели — групповая сессия 2 часа (ротация окон входа когорты):

- Знакомства
- Прогон ценностей + Charter (группой)
- Групповое упражнение R12 paired-frame (демо Schelling-координации)
- Q&A
- Назначение buddy-пары (каждый новичок получает действующего участника как проводника
  на первый месяц)

**Это опция, не обязаловка** (решение §9.E: ритуал mandatory или async optional). Если
выбирается ритуал — он **анти-культовый по дизайну**: знакомство и взаимопомощь, не
«посвящение». Buddy-пара = поддержка, не «надзиратель-куратор».

## §6 🔁 Ритуалы перехода (между ролями)

Переходы из Phase 3 — с лёгкими «вехами», не экзаменами:

| Переход | Веха |
|---|---|
| Observer → Contributor | 1 неделя тени + приём первой задачи |
| Contributor → PM | 3-месячный track record + Steward review |
| Contributor → Investor | формальное обязательство (капитал/время) + поправка Charter |
| Mentor → Steward | 12-месячный track record + peer-Steward номинация + R12-аудит pass |

Это перекликается с **МИМ-лестницей** Левенчука: рост измеряется не экзаменом, а
непрерывным наблюдением (mentor + cohort триангуляция) на реальных проектах. Переход — это
признание уже сделанного, не «сдача теста».

[src: prompt §8.E + Levenchuk МИМ continuous measurement]

## §7 🚫 Анти-паттерны онбординга

Чего не делаем **никогда**:

- ❌ «Выпей Kool-Aid» — клятва верности на входе
- ❌ Эскалация пред-обязательств («просто подпиши, детали потом»)
- ❌ Информационная перегрузка в Неделю 1 (выгорание когорты) — потому и лимит 8-12ч
- ❌ Не упомянут выход (опция уйти должна звучать **первой**, не последней)
- ❌ Buddy-пара как «надзиратель» вместо проводника
- ❌ «Когорта-основатель» как элитарный клуб вместо признания вклада
