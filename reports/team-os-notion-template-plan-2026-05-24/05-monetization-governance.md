# Phase 5 — Деньги и управление (честно, с защитой) ⭐⭐⭐

> **Простыми словами.** Самая ответственная часть. Когда люди вместе зарабатывают, надо
> заранее договориться: как делим доход, как считаем вклад, что подписываем (Charter), как
> принимаем решения и как проект проходит контрольные точки. И — главное — как сделать так,
> чтобы **никого нельзя было «обобрать» сверх договорённого и никого нельзя было запереть**.
> Здесь — 4 шаблона revenue share, 4 типа инвесторов, учёт вклада, Charter, Stage Gates, и
> сквозной 8-пунктовый этический чек на **каждый** денежный шаблон.
>
> ⚠️ На этой фазе автоматически включается **influence-ethics-expert** (получатель R12) —
> потому что деньги + статус + власть = зона максимального риска.

## §1 🎯 Главный принцип в одну строку

**Большая часть дохода (75-90%) — человеку напрямую. Меньшая (10-25%) — в Foundation на
развитие и поддержку всех. Потолок неравенства 5:1 (как в кооперативе Mondragón). Уйти
можно в любой момент со своей долей, без штрафа. 30 дней на выход при любом изменении
правил.**

Это и есть R12 на человеческом: **система не доит и не запирает.**

**Точная каноническая модель (для справки):** в Economic Model V10 (LOCKED) это
формализовано как «каждый отдаёт 25% → институциональный пул, оставляет 75%», что в
замкнутом цикле даёт 75% workers / 16.67% управленцы / 8.33% Ruslan cumulative. Диапазон
«75-90%» — это human-language обёртка (точный % зависит от тира и Charter, 10-25% take rate).

## §2 💰 Четыре шаблона revenue share

### Template 1 — Стандартный проект (revenue share)

- **Разбивка валового дохода проекта:**
  - **75-90%** — активным контрибьюторам (по учёту вклада)
  - **5-15%** — Foundation (поддержка инфраструктуры Jetix)
  - **5-15%** — капитальным инвесторам (если есть)
- **Mondragón 5:1 cap:** максимальная доля ≤ 5× минимальная доля
- **Fork-and-leave:** любой может выйти в любой момент, забрать накопленную долю, без штрафа
- **30-дневное окно** для новых участников
- **Charter обязателен** перед активацией

### Template 2 — Проект с капитальным инвестором

- **Капитальная часть:** 20-40% дохода (по размеру вклада)
- **Активные контрибьюторы:** оставшиеся 60-80%
- **Mondragón 5:1** всё равно применяется (капитал + труд вместе)
- **Liquidity events:** явные клаузулы Charter

### Template 3 — Когортная программа (формат мастерской)

- **L7-trial когорта** ≈ **€1500/мес** с участника (per Workshop / EXECUTION-PLAN step-5)
- **Разбивка:** 70-80% фасилитаторам / 10-20% Foundation / 0-10% хостам L6-когорты
- **Mondragón** применяется между тирами фасилитаторов

### Template 4 — Вклад знанием / IP

- **Mentor / автор методологического фреймворка:** 5-15% ретейнер
- **Без доли проекта**, пока не перейдёт в Contributor / PM
- **Лицензия:** open-source-ish с анти-fork-prevention (нельзя «запереть» чужой фреймворк)

## §3 🎟️ Четыре типа инвесторов

**⚠️ Мэппинг к V10:** каноническая модель использует **3 роли** (worker / investor /
promoter). Здесь «investor» раскладывается на **4 операционных вкуса вклада** — для удобства
учёта. Это не новая экономика, а декомпозиция одной роли.

| Тип | Что вкладывает | Как учитывается | Защита |
|---|---|---|---|
| **Capital** | деньги | сумма + транш-график; equity/revenue-share/convertible; voting по доле | Mondragón cap (не >5× минимума); exit terms в Charter |
| **Time** | часы + навык | часы × ставка (self-declared + peer-validated); трекинг (Toggl-аналог) | **потолок ≤50 ч/нед** (анти-выгорание) |
| **Network** | сеть / аудитория | интро (число + цели); плоская доля ИЛИ per-intro outcome | анти-audience-extraction R12 клаузула |
| **Knowledge / IP** | фреймворк / IP | вклад фреймворка; ретейнер; tier участия | лицензия open-ish + анти-fork-prevention |

**Чек для каждого типа:** транш-график (для Capital), ставка часа (Time), число интро
(Network), условия лицензии (Knowledge) — всё фиксируется **в Charter до старта**, не
постфактум.

## §4 📊 Учёт вклада (contribution accounting)

Прозрачный ledger по каждому проекту — без скрытых сплитов:

| User | Role | Hours | Capital | Network value | Knowledge | Total units | Share % |
|---|---|---|---|---|---|---|---|
| (по строке на участника) | | | | | | | |

- **Обновление:** ежемесячно + на milestone проекта
- **Видимость:** каждый видит **свою** строку + **общий прозрачный ledger** проекта (без
  скрытых долей). Полная картина всех проектов — только Steward (надзор).
- **Конвертация в долю:** total units → share %, с проверкой Mondragón cap **перед**
  распределением.

## §5 📜 Charter — договор команды (шаблон-черновик)

Обязательные секции любого Charter (без этого проект не выходит на SG-2):

1. **Ценности** — ссылки на Foundation / общие принципы команды
2. **Управление:**
   - Принятие решений (консенсус / взвешенный голос / PM-authority — выбор команды)
   - Разрешение споров (Steward-медиация + эскалация)
   - Stage Gates lighter (SG-1…SG-4)
3. **Монетизация:**
   - Разбивка revenue share
   - **Mondragón 5:1 cap явно прописан**
   - Метод учёта вклада
   - Условия выхода (exit terms)
4. **R12-комплаенс** (центральная секция):
   - Нет извлечения сверх согласованной доли
   - **Fork-and-leave сохранён** (любая роль выходит в любой момент)
   - **30-дневное окно выхода**
   - Mondragón cap STRICT
   - Механизм HALT при нарушении wage-ratio
   - Запрет non-consensual distribution
   - Попытки fork-prevention ЗАПРЕЩЕНЫ
5. **Разрешение споров:**
   - Сначала Steward-медиация
   - Peer-Steward review, если спор касается Steward
   - Внешний медиатор, если внутренняя эскалация не сработала
6. **Programmable Ethereum overlay (опционально, Phase 2+):**
   - Mondragón cap → enforcement смарт-контрактом
   - Распределение дохода → on-chain прозрачно
   - Fork-and-leave → exit-token механизм

## §6 🚦 Stage Gates lighter (на проект)

Облегчённые контрольные точки (полная версия — KM MVP; здесь упрощённая для команды):

## §7 ⚖️ R12 8-пунктовый чек — на КАЖДЫЙ денежный шаблон (CRITICAL)

Это сердце фазы. **Любой** шаблон монетизации проходит 8-пунктовый этический чек **перед**
применением. Чек наследует 8-вопросный pre-touch gate из EXECUTION-PLAN.

**8 пунктов:**

1. **Стоимость для получателя ≤ выгода** — человеку выгоднее участвовать, чем нет
2. **Предложение конкретно, не размыто** — точные %, сроки, условия
3. **Запрос пропорционален глубине отношений** — не просим партнёрство у незнакомца
4. **Уместно по стадии** (Bloom-прогрессия) — не пушим деньги на ступени «только услышал»
5. **Уместно по каналу** — деньги обсуждаются там, где это нормально
6. **R12 paired-frame PASS** — нет извлечения / lock-in / нарушения wage-ratio / fork-prevention
7. **Анти-CTA чек** — нет манипуляции / МЛМ / культа / нарушения порядка ступеней
8. **Halt-Log-Alert готов** (≤5 сек грейд F4, если нарушение)

### §7.A Применение чека к 4 revenue-шаблонам

| Пункт | T1 Стандарт | T2 Капитал | T3 Когорта | T4 Знание/IP |
|---|---|---|---|---|
| 1 Cost ≤ benefit | ✅ 75-90% человеку | ✅ 60-80% труду | ✅ ценность мастерской > €1500 | ✅ ретейнер за реальный фреймворк |
| 2 Конкретно | ✅ % в Charter | ✅ транш+equity явно | ✅ цена+разбивка явно | ✅ лицензия+ретейнер явно |
| 3 Пропорционально | ✅ по вкладу | ✅ по размеру вклада | ✅ по тиру фасилитатора | ✅ по tier участия |
| 4 По стадии | ✅ только SG-2+ | ✅ только SG-2+ | ✅ только step-5+ | ✅ при реальном вкладе |
| 5 По каналу | ✅ Charter | ✅ Charter | ✅ программа | ✅ Charter |
| 6 R12 pass | ✅ 5:1 + fork-leave | ✅ 5:1 капитал+труд | ✅ 5:1 между тирами | ✅ анти-fork-prevention лицензия |
| 7 Анти-CTA | ✅ нет МЛМ | ✅ нет lock-in | ✅ нет культа | ✅ нет IP-захвата |
| 8 Halt готов | ✅ Steward F4 | ✅ Steward F4 | ✅ Steward F4 | ✅ Steward F4 |

### §7.B 4 RUSLAN-LAYER action classes — явный мэппинг

Каждый шаблон проверяется на 4 классах действий-нарушений (из CLAUDE.md §4.2 /
`.claude/config/default-deny-table.yaml`):

| Action class | Что это | Как ловим в шаблоне |
|---|---|---|
| `extraction_beyond_share` | извлечение сверх согласованной доли | ledger прозрачен; любой сплит сверх Charter → HALT |
| `wage_ratio_violation` | нарушение потолка 5:1 | проверка max/min ≤ 5 **перед** распределением; превышение → HALT |
| `non_consensual_distribution` | распределение контента/IP/вклада без согласия | публикация чужого вклада требует consent_recorded; нет → HALT |
| `fork_prevention_attempt` | попытка запереть (язык lock-in) | детект формулировок «ты заперт после подписи»; найдено → HALT |

**На человеческом:** четыре «детектора жадности и принуждения». Если шаблон пытается
(1) забрать сверх договорённого, (2) сделать одного в 10 раз богаче другого, (3)
опубликовать чужое без спроса или (4) сказать «теперь не уйдёшь» — Steward останавливает
это за секунды.

## §8 ⛓️ Programmable Ethereum overlay (Phase 2+, опционально)

Сейчас (Phase 1) всё держится на **тексте + человеческом надзоре** (Steward + Charter). В
будущем (Phase 2+) это можно усилить смарт-контрактом — **по желанию каждой команды**
(per-Clan opt-in через Charter), не обязательно:

| Что | Phase 1 (сейчас) | Phase 2+ (опционально) |
|---|---|---|
| Mondragón 5:1 | проверка Steward | on-chain: tx reverts если ratio > 5 |
| Распределение дохода | прозрачный ledger | QF (quadratic funding) on-chain |
| Fork-and-leave | договор + Steward | RageQuit exit-token, proportional treasury |

**Стандарты (для справки, Phase 2+):** Ethereum L2 (Optimism/Base), ERC-1155 (worker/
investor/promoter NFT), ERC-5114 (soulbound 1-identity-1-vote), multisig 3/5 + 7-дневный
time-lock. Аудит-бюджет $50-200K. **Важно:** программируемая версия = более **сильная**
защита R12 (исполнение проверяемо), но Phase 1 текстовая версия работает и сохраняется.

## §9 🚫 Анти-паттерны монетизации

Чего не делаем **никогда** (это и есть граница R12):

- ❌ Скрытые сплиты / выплаты «мимо ledger»
- ❌ Wage ratio выше 5:1 (нарушение Mondragón)
- ❌ Язык «после подписи ты заперт» (fork-prevention)
- ❌ Культовые «эскалаторы преданности» (чем больше отдал, тем «свой»)
- ❌ Обязательная эксклюзивность (анти-fork-and-leave)
- ❌ Преждевременный партнёрский CTA (нарушение порядка ступеней / Bloom)
- ❌ Извлечение из сети сверх пред-согласованной доли (audience-extraction)
