## §3 Plain English version

### 3.1 Зачем именно strict order?

text_003 эксплицитно — Ruslan описывает **engineering reasoning**, не arbitrary preference:

- L0 = **substrate** для всего вышестоящего. Если L1 platform не имеет FPF foundation, она построена на ничём (vapor).
- L1 = **interface** для работы с L0. Без working interface — L0 = paper artefact.
- L2 = **applications** poверх L1. Universities учат **how to use** what L0+L1 provide.
- L3 = **values / mindset** что окружают L2 applications. «Ответственный подход к информации» обретает смысл только когда есть information system (L0+L1).
- L4 = **full methodology corpus** распределённый в users. Это требует все нижние layers для anchoring.

**Inversion examples** (R1 violations):
- ❌ Build L3 philosophy without L0 FPF describe → philosophical claims без operational substrate
- ❌ Distribute L4 ШСМ without L1 platform → no anchor for distribution
- ❌ L2 business overlay without L1 working prototype → vapor business
- ❌ Skip L0, jump к L1 «потому что Claude Code быстро» → builds platform on undescribed system

### 3.2 Где сейчас находимся

- **L0:** in-progress. Phase 0 master = 14 objects inventory; **не yet «smooth FPF describe»** (это deliverable «smooth artefact» — see vision/01 §3.4).
- **L1:** vapor. Voice batch 17.05 «D-05 work-plan» surfaces work items related к platform; no scope draft.
- **L2-L4:** vapor. Each surfaces в Strategic Insights (H1-H8) и FUNDAMENTAL, но not as L2/L3/L4 deliverables.

### 3.3 Что **ещё не** существует

- **L0 deliverable definition** — какой exact artefact считается «L0 acked»?
- **L0→L1 handoff criteria** — какой signal triggers «time to build platform»?

- **Layer ownership** — Ruslan ownership for all layers? Specific delegates per layer?
- **Timeline horizons** — L0 weeks; L4 years? Open.
