- **Анатолий Левенчук** — FPF author / FPF spec authority / ШСМ founder. Без его engagement «FPF describe Jetix» = Jetix-fork-of-FPF без original-author validation. R12 anti-extraction relevance: Jetix uses FPF (open spec); proper attribution + collaboration = baseline respect.

### 3.2 Roadmap stages (R1 surface; sequencing logic)

**Stage 1 — Pre-engage (NOW; R1 allowed).**
- Charter circulation к Anatoly + Tseren (Charter v0 LOCKED ready)
- Vision docs (this set) share для context
- Warm conversation: «вот наш direction, готовы ли участвовать в L0 describe?»
- No commitment ask yet.

**Stage 2 — L1 ack-to-describe (after Pre-engage).**
- Anatoly + Tseren confirm readiness to participate (Commitment A.2.8)
- Scope clarified: «describe methodology по FPF as applied к Jetix system»
- Cadence agreed (weekly? bi-weekly? batch sessions?)
- Compensation / equity / partnership terms — separate decision (out-of-scope для этого doc)

**Stage 3 — L0 co-author cycle.**
- Joint FPF describe sessions (Ruslan + Anatoly + Tseren)
- Single source of truth artefact (vision/01 §3.3 H2F2H workflow)
- Iterative refinement; «утвердить, проработать, зафиксировать» (text_003 ¶1)
- L0 ack-event triggered by all three confirming «smooth artefact ready»

**Stage 4 — L1 → first clan bridge.**
- After L0 acked: expand к first clan (5-10 signatories per Charter)
- L1 core may help recruit / bless candidates
- First clan participate с L0+L1 substrate как baseline
