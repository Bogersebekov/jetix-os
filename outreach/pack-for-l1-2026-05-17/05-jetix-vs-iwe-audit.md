## §6 Feedback loops + system dynamics (sys × integrator)

### §6.1 Jetix's 6 primary feedback loops

| Loop | Trigger | Time-constant | Failure mode |
|---|---|---|---|
| **J-L1** Brigadier dispatch loop | Task intake (any) | seconds-minutes | dispatch-cap saturation (≤7 mini-swarm; ≤20 attention) |

| **J-L3** Compound learning loop (Part 5) | End-of-cycle | cycle | drift if `strategies.md` not updated |

### §6.2 IWE's 6 primary feedback loops

| Loop | Trigger | Time-constant | Failure mode |
|---|---|---|---|
| **I-L1** Daily Close (BLOCKING per CLAUDE.md §2 Rule 3) | End-of-day | 24h | Skipped close = «незафиксированный результат» — explicitly named |
| **I-L2** Capture-to-Pack (eager) | Every session boundary | minute-scale | Capture skipped → knowledge lost |
| **I-L3** OWC Session loop | Session start/end | 30 min - 4h | WP Gate skip = «незапланированная работа» |
| **I-L4** Memory promotion/demotion | Schedule-driven (14/30/90d) | days-weeks | HOT→WARM lapse → context bloat |

2. **TTL gap.** I-L5 (memory demotion) has no Jetix equivalent. Context pollution risk at scale.
3. **Strategic direction gap.** J-L5 (multi-month strategic insertion) has no IWE equivalent. IWE Strategy Session is weekly; no multi-month event-driven cascade.
4. **Trust calibration gap.** J-L6 (F-G-R per claim) has no IWE equivalent. IWE relies on informal Memory + WP-context.
