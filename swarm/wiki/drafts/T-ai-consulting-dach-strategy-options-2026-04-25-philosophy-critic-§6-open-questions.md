### D-CD-Manufacturing-vs-Cash — Manufacturing Mittelstand sales-cycle vs Phase-1 cash-timeline floor

**Position A:** H1 Manufacturing Mittelstand is the structurally strongest hypothesis —
highest GDPR moat, highest pricing tolerance (€5K-€18K), loyal Mittelstand culture with
strong renewal rates (60-70% yr1), and Beziehungspflege-driven referral potential. The
knowledge-loss pain (Fachkräftemangel + retirement wave) is urgent and demonstrably felt
by decision-makers. Investing in this vertical is the highest-LTV Phase-1 choice.
- F: F3 (C-2 arithmetic: LTV €22.5K, LTV:CAC 10.3:1 cold-weighted)
- ClaimScope: Manufacturing Mittelstand €5M-€50M; holds at steady outreach; does NOT hold
  under Phase-1 cash pressure where €0 revenue for 8-12 weeks causes R-1 materialization
- R: validated if first Manufacturing client closes within 6 weeks of outreach launch;
  invalidated if zero discovery calls within first 30 outreach contacts

**Position B:** Manufacturing Mittelstand sales cycles of 8-16 weeks (C-2 §3 H1 explicit)
directly conflict with the Phase-1 cash-floor constraint (M2 = Month 2; €80 above kill per
L7 §7). With M2 as the minimum viable gate, the first revenue must materialize within 6-8
weeks of outreach start. A 12-week manufacturing sales cycle is structurally incompatible
with M2 as a strict survival gate. H3 Startupper (1-3 week close) or H2 Professional Services
(2-4 week close) are the cash-timeline-compatible choices.
- F: F3 (C-1 §3 H1 cons: «4-8 weeks minimum from first contact to decision»; C-2 §3 H1:
  «Mittelstand sales cycle 8-16 weeks; extends outreach-to-close horizon»)
- ClaimScope: Phase-1 solo-Ruslan under M2 cash constraint; relaxes if Ruslan has 3+ months
  cash runway before M2 gate pressure materializes
- R: validated if first Manufacturing contact cycle actually closes at <6 weeks (possible
  with warm Verband intro); invalidated by empirical close-cycle data from first 3 outreach attempts
  [src:decisions/LAYER-7-BUSINESS-TRAJECTORY-DEEP-DIVE.md#§7]

**Reconciliation (no averaging):** Both positions preserved. The trade-off space is:
Ruslan can hold both simultaneously — run fast-close verticals (H2/H3) for M2 cash
survival while seeding H1 Manufacturing in parallel for M3-M6 revenue. The dissent
dissolves if Ruslan has adequate runway; it sharpens if M2 is a hard kill gate.
Ruslan's actual cash position at outreach launch is the determining variable — which
is why brigadier cannot resolve this dissent arithmetically. It requires Ruslan's
personal financial-position ack.

### D-CD-PathC-Proactive-vs-Reactive — Path C proactive offer vs reactive-only discipline

**Position A:** C-2 §5.4 arithmetic shows that proactively offering Path C to qualifying
enterprise prospects (one pre-qualification question at discovery) lifts expected revenue per
qualified lead by +€5,850 (0.30 × €19.5K uplift). For H4 Medizintechnik (where Path C
may be the majority path per C-5), restricting to reactive-only means systematically
leaving revenue on the table when the client profile clearly warrants Path C framing.
Proactive Path C offer is the revenue-maximizing posture.
- F: F3 (C-2 arithmetic model; 30% Path C trigger probability = estimate, not measured)
- ClaimScope: H4 Medizintechnik and enterprise H1 Manufacturing; does NOT apply to H3
  Startupper where Path C is architecturally out of scope
- R: validated if proactive Path C qualification in first 5 discovery calls produces
  ≥1 Path C engagement commitment; invalidated if all 5 reject or indicate Path B preference

**Position B:** C-2 §5.4 also notes that Option B (proactive Path C) requires solo-Ruslan
to manage two different sales conversations and two delivery tracks simultaneously in Phase-1.
C-5 §5.1 H1 states: «Solo-Ruslan capacity verdict: maximum 1 concurrent Path C client
Phase-1.» Capacity ceiling is the binding constraint. Proactively offering Path C when you
can only deliver 1 client at a time risks overselling or causing a waitlist problem before
the product exists. Reactive-only Path C (wait for explicit GDPR-audit demand) is the
D18-consistent, bandwidth-disciplined posture: productize Path B first, then layer in
Path C when capacity exists.
- F: F3 (C-5 §5.1: 15-20 hrs setup + 4-6 hrs/week ongoing monitoring; capacity verdict
  «1 concurrent Path C client Phase-1»)
- ClaimScope: Phase-1 solo-Ruslan; capacity constraint relaxes at Phase-2 with first
  contractor hire
- R: validated if first Path C engagement consumes ≥80% of Phase-1 delivery budget
  (confirming capacity concern); invalidated if Path C client requires <15 hrs/week total

**Reconciliation (no averaging):** Both positions preserved. The resolution path is
vertical-dependent: if Ruslan selects H4 Medizintechnik as primary (high natural Path C
rate), the proactive posture with a capacity cap (max 1 Path C simultaneously, explicitly
stated in sales process) is the rational synthesis. If H1/H2/H3 are primary, reactive
Path C is simpler and appropriate. This is Ruslan's strategic choice — not brigadier's
to resolve.

**Position A:** H3 Startupper is the highest-velocity acquisition opportunity in the

persuasive here precisely where it is easiest to sell. Phase-1 H3 case studies from

H3 as a fast-cash track while running H2 as the positioning track in parallel. Bandwidth
constraint is the binding variable.
