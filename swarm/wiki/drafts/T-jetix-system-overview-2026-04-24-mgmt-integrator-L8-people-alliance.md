# L8 — People & Alliance

## §1 Миссия слоя

L8 — это слой **людей-как-коллаборантов**, внешних сетей и альянсов Jetix. Он принципиально отличается от двух смежных слоёв: L4 (агенты — AI-оркестровщики и исполнители) и L6 (ICP-фильтр — критерии отбора клиентов). L8 — ответ на вопрос «кто стоит за Jetix и вокруг него?», не «что делают агенты» и не «кому мы продаём».

В L8 три категории людей:

1. **Команда** — сам Руслан сейчас, будущие хайры по D26, Cloud Cowork как текущий рабочий партнёр.
2. **Alliance-партнёры** — участники roy-replication сети: каждый использует методологию Jetix, коммитится на ≥5 часов в неделю, постепенно переходит «внутрь» Jetix.
3. **Community-члены** — Secure Network (Telegram → платформа Phase-2+), тематические субсети по архетипам, члены с цифровыми портретами, участники fork-community.

L8 питается данными из L6 (кто прошёл ICP-фильтр) и из L4 (какие задачи matchmaker-агент уже нашёл). L8 пишет в L5 (Alliance-продуктовые требования), в L-B (сегментация аудитории) и в L-R (учёт ресурсных обязательств — 5h/week коммитменты). L8 обращается в L9 (governance) по вопросам fork-and-merge upstream решений.

## §2 Что живёт в L8

### 2.1 Команда 50-100 (D26 — locked)

**D26 verbatim:** *«Jetix будет не one person company, а команда из 50-100 человек, работающая как холдинг»* + *«у нас тут большая компания с мощными дядями ребятами у нас enterprise но только enterprise который довольно быстрый»* [src:decisions/LOCKS-D25-D28-ADDENDUM-2026-04-24.md#D26]

D26 явно отменяет ранее звучавший фрейм «one-person company» и «One Million Company aesthetic». Jetix — enterprise-fast, целевой steady-state: 50-100 человек, холдинговая структура. Это не solo с агентами на максималках — это корпорация, которая строится с Day 1 с командной-ready инфраструктурой (D2 Solo-with-team-ready scaffolding).

Фазовая траектория команды согласно D26 + D15 (revenue-gated hires):

- **Phase 1 (€0–€50K):** Руслан solo + Cloud Cowork. Ноль найтов. Инфраструктура team-ready, но команда = один человек.
- **Phase 2 начало (€50K–€200K):** первые 2-3 найта. Продажи + операции. Revenue-gate D15 обязателен.
- **Phase 2 mid–Phase 3 (€200K–€1M):** команда 5-10. Направления специализируются. Holding-структура D19 кристаллизуется.
- **Phase 3 (€1M–$100M):** команда 20-50, по D26 trajectory. Первые roy-leaders внутри Jetix (по 10 человек каждый). Alliance-члены численно превышают core team.
- **Phase 4 (>$100M):** steady-state 50-100 по D26. Alliance 100-500+ участников. Multi-roy ecosystem.

[src:decisions/LOCKS-D25-D28-ADDENDUM-2026-04-24.md#D26]

### 2.2 Alliance + 5h/week roy-replication (D21)

Alliance — это сеть специалистов, каждый из которых использует методологию Jetix в своей нише и коммитится минимум на 5 часов в неделю вклада в общий roi (roy of interest). Механика D21: «Jetix строит свой рой из 10 людей, зарабатывающих $10M–$100M; затем распаковывает методологию как инструмент/систему; затем другие рои emerge в других вертикалях/нишах/странах» [src:decisions/JETIX-VISION.md#D21].

5h/week commitment — это не условие оплаты (компенсация-токены D23 — Phase-2+ unlock), а принцип антипаразитизма. Alliance как сеть имеет ценность только при активном взаимодействии всех участников [src:reports/review_2026-04-24.md#audio_478: «Эффект сетевой ценности сообщества: ценность есть только пока все участники активны»].

Specialists accretely переходят «внутрь» Jetix: начинают как внешние контрагенты, становятся Alliance-партнёрами, затем — roy-leaders. Это органический найтинг через доверие и proof-of-contribution. [src:raw/voice-transcripts/2026-04-24-ruslan-chat-voice-usb-c-positioning.md#Voice2: «нарастают контакты — специалисты начинают работать с нами, переходят к нам»]

### 2.3 Secure Network — Telegram-based Фаза-1, платформа Фаза-2+

Secure Network — quality-gated пространство для умных, adequate, заряженных. Антилинкедин: не «быдло ищет рабов» [src:decisions/JETIX-VISION.md#§5], а глубокая кооперация между профессионалами, прошедшими D22 5-критерийный ICP-фильтр.

**Decision 5 evolution:**

- **Phase 1 (простой chat):** Telegram-чат invite-based, 5-20 знакомых face-to-face, без anti-free-riding механик.
- **Phase 2+ (Secure Network platform):** тематические субсети по архетипам (предприниматели / инженеры / философы / политики / ресёрчеры / продавцы — каждый в своём sub-channel). Tool-sharing складчина: Perplexity-max, Claude Code, expensive research tools — pool через membership. Expertise-sharing: последние AI-наработки + экспертиза всех участников.
- **Phase 3+:** полноценная платформа с токен-экономикой D23, cross-network matchmaking, international sub-networks.

Качество-gate: проходит только тот, кто соответствует 5 критериям D22 (startupper mindset, entrepreneurial drive, stable, adequate, upward-direction) [src:decisions/JETIX-VISION.md#§7.2]. Мембрана — invite + self-selection через framing «самая большая авантюра века» [src:reports/review_2026-04-24.md#audio_506].

[src:decisions/STRATEGIC-INSIGHT-JETIX-AI-BIOS-MOMENT-2026-04-24.md#§4]

### 2.4 Цифровые портреты Alliance-членов (audio_524)

Каждый Alliance-член имеет curated digital profile — machine-readable структурированная карточка: навыки / интересы / available-hours / past-roy-contributions / archetype(s) / ICP-tier.

Портреты — топливо для matchmaker-маршрутизации (§2.6 ниже). Без структурированных портретов matchmaker работает ad-hoc «в голове Руслана»; с портретами — становится scalable и передаётся агентам L4.

Формат цифрового портрета (концептуально, Phase-2+ design):
- Имя + роль + нишевой архетип (из 11 JETIX-VISION §7.1)
- Доступные часы/неделю + preferred collaboration modes
- Скиллы (structured tags, не prose)
- Прошлые roy-contributions (linked commits / deliverables)
- Открытые потребности (чего ищет в Alliance)

[src:reports/review_2026-04-24.md#audio_524 task#464: «Создать минимальный сайт с описанием "кто я и чем занимаюсь" как основу цифрового портрета»]

Сейчас: цифровых портретов нет. Фаза-2+ разработка.

### 2.5 Fork-community (D27 — locked)

**D27 verbatim:** *«Jetix должен стать базовой стабильной системой, которую пользователи могут форкать и адаптировать под себя, а лучшие мутации возвращать в основную ветку»* [src:decisions/LOCKS-D25-D28-ADDENDUM-2026-04-24.md#D27]

GitHub-workflow применён к методологии:

- **Canonical upstream:** Jetix — stable, curated, maintained core.
- **Downstream forks:** каждый client / partner / ecosystem participant форкает методологию, адаптирует под свою нишу, своё вертикаль, свои процессы.
- **Merge-back:** лучшие мутации возвращаются upstream через PR-style contribution. Upstream держит quality bar + integration.
- D13 Closed core / Open surface усилен: core = canonical Jetix upstream, surface = allowed fork-and-merge зона.
- D21 Roy-replication уточнён: каждый roy в другой нише = fork + adaptations + потенциальный merge-back.

[src:reports/review_2026-04-24.md#audio_519 idea#201: «Создание сети пользователей Jetix, которые адаптируют систему под свои навыки и потребности, создавая экосистему форков и улучшений»]

**Governance open questions (из D27, downstream decisions pending):**

- Конкретный merge-back protocol (PR review / vote / BDFL?)
- Governance model: BDFL (Руслан) / committee / meritocracy — решение в Phase-3+
- IP licensing: MIT-like / proprietary с исключением / dual-license — открытый вопрос
- Timing когда открывать: Phase-2+ (€200K validity unlock) или раньше?

Эти вопросы явно pending; D27 зафиксировал архитектурный принцип, но не governance детали. Downstream decisions идут в SYSTEM-OVERVIEW и L9-governance. [src:decisions/LOCKS-D25-D28-ADDENDUM-2026-04-24.md#D27]

### 2.6 Matchmaker — вручную сейчас, платформа Phase-2+ (D21)

D21 механика: Jetix матчит «сложная задача» + «specialist who can solve it» + «AI-агенты смазывают coordination» [src:decisions/JETIX-VISION.md#D21].

Руслан voice-2: *«просто соединяется — надо какой-то сложный клиент, другому подрядчик — AI-агентами смазывается, происходит быстрее, качественнее, адекватнее, плюс фиксируется всё адекватно»* [src:raw/voice-transcripts/2026-04-24-ruslan-chat-voice-usb-c-positioning.md#Voice2]

Текущее состояние: Руслан делает это ad hoc вручную. Нет системы. Нет протокола.

Phase-2+ platform включает:
- Search over цифровых портретов (§2.4) по задачам
- AI-агент (L4) для маршрутизации задач → specialist profiles
- Contract-fixing механика: quick + адекватные touch-points, не многонедельные переговоры
- Network effects: чем больше портретов → тем точнее match → тем больше ценность Alliance для каждого участника

### 2.7 Mittelstand AI Alliance — EISA-moment (STRATEGIC-INSIGHT §4-5)

**Концепция:** Jetix позиционирует Alliance как sovereign European AI consortium — EISA-момент: открытый стандарт для Mittelstand + безопасная implementation + лицензируемая методология.

STRATEGIC-INSIGHT §4: *«Mittelstand AI Alliance = EISA moment — positioning as sovereign European AI consortium»* [src:decisions/STRATEGIC-INSIGHT-JETIX-AI-BIOS-MOMENT-2026-04-24.md#§4]

В 2026 рынок AI = рынок PC в 1982: бизнес ждёт стандарт, боится утечки данных (GDPR), vendor lock-in, compliance. Jetix's play: занять позицию через local-first AI + client-private KB architecture. Каждый Mittelstand-клиент — node federated network. Jetix orchestrates без централизации данных [src:decisions/STRATEGIC-INSIGHT-JETIX-AI-BIOS-MOMENT-2026-04-24.md#§5].

**Potential legal structure** (open question из STRATEGIC-INSIGHT §9 Q3): Linux Foundation analog / ARM Holdings analog. Формализация timing — Phase-2 открытый вопрос.

**Alliance как ответ на «7 конкурентных advantages» (STRATEGIC-INSIGHT §2):**
- Mittelstand-specific знания — non-cloneable clean-room
- Compliance (GDPR / EU AI Act) — built-in, не added-on
- Network effects: каждый new Mittelstand member усиливает весь Alliance

Текущий статус: concept-only. Нет юридической структуры. Нет formalized members. STRATEGIC-INSIGHT §10 action: добавить в backlog «обсуждение с Русланом open questions §9». [src:decisions/STRATEGIC-INSIGHT-JETIX-AI-BIOS-MOMENT-2026-04-24.md#§7]

## §6 Эволюция по воротам

### G0 → G1 (€0–€50K, текущая фаза)

- Solo Руслан + Cloud Cowork
- Первый Telegram-чат invite-based, 5-20 человек лично известных
- Matchmaker работает ad hoc в голове Руслана
- 3-audience лендинги строятся (задача #439, высокий приоритет)
- Alliance-портреты отсутствуют; все «портреты» — в голове основателя
- Fork-community: нет

### G1 → G2 (€50K–€200K)

- Первые 2-3 найта (D15 revenue-gate: после первых €50K+)
- Secure Network: design-фаза — архитектура, протокол качество-gate
- Matchmaker-платформа: concept-stage, не в production
- Цифровые портреты: шаблон + первые 5-10 Alliance-членов заполняют профили
- Alliance: первые 5-10 внешних специалистов с формальным 5h/week коммитментом
- Mittelstand AI Alliance: обсуждение юридической структуры с Русланом (STRATEGIC-INSIGHT §9 Q3)

### G2 → G3 (€200K–€1M)

- Команда 5-10 человек
- Secure Network platform запущена (Phase-2 activation per D5 evolution)
- Цифровые портреты запущены в production, machine-readable
- Fork-community открыта для Alliance-членов (per D27 timing: Phase-2+ validity unlock)
- Matchmaker-платформа: в production, AI-агенты L4 уже маршрутизируют через портреты
- Mittelstand AI Alliance: первые formalized члены; юридическая структура определена
- Tool-sharing складчина: активна внутри Secure Network

### G3 → G4 (€1M–$100M)

- Команда 20-50, по D26 trajectory
- Alliance 100-500 участников; первые roy-leaders вне core Jetix team
- Roy-replication external: первые рои в других нишах / вертикалях / странах, каждый = fork + adaptations
- Secure Network: международные субсети, multi-language
- Fork-and-merge: governance process устоялся; первые merge-back contributions от external roys

### G4 → G5 ($100M–$100B–$1T)

- Команда 50-100 per D26 steady-state
- Alliance 1000-5000+ участников
- Multi-roy ecosystem: Jetix meta-coordinates network of swarms in multiple verticals/geographies
- Secure Network: full platform, token economy Phase-3+ (D23 public layer, если Option B/C approved)
- Mittelstand AI Alliance: mature consortium с regulatory involvement (USB-C D20 + $1T trajectory D19)
- Цифровые портреты: full graph — relationships / contributions / trust-scores / AI-assisted matching
