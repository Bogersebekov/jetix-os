## §0 Summary — Epistemic Posture Surface (≤300 words)

**Central observation.** The 14 Jetix objects (O-01..O-14) operate at heterogeneous FPF levels — some are instantiated systems (U.System / U.Work), others are design documents (U.Episteme), others are vapor-level concepts (U.WorkPlan uninstantiated). Comparing these against L1 sources (FPF Spec, IWE template, Levenchuk books, blocked B2 and R0-R2) requires matching the comparison level to the FPF type-face of each object. Mismatched level = apples-vs-oranges categorical error.

**Three dominant flags across all 14 objects:**

1. **EP-1 Artefact-system gap (ubiquitous).** Most Jetix objects carry LOCKED document status (A.16 language-state) while operational status is F2-F4 or vapor. Any comparison must specify which face is being compared: document vs operational system. Comparing a LOCKED Jetix document to an FPF Spec section is valid at Artefact level; comparing it to operational FPF intelligence amplification requires the operational face — which for most Jetix objects is absent or partial.

2. **B2 + ШСМ blocked.** Two of five L1 sources are blocked (Aisystant paid guide, ШСМ residency materials). Any cell claiming «Jetix vs IWE» must be scoped to «public template only». Any cell claiming «vs ШСМ» is hypothetical comparison only — clearly flagged.

3. **FPF Spec scale.** FPF Spec is 72,800 lines covering a comprehensive intelligence amplification framework. Jetix is a single-owner operational AI OS in Phase A. System-level comparison is valid (two AI-OS ecosystems), but must not conflate system-level similarity with mechanism-level equivalence.

**What this matrix does NOT do:** it does not assert which comparison is more valuable, it does not recommend which levels to pursue, it does not evaluate FPF vs IWE vs Levenchuk quality. That is Ruslan's domain.
