# Повышение мощностей: Claude + Gemini + Notion

## Конечная цель
Создать связку Claude + Gemini + Notion, где каждый инструмент делает то, в чём он лучший, а Notion — единое хранилище и мост. Скорость и качество работы x2-3.

## Архитектура: кто что делает

### Claude (COO — Операционный директор)
**Сильные стороны:** Notion MCP, структурирование, выполнение задач
**Роль:**
- Работа с Notion напрямую через MCP
- Структурирование информации
- Выполнение конкретных задач
- Подготовка данных для Gemini
- Занос результатов обратно в Notion
- Управление pipeline рабочего дня

### Gemini (Стратег / Chief Analyst)
**Сильные стороны:** огромный контекст (1M+ токенов), синтез из множества источников
**Роль:**
- Bird's Eye View: загрузить 5-10 страниц → увидеть картину целиком
- Стратегический анализ, выявление противоречий
- Синтез бренда/позиционирования
- Deep research: книги, курсы, длинные документы
- Мета-анализ: какие решения принять

### Notion (Единый источник правды)
- Хранение всех данных, решений, планов
- Мост между Claude и Gemini
- Версионность, БД проектов/клиентов/знаний

## Потоки данных

**Поток 1: Claude → Notion → Gemini** (подготовка к анализу)
Claude читает Notion → собирает дамп → Руслан копирует в Gemini → анализ

**Поток 2: Gemini → Руслан → Claude → Notion** (результаты)
Gemini выдаёт выводы → Руслан передаёт Claude → структурирует → заносит в Notion

**Поток 3: Claude ↔ Notion** (ежедневная операционка)
Обновление логов, создание страниц, управление БД

**Поток 4: Gemini** (автономный deep research)
Файлы/книги → Gemini → синтез → через Claude в Notion

## Рекомендация: Вариант A + D (гибрид)

**Стандартный workflow:**
1. Руслан → задача в Claude
2. Claude → читает Notion, готовит дамп-файл с контекстом + вопросами для Gemini
3. Руслан → скачивает файл, загружает в Gemini
4. Gemini → анализирует, выдаёт результат
5. Руслан → копирует результат в Claude
6. Claude → структурирует и заносит в Notion

## Шаблоны для типовых задач

### Шаблон 1: Стратегический анализ (Bird's Eye View)
**Когда:** общая картина, противоречия, решение
**Claude:** дамп всех страниц + вопросы
**Gemini:** анализ, выводы, рекомендации

### Шаблон 2: Синтез бренда/позиционирования
**Когда:** единое видение из всех источников
**Claude:** все страницы про бренд + ICP + кейсы
**Gemini:** синтез, неочевидные связи

### Шаблон 3: Deep Research
**Когда:** фреймворки из длинного материала
**Руслан:** файл в Gemini напрямую
**Gemini:** извлечение, структурирование
**Claude:** результат в Notion

### Шаблон 4: Мета-анализ ICP/рынка
**Когда:** перекрёстный анализ данных
**Claude:** все ICP + рыночные данные
**Gemini:** перекрёстный анализ, валидация гипотез

## Первый тест: Company Hub
Источники (7 шт): Документ «Кто я + Jetix», текущий Company Hub, R&D-лаборатория, инсайты, рыночный контекст, голосовые заметки, мета-анализ Gemini.

**Решение Руслана:** берём базовые повторяющиеся паттерны (хищник + спортсмен + инженер). Глубокий анализ философии = будущее.

- Стандартизировать формат ответа Gemini (JSON/markdown)
- Автоматизация через Make/Zapier (Claude → файл → Drive → Gemini)
