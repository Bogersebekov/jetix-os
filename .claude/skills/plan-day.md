# Skill: /plan-day

## Описание
Утреннее открытие рабочего дня. Загрузка контекста, генерация плана.

## Триггер
Пользователь пишет `/plan-day` или начинает утреннюю сессию.

## Алгоритм

### 1. Загрузить контекст
Прочитать файлы в таком порядке:
1. Вчерашний daily-log (`daily-log/YYYY-MM-DD.md`) — секция "На завтра"
2. `command-center/resources.md` — доступное время, бюджет
3. Все `projects/*/overview.md` со статусом `status: active` — поле `next_action`

### 2. Собрать pending
- Незакрытые `- [ ]` задачи из вчерашнего лога
- `next_action` из каждого активного проекта

### 3. Создать файл дня
Создать `daily-log/YYYY-MM-DD.md` из шаблона `templates/tpl-daily.md`.
Заполнить автоматически:
- Секция "Статусы активных проектов" — `next_action` из overview.md
- Секция "Pending из вчера" — незакрытые задачи
- Секция "Ресурсы" — из resources.md

### 4. Предложить план
На основе приоритетов проектов и pending задач предложить:
- 1-2 главных задачи (секция "Приоритет")
- 1-2 задачи "если останется время"
- Фокус дня (одно предложение)

### 5. Дождаться подтверждения
Показать план Руслану. После подтверждения — записать в файл.

## Выход
Файл `daily-log/YYYY-MM-DD.md` создан и заполнен.
Руслан знает план на день.
