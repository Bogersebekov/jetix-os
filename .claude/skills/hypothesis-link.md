# /hypothesis-link

Bidirectional link: hypothesis ↔ artefact.

## Arguments

| Flag | Required | Описание |
|---|---|---|
| `<H-NNN>` | yes | Hypothesis id |
| `<artefact-path>` | yes | Path к артефакту (wiki/concepts/X.md / decisions/Y.md / etc.) |
| `--add-only` | no | Add only; skip reverse-side update |
| `--remove` | no | Remove instead of add |

## Workflow

1. Locate H-NNN; parse frontmatter
2. Read artefact file; parse frontmatter
3. Mutual update:
   - Hypothesis frontmatter `linked_artefacts: [..., <path>]`
   - Artefact frontmatter `linked_hypotheses: [..., H-NNN]` (Layer 3 CRM-style overlay)
4. Если artefact file не имеет frontmatter — halt + report (требуется первичное YAML)
5. Write back both files
6. Update `updated:` field на обоих
7. Append к `hypotheses/_log.md`:
   ```
   ## YYYY-MM-DD HH:MM — H-NNN linked
   - Artefact: <path>
   - Bidirectional: yes/no
   ```
8. Output: confirmation обе stороны updated

## Examples

```bash
/hypothesis-link H-001 wiki/concepts/method-systems-thinking.md
/hypothesis-link H-002 decisions/REFLECTION-INBOX-2026-05-16.md
/hypothesis-link H-005 wiki/concepts/method-as-1st-class-object.md --remove
```

## Reverse view

После linking, see `crm/hypothesis-views/by-hypothesis.md` (generated by `tools/build-hypothesis-views.py`) для aggregate view all linked artefacts per hypothesis.

## Validation gate

- Artefact file must exist
- Artefact must have YAML frontmatter (Layer 3 invariant)
- Default-Deny mass-link operations (no `--bulk` flag — per R11)
