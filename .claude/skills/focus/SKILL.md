# /focus {project} — Переключение на проект

## Что делать

Аргумент $ARGUMENTS — это имя проекта (название папки в projects/ или ключевое слово).

## Шаг 1: Найти проект
1. Если $ARGUMENTS совпадает с папкой в projects/ — используй её
2. Если нет точного совпадения — поищи grep по overview.md файлам (title в frontmatter)
3. Если не нашёл — покажи список доступных проектов и спроси

## Шаг 2: Загрузить контекст из файловой системы
Прочитай из projects/{project}/:
- overview.md — зачем, точка А/Б, текущий статус, next_action
- plan.md — план действий (если есть)
- log.md — последние 5-10 записей
- questions.md — открытые вопросы (если есть)

## Шаг 3: Загрузить контекст из Notion (если есть Notion ID)
Если в overview.md frontmatter есть поле notion_id — прочитай эту страницу через Notion MCP.

## Шаг 4: Показать сводку
Покажи компактно:
- Зачем этот проект
- Текущий статус (Точка А)
- Цель (Точка Б)
- Последние действия (из лога)
- Next action
- Открытые вопросы

## Шаг 5: Предложить следующий шаг
На основе next_action и open questions — предложи что делать в этой сессии. Жди подтверждения.

## Правила
- Если файл не существует — скажи, не выдумывай
- Сводка должна быть компактной (не больше 20 строк)
- Не начинай работу без подтверждения пользователя
