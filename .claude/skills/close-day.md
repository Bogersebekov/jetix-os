# Skill: /close-day

## Описание
Вечернее закрытие рабочего дня. Фиксация результатов, обновление логов, git push.

## Триггер
Пользователь пишет `/close-day` или завершает вечернюю сессию.

## Алгоритм

### 1. Прочитать текущий daily-log
Открыть `daily-log/YYYY-MM-DD.md` — посмотреть план и что записано в "Что сделано".

### 2. Спросить итоги (если не заполнены)
Если секция "Что сделано" пуста — спросить Руслана:
- Что сделал сегодня?
- Какие инсайты?
- Что на завтра?
- Энергия (1-10)?

### 3. Заполнить daily-log
Обновить в `daily-log/YYYY-MM-DD.md`:
- "Что сделано" — список результатов
- "Инсайты дня" — если есть
- "На завтра" — задачи на следующий день
- "Итог дня" — главный результат, что не сделано, энергия
- frontmatter: `projects_touched`, `energy`

### 4. Обновить логи проектов
Для каждого проекта из `projects_touched`:
- Добавить запись в `projects/{name}/log.md` (сверху):
  ```
  ### YYYY-MM-DD — [Что сделано]
  - Результат
  - Следующий шаг: [описание]
  ```
- Обновить `next_action` в `projects/{name}/overview.md` (если изменился)
- Обновить `updated:` в frontmatter

### 5. Git commit + push
```bash
git add -A
git commit -m "[daily] close day YYYY-MM-DD"
git push
```

## Выход
- Daily-log заполнен
- Логи затронутых проектов обновлены
- Всё закоммичено и запушено в GitHub
