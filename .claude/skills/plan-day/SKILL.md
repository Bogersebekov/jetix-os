# /plan-day — Утреннее открытие дня

## Шаг 1: Собрать контекст (автоматически)
1. Прочитай вчерашний daily-log (если есть): `daily-log/` — найди последний файл по дате
2. Прочитай command-center/resources.md — текущие ресурсы
3. Пройдись по projects/ — в каждом overview.md найди поле next_action в frontmatter
4. Через Notion MCP прочитай Командный центр (ID: 3322496333bf8161b6d3ea789d039356) — актуальные приоритеты

## Шаг 2: Показать сводку
Покажи мне:
- Что было вчера (из daily-log)
- Текущие next_action по каждому активному проекту
- Приоритеты из Notion

## Шаг 3: Предложить план
- Предложи 1-2 главных задачи на сессию (на основе приоритетов и next_action)
- Предложи фокус дня
- Жди моего подтверждения или корректировки

## Шаг 4: Создать daily-log
После подтверждения:
1. Создай файл daily-log/YYYY-MM-DD.md из шаблона templates/tpl-daily.md
2. Заполни: дату, план сессии, фокус дня
3. Git add (но НЕ commit — это сделает /close-day)

## Правила
- Не выдумывай данные. Если файл не существует — скажи об этом
- Если нет вчерашнего лога — начни с чистого листа
- Будь кратким в сводке, без воды
- Формат даты: YYYY-MM-DD
