### Phase 3 — §C Основная функция — БД (Karpathy-style wiki) + двойная фильтрация (lens-driven)

**Tasks — это самая важная Phase per Ruslan brief:**

#### C.1 Это не «заметки», это база данных
- Karpathy LLM Wiki pattern (referenced)
- 9 entity types (concepts / entities / sources / topics / ideas / experiments / claims / summaries / foundations)
- Graph structure (edges.jsonl — 9 typed edges)
- Каждая идея = entity with metadata (F-G-R triple + cross-refs + sources)

#### C.2 Concept extraction + auto-linking
- Voice note → extracted items (Claude через `extract.py`)
- Per item → tagged с existing concepts (фuzzy match + semantic)
- Новые concepts auto-suggested (если не match существующим)
- Auto-link через `wiki/graph/edges.jsonl`
- Ручной promote (DRAFT → ready через review report)

#### C.3 Двойная фильтрация (lens-driven) — критическая feature
- **Filter 1: Base wiki integration**
  - Каждая новая идея проходит через **всю существующую wiki** (1100+ entities)
  - Auto-detect similarities / contradictions / extensions
  - Если идея новая — добавляется как entity + linked к related
  - Если идея уже была — flagged as duplicate / reinforcement / refinement
  - Output: enriched extraction (не just text но связанный context)
- **Filter 2: Lens-driven dynamic filter**
  - Define «линза»: текущий focus / вопросы недели / гипотеза которую тестируем / specific project
  - Run lens против накопленных заметок → pull только relevant
  - **Use cases:**
    - «Какие insights я записывал по теме workshop-Mastery последние 30 дней?» → lens pulls only those
    - «Вопросы которые ещё не ответил» → lens identifies open questions
    - «Гипотезы которые подтвердились / опровергнуты накопленным опытом» → lens cross-references
  - Lens = программируемый (Claude prompt + filtered query); раз в несколько дней re-run для накопления

#### C.4 Worked example (vivid)
- Day 1: voice note «думал о том как Mondragón 5:1 reconciles с QF distributions»
- Extraction: item «Mondragón × QF conflict question»
- Filter 1: linked to existing concepts (Economic V10 / R12 / Mondragón / QF)
- Stored as `wiki/ideas/mondragon-qf-conflict-question.md`
- Day 30: lens «open questions in Economic V10 design»
- Lens pulls: эта item + 5 related questions + 3 partial answers
- Synthesis: pattern emerges (Mondragón cap = hard floor; QF = soft distribution; need redistribute formula)
- New concept proposed: `wiki/concepts/mondragon-qf-reconciliation-formula.md`
