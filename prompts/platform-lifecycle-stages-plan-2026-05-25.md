### Phase 2 — 3 stage definitions (core lens) ⭐⭐⭐

**Цель:** define Build / Run / Scale clearly.

**Output:** `reports/platform-lifecycle-stages-plan-2026-05-25/03-three-stages-definitions.md`

**Per stage в формате:**

#### §A Build Stage (Foundation building)

- **FPF definition:** system-under-construction; substrate accumulation; founder-solo + 1-on-1 partner trials; methodology validation pending
- **Visual markers (что я вижу когда в этом stage):**
  - Большая часть работы — founder solo
  - Partner conversations = 1-on-1, не group
  - Revenue = 0 или founder savings
  - Methodology fixated на paper, не tested by 3rd parties
  - Notion templates = design level, не implemented для multiple users
- **Entry criteria (что должно быть зафиксировано для вступления в Build):**
  - Foundation v1.0 LOCKED ✅
  - 4 LOCKED canonical existence ✅
  - Method V2 fixated ✅
  - Founder commitment to project ✅
- **Exit criteria (что должно быть достигнуто для перехода в Run):**
  - 3-5 видео записано (A method / B education / C platform)
  - Notion templates implemented (Personal OS Week 1 core 5 DBs + Team OS demo workspace)
  - Charter draft text ready
  - 1-2 партнёра T1 confirmed (methodology подтверждена)
  - 3-5 testers T3 active (Дмитрий / Сева / ближний круг)
  - Discovery call template practiced
  - Юр оформление initiated (Einzelunternehmen / GmbH decision сделано, документы поданы)
  - Basic financial system (бизнес-счёт / bookkeeping start)
- **Анти-паттерны (что путать с Build):**
  - «Substrate накопление» = NOT exit criterion (нужно exit toward execution)
  - «Idealize methodology» = perfectionism trap (good enough → ship)
  - «Wait for ideal partner» = waiting trap (1 confirmed > 10 considered)
- **Mermaid: Build stage state diagram**

#### §B Run Stage (Platform Operational)

- **FPF definition:** cybernetic feedback loop active; Method V2 §H meta-control работает; cohort 5-50 первых founders; revenue flowing; methodology validated; Charter signed
- **Visual markers:**
  - Workshop running с участниками (не только founder)
  - Revenue flow active (ступени 5 €1500/мес начали платить)
  - Cohort 5-50 founders в активной trial
  - Charter signed несколько участников
  - Methodology тестируется N людей simultaneously
  - 1-2 партнёра T1/T2 actively co-creating
- **Entry criteria:**
  - Все Build exit criteria ✅
  - + 1-st Workshop cohort launched (5+ участников)
  - + Charter signed by 3+ participants
  - + First revenue collected
- **Exit criteria (для перехода в Scale):**
  - 1K+ users (substrate consumers, не all founders)
  - Multiple cohorts running parallel (3+)
  - Revenue €10K/мес+ stable
  - Methodology has 100+ documented case studies
  - First clan / sub-community emerges organically
  - Programmable Ethereum overlay (Phase 2+) implemented если выбран path
  - Founding cohort transitioned partly to consultant/mentor roles (T4 active)
- **Анти-паттерны:**
  - «Бесконечно ждать perfect methodology» = blocker; ship at 80%
  - «Только сам ведёт workshops» = founder bottleneck; transition к partner-led
  - «Закрытое сообщество» = R12 violation; fork-and-leave preserved
- **Mermaid: Run stage state diagram**

#### §C Scale Stage (Self-propagating)

- **FPF definition:** self-propagating system; Mondragón cooperative governance; R12 anti-extraction protected; fork-and-leave preserved; кланы / mega-corporation trajectory
- **Visual markers:**
  - Multiple клан/sub-community parallel (не один центральный)
  - Audience grows через blogger/businessman leverage (не founder outreach)
  - Revenue distributed via revenue-share templates per Economic V10
  - Mondragón 5:1 ratio enforced
  - Programmable Ethereum overlay в работе (если включён)
  - Founder = one of many stewards, не sole authority
- **Entry criteria:**
  - Все Run exit criteria ✅
  - + Cooperative governance structure legalized
  - + Charter mass-signed (50+)
  - + Multiple clans active
- **Exit criteria (для трансформации в next stage — mega-corp / mass platform):**
  - 100K+ users
  - Self-governing clans без founder oversight
  - Recursive cohort spawning (cohort spawns cohort)
  - Trajectory toward $1T market cap анchored
- **Анти-паттерны:**
  - «Founder retains control» = corruption (Mondragón violated)
  - «Скрытые шорткаты» (быстрый growth) = R12 violation, secta risk
  - «Все клан одинаковые» = monoculture, нет niche diversity
- **Mermaid: Scale stage state diagram + Mondragón / R12 protections diagram**

**Mermaid PL-1:** 3-stage progression Build → Run → Scale с entry/exit criteria visible.
