### §7.1 Failure modes (≥5 deep)

#### Failure mode 1 — humans-as-bugs caveat
- Debug metaphor implies humans = errors needing fixing
- Politically loaded: dehumanising / authoritarian undertones
- Specific examples (Soviet «engineering of human souls» / authoritarian social engineering)
- Where Jetix metaphor risks this read

#### Failure mode 2 — Determinism trap
- «society-as-code» suggests deterministic execution
- Ignores emergence + agency + free will
- Hayek «pretence of knowledge» (1974 Nobel speech) — economy as computable trap
- Where Jetix metaphor risks this

#### Failure mode 3 — Agency-as-irreducible
- Moral / political agency cannot be «coded»
- Kant categorical imperative + Arendt action — agency irreducible
- Where Jetix metaphor risks reducing agency

#### Failure mode 4 — Cultural diversity (universalism trap)
- Universalist code framing ignores plural cultural ontologies
- Postcolonial critique: «whose code? whose debugging criteria?»
- Where Jetix metaphor risks cultural imperialism

#### Failure mode 5 — Marxist counter
- Code-society ignores power relations + material conditions (capital + labor)
- Class struggle not «debuggable»
- Material base vs ideological superstructure (Marx)
- Where Jetix metaphor risks ahistorical idealism

#### Failure mode 6 — Phenomenological counter
- Embodied experience missing from code abstraction
- Husserl / Heidegger / Merleau-Ponty lineage
- «Code» = third-person abstraction; lived experience = first-person primary
- Where Jetix metaphor risks technocratic erasure

#### Failure mode 7 — Morozov solutionism critique
- «every social problem becomes a coding problem» trap
- «To Save Everything, Click Here» (2013) critique
- Where Jetix metaphor risks solutionism

### §7.2 Per-failure-mode counter-argument readiness
- For each failure mode: what Jetix counter-argument could address it?
- Inventory ≥5 counter-arguments per mode

### §7.3 Strongest critique aggregation
- Top 3 strongest critiques aggregated с per-critique severity F-grade
