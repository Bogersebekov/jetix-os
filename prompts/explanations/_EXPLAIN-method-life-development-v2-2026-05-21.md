## §6 Tone + style spec (CRITICAL)

**Human-language, Feynman-style:**
- Explain to curious adult с no Jetix background
- Pretend reader nothing about Foundation v1.0 / ROY swarm / Hypothesis arch (architecture-level details = v1 reference only)
- Use everyday analogies (e.g., «информация как вода в реке» / «метод как маршрут на карте»)
- Concrete examples перед abstract concepts
- Multiple ways to explain same idea (multiple angles for robustness)
- Anti-jargon — define every technical term inline first use
- Storytelling preferred (Heath SUCCES Stories letter)
- Russian primary; English technical terms preserved + Russian explanation parenthetical

**NOT acceptable:**
- ❌ «brigadier-scribe surfaces» (jargon)
- ❌ «F-G-R triple discipline» (assumes context)
- ❌ Dense architecture-first description (that's v1)
- ❌ Constitutional posture annotations inline (footnotes OK; not main flow)

**Acceptable:**
- ✅ «представь систему которая может смотреть на саму себя и решать как ей лучше работать»
