## §2 Phase 1 — Team OS overview ⭐

**Source:** Personal OS plan substrate + Foundation Part 4 / 11 + Workshop concept + Strategic Insights Partnership.

**What to write:**

### §2.A Что такое Team OS (one-paragraph plain)

Personal OS = personal layer (individual). Team OS = collaboration layer на нём. Когда N человек с Personal OS объединяются → Team OS соединяет их workspaces в shared workspace, добавляет roles / projects catalog / marketplace / monetization / daily brief.

### §2.B Что survives из Personal OS

- DBs (Daily Log / Projects / Knowledge / CRM / Reviews / Strategic / Life Pulse / Reference) — каждый user имеет свой набор
- Frontmatter mirror discipline
- Voice intake DRAFT-only
- Claude Code integration patterns
- Fork-friendly architecture (Layer 1 universal + Layer 2 niche overlay)

### §2.C Что NEW в Team OS

- **Shared workspace** с собственными shared DBs
- **Roles + permissions** layer
- **Project Catalog DB** (joint projects browse / filter / join / propose)
- **Skills/Needs Marketplace DB** (offer/need exchanges)
- **Charter + Stage Gates lighter** для joint projects
- **Monetization templates** (revenue share / investors / contribution accounting)
- **Daily CC pass per user** (личный daily brief)
- **Onboarding ritual** (cohort cohesion)

### §2.D Когда single Personal OS достаточно vs когда Team OS оправдан

- **Personal OS only:** Daily life management / solo projects / personal learning / individual hypotheses
- **Team OS needed:** Joint projects 2+ people / cohort programs / partnership formation / revenue share scenarios / cohort accountability

### §2.E Cross-fork relationship

- Personal OS fork-friendly: Layer 1 universal + Layer 2 personal niche
- Team OS fork-friendly: Layer 1 universal + Layer 2 cohort niche (e.g., «education cohort» / «research collective» / «startup founding team»)
- Multiple Team OS workspaces возможны (один user может быть в нескольких)

## §4 Phase 3 — Roles + permissions taxonomy ⭐⭐⭐

**Source:** Foundation Part 4 Role Taxonomy + CRM 24 roles + Strategic Insights Partnership/Tyson + lev-master Phase 6 МИМ 8-level + execution-plan T1-T4 partner types.

**Per role — FULL spec (≥6 attributes):**

### §4.A Roles baseline (7-10 ready-to-use)

#### Role 1 — Project Manager (PM)

- **Scope:** Coordinator 1 specific project (one project = one PM)
- **Permissions:** EDIT Project Workspace / EDIT Decisions Queue (own project) / VIEW all shared DBs
- **Expected output:** Project velocity / Stage Gate progression / blockers surface
- **Contribution units:** PM time tracked → contribution accounting
- **Revenue share default:** 10-20% per project (negotiable per Charter)
- **R12 audit:** MEDIUM risk — coordinating power → must rotate per project / Charter clause «no PM-for-life»
- **Capabilities reference:** Foundation Part 4 manager role abstract

#### Role 2 — Investor Capital (Inv-Cap)

- **Scope:** Money investor in specific project
- **Permissions:** VIEW + COMMENT Project + Revenue Accounting (own stake) / VOTE Decisions Queue
- **Expected output:** Capital deployment + financial discipline check
- **Contribution units:** Money amount + duration → contribution share
- **Revenue share default:** Per Economic V10 — capital portion of project revenue (e.g., 20-40% per investor stake size)
- **R12 audit:** HIGH risk — capital control → Mondragón 5:1 cap mandatory + fork-and-leave + 30-day opt-out
- **Influence-ethics paired:** YES (auto-pair)

#### Role 3 — Investor Time (Inv-Time)

- **Scope:** Time/skills investor — hours worked tracked
- **Permissions:** EDIT (assigned task) / VIEW Revenue Accounting (own stake) / VOTE Decisions
- **Expected output:** Hours delivered + skill quality
- **Contribution units:** Hours × skill rate
- **Revenue share default:** Per Economic V10 — labor portion
- **R12 audit:** LOW-MEDIUM — must have hour cap protection (no 80h/week exploitation)

#### Role 4 — Investor Network (Inv-Net)

- **Scope:** Network / audience / introductions investor
- **Permissions:** VIEW Project + Revenue Accounting (own stake) / VOTE Decisions
- **Expected output:** Intros made / audience access provided
- **Contribution units:** Flat pre-negotiated share OR intros-delivered tracking
- **Revenue share default:** 3-10% per significant intro (negotiable Charter)
- **R12 audit:** MEDIUM — audience-extraction risk → R12 paired-frame mandatory

#### Role 5 — Contributor

- **Scope:** Active worker (non-investor, doesn't have capital/network stake — pure delivery)
- **Permissions:** EDIT (assigned task) / VIEW Project + Marketplace / COMMENT Decisions
- **Expected output:** Task completion
- **Contribution units:** Tasks × difficulty × quality
- **Revenue share default:** Hourly rate OR project share (Charter)
- **R12 audit:** LOW — standard employment-style, R12 paired-frame baseline

#### Role 6 — Advisor

- **Scope:** Light consultative role (1-2h per month per project)
- **Permissions:** VIEW Project + Marketplace / COMMENT Decisions
- **Expected output:** Strategic advice / blockers surface
- **Contribution units:** Sessions delivered
- **Revenue share default:** 0.5-2% flat OR per session retainer
- **R12 audit:** LOW

#### Role 7 — Facilitator

- **Scope:** Workshop / cohort session host
- **Permissions:** EDIT (sessions) / VIEW Project + Marketplace / COMMENT Decisions
- **Expected output:** Session quality + cohort engagement
- **Contribution units:** Sessions hosted × participant feedback
- **Revenue share default:** Session-based + (optional) cohort progress bonus
- **R12 audit:** MEDIUM — facilitator power → no cult dynamics / Charter Schelling check

#### Role 8 — Mentor

- **Scope:** Long-term guidance (3-12 months per mentee)
- **Permissions:** VIEW + COMMENT Mentee's Personal OS (with mentee consent) / VIEW Shared
- **Expected output:** Mentee skill progression + life-direction support
- **Contribution units:** Months × depth tier (Tier 1 1h/mo / Tier 2 4h/mo / Tier 3 weekly)
- **Revenue share default:** Per-engagement retainer (no project share unless mentor → contributor transition)
- **R12 audit:** HIGH — mentor-power dynamics → mandatory R12 boundary list (no extraction / no exclusive lock-in / fork-and-leave)

#### Role 9 — Observer

- **Scope:** View-only (potential member / curious / pre-onboarding)
- **Permissions:** VIEW Project Catalog / VIEW Marketplace / NO Decisions / NO Revenue Accounting
- **Expected output:** None (passive)
- **Contribution units:** N/A
- **Revenue share default:** N/A
- **R12 audit:** LOW
- **Transition:** Observer → Contributor / Investor / etc. after pre-onboarding

#### Role 10 — Steward (R12 enforcement)

- **Scope:** R12 paired-frame compliance enforcement across Team workspace
- **Permissions:** EDIT Charter / OVERSIGHT all Decisions Queue / VIEW Revenue Accounting (all projects) / REVIEW R12 audits
- **Expected output:** R12 violations halted ≤5s F4; Charter compliance verified
- **Contribution units:** R12 audits delivered × violations caught
- **Revenue share default:** Flat retainer (NOT project-share to avoid conflict of interest)
- **R12 audit:** META — Steward must rotate / have peer-Steward check / cannot self-promote
- **Influence-ethics analog at user-level**

### §4.B Role transitions

(Per-role transitions diagram в Phase 8 mermaid TM-3)

- Observer → Contributor / Investor (after onboarding + skills/needs match)
- Contributor → Inv-Time (если starts taking equity)
- Inv-Cap → Mentor (light involvement)
- Contributor → PM (after demonstrated coordination capacity)
- Advisor → Inv-Net (если scales their network value)
- Mentor → Steward (after 12+ months track record)

### §4.C Multiple-role hat (same user)

User может занимать multiple roles одновременно в разных проектах:
- User A = PM в Project 1 + Contributor в Project 2 + Advisor в Project 3
- Per project — separate accounting + permissions

### §4.D Constitutional anchors

- IP-1 STRICT: Roles = abstract Layer 3 (e.g., «Project Manager»); specific people (User A) = Layer 2 instance overlay binding
- Foundation Part 4: hub-and-spoke discipline preserved (PM = single dispatcher per project)

### §6.A Revenue share templates

**Template 1 — Standard project revenue share**
- Project gross revenue split:
  - 75-90% to active contributors (per contribution accounting)
  - 5-15% to Foundation (sustain Jetix infrastructure)
  - 5-15% to capital investors (if any)
- Mondragón 5:1 cap: highest contribution share : lowest contribution share ≤ 5:1
- Fork-and-leave: anyone can exit anytime, take their share to date, no penalty
- 30-day opt-out window for new joiners
- Charter clause REQUIRED before activation

**Template 2 — Investor capital project**
- Capital portion: 20-40% of project revenue (per stake size)
- Active contributors: remaining 60-80%
- Mondragón 5:1 still applies (capital + labor combined)
- Liquidity events: explicit Charter clauses

**Template 3 — Cohort program (Workshop format)**
- L7 trial cohort €1500/month per participant
- Revenue split: 70-80% facilitators / 10-20% Foundation / 0-10% L6 cohort hosts
- Mondragón applies across facilitator tiers

**Template 4 — Knowledge/IP contribution**
- Mentor / methodology framework contributor gets 5-15% retainer
- No equity unless transition to Contributor / PM
