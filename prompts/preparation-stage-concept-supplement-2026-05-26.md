### Phase 1 — Preparation Stage Concept full spec (deep)

**Tasks — exhaustive standalone concept spec:**

#### §A Что такое Этап Подготовки (Preparation Stage)
- One-liner: «Preparation = first stage of work, не pre-work»
- 1-paragraph expansion
- 1-page deeper narrative — какое feel / experience есть в preparation stage
- Контраст: подготовка-как-обязанность (cargo-cult) vs подготовка-как-творческий-фундамент (mastery)

#### §B Подготовка = неотъемлемая часть результата
- Why result starts where preparation starts (vivid argument)
- Examples: hospital surgery prep / chef mise en place / engineer requirements gathering / writer research / sales discovery call prep / consulting client onboarding
- Anti-example: «прыжок в действие без prep» — typical failure modes

#### §C Method of methods primary check — «нужна prep или нет?»
- First gate question: «эту задачу можно сразу делать (existing method) или нужна подготовка (build context first)?»
- Decision tree (P-04 triage):
  - **Speed-mode:** надкибись, низкий impact, есть существующий метод → minimal prep / direct action
  - **Depth-mode:** high impact / unique situation / cascading consequences → preparation required
- Calibration questions (P-07):
  - Сколько подготовки? (10 min / 1 hour / 1 day / 1 week)
  - Насколько детально? (rough sketch / structured plan / detailed protocol)
  - Когда? (immediate / scheduled / before specific trigger)
  - Зачем? (что узнать / что построить / что прояснить)
  - Куда? (направление / scope / boundaries)

#### §D Sequence — prep → action → feedback → improvement loop
- Sequence: 1) detect prep need → 2) select prep method → 3) execute prep → 4) build picture → 5) select action method (potentially custom) → 6) execute action → 7) collect feedback → 8) improve → next loop
- Loop visualization
- Where mastery accumulates (transitions specifically)

#### §E The Trick — Unique Method Creation through Preparation (P-06 critical)
- Pattern: «сразу делать = выбираем из existing methods» → constrained to repertoire
- Pattern: «делаем preparation → появляется picture → можем создать unique custom method» → unlocked creative ground
- Why это критично — каждая ситуация unique (per Note 2 mastery deepening), repertoire never enough; preparation = creative substrate
- Worked examples:
  - Sales: discovery call prep → understanding client → custom proposal (not template)
  - Research: literature review → unique research question
  - Engineering: requirements gathering → custom architecture
  - Coaching: client intake → adapted methodology
  - Consulting: company audit → bespoke recommendations
- Cross-link к Mastery Concept (templates × unique tasks dualism) — preparation enables unique side

#### §F Mastery Visible at Transitions (P-08, P-10)
- 3 critical transitions:
  - **Preparation → Action** (when to stop preparing, start doing)
  - **Study → Action** (knowledge → applied skill)
  - **Action → Feedback → Improvement** (learning loop closure)
- Mastery = quality + speed + accuracy at these transitions
- Анти-паттерны at transitions:
  - Endless preparation (analysis paralysis) — never crossing prep→action
  - Endless study (collectors of knowledge) — never crossing study→action
  - No feedback loop closure — repeating mistakes / no growth
- Skill development at transitions = where mastery accumulates measurably

#### §G Operational checklist для applying preparation stage
- Before any non-trivial task — 5 calibration questions
- Quick triage: speed-mode vs depth-mode (P-04)
- If depth-mode → preparation method selection
- After preparation → custom method assembly OR existing method selection
- Action execution
- Feedback collection at end (≤5 min)
- Improvement note for next loop

#### §H Anti-patterns
- Treating preparation as «not real work» (lose mastery substrate)
- Skipping preparation always (low quality output)
- Over-preparing always (analysis paralysis)
- No calibration (same prep depth для everything)
- No transition awareness (mastery doesn't develop)
- No feedback loop (no learning)
