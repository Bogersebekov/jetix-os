## <span id="id3021172"></span>Appendix A. Glossary of Abbreviations

</div>

</div>

The most important abbreviations and acronyms used in the main text are
defined here.

<div class="glosslist">

API  
<span class="emphasis">*Application Programming Interface.*</span> The
set of procedure calls that communicates with a linkable procedure
library or an operating-system kernel or the combination of both.

BSD  
<span class="emphasis">*Berkeley System Distribution*</span>; also
<span class="emphasis">*Berkeley Software Distribution*</span>; sources
are ambiguous.<span id="id3021215" class="indexterm"></span> The generic
name of the Unix distributions issued by the Computer Science Research
Group at the University of California at Berkeley<span id="id3021225"
class="indexterm"></span> between 1976 and 1994, and of the open-source
Unixes genetically descended from them.

CLI  
<span class="emphasis">*Command Line Interface.*</span> Considered
archaic by some, but still very useful in the Unix world.

CPAN  
<span class="emphasis">*Comprehensive Perl<span id="id3021270"
class="indexterm"></span> Archive Network.*</span> The central
<a href="http://cpan.org/" target="_top">Web repository</a> for Perl
modules and extensions.

<span id="gnu_project"></span>GNU  
<span id="id3021299" class="indexterm"></span>

<span class="emphasis">*GNU's Not Unix!*</span> The recursive acronym
for the Free Software Foundation's project to produce an entire
free-software clone of Unix. This effort didn't completely succeed, but
did produce many of the essential tools of modern Unix development
including Emacs and the GNU Compiler Collection.

GUI  
<span class="emphasis">*Graphical User Interface.*</span> The modern
style of application interface using mice, windows, and icons invented
at Xerox PARC during the 1970s, as opposed to the older CLI or roguelike
styles.

IDE  
<span class="emphasis">*Integrated Development Environment.*</span> A
GUI workbench for developing code, featuring facilities like symbolic
debugging, version control, and data-structure browsing. These are not
commonly used under Unix, for reasons discussed in
[Chapter 15](toolschapter.html "Chapter 15. Tools").

<span id="ietf"></span>IETF  
<span id="id3019971" class="indexterm"></span>

<span class="emphasis">*Internet Engineering Task Force.*</span> The
entity responsible for defining Internet protocols such as TCP/IP. A
loose, collegial organization mainly of technical people.

IPC  
<span class="emphasis">*Inter-Process Communication.*</span> Any method
of passing data between processes running in separate address spaces.

<span id="mime"></span>MIME  
<span id="id3020025" class="indexterm"></span>

<span class="emphasis">*Multipurpose Internet Mail Extensions.*</span> A
series of RFCs that describe standards for embedding binary and
multipart messages within RFC-822 mail. Besides being used for mail
transport, MIME is used as an underlevel by important application
protocols including HTTP and BEEP.

<span id="oo"></span>OO  
<span id="id3020062" class="indexterm"></span>

<span class="emphasis">*Object Oriented.*</span> A style of programming
that tries to encapsulate data to be manipulated and the code that
manipulates it in (theoretically) sealed containers called objects. By
contrast, non-object-oriented programming is more casual about exposing
the internals of the data structure and code.

OS  
<span class="emphasis">*Operating System.*</span> The foundation
software of a machine; that which schedules tasks, allocates storage,
and presents a default interface to the user between applications. The
facilities an operating system provides and its general design
philosophy exert an extremely strong influence on programming style and
on the technical cultures that grow up around its host machines.

<span id="pdf"></span>PDF  
<span id="id3020124" class="indexterm"></span>

<span class="emphasis">*Portable Document Format.*</span> The PostScript
language for control of printers and other imaging devices is designed
to be streamed to printers. PDF is a sequence of PostScript pages,
packaged with annotations so it can conveniently be used as a display
format.

<span id="pdp11"></span>PDP-11  
<span id="id3020161" class="indexterm"></span>

<span class="emphasis">*Programmable Data Processor 11.*</span> Possibly
the single most successful minicomputer design in history; first shipped
in 1970, last shipped in 1990, and the immediate ancestor of the VAX.
The PDP-11 was the first major Unix platform.

<span id="png_def"></span>PNG  
<span id="id3020200" class="indexterm"></span>

<span class="emphasis">*Portable Network Graphics.*</span> The World
Wide Web Consortium's standard and recommended format for bitmap
graphics images. An elegantly designed binary graphics format described
in [Chapter 5](textualitychapter.html "Chapter 5. Textuality").

RFC  
<span class="emphasis">*Request For Comment.*</span> An Internet
standard. The name arose at a time when the documents were regarded as
proposals to be submitted to a then-nonexistent but anticipated formal
approval process of some sort. The formal approval process never
materialized.

RPC  
<span class="emphasis">*Remote Procedure Call.*</span> Use of IPC
methods that attempt to create the illusion that the processes
exchanging them are running in the same address space, so they can
cheaply (a) share complex structures, and (b) call each other like
function libraries, ignoring latency and other performance
considerations. This illusion is notoriously difficult to sustain.

<span id="tcpip"></span>TCP/IP  
<span id="id3020293" class="indexterm"></span>

<span class="emphasis">*Transmission Control Protocol/Internet
Protocol.*</span> The basic protocol of the Internet since the
conversion from NCP (Network Control Protocol) in 1983. Provides
reliable transport of data streams.

UDP/IP  
<span class="emphasis">*Universal Datagram Protocol/Internet
Protocol.*</span> Provides unreliable but low-latency transport for
small data packets.

UI  
<span class="emphasis">*User Interface.*</span>

VAX  
Formally, <span class="emphasis">*Virtual Address Extension*</span>: the
name of a classic minicomputer design developed by Digital Equipment
Corporation (later merged with Compaq, later merged with
Hewlett-Packard) from the PDP-11<span id="id3020372"
class="indexterm"></span>. The first VAX shipped in 1977. For ten years
after 1980 VAXen were among the most important Unix platforms.
Microprocessor reimplementations are still shipping today.
