## <span id="id2808024"></span>Our Case Studies

</div>

</div>

A lot of books on programming rely on toy examples constructed
specifically to prove a point. This one won't. Our case studies will be
real, pre-existing pieces of software that are in production use every
day. Here are some of the major ones:

<div class="variablelist">

<span class="term">*cdrtools*/*xcdroast*<span id="id2808063" class="indexterm"></span></span>  
These two separate projects are usually used together. The *cdrtools*
package is a set of CLI tools for writing CD-ROMs; Web search for
“cdrtools”. The *xcdroast* application is a GUI front end for cdrtools;
see the
<a href="http://www.xcdroast.org/" target="_top">xcdroast project
site</a>.

<span class="term">*fetchmail*<span id="id2808124" class="indexterm"></span></span>  
The *fetchmail* program retrieves mail from remote-mail servers using
the POP3 or IMAP post-office protocols. See the
<a href="http://www.catb.org/~esr/fetchmail" target="_top">fetchmail
home page</a> (or search for “fetchmail” on the Web).

<span class="term">*GIMP*<span id="id2808168" class="indexterm"></span></span>  
The *GIMP* (GNU Image Manipulation Program) is a full-featured paint,
draw, and image-manipulation program that can edit a huge variety of
graphical formats in sophisticated ways. Sources are available from the
<a href="http://www.gimp.org/" target="_top">GIMP home page</a> (or
search for "GIMP" on the Web).

<span class="term">*mutt*<span id="id2808212" class="indexterm"></span></span>  
The *mutt* mail user agent is the current best-of-breed among text-based
Unix electronic mail agents, with notably good support for
MIME<span id="id2808233" class="indexterm"></span> (Multipurpose
Internet Mail Extensions) and the use of privacy aids such as PGP
(Pretty Good Privacy) and GPG (GNU Privacy Guard). Source code and
executable binaries are available at the
<a href="http://www.mutt.org" target="_top">Mutt project site</a>.

<span class="term">*xmlto*<span id="id2808269" class="indexterm"></span></span>  
The *xmlto* command renders DocBook and other XML documents in various
output formats, including HTML and text and PostScript. For sources and
documentation, see the
<a href="http://cyberelk.net/tim/xmlto/" target="_top">xmlto project
site</a>.

</div>

To minimize the amount of code the user needs to read to understand the
examples, we have tried to choose case studies that can be used more
than once, ideally to illustrate several different design principles and
practices. For this same reason, many of the examples are from my
projects. No claim that these are the best possible ones is implied,
merely that I find them sufficiently familiar to be useful for multiple
expository purposes.

</div>
