# Потоки информации в системе

> Единые правила: какая информация куда попадает, по какому маршруту, и где её потом искать. Никаких случайных страниц.

**Визуальная схема:** [Miro — AS IS и TO BE](https://miro.com/app/board/uXjVG2-05Ns=/)

---

# Три уровня информации

## Уровень 1: Оперативный (проекты)

**Что:** задачи, решения, логи, статусы, следующие действия
**Где живёт:** Страницы проектов (8 шт) + Daily Log + Командный центр + Недельные планы + Журнал чатов
**Срок жизни:** пока проект активен

## Уровень 2: Тематический (второй мозг)

**Что:** инсайты, идеи, конспекты, исследования, фреймворки
**Где живёт:** Research Hub (БД) + Банк идей
**Срок жизни:** долгосрочно, обрастает связями

## Уровень 3: Ритуальный (обслуживание)

**Что:** еженедельное ревью, обработка инбокса, закрытие дня
**Где живёт:** Pipeline + Анализ недели + Журнал чатов

# Правила маршрутизации: куда что записывать

| Тип информации | Куда | Пример |
|---|---|---|
| Короткая мысль / идея | Банк идей (с тегом темы) | "А что если использовать AI для прайсинга?" |
| Инсайт по конкретному проекту | Страница проекта → «Идеи и инсайты» | "Для оффера лучше начать с аудита" |
| Глубокое исследование / конспект | Research Hub (с темой + relation на проект) | "Анализ моделей AI-консалтинга в Европе" |
| Фреймворк / методология | Research Hub (тип: фреймворк) | "Value-based pricing — как работает" |
| Решение по проекту | Страница проекта → «Решения приняты» | "Начинаем с SMB, не enterprise" |
| Полезная ссылка | Страница проекта → «Ресурсы» | Ссылка на статью про AI pricing |
| Вопрос без ответа | Страница проекта → «Открытые вопросы» | "Как оценить ROI для клиента?" |
| Результат рабочей сессии | Страница проекта → «Лог работы» | "Синтез блока A завершён" |
| AI создал страницу (оперативная) | Подстраница проекта | Инструкция, чеклист, план |
| AI создал страницу (исследовательская) | Research Hub | Анализ рынка, обзор конкурентов |
| Рефлексия / состояние | Daily Log → инсайты + Недельный план → мозг/тело | "Мало спал, продуктивность упала" |

# Research Hub — структура БД

**Свойства записи:**

- **Name** (title) — название материала
- **Тема** (multi-select) — из 15 тем
- **Проекты** (relation → БД Проектов) — к какому проекту относится
- **Тип** (select: инсайт / исследование / конспект / фреймворк)
- **Дата** (date)
- **Статус** (select: черновик / готов / архив)
- **Источник** (select: Inbox / AI-чат / сам написал / внешний)

**Вьюшки:**

- Все материалы (сортировка по дате)
- По темам (группировка по Тема)
- По проектам (группировка по Проекты)
- По типу (фильтр по Тип)

**15 тем** (предварительные, уточнить с Русланом):
*Нужно выписать конкретные 15 тем из Банка идей или спросить у Руслана*

# Подстраницы-этапы в проектах

Большие проекты делятся на этапы. Каждый этап = подстраница внутри страницы проекта.

**Шаблон этапа (7 секций):**

1. **Зачем** — зачем этот этап, что он даёт проекту
2. **Точка А** — где мы сейчас по этому этапу
3. **Точка Б** — результат: что должно быть готово
4. **План действий** — конкретные шаги
5. **Показатели** — время, дедлайн
6. **Лог работы** — `[Дата] — Что сделано → Где остановились → Что дальше`
7. **Открытые вопросы** — что непонятно

**На странице проекта** добавляется секция «Этапы» с обзором:

| Этап | Цель | Статус | ~Время |
|---|---|---|---|
| Этап 1: ... | ... | ✅/⏳/⬜ | ... |
| Этап 2: ... | ... | ⬜ | ... |

**Правило отчётности (два уровня):**

- **Лог этапа** (подробный) — каждый день работы над этапом записывается в Лог работы на странице этапа. Здесь детали: что именно сделано, на чём остановился, что дальше.
- **Лог проекта** (краткий) — на странице проекта записывается только выжимка: `[Дата] — [Этап X] — Ключевой результат`. Без деталей — детали на странице этапа.

Таким образом: страница проекта остаётся чистой и обзорной, а подробности живут в этапах.

# Правило для AI: куда сохранять страницы

AI НИКОГДА не создаёт страницы в случайном месте. Правило:

1. **Оперативная страница** (инструкция, чеклист, план) → подстраница соответствующего проекта или Командного центра
2. **Исследовательская страница** (анализ, обзор, конспект) → запись в Research Hub
3. **Если непонятно куда** → спросить у Руслана

# Потоки информации (схема)

## Источники → Система

**Telegram Inbox** → еженедельная обработка → сортировка по типу → нужная БД/страница
**Claude AI (рабочие чаты)** → результаты в проект, исследования в Research Hub, саммари в Журнал чатов

## Внутри системы

**Daily Log** (черновик) → вечернее закрытие → Страницы проектов (чистовик)
**Проекты** → еженедельное ревью → Недельный план + Командный центр
**Research Hub** ↔ Проекты (relation) — двусторонняя связь
