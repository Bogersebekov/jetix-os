# 🔄 Pipeline рабочего дня — инструкция

# Pipeline рабочего дня
> Конкретная пошаговая инструкция: что делать утром, днём и вечером. Открыл → сделал по шагам → закрыл. Никаких "а куда это записать?".
---
# 🌅 УТРО: Открытие дня (15-20 мин)
## 1. Оценка состояния
Просыпаюсь → оцениваю:
- **Энергия:** высокая / средняя / низкая
- **Время на работу:** сколько часов реально есть
- **Контекст:** что было вчера, что в голове
От этого определяю **Day Type:**
<table header-row="true">
<tr>
<td>Day Type</td>
<td>Когда</td>
<td>Фокус</td>
</tr>
<tr>
<td>🔥 Deep Work</td>
<td>Энергия высокая, 3+ часов</td>
<td>Основная работа по проектам</td>
</tr>
<tr>
<td>🧩 Orientation</td>
<td>Нужно разобраться / настроить систему</td>
<td>Системная работа, планирование</td>
</tr>
<tr>
<td>🌊 Light</td>
<td>Мало энергии или времени</td>
<td>Лёгкие задачи, обработка инбокса</td>
</tr>
<tr>
<td>🔋 Off</td>
<td>Выходной / восстановление</td>
<td>Отдых, без рабочих задач</td>
</tr>
</table>
## 2. Фокус дня
Одно предложение — главное действие дня. Пример: *"Синтез блока A — довести до финального документа"*
## 3. Создание записи дня в Daily Log
**Где:** Daily Log БД → новая запись
**Что заполнить:**
- Name: `[дата] — [день недели]`
- Day Type: выбрать из списка
- Key Action (Plan): фокус дня одним предложением
- Date: сегодняшняя дата
- Проекты: выбрать проекты, над которыми работаю сегодня
**Контент страницы дня** (по шаблону):
- Точка А (начало дня) — кратко, 2-3 строки: где я сейчас
- Точка Б (цель на вечер) — что должно быть сделано
- План по шагам — конкретные шаги (вытянуть из проектов)
- Бюджет времени — сколько на каждый шаг
- НЕ-список — что сегодня НЕ делаю
## 4. Вытягивание задач из проектов
**Где:** Командный центр → БД Проекты → фильтр "Активные"
**Как:**
1. Открыть 1-2 проекта в фокусе
2. Посмотреть секцию «Следующее действие» и «План действий»
3. Скопировать конкретные шаги в план дня
4. Расставить по приоритету и времени
## 5. Настройка AI
**Где:** Claude → Командный центр (этот чат)
**Как:**
1. Открыть чат Командного центра
2. Сказать: "Доброе утро. Прочитай Командный центр и страницу дня \[ссылка\]"
3. AI читает → видит Day Type, план, контекст проектов
4. Уточнить план если нужно → AI готов
## 6. Запуск Toggl
**Как:** Выбрать проект + тег + описание задачи → Start
---
# 💪 ДЕНЬ: Работа
## 7. Рабочие сессии
**Где:** Рабочий чат Claude (один чат = одна тема)
**Как:**
1. Открыть рабочий чат для конкретного проекта/задачи
2. Дать контекст: "Работаем над \[проект\]. Прочитай страницу проекта \[ссылка\]."
3. Работать по плану
4. Toggl трекает время
**Правила рабочего чата:**
- Один чат = одна тема. Не смешивать проекты
- Если чат длинный (15+ сообщений) → закрыть, открыть новый
- Перед закрытием: попросить AI записать результаты в Notion
## 8. Переключение между задачами
Если переключаюсь на другой проект:
1. Остановить Toggl → запустить новый таймер
2. Открыть новый рабочий чат (или вернуться в существующий)
3. Дать контекст нового проекта
---
# 🌙 ВЕЧЕР: Закрытие дня (15-20 мин)
## 9. Фиксация результатов
**Где:** Чат Командного центра
**Как:** Сказать AI: "Закрываем день. Обнови всё."
AI делает (или я делаю вручную):
### 9а. Страница дня (черновик)
- [ ] Секция «✅ Что сделано» — всё что реально сделано
- [ ] Deep work минуты
- [ ] Инсайты дня
- [ ] Вопросы которые возникли
### 9б. Свойства Daily Log
- [ ] Actual Main Action — что реально было главным
- [ ] Reflection — одно предложение про день
- [ ] deep work min — количество
### 9в. Страницы проектов (чистовик)
В каждый затронутый проект → секция «Лог работы»:
```javascript
[Дата] — Что сделано → Где остановились → Что дальше
```
- [ ] Обновить «Следующее действие» в БД Проектов
- [ ] Обновить «Статус» проекта (если изменился)
### 9г. Toggl
- [ ] Все таймеры остановлены
- [ ] Проверить записи: проект + тег + описание
### 9д. Журнал чатов
- [ ] Записать в БД «Журнал чатов»: какие чаты были, что обсуждали, ключевые результаты
### 9е. Память AI
- [ ] Если появилась новая важная информация → обновить memory edits
- [ ] Если изменились проекты/приоритеты → обновить Project Instructions

# 📊 РАЗ В НЕДЕЛЮ: Ревью (воскресенье/понедельник)
## 10. Еженедельное ревью
🔗 **Полная инструкция:** [📊 Анализ недели](https://www.notion.so/3342496333bf814a88e1d0699d1d469d)
**Кратко (3 части):**
1. **Анализ:** что сделано по проектам → план vs реальность → время (Toggl) → инсайты → состояние (мозг/тело)
2. **Обработка:** Telegram-инбокс → распределить по проектам
3. **Планирование:** Точка А → Точка Б → фокусные проекты → НЕ-список → обновить статусы
