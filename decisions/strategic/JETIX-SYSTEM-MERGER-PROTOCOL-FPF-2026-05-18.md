## §3 Two sub-protocols

### §3.1 «Намордник» (constraint imposition)

**Process:**
1. Host system catalogues required-constraints (e.g., R12 anti-extraction, FPF discipline, F-G-R per claim).
2. Incoming system commits к constraint set as condition of merger.
3. Programmable enforcement: smart-contract / governance / audit-trail layer.
4. Periodic compliance audit.

**Cybernetic frame:** Ashby requisite variety — host imposes constraint set sufficient to match incoming system's behavioural variety. Per research/deepening-2026-05-18/03-ashby-requisite-variety.md.

**Programmable enforcement:** Per Ethereum architecture acked 18.05 — R12 programmable Option D Hybrid (Mondragón wage ratio cap + QF revenue distribution + fork-and-leave exit tokens). Applies к merger constraint enforcement.

### §3.2 «USB-C порт переделка» (interop bridge)

**Process:**
1. Incoming system retrofits FPF-compatible interface (lingua franca translation layer).
2. Host system exposes standard FPF endpoint.
3. Data + role + commitment + work-artefact translation through FPF.
4. Bidirectional protocol-mediated interaction.

**Cross-precedent — universal connector evolution:**
- USB-A (1996) → USB-C (2014): physical + protocol unification.
- TCP-IP (1974): network layer protocol.
- HTTP (1991): application layer protocol.
- W3C / IETF: open standards governance.

**FPF candidate:** universal merger language at info-system layer.

## §5 Cross-precedent triangulation

### §5.1 USB-C history (2014 → present)
- 14-year journey USB-A → USB-C (universal connector).
- 2014 standardization → 2018+ mainstream adoption.
- **Lesson:** standardization 5+ years to ubiquity; needs major player anchor.

### §5.2 TCP/IP (1974 → present)
- DARPA project → ARPANET → global internet.
- Open protocol governance (IETF).
- **Lesson:** open-protocol + governance + reference implementation = adoption.

### §5.3 M&A studies (corporate)
- Cisco acquisition pattern (1990s-2000s): 200+ acquisitions integrated through standardized M&A playbook.
