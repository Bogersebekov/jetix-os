## §1 Multi-tradition SOTA concept decomposition

(Phase 1 deep extraction Левенчука + Phase 2 + Phase 3 integration)

### §1.1 Что SOTA значит — 7 ortho axes

Across all traditions, SoTA-claim factorises на 7 ortho axes:

| Axis | Что значит |
|---|---|
| **WHAT** | Объект (method / discipline / instrument / theory / mastery / artefact) |
| **WHEN** | Year-stamp (Левенчук: «SoTA всегда имеет дату» [Метод 2025 :6485]) |
| **WHERE** | Domain scope (ML vision / methodology composition / cooperative governance) |
| **WHO** | Community of recognition (Kuhnian paradigm-relative) |
| **WHY-better** | Predictive advantage + universality + compactness [Левенчук §15] |
| **HOW-validated** | Mechanism (benchmark / replication / community uptake) |
| **HOW-LONG** | Expected lifespan / time-decay (5+5=10y Левенчук default) |

**Каждая honest SoTA-claim должна specify все 7.** Без любого = vague claim, potentially dishonest.

### §1.2 8 paradigms × 7 axes — matrix dimensionality

| Paradigm | Primary axis emphasis |
|---|---|
| Popper | WHY-better (predictive risk) + HOW-validated (falsification) |
| Kuhn | WHO (paradigm community) + WHEN (revolutions) |
| Lakatos | WHAT (programme structure) + HOW-LONG (progressive lifespan) |
| Feyerabend | (denies single SOTA) — multiple WHERE + WHO simultaneous |
| Laudan | WHY-better (problem-solving) |
| Hacking | WHAT (instrument autonomy) + HOW-validated (intervention) |
| Longino | WHO (4 social norms) |
| Chalmers (Bayesian) | HOW-validated (probabilistic update) |

**Net:** Jetix substrate должен **держать все 8** как simultaneously valid axes для different decision contexts.

| Field | SoTA decay | Authority | Mechanism |

| Classical engineering | years | standards bodies | ISO + IEEE |
| Software (general) | 3-5 years | open-source community | GitHub + Hacker News |
| **ML/AI 2020+** | **weeks-months** | **mix academy + Big Tech** | **PWC + arXiv + HF + benchmarks** |

### §4.2 Goodfellow + Huyen — operational SOTA-discipline для ML

**Goodfellow Deep Learning (2016):**
- Benchmark = falsification test (Popperian operational)
- Benchmark staleness through reuse [line 3534]
- Multiple datasets per area (MNIST → CIFAR → ImageNet) = Lakatosian progressive shift through benchmark replacement
- Architecture changes «every few weeks to months» — continuous tracking required

**Huyen Designing ML Systems (2022) — explicit «Avoid SOTA trap»:**
- Research SoTA vs Production SoTA = different objects [Ch.1]
- «Static benchmark ≠ your data» [line 5989]
- «Simple baseline first» [line 5995]
- Melis 2018 — hyperparameter tuning rivals architecture [line 6538]
- «Appear cutting edge» = bullshit motivator, не engineering [line 5984]
