# Метод и Мастерство — собрать нужные методы + применить в нужный момент

## One-liner

**Мастерство = накопить нужные методы + выбрать и применить нужный метод в нужный момент.**

Не «знать много методов» (это библиотека). Не «применять один метод универсально» (это привычка). Именно **умение выбирать** — что применить **здесь и сейчас** под **эту конкретную ситуацию**.

## Откуда это (inheritance, честно)

**База — Method V2 §J meta-method** + 8-component composition (наша canonical methodology, building поверх Левенчуковской FPF и OMG-MMK lineage). Не оригинальный вклад — это переработка известной методологической base.

**Ericsson deliberate practice** + **Carol Dweck growth mindset** + восточные традиции мастерства (без идеализации) + **engineering virtuosity** (Knuth / Hoare / Levenchuk МИМ) = source pool.

**Honest:** большая часть концепций — это синтез + переформулировка известного. Оригинальный вклад (если есть) — **в синтезе + AI-augmentation + Workshop frame + Mastery at transitions explicit + Templates × Unique dualism explicit + Preparation Stage explicit**.

## Что в этом синтезе свое (4 точки)

### 1. Workshop как foundational metaphor (не «платформа»)

Jetix описывается не как **«платформа для развития»** (abstract), а как **мега-мастерская** (concrete пространство). Workshop = body of Vision. Это меняет communication frame с marketing на experiential — «вот пространство, вот инструменты, заходи работать». Per анти-маркетинг discipline.

### 2. Mastery at transitions (не at peaks)

Мастерство видно и накапливается **на переходах**, не в peak performance:
- preparation → action (когда останавливать prep и стартовать)
- study → action (когда применить новое знание)
- action → feedback → improvement (закрытие цикла обучения)

Это operational definition mastery, проверяемый через portfolio (количество + тяжесть успешно пройденных transitions), не через сертификаты.

### 3. Templates × Unique tasks dualism

Mastery = балансировать две стороны:
- **Templates side:** автоматизировать / templatize максимум что повторяется (чтобы голову не ебать)
- **Unique side:** embrace inherent uniqueness ситуаций (каждая работа с клиентом / партнёром уникальна)

Не «один из двух», а **умение выбирать когда templating vs когда unique** — это сам по себе skill.

### 4. Preparation Stage explicit (Extended 8-step meta-method)

Расширили Method V2 §J 6-step procedure до 8 steps с **explicit preparation stage** перед action:
- Step 0: Preparation need detection
- Step 1: Preparation method selection
- Step 2: Preparation execution
- Step 3: Picture formation
- Step 4: Action method selection / CREATION (existing OR unique custom)
- Step 5: Action execution
- Step 6: Feedback collection
- Step 7: Improvement note → next loop

**The trick:** «когда хочешь сразу делать — берёшь из existing методов; когда делаешь preparation — после неё появляется picture → можешь создать **unique custom method** для конкретной ситуации» (более effective чем repertoire).

## AI-augmentation (where it fits)

Per O-182 stratification:
- **Preparation execution = AI heavy** (research / synthesis / structured queries / formatting / capture)
- **Action method selection / creation = HUMAN** (mastery moment — unique method born из preparation)
- **Transitions = HUMAN** (judgment calls)
- **Feedback aggregation = AI assist + HUMAN judgment**

AI = preparation amplifier. Human = action strategist + transitions master. Не «AI заменяет человека» — AI освобождает время для mastery moments.

## Чем это НЕ является

- **НЕ peer-reviewed methodology** (yet) — персональный voice-to-canon процесс + AI synthesis; для peer-review нужен отдельный shaping process через partners
- **НЕ новая фундаментальная methodology** — переформулировка + синтез + extension существующего (Method V2 §J / Левенчук / Ericsson / Dweck)
- **НЕ универсальная инструкция «делай так»** — это descriptive фреймовка, не prescriptive алгоритм
- **НЕ замена для systematic methodology training** (МИМ / engineering schools / etc.)

- Frame для AI integration (preparation vs action разделение)
