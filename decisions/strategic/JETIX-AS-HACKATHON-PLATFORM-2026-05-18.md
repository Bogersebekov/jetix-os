## §5 Operational pattern formalization

### §5.1 Event orchestration cycle

1. **Initiate** — sponsor commits + project surfaced + Strategic file shared («strategy file» per text_009 Thread 3 ¶3).
2. **Match** — participants (прошаренные + крутые first; bloggers + general second-cohort) + mentors + tools auto-assigned via FPF protocol.
3. **Solve** — multi-day/month marathon с continuous mentor pairing (TPS pattern per research/deepening §14).
4. **Reward** — QF distribution per Tang+Weyl mechanism (Ethereum architecture acked).
5. **Recurse** — winners feedback к platform improvement (gratitude loop per text_009 Thread 7).

### §5.2 Multi-rhythm cycle taxonomy

| Rhythm | Duration | Project complexity | Audience |
|---|---|---|---|
| Day | 1 day | Low-medium (clearly-scoped, blueprint-class) | bloggers + general |
| Month | 1 month | Medium (sponsor projects, integrated systems) | engineers + sponsor partners |
| Year | 1 year | High (major collaborations, Master Workshop apprenticeship) | L1 engineers + Master Workshop |

### §5.3 Stack baseline (per text_008 + text_009)

- **Compute:** Cloud Cowork + internet access + serverless options.
- **FPF protocol:** 72K строк vendored; participant onboarding via FPF (lingua franca).
- **Mentors:** mentor pairing; per-cohort allocation.
- **AI наработки:** best practices library + AI tooling.
- **Protocols:** FPF + ROY swarm orchestration + Ethereum substrate.
