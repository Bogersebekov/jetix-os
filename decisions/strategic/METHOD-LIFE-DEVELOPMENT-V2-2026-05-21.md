## §1 Фундаментальная онтология «всё = информация + методы переработки»

**[Полная глубина: `reports/method-v2-2026-05-21/01-fundamental-ontology.md`]**

### Что значит «всё = информация»

Руслан 21.05 evening: «отталкиваясь от того что все есть информация».

Простыми словами: информация — это **любая разница, которая что-то значит для
системы** (Бейтсон 1972: «a difference which makes a difference»). Не «маленькая
часть мира», а **способ описать мир**. Шеннон 1948 дал математическую меру через
entropy (биты). Винер 1948 ввёл противопоставление «информация vs шум» как ключ
к понятию упорядоченных систем.

### Что такое метод

Метод — это **способ что-то сделать с информацией**. Чтение — метод; программа —
метод; привычка — метод. Левенчук MG4 ⭐⭐⭐: **метод как объект 1-го класса** —
методы сами можно изучать, передавать, комбинировать. Это огромный сдвиг от
«живём методами» к «думаем о методах».

### Информация о методах = методология

Третий слой — meta-уровень. Знание о том, **какие методы существуют и когда
какой применять**. Это и есть **методологическая основа Jetix**:
- Wiki содержит факты (информация уровня 1)
- Wiki содержит how-to статьи (методы уровня 2)
- Wiki содержит сборники паттернов (информация о методах = методология уровня 3)

## §2 Самоуправляющиеся системы

**[Полная глубина: `reports/method-v2-2026-05-21/02-self-managing-systems.md`]**

### Что значит «адекватно управлять собой»

Руслан 21.05: «система может вот адекватно управлять собой там ставить себе
вот какие-то цели задачи и так далее то она может вот хорошо развиваться».

Четыре связанные способности:
1. **Ставить цели** — куда я хочу прийти
2. **Ставить задачи** — что для этого делать
3. **Отслеживать движение** — где я сейчас
4. **Корректировать** — менять курс, если уходишь

Это **петля управления с обратной связью** в чистой кибернетической форме
(Винер 1948). Не метафора — математически описанный механизм. Простейший
пример — термостат.

### Винер — кибернетика 1948

Норберт Винер впервые объединил в единую теорию управление в живых организмах,
машинах, обществе. До Винера это были три дисциплины. После Винера — один
класс явлений на разных носителях. Negative feedback (стабилизирующая) vs
positive feedback (усиливающая) — оба нужны.

### Эшби — закон необходимого разнообразия

«Только разнообразие в управляющей системе может контролировать разнообразие
в управляемой» (Эшби 1956). 10 сценариев требуют ≥10 ответов. **ROY swarm
5 экспертов** = applied этого закона.

## §4 Потребление информации + накопление опыта

**[Полная глубина: `reports/method-v2-2026-05-21/04-info-consumption-experience.md`]**

Руслан 21.05: «процесс жизни это процесс» + «потребил много информации
соответственно у меня там опыт накопился».

### Жизнь — процесс, не состояние

Минута за минутой — информация входит в систему. 95-98% обработки **ниже
уровня сознания** (Канеман System 1). Сознательно контролируем: куда направить
внимание, что выбираем читать, с кем общаемся, какие методы включаем.

### Compound interest в накоплении опыта

Если N новых единиц/день связываются с K из накопленных — полезность растёт
**экспоненциально**, не линейно. Большинство людей живут десятилетиями без
compound — поверхностное потребление / отсутствие связывания / нет внешней
памяти / нет рефлексии. Метод развития = **активная борьба** с этими
антипаттернами.

### Tacit knowledge (Polanyi)

«Мы знаем больше, чем можем рассказать». Большая часть мастерства = tacit
(кататься на велосипеде / узнавать лицо / готовить «по вкусу»). Передаётся
через **apprenticeship + действие + наблюдение**, не через тексты. Поэтому
Jetix Workshop = hands-on, не лекции.

### Каналы потребления

```mermaid
mindmap
  root((Каналы потребления))
    Чтение
    Слушание
    Смотрение
    Действие опыт
    Рефлексия
    Беседа эксперт
    Преподавание
    Сон консолидация
```

**Преподавание = лучший способ выучить** (Феинмановский test of understanding).
В Jetix через 3-tier funnel — продвинутые обучают новичков, углубляя своё
понимание.

### DIKW pyramid (Ackoff 1989)

Данные → Информация → Знание → Мудрость. Каждый переход = переработка, не
хранение. Современная экономика застревает на data/information; **knowledge**
и **wisdom** — дефицит.

**Что отсюда следует.** Compound работает с правильной техникой. Tacit
knowledge — большая часть мастерства; hands-on неизбежен. Качество каналов >
количество. Активное потребление в десятки раз эффективнее пассивного.
Внешняя память — мультипликатор. Сон — глубокая обработка, не «потеря времени».

### 6-шаговая процедура мета-метода (Ruslan voice)

1. **Identify системы влияния** — окружающие system (people / contexts /
   tools / resources)
2. **Целесообразно собирать информацию** — достаточно для действия, не больше
   (gradient достаточности; satisficing per Simon)
3. **Целесообразно обрабатывать** — что важно / что фоновое; pattern matching
4. **Ставить цель целесообразно** — не «хочется», а «надо для жизни»
5. **Выбрать лучший метод** — на основе шагов 1-4
6. **Apply + measure + adjust** — execute, observe, update method library

### Gradient достаточности

Не perfect information, а **достаточная** для конкретного действия. Simon
1957 bounded rationality + satisficing. Кривая «качество × объём info»
загибается — есть момент, после которого больше research = убывающая отдача.
