> **Цель раздела:** дать ОБРАЗ системы. Метафора = быстрое понимание.

## 2.1 Почему мастерская
*[Заполнить из Workshop concept LOCKED v1 — почему именно мастерская. Не завод. Не университет. Не офис.]*

## 2.2 Что есть в мастерской
- **Фундамент** — пол, стены, крыша (базовая инфраструктура)
- **Охрана** — фильтрация что попадает внутрь
- **Склад** — хранилище материалов и готовых изделий
- **Столы и станки** — рабочие инструменты (adaptable)
- **Front-office** — взаимодействие с внешним миром
- **Планировочный стол** — стратегия / приоритизация

> *[Каждый элемент развернуть на абзац. Как именно проявляется в системе управления.]*

## 2.3 Адаптивность станков ⭐
*[Заполнить: главное преимущество — станки можно быстро добавлять, переделывать, удалять под текущую задачу. Примеры: D2 внедрён за 1 день, MCP, Plan Mode, новые agents. Это НЕ статичная система.]*

## 2.4 Роль владельца
*[Заполнить: владелец мастерской НЕ исполнитель. Он — архитектор + арбитр + дегустатор результата. Адаптирует роль по фазе развития.]*

## 2.5 Философия — "система для мастеров, кастомизируется под кейс"
*[Заполнить: эта система не "одна для всех". Она — каркас, который каждый адаптирует под себя. Investor → одни станки. Инженер → другие. Health-coach → третьи. Принципы общие, конкретика разная.]*

# 4. 🏗️ Архитектура системы — слои

> **Цель раздела:** как это устроено technically/conceptually.

## 4.1 Слой 0 — Foundation (фундамент)
- **Storage** — где живут данные (filesystem / wiki / DBs)
- **Index** — как найти что нужно
- **Conventions** — единый язык (datetime / naming / tags)

## 4.2 Слой 1 — Information processing
- **Inputs** — voice / chat / web / docs
- **Filters** — что впускаем / что отбрасываем
- **Digestion** — что осмысливаем / куда складываем
- **Outputs** — что наружу (decisions / artifacts / messages)

## 4.3 Слой 2 — Resource management
- **Attention tracking** — что съедает внимание
- **Time tracking** — куда уходит время
- **Knowledge tracking** — что узнал / куда положил
- **Connection tracking** — кто рядом / какой leverage

## 4.4 Слой 3 — Adaptive tools (станки)
- Можно добавлять / удалять / переделывать
- Примеры: skills / agents / workflows / scripts

## 4.5 Слой 4 — Strategic layer
- **Planning** — куда движемся
- **Review** — где мы относительно плана
- **Adjustment** — корректируем курс

- **Минимум:** Foundation + Information processing
- **ROI:** освобождение 5-15h/неделю + меньше losses

## 9.4 Investor / Capital allocator
- **Цель:** видеть состояние portfolio + thesis evolution
- **Что взять:** Foundation + visibility слой
- **ROI:** faster better decisions

## 9.5 Researcher / Knowledge worker
- **Цель:** накопить персональный knowledge graph
- **Что взять:** Foundation + knowledge accumulation
