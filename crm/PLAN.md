## ¬І3. Schema (frontmatter)

See `crm/README.md ¬І3` for full schema, or authoritative spec at
`crm/_schema/frontmatter.yaml`.

**Required:** `name`, `slug`, `type`, `roles`, `created`, `updated`.

**Optional buckets:** identity / background / contact / audience / tools /
origin / icp (D22 fit) / pipeline / value / meta / org-extras.

**Auto-fix:** `icp.total = sum(azart+stable+adequate+upward)`,
`updated >= created`.

See `crm/README.md ¬І4` for cheatsheet, `.claude/skills/crm-*.md` for

## ¬І5. Strategy hooks (¬І7 ¬І8 auto-fill)

6 OFFERS + 6 ASKS in `crm/_schema/strategy-hooks.yaml`. Filter operators:
`roles`, `tags`, `icp_startupper`, `icp.<field>_min`,
`audience.total_followers_min`, `location_country`, `<field>_min`.

`/crm-add` and `/crm-update --resync-hooks` evaluate filters against
contact frontmatter and prefill ¬І7/¬І8 with `[auto-suggested, edit me]`
markers. Always reviewed before outreach.

Hooks reference live strategy docs (`decisions/`, `directions/_active/`,
`swarm/wiki/cycles/`). Update file as strategy evolves.

## ¬І6. Voice pipeline integration

`tools/run_pipeline.sh` extracts items with `target: crm` from voice memos.
`crm/_scripts/voice_router.py` routes:

- `intent: add` вЖТ creates `<slug>-DRAFT.md` (status=`draft_from_voice`).
- `intent: touch` вЖТ fuzzy-match вЖТ if 1 candidate: append ¬І11 entry.
- `intent: update` вЖТ fuzzy-match вЖТ if 1 candidate: update fields.

**DRAFT-only invariant:** never auto-overwrites prod records. Ambiguous
matches reported in summary for human resolution.

## ¬І7. Stuck detection

Defined in `_schema/statuses.yaml`:
- Active statuses: `warm`, `contacted`, `discovery_call`, `proposal`, `negotiation`
- Threshold: 14 days since `last_touch_date`

Surfaced via `/crm-stuck` and weekly review.
