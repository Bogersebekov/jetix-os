# `hypotheses/tables/` — Layer 7

Excel/CSV table layer. Filesystem-authoritative MDs = source of truth; Excel/CSV = derived view.

## Files

| File | Purpose | Auto-generated by |
|---|---|---|
| `hypotheses.xlsx` | Master Excel — все гипотезы; columns = frontmatter fields | `_build-table.py` |
| `hypotheses.csv` | CSV mirror (filesystem-friendly; git-diff-able) | `_build-table.py` |
| `alphas-state-graph.xlsx` | Per-alpha sheets с current states per hypothesis | `_build-table.py --alphas-table` |
| `_build-table.py` | Rebuild script (openpyxl + pandas + PyYAML) | — |

## Usage

```bash
cd ~/jetix-os

# Full rebuild — default
python3 hypotheses/tables/_build-table.py

# CSV only (if openpyxl unavailable)
python3 hypotheses/tables/_build-table.py --csv-only

# Skip samples
python3 hypotheses/tables/_build-table.py --no-include-samples

# Skip alphas table
python3 hypotheses/tables/_build-table.py --no-alphas-table
```

Or via skill:
```bash
/hypothesis-build-table
```

## How it works

1. Walks `hypotheses/{active,testing,confirmed,refuted,paused,samples}/`
2. Parses YAML frontmatter каждого `H-*.md` file
3. Aggregates rows; superset of frontmatter fields → columns
4. Writes:
   - `hypotheses.csv` (canonical fields first, then alphabetical)
   - `hypotheses.xlsx` (bold header, autowidth, freeze panes)
   - `alphas-state-graph.xlsx` (sheet per alpha, sorted by hypothesis updated)

## Excel structure

### `hypotheses.xlsx` — "Hypotheses" sheet
Columns (canonical order):
1. id
2. title
3. slug
4. status
5. _status_dir (which dir file lives in)
6. category
7. lifecycle
8. owner
9. created / updated
10. fpf_F / fpf_G / fpf_R (Layer 5)
11. strategic_region
12. test_method / success_criteria / refutation_criteria
13. outcome / learning_extracted
14. alphas (flattened list of {alpha, state})
15. linked_artefacts / linked_hypotheses
16. _file (filesystem path)

Plus any custom frontmatter fields в alphabetical order.

### `alphas-state-graph.xlsx` — 7 sheets (one per alpha)
Each sheet:
- H-NNN
- Hypothesis title
- Current state в this alpha
- Status (hypothesis overall)
- File
- Last updated

## Workflow с Claude Code

Claude can edit `.xlsx` через openpyxl:

```python
import openpyxl
wb = openpyxl.load_workbook("hypotheses/tables/hypotheses.xlsx")
ws = wb["Hypotheses"]
# Read row, modify, save
```

**BUT:** edits to xlsx are NOT authoritative. They get overwritten on next `_build-table.py` run. **MD files = source of truth.** Use Excel/CSV для:
- Visual scanning / filtering
- Sharing snapshot с stakeholders
- Spreadsheet-style analysis (pivot tables, charts)

## CSV fallback (no openpyxl)

If openpyxl unavailable, script writes only `hypotheses.csv`. Filter/sort via:
```bash
# Quick filter: only active
awk -F, 'NR==1 || $4=="active"' hypotheses/tables/hypotheses.csv
```

## Idempotency

Safe to re-run any time. Output deterministic based on MD frontmatter state.
No data loss возможен — regenerates from MDs.
