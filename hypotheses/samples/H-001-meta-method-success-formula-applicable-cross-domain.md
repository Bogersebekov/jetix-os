# H-001 — Meta-method (cycle + discipline + output + reflection) applicable cross-domain

## Гипотеза

Meta-method, identified как «cycle defined → discipline → output → reflection» паттерн в инженерном контексте, **generalizes beyond engineering** к non-engineering доменам с сохранением структуры и продуктивности.

Если гипотеза подтверждена, это означает:
- Meta-method = first-class primitive (per Левенчук MG4)
- Любой domain может быть скаффолдирован через этот pattern
- Foundation для methodology-as-a-service consulting offering

## Метод теста

Multi-domain replication test:

### Domain 1 — Education (Я как студент)
- Pick новый навык (например, «advanced LaTeX»)
- Define cycle: lesson → exercise → reflection → next lesson
- Run 5+ iterations
- Write reflection: did pattern hold? where did it strain?

### Domain 2 — Personal-dev (Daily habit)
- Pick habit (например, «morning meditation 15min»)
- Define cycle: trigger → execute → log → reflect → adjust
- Run 30 days
- Write reflection.

### Domain 3 — Pitch development
- Pick pitch artefact (Karpathy outreach example existing)
- Define cycle: draft → test (read aloud OR send to 1 trusted) → reflect → revise
- Run 5+ iterations
- Write reflection.

### Aggregation
После 3 domains complete:
- Identify common structure?
- Where did pattern strain? (domain-specific exceptions)
- Compound learning: 1-3 line invariant per domain + 1 cross-domain invariant

## Условия / scope

- Catallactic strategic region (не игровые / не adversarial домены)
- Min lifecycle ≥3 months (each domain needs ≥1 month deep dive)
- Single agent execution (Ruslan); not requires team
- Excludes: high-stakes domains (legal / medical) — overly constrained для cycle definition
