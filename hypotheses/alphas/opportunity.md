# Alpha: Opportunity

## Purpose
Reason for endeavor — problem / need / chance что motivates работу.

## State graph
```
identified → solution_needed → value_established → viable → addressed → benefit_accrued
```

## State definitions

| State | Meaning |
|---|---|
| **identified** | Возможность / проблема noticed; pre-conceptual |
| **solution_needed** | Подтверждено что нужно решение — есть pain или value-opportunity |
| **value_established** | Конкретная value proposition articulated (для каких stakeholders какой outcome) |
| **viable** | Solution path technically/financially viable; resources realistically available |
| **addressed** | Solution implemented; opportunity activly being captured |
| **benefit_accrued** | Measurable benefit realized — opportunity successfully captured |

## Jetix application

Для hypothesis substrate, opportunity = problem-space hypothesis targets:
- **Outreach hypothesis:** opportunity = «L2 founders ищут methodology partner»
- **Method hypothesis:** opportunity = «meta-method generalization unlocks 10× domains»
- **Engineering hypothesis:** opportunity = «pipeline X reduces friction Y»
- **Business hypothesis:** opportunity = «3-tier funnel предлагает better unit economics»

## Example transition

```
2026-05-20 — opportunity → identified: Meta-method might generalize beyond engineering
2026-06-10 — opportunity → solution_needed: cross-domain testing confirms broad applicability
2026-07-01 — opportunity → value_established: «K methodology consulting» value prop defined
2026-09-XX — opportunity → viable: 3-pilot domains complete; resources clear
```
