# Brain Extension Skill Test

## §1 Core claim

Skill tests where **cheat-sheet / formulas-instruction / tooling allowed** = measures **reasoning + tool-use + judgment**, NOT raw memorization. Cheat-sheet = «удлиняет твою память». Test outcome → AI-augmented capability metric. Applicable: Jetix hackathon assessment / engineer cohort screening / education-layer pedagogy. Aligns с brain-extension thesis (AI = continuation of brain compute).

**F-grade:** F4 (voice anchor F4 + multiple precedent — открытые экзамены MIT EECS / actuarial / pilot checkrides; existing methodology adaptable; needs operational pilot at scale)

## §2 Voice anchor (verbatim)

> «Вот потом еще одна идея это то, что вот когда тебя там базово научили чему-то, и ты там проходишь какой-то тест, да, и у тебя есть вот инструкция, ну, чтобы там помнить какие-то формулы, допустим, ну, или такой вот небольшой этот книжечка с подсказками, ну, то есть просто, ну, показывает, что где, то есть удлиняет твою память. а именно от скилла человека.»

## §3 Expansion + cross-refs

### Core principle
- **Raw memory ≠ skill**. Memory можно extend (книжка / Anki / GPT). Skill (reasoning, decomposition, judgment, tool selection) — primary metric.
- **«Удлиняет твою память»** = brain-extension framing — direct lineage с [[wiki/ideas/ai-kak-prodolzhenie-mozga-rasshirenie-vychislitelnykh-m]]

### Precedent — open-book / cheat-sheet baselines
- **MIT EECS / Stanford CS**: many qualifying exams allow notes / textbooks — measures synthesis, не recall
- **Actuarial exams**: formula sheets provided; measures application
- **Pilot checkrides**: aircraft manuals during oral; measures decision-making
- **Bar exam (UK / parts of US)**: codes / statutes allowed; measures legal reasoning
- **PhD viva**: literature open; measures argument structure

### Strategic implications для Jetix
- **Hackathon assessment design**: equal-conditions + brain-extension allowed → measure actual skill (anti-donaters, anti-lucky, anti-memorizers)
- **Engineer cohort screening (BL-1 unblock)**: open-resource technical screen — filter for «can use tools + reason» NOT «memorized algorithms»
- **Education Layer**: pedagogy emphasizes meta-skill (judgment, decomposition, tool-orchestration) over recall
- **AI-augmented capability metric** = quantifiable: skill = output quality given (knowledge + tools + cheat-sheet) constants across cohort

### Adversarial counters (R1 surface)
1. **Cheat-sheet may inflate low-skill scores** (good cheat-sheet > weak skill?) — counter: hard problems require synthesis cheat-sheet can't pre-compute
2. **AI tools makes reasoning indistinguishable from prompt-skill** — counter: NOT a bug; prompt-skill = primary AI-era skill
3. **Gameable via cheat-sheet pre-fabrication** — counter: standardize cheat-sheet (provided, not bring-your-own)
4. **Domain dependence**: brain-extension works для math / code less для creative / values judgments — domain-by-domain calibration needed

## §5 Test design / refutation conditions (F-G-R)

**F = F4.** Promotion к F5+ requires:
- Pilot run на hackathon Q3 2026 — cheat-sheet allowed; compare skill ranking vs traditional closed-book ranking
- Inter-rater reliability ≥ 0.7 on «skill» scoring (3+ judges)
- Skill ranking predicts hackathon outcome (build quality / 3-month retention) better than closed-book ranking

**G = idea-brain-extension-skill-test** (tech niche; G-scope: assessment design)

**R = REFUTED if:**
- Cheat-sheet inflates low-skill signal (good cheat-sheet → high score regardless of skill)
- Reasoning task indistinguishable from memory in test design (operationalization failure)
- Test gameable via pre-fabricated cheat-sheets
- Skill ranking does not predict downstream outcome

**Open questions:**
- Standardized cheat-sheet (provided) vs free cheat-sheet (bring-your-own) — which design?
- Time-bounded vs open-time (recall: pilot checkride bounded; PhD viva open)
- Domain calibration: where does brain-extension framework break down?
- Integration с AI-as-tool: GPT-5 allowed? On what tier of test?
