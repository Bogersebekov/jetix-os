# Цикл развития: невежество → поиск → действие → новая точка А → новые проблемы

## Одна строка

Цикл: признание невежества → поиск решений → воплощение → новая точка А → новые проблемы → признание невежества.

## Структура цикла

```
Признание невежества (диагноз)
       ↓
Поиск решений (рекогносцировка)
       ↓
Подготовка (план + инструменты)
       ↓
Действие (воплощение)
       ↓
Новая точка А (новый baseline)
       ↓
Новые проблемы (открываются после устранения старых)
       ↺
```

## Принцип

Все компоненты должны быть **вместе как единое целое**. Пропустить любой = застрять:

- Без диагноза → решаешь не ту проблему.
- Без поиска → используешь старые решения для новых проблем.
- Без подготовки → действуешь хаотично.
- Без действия → циклы остаются на бумаге.

## Применимость в Jetix

Использовать как **мета-модель** для всех проектов. Любой stuck-state → пройтись по компонентам, найти, какой пропущен.
