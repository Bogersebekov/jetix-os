# Curated community access — курированный допуск в сообщество

## Суть в одной строке

Ценность сообщества определяется не массовостью, а качеством отобранных участников.

## Определение

Модель сообщества, где попасть внутрь могут только те, кого одобрили (обычно — существующие участники или команда платформы). Альтернатива массовому открытому доступу (Slack/Discord без фильтра) и alternative приватным «клубам по подписке».

## Ключевые свойства

- Входной фильтр — адекватность, релевантность, готовность контрибьютить.
- Доверие по умолчанию внутри сообщества: каждого проверили.
- Участник не просто потребляет, а вносит вклад.
- Репутация и вклад — валюта внутри.

## Применимость

**Когда использовать:**

- Сообщество должно работать над реальными задачами (проекты, коллаборация), не просто болтать.
- Качество участников важнее охвата.
- Нужен эффект «все здесь понимают, о чём речь».

**Когда НЕ использовать:**

- Когда цель — массовая образовательная платформа (инфобизнес).
- Когда нужен быстрый рост MAU/DAU для инвестиций.

## Прецеденты

- Apache Software Foundation — commiter-статус выдают за вклад.
- Linux Kernel — maintainers фильтруют PR.
- Y Combinator — отбирает стартапы перед YC Demo Day.
- Inner Circle / RaveSG / Founders Fund — закрытые сети предпринимателей.
