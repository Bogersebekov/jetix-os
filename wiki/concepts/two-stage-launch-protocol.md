# Two-Stage Launch Protocol

## Суть в одной строке

Stage 1: **элит-engineer plan-mode** (~year stealth-on-core + public-on-narrative) building platform + virtual state legal foundations + currency/economy. Stage 2: **mass spread** через gamification + bloggers + Mr. Beast-style spectacle.

## Stage 1 (capability layer)

| Element | Value |
|---|---|
| Cohort | Elite engineers (full-time, NO outsource) |
| Mode | Plan-Mode (build first, don't market early) |
| Timeline | ~год minimum (audio_685) |
| Output | Platform substrate + legal foundations + main currency / economy |
| Public posture | Stealth-on-core, public-on-narrative (hybrid per engineering cell 4.1) |
| Venue | Workshop (O-30 6-element offline workspace) |

## Stage 2 (distribution layer)

| Element | Value |
|---|---|
| Mode | Mass spread |
| Mechanisms | Gamification + local myths + bloggers (Mr. Beast / streamers) |
| Public posture | MAX public + branded + documentary-live (audio_686) |
| Tone | Workhata format (workshops as spectacle) |
| Target | 1M users / 100M user-hours by EOY 2026 (O-36) |

## Phase transition criterion

**Capability-based, NOT time-based** (per cybernetics cell 4.4):
> Stage 1 → Stage 2 transition when MVP-substrate handles 1000 concurrent users.

NOT «после года» rigid — capability gate.
