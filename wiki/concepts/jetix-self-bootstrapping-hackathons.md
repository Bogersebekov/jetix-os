# Concept: Jetix self-bootstrapping hackathons (Hypothesis C)

> **R1 wiki concept.** Vapor-stage; Hypothesis C ranked Phase 2 priority per `research/hackathon-deep-2026-05-18/03-jetix-hypotheses-deep.md` §5.

## Definition

**Self-bootstrapping hackathon series** = sequence of Jetix-hosted hackathons где each event builds substrate that compounds Jetix capability. Next event leverages prior outputs; recursive amplification across cycle.

## Mechanism — recursive amplification (Harari Nexus thesis literal match)

```
Hackathon-1 (build Workshop UI)
  └─> Workshop UI deployable
       └─> Workshop UI хosts Hackathon-2 (build FPF tooling)
            └─> FPF tooling enables better Workshop curriculum + Hackathon-3 efficiency
                 └─> Hackathon-3 (Pattern Language artefact) more productive
                      └─> Pattern Language artefact = Jetix substrate signal externally
                           └─> Hackathon-4 attracts wider talent (SBT role-attestation)
                                └─> Hackathon-5 tests QF prize distribution mechanism
```

## Pre-mortem (anti-cult-formation discipline per direction 02 Cybersyn)

**Failure modes:**
- Inner-circle dynamics develop (Jetix members building for each other)
- External value validation deferred (compounds without ROI)
- Cult-formation risk
- Workshop attendees subsidize internal substrate без recognition

**Mitigation discipline:**
- Each hackathon includes **«external value validation» track** (output deployable beyond Jetix)
- Independent quality audit per major release (per direction 12 cross-domain transfer)
- Open-source mandatory + community use metrics tracked
- Workshop attendees compensated explicitly (recognition + revenue share via QF)
- Public progress reporting (FPF community wiki updates)
- Year-2 milestone: 50%+ external participants (escape velocity check)
