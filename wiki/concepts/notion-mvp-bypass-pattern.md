---

## §1 Что это

**Принцип:** не строить full platform до того как первый cohort-member получит actionable substrate. Вместо этого — **Notion-шаблон со стратегическим документом + ресурсами + checklist** = first-cohort MVP delivery.

**Pattern:** «substrate готова → Notion-обёртка → отдача первому → feedback → platform later».

**Anti-pattern (rejected):** «platform built first → then invite users».

---

## §2 Почему важно

### §2.1 Decouples promotion-mode start from platform-build delay

Promotion-mode start ([[development-promotion-mode-transition]]) НЕ должен ждать full platform readiness. Filesystem = source of truth (per CLAUDE.md Global Rule 4 → §4.2). Notion = view, не authoritative. Можно создать **view-only Notion-шаблон** который exports / mirrors filesystem substrate для shareable consumption.

### §2.2 Human-pace fit

Дмитрий = humanities-bridge audience. Reading filesystem directly = engineer-pattern. Notion-шаблон = humanities-pace + visual + commentable interaction model. Match audience cognitive style.

### §2.3 Tests substrate transmissibility

Если Notion-шаблон transmits substrate effectively → confirms «substrate sufficient for promotion» hypothesis ([[development-promotion-mode-transition]] §6.1 + O-163). Если recipient блокируется на конкретном substrate gap → surfaces gap explicitly.

## §4 Components of Notion-шаблон

1. **Title + one-liner** (canonical Jetix-метод articulation)
2. **3 visual layers** — what / why / how
3. **Concrete next-step** — обязательный element (per [[external-system-cybernetic-principle]] §3.2 self-coaching pattern; external system gives main system concrete next-action)
4. **Resources** — books / docs / podcasts / sample recordings
5. **Checklist** — actionable items
6. **Feedback channel** — explicit comment / message / call invitation
7. **Voluntary opt-in clause** — R12 paired-frame requirement
8. **Fork-and-leave clause** — R12 RUSLAN-LAYER action class preservation

---

## §5 Risks

### §5.1 Notion as platform-lock-in

**Risk:** Notion = SaaS lock-in concern. Cohort members might prefer markdown / IPFS / GitHub.

**Mitigation:** Notion = view only; export-to-markdown discipline preserved (filesystem = source of truth invariant). Export script `tools/notion-export.py` if scaling.

### §5.2 Visual polish vs substrate density tradeoff

**Risk:** Notion encourages visual polish; visual polish может скрыть substrate-gap.

**Mitigation:** content-first authoring discipline; visual minimal until substrate quality validated.

### §5.3 Comment-mode feedback management

**Risk:** Notion comments → overflow inbox.

**Mitigation:** dedicated CRM § references на comments; weekly aggregate review.
