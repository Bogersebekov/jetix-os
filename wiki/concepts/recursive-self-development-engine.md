# Recursive Self-Development Engine

## Суть в одной строке

Plan-mode + execute-mode auto-toggle engine where ROY swarm generates direction candidates → Ruslan acks → execution proceeds; recursive cycle с reflection. **IP-1 STRICT** — NOT autonomous runtime self-modification.

## Определение

Per text_009 Thread 1 (¶1): «система развивается уже с самого начала, ну, как бы сама себя же развивала, снова-таки говорила, как себя развивать... выделяла на это ресурсы... искала людей».

**Engine pattern (R1 + IP-1 preserved):**
1. **Plan-mode toggle** — brigadier generates direction candidates via ROY swarm cell dispatch (5 experts × 4 modes).
2. **Ruslan ack** — owner authority approves direction (per Pillar C Tier 2 rule 1: AI does NOT strategize).
3. **Execute-mode toggle** — brigadier dispatches cells; resources allocated; participants engaged.
4. **Reflection cycle** — outcome reviewed; strategies file appended (gated cycle output); next plan-mode informed.

## IP-1 CAVEAT (CRITICAL — surface foregrounded)

### Что engine **НЕ** означает:
- ❌ Autonomous runtime self-modification of agent system.md.
- ❌ Autonomous Foundation modification (Parts 1-11 / Pillar A/C).
- ❌ Autonomous strategic prose generation (Tier 2 rule 1).
- ❌ Autonomous Default-Deny table modification.

### Что engine **ДА** означает:
- ✅ Brigadier surfaces direction candidates.
- ✅ Pre-authorized action classes execute autonomously within Default-Deny table scope.
- ✅ Strategies files updated per-cycle (gated cycle output).
- ✅ AWAITING-APPROVAL packets emerge per Foundation-touching candidate.

**Pillar C Tier 2 rule 9 «AI does NOT self-modify at runtime» = PRESERVED STRICT.**
