# FPF as Info-Transfer Vocabulary

## Суть в одной строке

FPF = эффективный словарь для передачи, создания и фиксации информации между системами (человек↔компьютер, человек↔человек, фирма↔фирма).

## Определение

**FPF (Foundational Primitive Framework)** = стандартизированный словарь / язык / протокол для эффективной передачи, создания и фиксации информации между системами. По Ruslan voice anchor (verbatim claim 11): «FPF просто помогает это эффективно передавать, создавать и фиксировать… удобный словарь охуенно». [src: audio_696@19-05-2026_17-05-37.ogg claim 11]

Аналог: ISO 9001 / IEEE protocols / GraphQL schema / OpenAPI / W3C standards — но позиционируется не только для технического обмена, а для inter-organisation (включая non-technical) communication.

## Ключевые свойства

- **Vocabulary compression** — N-systems × M-vocabularies → N-systems × 1-vocabulary (quadratic → linear)
- **Multi-direction usage** — info transfer + creation + fixation (three functions in one substrate)
- **Multi-system applicability** — человек↔компьютер, человек↔человек, фирма↔фирма
- **Self-referential capability** — FPF может описать FPF (recursion)
- **Convenience («удобный») — Ruslan ack** — usability emphasis, not formal proof
- **Vocabulary, not theory** — distinguishes from epistemological claims (Kuhn paradigms, Wittgenstein language-games — adjacent traditions)

## Применимость

**Когда использовать:**
- Inter-organisation communication (consulting, hackathons, workshops)
- Cross-discipline projects (engineering ↔ research ↔ humanities)
- Standardising existing ad-hoc info-transfer patterns
- Karpathy/engineer cohort outreach positioning anchor

**Когда НЕ использовать:**
- Mathematical formal proofs (FPF = vocabulary, not theorem-prover)
- Domain-specific deep technical specs (FPF = meta-substrate, not replacement)
- Without explicit FPF-Steward review for primitive additions
- When informal language suffices (no benefit from standardisation overhead)
