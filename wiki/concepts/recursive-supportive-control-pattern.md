# Recursive Supportive Control Pattern (O-71)

## Суть в одной строке

**Multi-tier control hierarchy = supportive, не handcuffing.** Recursive: каждый tier enables next-lower tier, не constrains.

---

## Pattern

**Per-tier responsibility:**
1. **Enable** lower tier — provide resources / info / vocabulary / guidance
2. **Escalate** when blocked — surface obstacles upward, не silently absorb

**Recursive property:** the same enable+escalate logic applies at every tier (T0 → T1 → T2 → ... → owner).

**Result:** control distributes без power-concentration; every layer has agency но не autonomy-from-system.

---

## Анти-паттерн: Handcuffing Control

| Supportive control | Handcuffing control |
|---|---|
| Provides vocabulary | Removes vocabulary |
| Surfaces options | Filters options |
| Enables escalation | Blocks escalation |
| Respects exit | Imposes lock-in |
| Defaults reversible | Defaults committed |

Handcuffing = restrictive constraint без enabling reciprocal — violates corrigibility (FUNDAMENTAL §4.3) + R12 anti-extraction.
