# Concept: Hackathon as information-flow event (Harari Nexus lens)

> **R1 wiki concept.** NOT Strategic Insight. Promote pattern для future Jetix work.

## Definition

**Hackathon = compressed information-flow event.** Per Harari Nexus framework (2024) + Jetix FPF mapping (per `research/hackathon-deep-2026-05-18/02-fpf-mapping.md`):

A hackathon is a **bounded ritual** that compresses 6+ simultaneous information streams (ideas / methods / connections / hiring / investment / cultural memes) into 24-72h. Velocity advantage: 10-1000× over traditional channels (formal education / industry conferences / online courses / accelerators).

## Mechanism

```
6 info streams × 9 stakeholder transformations × N output artefacts
```

**Mythology + bureaucracy dual stack** (per Nexus C-N-1):
- Mythology layer: theme · sponsor narrative · winner stories · team unity metaphors
- Bureaucracy layer: rules · judging criteria · IP handling · prize allocation

**Self-correction signals** (per Nexus C-N-3): ✓ open-source submission / ✓ multi-judge / ✓ post-event feedback / ✗ sponsor lock-in / ✗ winner bias / ✗ mentor asymmetry

**Dataism risk** (per Homo Deus critique): output-volume worship / time-commodification / build-for-build's-sake / talent extraction / sponsor theme dilution / burnout normalization.
